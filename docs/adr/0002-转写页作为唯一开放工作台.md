# ADR 0002：转写页作为唯一开放工作台

## 背景

仓库曾同时保留转写、标注、分析、写作、词典等多个页面名义上的“工作台”定位，但真实落地能力高度集中在转写页，其他页面更多承担规划说明或只读入口角色。

## 决策

1. 当前阶段只把转写页视为唯一开放工作台。
2. 标注、分析、写作、词典等页面在复用边界成熟前维持规划态或受限入口。
3. 后续开放其他工作台前，必须先满足 controller/runtime 可复用、语义边界稳定和发布证据完整。

## 影响

1. 近期研发排期优先服务于转写页内核收口，而不是横向扩写页面。
2. 规划文档中的“多工作台并进”表述一律按历史语境理解。
3. 与开放条件相关的具体执行口径，统一落在 `docs/execution/plans/` 与 `docs/execution/release-gates/`。

## 被放弃的备选方案

1. 按页面平均推进，把多个工作台同时维持为活跃交付面。
2. 先补齐空壳页面，再回头处理转写页耦合问题。

## 后续回顾点

1. 若其他工作台出现稳定的 controller/runtime 复用核，再新增 superseding ADR。
2. 每轮发布前都需要核对该决策是否仍与产品表述一致。
3. **写作工作台**：从 lab 升格为「开放」时，须同时满足 [独立工作台开放门槛](../execution/plans/独立工作台开放门槛与复用边界-2026-03-30.md) 与 [ADR 0013 · 写作 lab 毕业门禁](./0013-writing-lab-graduation-criteria.md)；升格后可在本 ADR 增补 superseding 段落或修订「唯一开放」表述（由产品定夺）。
