# ADR 0013：语料库（原写作）lab 毕业为产品开放工作台

## 背景

[ADR 0002](./0002-转写页作为唯一开放工作台.md) 冻结「当前仅转写页为**开放**工作台」。写作长期以 **lab + flag** 交付；若缺少**可勾选**毕业条件，易与「占位页已换壳但门槛未满足」冲突。本 ADR 定义写作从 **NO-GO / lab** 到 **可宣告开放** 的**最小门禁**（不替代门槛文档全文）。

## 决策（骨架）

1. **与 0002 关系**：满足本 ADR **且** [独立工作台开放门槛](../execution/plans/独立工作台开放门槛与复用边界-2026-03-30.md) **go** 条款后，方可将写作从 **lab** 升格为与转写并列的 **开放工作台**；升格时须在 0002 **后续回顾点** 记一笔或另发 superseding 修订（由产品定夺）。
2. **导出与预览统一**：已实现 [ADR 0010](./0010-writing-workspace-export-runtime.md) 规定的 **Pandoc 单一预览核**（HTML 预览与 docx/html/tex 导出同源 `defaults`），并有 **CI golden** 覆盖 fixture；**PDF（Typst）** 与 **Typst CLI** 探针/降级 UX 已按 0010 **A/B/C 分档**落地。  
2b. **块语义正本**：已实现路线图 [2.5 · 写作合同包](../execution/plans/写作页开发路线图-2026-04-22.md#25-写作合同包全仓块语义单一正本) 规定的 **单一路径**（`kind` / Zod / 工具与 Dexie 元数据**不得**在业务中双轨类型）；**或** 经 ADR/修订 0012 书面豁免并附迁移期。
3. **Citation Resolver Core**：[ADR 0011](./0011-writing-corpus-ref-and-citation-jump.md) 核心已合入，写作与导出 **仅**依赖公开 API；转写 `citationJump` 为薄封装。
4. **协同一期**：已实现 [ADR 0012](./0012-writing-workspace-dexie-schema.md) **G4 一期**（章级/文档级快照 + Supabase 协议回放），且 **载荷与守卫** 与 [ADR 0016](./0016-writing-collab-object-mapping.md) **冻结草案一致**；**字符级实时** 不作为毕业硬条件（见 0012）。
5. **门禁与观测**：`layoutGuard` 双分支、`feature flag` 矩阵、`check:build-budgets` 写作阈值、**Pandoc/一键环境** 文档路径可查（见 0010 **官方一键环境**）。
6. **路线图 §2.7 子集（升格前须满足）**：[写作页路线图 · 2.7](../execution/plans/写作页开发路线图-2026-04-22.md#27-体验安全迁移与生态补充) 中 **2.7.1（i18n / 稳定错误码，与诊断字段可对照）**、**2.7.2（`writing_*` Dexie 迁移剧本）**、**2.7.3（子进程 / spawn 安全自检）**、**2.7.5（`/writing` CSP v0）** 须在升格前 **书面 + 可检**；**2.7.4（TTFF）** 与路线图 [第七节 · 性能/体验](../execution/plans/写作页开发路线图-2026-04-22.md#七验收指标汇总须可测) **同轨验收**。**2.7.6（依赖 bot）**、**2.7.7（遥测，默认关）**、**2.7.8（对外模板贡献）** **不作**升格硬条件；若已开启遥测或接受外部模板贡献，则须符合路线图对应条目的约束。
7. **路线图 §2.8 子集（升格前须满足）**：[写作页路线图 · 2.8](../execution/plans/写作页开发路线图-2026-04-22.md#28-可访问性事故手册大文档边界与横向收口) 中 **2.8.1（a11y / 键盘主路径，与 [F.4](../execution/plans/写作页开发路线图-2026-04-22.md#f4-可访问性与焦点) 可对照）**、**2.8.2（lab 事故运行手册定稿）**、**2.8.8（手工验收检查表 ≥10 条且发布前留痕）** 须在升格前 **书面 + 可勾**。**2.8.3（Worker / 大文档上限）** 与第七节 **Worker / 长篇** 行须有 **可引用数字或排队策略**（可与 TTFF 同里程碑迭代）。若已支持 **富 HTML 粘贴（2.8.4）**、已 **随发行打包字体或许可证资源（2.8.5）**、或 **`writingDocAdapter`/工具公开面已导出（2.8.6）**，则分别满足对应条目的 **自检表**。**2.8.7（协同与隐私）**：**presence、分享链接或等价能力任一上线前** 须有 **书面边界**；未上线则 **不作**升格硬条件。

## 影响

- 产品「写作开放」营销与路由默认行为须与本 ADR + 0002 一致。
- 未满足前，`CorpusLibraryPage`（语料库；原 `WritingPage`）默认仍为占位或 lab，由 flag 控制。

## 被放弃的备选方案

1. 仅以「UI 可点」为开放证据、不要求导出与 Resolver 就绪（不采纳）。

## 后续回顾点

1. 首次满足门禁后，更新本 ADR `status` 并同步路线图 [第一节](../execution/plans/写作页开发路线图-2026-04-22.md) 事实表。
2. 若引入字符级协同二期，单独 ADR 评审是否修订毕业条件。
