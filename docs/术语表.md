# 术语表（Layer-first，1页版）

## 目标
- 统一术语，减少 layer / tier / track / lane 混用。
- 明确边界：业务域 layer-first，互操作域 tier-specialized。

## 术语定义
- Layer：业务层（转写层/翻译层），用于日常编辑、筛选、统计、权限。
- Tier：语言学/外部格式结构层（EAF/TextGrid/层级约束），用于导入导出和结构校验。
- Track：时间轴显示模式（single/multi-auto/multi-locked）。
- Lane：时间轴 UI 行与行号分配（锁定、重排、折叠等）。
- Utterance：语段实体（时间边界与说话人上下文）。

## 边界规则
- 业务域（hooks/components/pages）默认只使用 layer 语义。
- tier 语义仅允许出现在桥接/互操作/语言学校验模块。
- 新增业务字段命名统一使用 layerId；不新增 tierId 作为业务字段名。
- tierId 仅用于 tier 原生语义或桥接上下文。

## 代码落地约束
- TierDefinitionDocType 的导入受静态检查约束（见 scripts/check-tier-boundary-imports.mjs）。
- TierBridgeService 是 tier 概念对业务域暴露的主要边界之一。
- 互操作特例文件（导入导出/快照恢复）允许保留 tierId：`useImportExport`、`useTranscriptionSnapshotLoader`。
- tierId 仅允许在互操作/桥接/测试代码中出现；业务域（hooks/components/pages）禁止新增 tierId。

## 阶段封版标准（术语与边界）
- `npm run check:tier-boundary-imports` 通过。
- `npm run check:tierid-diffusion` 通过，且 `offBoundaryMentions=0`。
- 新增代码在业务域仅使用 layer 语义（`layerId` / `LayerLinkEdge`）。
- 若需保留 tierId，必须在 PR 说明中标注“互操作特例”并给出回收计划。
