# 发布说明：轨道分配与聚焦增强（2026-03-24）

## 版本范围
- 模块：转写时间轴、轨道分配、说话人管理、Toast 反馈
- 目标：提升多人重叠场景的可读性与可控性，降低“跳轨”“点不中”的操作成本

## 主要变更
1. 轨道模式升级为三态
- `single`
- `multi-auto`
- `multi-locked`

2. 轨道锁定能力（MVP）
- 新增 `laneLockMap`（`speakerId -> laneIndex`）
- 分配优先级收敛为：锁定轨道 > 说话人已偏好轨道 > 可复用轨道 > 新建轨道

3. 重叠策略收敛
- `single`：保持不允许重叠编辑
- `multi-auto` / `multi-locked`：允许重叠编辑

4. 聚焦说话人
- 模式：`all` / `focus-soft` / `focus-hard`
- 新增显式聚焦目标下拉（可选“跟随当前选中”或指定说话人）
- 新增媒体维度记忆：切换媒体后自动恢复该媒体上次聚焦目标

5. 重叠区命中优化
- 同点重复点击可在重叠候选间轮换
- 注释项显示 `x/y` 位次提示
- 轮换时弹出轻量 Toast（`重叠候选 x/y`）

## 当天补丁（稳定性收口）
1. 聚焦目标按当前媒体过滤
- 显式聚焦目标仅允许当前媒体可选说话人
- 目标失效时自动回退到“跟随当前选中”（避免 `focus-hard` 下全空白）

2. 锁定冲突可见化
- 轨道分配新增锁定冲突统计（`lockConflictCount`）
- 右键菜单新增冲突提示：`锁定冲突 X 项（已回退自动分配）`

3. 重叠候选计算优化
- 从逐条全量扫描改为活动窗口预处理，降低重叠密集场景渲染开销

## 阶段 2（体验与可观测）已执行
1. 顶部冲突轻提示
- 工具栏新增可点击冲突徽章：`锁冲突：N`
- 点击后弹出轻量 Toast，展示冲突计数与说话人摘要

2. 重叠轮换埋点
- 新增轮换指标累计：触发次数、平均轮换步数、平均候选规模
- 在页面日志中输出 `transcription.overlap_cycle` 指标快照

3. 性能基线
- 新增轨道分配性能基准测试：`src/perf/TrackLayoutPerformanceBaseline.test.ts`
- 新增命令：`npm run perf:track`

## 阶段 3（轨道实体化 PoC）已执行
1. 新增轨道实体本地持久化模块
- 文件：`src/services/TrackEntityStore.ts`
- 结构：`mediaId -> { mode, laneLockMap, updatedAt }`

2. Orchestrator 已接入媒体维度持久化读写
- 切媒体自动恢复该媒体轨道模式与锁定映射
- 无历史值时回退默认（`single` + 空锁定）

3. 落地设计文档
- `docs/规划-轨道实体化落地方案-2026-03-24.md`

## 影响面
- 仅影响转写层的显示与交互行为
- 不修改数据库 Schema
- 翻译层继续保持与说话人布局逻辑解耦
