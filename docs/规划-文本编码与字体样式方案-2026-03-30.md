> 2026-03-30 · 解语 (Jieyu) · 包含编码安全、字体回退、层级字体/样式配置 UI

## 〇、业内参考方案对比

在设计之前调研了四个同领域工具的字体/样式配置方案：

### ELAN（MPI，语言标注）

- **粒度**：Tier（层）级别
- **入口**：Add/Change Tier 对话框 → "More Options..." 按钮 → 弹出字体选择器 + 颜色选择器
- **能力**：每层可独立设置字体（系统字体列表）、字号、层名称颜色、高亮颜色
- **批量操作**：可选"应用到同类型的所有层 / 所有子层 / 同一说话人的所有层"
- **持久化**：存储在 `.eaf` XML 文件中（`<TIER>` 元素的属性）

### Aegisub（字幕编辑器）

- **粒度**：Style（样式）级别，一个样式可被多行复用
- **入口**：Subtitles → Styles Manager → 双击样式 → Style Editor 弹窗
- **能力**：字体族下拉（系统全部字体）、字号、**粗体/斜体/下划线/删除线** 复选框、四色（主/辅/描边/阴影）、边距、对齐、旋转、缩放
- **覆盖**：行内可用 override tag（`\b1` `\i1` `\fn字体名`）局部覆盖样式
- **持久化**：ASS 格式内的 `[V4+ Styles]` 段

### Praat（语音学）

- **粒度**：全局，无 per-tier 字体配置
- **入口**：无专门 UI，依赖系统字体 + 反斜杠特殊符号输入（`\ct` → ɕ）
- **能力**：仅支持 IPA 符号通过特殊记号输入，无字体/粗体/斜体选择
- **启示**：Praat 面向音系学研究，功能取向不同；但"缺乏 per-tier 字体"是用户常见痛点

### FLEx / FieldWorks（SIL，词典/语法标注）

- **粒度**：Writing System（书写系统）级别
- **入口**：项目设置 → Writing Systems → 每个 writing system 绑定一个字体 + 键盘输入法
- **能力**：每种书写系统（vernacular / analysis / national）各自独立字体、输入法、排序规则
- **特色**：与语言 metadata 强绑定，不是"视觉样式"而是"语言系统 → 字体"的映射
- **持久化**：LDML（Locale Data Markup Language）文件

### NLLB / SIL / Ethnologue / WALS — 外部数据源与参考

除了 ELAN/Aegisub/Praat/FLEx 四个同类工具外，以下四个语言学基础设施为本方案的"语言 → 脚本 → 字体"映射提供了数据依据和设计参考。

#### Meta NLLB（No Language Left Behind）— 语言代码规范

- **代码格式**: FLORES-200 采用 `{iso639-3}_{script}` 双段标识（如 `ace_Latn`、`kas_Deva`、`taq_Tfng`）
- **覆盖**: 200 种语言，其中 4 种语言提供双脚本变体（Acehnese Latin/Arabic、Banjar Latin/Arabic、Kashmiri Devanagari/Arabic、Minangkabau Latin/Arabic）
- **对本方案的启示**:
  - 我们的 `resolveScriptForLanguage()` 回退表 `LANG_DEFAULT_SCRIPT` 应覆盖 NLLB-200 中的全部脚本标签，确保低资源语言也能正确匹配字体
  - 对于多脚本语言（如 Kashmiri 同时用 `Arab` 和 `Deva`），应尊重 `OrthographyDocType.scriptTag` 而非硬编码，与 NLLB 的设计一致
  - FLORES-200 语言列表可直接扩展到 `LANG_DEFAULT_SCRIPT` 内置表，确保解语支持的语言包含 NLLB 覆盖范围

#### SIL 生态系统 — 字体目录 + ScriptSource + SLDR

- **SIL 字体目录**: 按地域分 6 区（Global / America / East & SE Asia / South Asia / Middle East / Africa），提供 30+ OFL 字体族
  - Global: Andika（识读教学无衬线）、Charis SIL（衬线）、Doulos SIL（类 Times）、Gentium（多脚本衬线）
  - 区域专用: Padauk（Myanmar）、Annapurna SIL（Devanagari）、Scheherazade New（Arabic）、Abyssinica SIL（Ethiopic）、Busra/Mondulkiri（Khmer）、Tai Heritage Pro（Tai Viet）、Awami Nastaliq（Nastaliq Arabic）
  - **对本方案的影响**: `SCRIPT_FONT_MAP` 应将 SIL 字体作为优先推荐项（尤其是少数民族语言脚本），因为 Noto 系列可能不覆盖所有正字法变体
- **ScriptSource**: ISO 15924 脚本注册数据库，152 种现行脚本 + 110 种历史/虚构脚本，含每种脚本的语言列表、Unicode 状态、字体推荐
  - **可作为数据源**: 后续可通过 ScriptSource API 验证 `LANG_DEFAULT_SCRIPT` 的正确性
- **SLDR（SIL Locale Data Repository）**: LDML 格式存储每种书写系统的 exemplar 字符集、默认脚本、字体推荐
  - 数据结构: `languageId → defaultScript → defaultFont + sortRules + exemplarCharacters`
  - 与 FLEx 的 Writing System 模型一脉相承，本方案的 `resolveScriptForLanguage()` 在概念上对应 SLDR 的 locale 解析路径
- **SIL Line Metrics 文档** (writingsystems.info): 详述字体内部行距设置（`hhea.Ascender`、`OS/2.sTypoAscender`、`usWinAscent`），核心洞见:
  - 不同脚本的理想行距差异显著（Latin ~120%、Devanagari ~140%、Tibetan ~160% of em-size）
  - "没有技术手段能在字体内部为不同脚本定义不同行距" — 需要应用层根据脚本调整行高
  - **直接启发了本方案 §5.8 的脚本行高因子机制**

#### Ethnologue — 语言 → 脚本权威映射

- Ethnologue（SIL 维护）是 ISO 639-3 的注册权威，为每种语言提供官方脚本分配
- 与 SLDR/ScriptSource 共享底层数据，通过 `iso639-3.sil.org` 和 `ethnologue.com` 公开
- 解语已引入 `iso-639-3` npm 包（来源于 SIL ISO 639-3 注册），天然继承了 Ethnologue 的语言元数据
- **注意**: Ethnologue 25 版后不再免费完整访问，但 ISO 639-3 代码表保持开放

#### WALS（World Atlas of Language Structures）— 类型学参考

- **定位**: 马普所进化人类学研究所维护的语言结构特征数据库（2,662 种语言、192 个特征、151 章节）
- **与本方案的间接关系**: WALS 主要记录音系/语法/词汇类型学特征（辅音数量、元音音质、语序等），不直接涉及字体/脚本配置
- **可用价值**:
  - WALS 的语言地理分布数据可辅助验证 `LANG_DEFAULT_SCRIPT` 的区域覆盖
  - 音系特征（如声调系统、辅音 inventory）可启发未来 IPA 字体推荐逻辑（声调语言可能更需要支持声调标记的字体如 Charis SIL）
  - WALS 数据集已完结（不再更新），可作为稳定的参考基线

### 解语方案定位

| 维度 | ELAN | Aegisub | FLEx | 解语（本方案）|
|------|------|---------|------|--------------|
| 样式粒度 | Tier | Named Style | Writing System | **Layer**（层）|
| 配置入口 | 弹窗对话框 | 弹窗对话框 | 项目设置面板 | **右键菜单子菜单** |
| 粗体/斜体 | ✗ | ✓ | ✗ | **✓** |
| 字号选择 | ✓ | ✓ | ✓ | **✓**（预设档位）|
| 字体选择 | 系统全部 | 系统全部 | 绑定书写系统 | **按语言动态推荐**（script→font 映射）|
| 行内覆盖 | ✗ | ✓（ASS tag） | ✗ | **✗**（层粒度足够）|

**设计决策依据**：

1. **层粒度而非行粒度**：解语的层本身就对应一种语言/语系（与 ELAN Tier、FLEx Writing System 类似），层粒度样式是最自然的映射
2. **右键菜单而非弹窗**：ELAN/Aegisub 都用模态弹窗，但解语已有成熟的 `ContextMenu` 嵌套子菜单组件，右键菜单更轻量、所见即所得
3. **按语言动态推荐字体 + 本地字体枚举**：首先根据 script → font 映射表推荐脚本适用字体（类似 FLEx 的 Writing System → Font 绑定）；同时在 Chrome/Edge 中通过 `queryLocalFonts()` 渐进增强，允许用户加载并选择设备已安装的任意字体（需明确授权，Firefox/Safari 不受影响，仅展示预设列表）
4. **不做行内 override**：Aegisub 的行内 tag 表达力强但增加格式复杂度；语言标注工具层粒度样式已满足核心需求

## 三、Part A — 编码安全

### 3.1 新增统一文本摄取模块

**文件**: `src/utils/textIngestion.ts`

```ts
/**
 * 统一文本摄取：BOM 检测 → XML 声明 → 严格 UTF-8 → 候选回退 → 用户提示
 * Unified text ingestion: BOM → XML declaration → strict UTF-8 → candidate fallback → user prompt
 */

export interface TextIngestionResult {
  text: string;
  detectedEncoding: string;
  confidence: 'high' | 'medium' | 'low';
}

export async function ingestTextFile(
  file: File | Uint8Array,
  options?: {
    /** 强制编码（跳过检测）| Force encoding (skip detection) */
    forceEncoding?: string;
    /** XML 模式：优先解析 <?xml encoding="..."> | XML mode: prioritize XML declaration */
    xmlMode?: boolean;
  },
): Promise<TextIngestionResult>;
```

**检测优先级**:
1. `options.forceEncoding` → 直接使用
2. BOM 嗅探（UTF-8 `0xEF 0xBB 0xBF` / UTF-16LE `0xFF 0xFE` / UTF-16BE `0xFE 0xFF`）
3. XML 声明 `<?xml ... encoding="xxx"?>` 解析（仅 `xmlMode: true`）
4. 严格 UTF-8 解码（`new TextDecoder('utf-8', { fatal: true })`）
5. 候选回退：按 `['gbk', 'big5', 'shift_jis', 'euc-kr', 'iso-8859-1']` 逐一尝试，取首个不含 `U+FFFD` 的结果
6. 兜底：UTF-8 replacement 模式 + `confidence: 'low'`

**不引入 jschardet**：候选集有限且可控，无需引入额外依赖。

### 3.2 收口现有入口

| 文件 | 改动 |
|------|------|
| `EafService.ts` | `readFileAsText(file, encoding)` → `ingestTextFile(file, { xmlMode: true })` |
| `AiChatCard.tsx` | `reader.readAsText(file)` → `ingestTextFile(file)` |
| `JymService.ts` | `new TextDecoder().decode(u8)` → `ingestTextFile(u8)` |
| `useImportExport.ts:312` | `readFileAsText(file)` → `ingestTextFile(file)`（非归档导入的统一入口） |

当 `confidence === 'low'` 时，在 UI 层弹出 toast 提示用户编码可能有误，可手动指定。

> **注意**：以下 TextDecoder 使用场景**不属于文本文件摄取**，不需要改动：
> - `src/ai/config/aiChatSettingsStorage.ts:102` — AES-GCM 密钥解密
> - `src/ai/providers/streamUtils.ts:10,82` — SSE 流式解析

### 3.3 Unicode 归一化

在 `src/utils/camDataUtils.ts` 中新增：

```ts
/** NFC 归一化存储 | NFC normalize for storage */
export function normalizeTextForStorage(text: string): string {
  return text.normalize('NFC');
}

/** NFC + lowercase 归一化搜索 | NFC + lowercase normalize for search */
export function normalizeTextForSearch(text: string): string {
  return text.normalize('NFC').toLowerCase();
}
```

在写入路径（`saveUtteranceText`、token/morpheme 创建）统一调用 `normalizeTextForStorage`。

>
> **已有变量对齐**：当前 `global.css:18` 已定义 `--header-brand-font: "JetBrains Mono", "SF Mono", ui-monospace, monospace;`，新增的 `--font-mono` 与其保持一致值。后续 header 品牌字体可改为 `font-family: var(--font-mono);` 统一引用。

**注意**：这些字体不打包到 bundle，依赖用户系统已安装；后续可做按需 `@font-face` CDN 加载。

### 4.4 字体选择原则

所有预设字体和回退链须遵循以下原则：

| 优先级 | 原则 | 说明 |
|--------|------|------|
| **P0** | **Noto 系列优先** | Google Noto 覆盖 150+ 脚本、统一设计语言、OFL 开源；每种脚本的首选字体应为 `Noto Sans {Script}` 或 `Noto Serif {Script}` |
| **P1** | **SIL 字体作为语言学补充** | Charis SIL / Doulos SIL / Gentium Plus / Andika 提供完整 IPA 支持；Padauk / Annapurna SIL 等覆盖 Noto 未达的正字法变体。全部 OFL |
| **P2** | **仅使用免费开源字体** | 所有候选字体必须为 OFL、Apache 2.0 或同等自由许可。严禁商业授权字体（Arial、Times New Roman、Helvetica 等不出现在预设列表中）|
| **P3** | **IPA 模式专属过滤** | 当正字法类型为 `phonemic` / `phonetic` 时，仅展示已验证支持完整 IPA Unicode 块的字体（§5.2.2 `ipaSupport: true`），避免用户选到无法显示 IPA 字符的字体 |
| **P4** | **可跨平台分发** | 字体列表不依赖特定 OS 的预装字体；独立安装包须能打包或按需下载（§4.5）|

**对现有方案的影响**：
- `--font-base` 中移除 `PingFang SC`、`Hiragino Sans GB`（商业/系统专有），改为纯 Noto 链
- `PdfViewerRenderer` 已在 §4.2 修复（移除 Arial）
- `SCRIPT_FONT_MAP` 中不出现 Arial / Times New Roman / Helvetica

### 4.5 独立安装包字体策略（Electron / Tauri）

当前解语运行在浏览器中，字体依赖用户系统已安装。未来打包为独立桌面应用时需考虑字体分发：

#### 4.5.1 分发层级设计

```
┌────────────────────────────────────────────────────────────────┐
│ Tier 0 — 内置打包（随安装包分发，约 2-4 MB）                    │
│   • Noto Sans (Latin/Cyrillic/Greek subset)  ~400 KB           │
│   • Charis SIL                               ~1.2 MB           │
│   • Noto Sans SC (简体中文 subset)            ~1.5 MB           │
│   总计: ~3 MB — 保证开箱即用的 IPA + 中文 + 拉丁覆盖           │
├────────────────────────────────────────────────────────────────┤
│ Tier 1 — 按需下载（首次选择某脚本字体时自动下载到本地缓存）      │
```

#### 5.2.4 解析脚本 + 生成字体列表

```ts
/**
 * 根据层的语言和正字法配置，解析出适用脚本
 * Resolve script tag from layer's language and orthography settings
 */
export function resolveScriptForLanguage(
  languageId: string,
  orthographies?: OrthographyDocType[],
): string {
  // 优先：用户在正字法中显式配置的 scriptTag
  const ortho = orthographies?.find((o) => o.languageId === languageId && o.scriptTag);
  if (ortho?.scriptTag) return ortho.scriptTag;
  // 回退：内置映射表
  return LANG_DEFAULT_SCRIPT[languageId] ?? 'Latn';
}

/**
 * 获取某脚本适用的字体列表（脚本专用 + 通用）
 * Get font presets for a script tag (script-specific + universal)
 * 
 * @param ipaMode 是否为 IPA 输入模式（正字法 type = phonemic/phonetic）
 *                When true, only fonts with ipaSupport: true are returned
 */
export function getFontPresetsForScript(
  scriptTag: string,
  ipaMode = false,
): FontPreset[] {
  const scriptFonts = SCRIPT_FONT_MAP[scriptTag] ?? SCRIPT_FONT_MAP['Latn']!;

  // 合并：系统默认 + 脚本专用 + IPA通用 + 等宽
  const allFonts = [
    UNIVERSAL_FONTS[0]!, // 系统默认 always first
    ...scriptFonts,
    ...UNIVERSAL_FONTS.slice(1), // Charis SIL + Mono
  ];

  if (!ipaMode) return allFonts;

  // IPA 模式：仅保留支持完整 IPA 的字体 | IPA mode: keep only IPA-capable fonts
  // 始终保留"系统默认"选项（带回退链兜底）再加上所有 ipaSupport: true 的字体
  return allFonts.filter((f) => f.key === '系统默认' || f.ipaSupport);
}
```

#### 5.2.5 本地字体枚举（渐进增强）

通过 `window.queryLocalFonts()` 在支持的浏览器（Chrome/Edge 103+）中读取设备已安装字体，作为脚本预设字体的补充。不支持的浏览器（Firefox/Safari）仅展示预设列表，功能无损。

```ts
// ── Types ────────────────────────────────────────────────────
interface LocalFontEntry {
  family: string;           // 字体族名 | Font family name (deduplicated)
  postscriptNames: string[]; // 含全部变体 | All style variants
}

interface LocalFontState {
  status: 'idle' | 'loading' | 'loaded' | 'denied' | 'unsupported';
  fonts: LocalFontEntry[];
}

// ── 能力检测 | Feature detection ─────────────────────────────
export function isLocalFontAccessSupported(): boolean {
  return typeof window !== 'undefined'
    && typeof window.queryLocalFonts === 'function';
}

// ── 字体枚举（去重 + 按 family 聚合）| Enumerate & deduplicate ──
let cachedLocalFonts: LocalFontEntry[] | null = null;

export async function queryLocalFontFamilies(): Promise<LocalFontEntry[]> {
  if (cachedLocalFonts) return cachedLocalFonts;
  if (!isLocalFontAccessSupported()) return [];

  const fontDataArr = await window.queryLocalFonts();
  // 按 family 去重聚合 | Group by family
  const familyMap = new Map<string, string[]>();
  for (const fd of fontDataArr) {
    const existing = familyMap.get(fd.family);
    if (existing) {
      existing.push(fd.postscriptName);
    } else {
      familyMap.set(fd.family, [fd.postscriptName]);
    }
  }
  cachedLocalFonts = Array.from(familyMap.entries())
    .map(([family, postscriptNames]) => ({ family, postscriptNames }))
    .sort((a, b) => a.family.localeCompare(b.family));
  return cachedLocalFonts;
}

/** 清除缓存（用于测试 / 权限变化后刷新）| Clear cache */
export function clearLocalFontCache(): void {
  cachedLocalFonts = null;
}
```

**React Hook**：

```ts
// src/hooks/useLocalFonts.ts
import { useCallback, useState } from 'react';
import {
  isLocalFontAccessSupported,
  queryLocalFontFamilies,
  type LocalFontEntry,
  type LocalFontState,
} from '../utils/layerDisplayStyle';

export function useLocalFonts() {
  const [state, setState] = useState<LocalFontState>({
    status: isLocalFontAccessSupported() ? 'idle' : 'unsupported',
    fonts: [],
  });

  const loadLocalFonts = useCallback(async () => {
    if (!isLocalFontAccessSupported()) return;
    setState((prev) => ({ ...prev, status: 'loading' }));
    try {
      const fonts = await queryLocalFontFamilies();
      setState({ status: 'loaded', fonts });
    } catch (err: unknown) {
      // 用户拒绝权限 | User denied permission
      const isDenied = err instanceof DOMException && err.name === 'NotAllowedError';
      setState({ status: isDenied ? 'denied' : 'idle', fonts: [] });
    }
  }, []);

  return { ...state, loadLocalFonts };
}
```

**安全与隐私考虑**：
- `queryLocalFonts()` 需要用户主动授权（浏览器弹出权限确认），不会静默收集
- 字体列表存在指纹识别风险（W3C spec 已声明），解语仅在用户点击"加载本地字体"时触发，不自动枚举
- 缓存仅在内存中（`cachedLocalFonts`），页面刷新后清空，不持久化字体列表
- 用户选中的字体名存入 `LayerDisplaySettings.fontFamily`，如果其他设备没安装该字体，CSS 会自然回退到 font-stack 下一项

**字体列表合并顺序**：
```
┌─ 系统默认            ← UNIVERSAL_FONTS[0]
├─ 脚本专用预设         ← SCRIPT_FONT_MAP[scriptTag]
├─ IPA / Mono          ← UNIVERSAL_FONTS[1..]
├─ ───────────────────
├─ 加载本地字体…       ← (status=idle 时显示，点击触发权限申请)
│   或
├─ [本地字体列表]      ← (status=loaded 后，按 family 名列出)
│   ├── Arial
│   ├── Courier New
│   ├── …
│   └── (超过 30 项时显示搜索框过滤)
└─ (Firefox/Safari: 此区域不显示)
```

#### 5.2.6 CSS 转换

```ts
/** 将 LayerDisplaySettings 转为行内 CSS | Convert display settings to inline CSS */
export function layerDisplaySettingsToStyle(settings?: LayerDisplaySettings): CSSProperties {
  if (!settings) return {};
  const style: CSSProperties = {};
  if (settings.fontFamily) {
    // 查所有脚本的字体映射，找到 css 值
    style.fontFamily = resolveFontCss(settings.fontFamily);
  }
  if (settings.fontSize) style.fontSize = `${settings.fontSize}px`;
  if (settings.bold) style.fontWeight = 'bold';
  if (settings.italic) style.fontStyle = 'italic';
  if (settings.color) style.color = settings.color;
  return style;
}
```

### 5.4 新增组件

**文件**: `src/components/LayerStyleSubmenu.ts`

纯数据构建函数（不是 React 组件），返回 `ContextMenuItem[]`。
**关键变化**：接收 `languageId` + `orthographies`，动态生成字体列表。

```ts
import type { LayerDisplaySettings, OrthographyDocType } from '../db';
import type { ContextMenuItem } from './ContextMenu';
import {
  resolveScriptForLanguage,
  getFontPresetsForScript,
} from '../utils/layerDisplayStyle';

const FONT_SIZE_OPTIONS = [
  { label: '小 (11px)', value: 11 },
  { label: '默认 (13px)', value: 13 },
  { label: '大 (15px)', value: 15 },
  { label: '特大 (18px)', value: 18 },
];

/** 构建"显示样式"子菜单项 | Build "Display Style" submenu items */
export function buildLayerStyleMenuItems(
  current: LayerDisplaySettings | undefined,
  languageId: string,
  orthographies: OrthographyDocType[],
  onUpdate: (patch: Partial<LayerDisplaySettings>) => void,
  onReset: () => void,
  /** 本地字体状态（来自 useLocalFonts hook） | Local font state from useLocalFonts hook */
  localFonts?: {
    fonts: LocalFontEntry[];
    status: 'idle' | 'loading' | 'loaded' | 'denied' | 'unsupported';
    load: () => Promise<void>;
  },
): ContextMenuItem[] {
  const currentFont = current?.fontFamily ?? '系统默认';
  const currentSize = current?.fontSize ?? 13;

  // 根据层语言动态获取适用字体 | Dynamically get fonts based on layer language
  const scriptTag = resolveScriptForLanguage(languageId, orthographies);

  // 检测 IPA 模式：正字法类型为 phonemic/phonetic 时过滤非 IPA 字体
  // Detect IPA mode: filter non-IPA fonts when orthography type is phonemic/phonetic
  const ipaMode = orthographies.some(
    (o) => o.languageId === languageId && (o.type === 'phonemic' || o.type === 'phonetic'),
  );
  const fontPresets = getFontPresetsForScript(scriptTag, ipaMode);

  // 合并字体列表：预设 + 本地字体 | Merge font list: presets + local fonts
  const fontMenuItems: ContextMenuItem[] = fontPresets.map((preset) => ({
    label: `${currentFont === preset.key ? '✓ ' : '  '}${preset.key}`,
    onClick: () => onUpdate({
      fontFamily: preset.key === '系统默认' ? undefined : preset.key,
    }),
  }));

  // 追加本地字体区域（仅 Chrome/Edge） | Append local fonts section (Chrome/Edge only)
  if (localFonts && localFonts.status !== 'unsupported') {
    if (localFonts.status === 'loaded' && localFonts.fonts.length > 0) {
      // 已加载：显示分隔线 + 设备字体列表 | Loaded: show separator + device font list
      let isFirstLocal = true;
      for (const lf of localFonts.fonts) {
        fontMenuItems.push({
          label: `${currentFont === lf.family ? '✓ ' : '  '}${lf.family}`,
          ...(isFirstLocal ? { separatorBefore: true } : {}),
          onClick: () => onUpdate({
            fontFamily: lf.family,
          }),
        });
        isFirstLocal = false;
      }
    } else if (localFonts.status === 'idle' || localFonts.status === 'denied') {
      // 未加载 / 权限被拒：显示"加载本地字体…" 入口 | Not loaded / denied: show load entry
      fontMenuItems.push({
        label: localFonts.status === 'denied'
          ? '  本地字体（权限被拒）'
          : '  加载本地字体…',
        separatorBefore: true,
        disabled: localFonts.status === 'denied',
        onClick: () => localFonts.load(),
      });
    } else if (localFonts.status === 'loading') {
      fontMenuItems.push({ label: '  加载中…', separatorBefore: true, disabled: true });
    }
  }

  return [
    {
      label: '字体',
      children: fontMenuItems,
    },
    {
      label: '字号',
      children: FONT_SIZE_OPTIONS.map((opt) => ({
        label: `${currentSize === opt.value ? '✓ ' : '  '}${opt.label}`,
        onClick: () => onUpdate({ fontSize: opt.value === 13 ? undefined : opt.value }),
      })),
    },
    {
      label: `${current?.bold ? '☑' : '☐'} 粗体`,
      shortcut: '⌘B',
      onClick: () => onUpdate({ bold: !current?.bold }),
    },
    {
      label: `${current?.italic ? '☑' : '☐'} 斜体`,
      shortcut: '⌘I',
      onClick: () => onUpdate({ italic: !current?.italic }),
    },
    {
      label: '重置样式',
      separatorBefore: true,
      danger: true,
      onClick: onReset,
    },
  ];
}
```

### 5.8 字体 ↔ 行高双向联动机制

#### 5.8.1 现状分析

当前行高系统的关键常量和路径：

| 量 | 值 | 来源 |
|---|---|---|
| `DEFAULT_TIMELINE_LANE_HEIGHT` | 54px | `useTimelineLaneHeightResize.ts` |
| `MIN_TIMELINE_LANE_HEIGHT` | 42px | 同上 |
| `MAX_TIMELINE_LANE_HEIGHT` | 180px | 同上 |
| 默认字号 | ~12.5px (0.78rem) | `transcription.css .timeline-annotation` |
| 行高存储 | `localStorage('jieyu:lane-heights')` | `useTranscriptionWorkspaceLayoutController` |
| 行高拖拽 | `useTimelineLaneHeightResize` — pointerdown/move/up 拖拽底边 | 组件 `TranscriptionTimelineMediaLanes` |
| 行高应用 | `--timeline-lane-height` CSS 变量，inline style 注入 | 同上 |

**关键洞见**（来自 SIL Line Metrics 文档）：
- 不同脚本的理想行距差异显著 — Latin 约 120% em-size、Devanagari 约 140%、Tibetan 约 160%
- 字体内部无法为不同脚本指定不同行距（`sTypoAscender` 等是全局值），**必须由应用层根据脚本调整行高**
- NLLB 覆盖的 200 种语言使用 40+ 种脚本，每种脚本的垂直 extent 不同

#### 5.8.2 设计目标

1. **字号 → 行高**：用户通过菜单改字号时，lane height 按比例自动调整，无需手动拖拽
2. **行高 → 字号**：用户拖拽改 lane height 时，字号按比例自动缩放，保持文本与 lane 的比例和谐
3. **脚本感知**：不同脚本有不同的行高因子（Tibetan 需要更多垂直空间，Latin 更紧凑）
4. **手动优先**：如果用户先改字号再手动拖行高，以最后操作为准（不产生循环联动）

#### 5.8.3 脚本行高因子

```ts
/**
 * 不同脚本的行高缩放因子（相对于 Latin 基线 1.0）
 * Line-height scale factor per script (relative to Latin baseline 1.0)
 * 
 * 依据：SIL Line Metrics 推荐 + NLLB 多脚本实测 + OpenType sTypoAscender/Descender 分析
 */
const SCRIPT_LINE_HEIGHT_FACTOR: Record<string, number> = {
  Latn: 1.0,   // Latin/Cyrillic/Greek 基线
  Cyrl: 1.0,
  Grek: 1.0,
  Hans: 1.05,  // CJK 方块字略高 | CJK square glyphs slightly taller
  Hant: 1.05,
  Jpan: 1.05,
  Kore: 1.05,
  Arab: 1.15,  // Arabic 长 ascender + descender | Long ascenders/descenders
  Deva: 1.12,  // Devanagari headline + descenders | Headline bar + descenders
  Thai: 1.15,  // 声调标记上下 | Tone marks above/below
  Tibt: 1.25,  // 藏文垂直堆叠最大 | Tibetan stacked marks, tallest vertical extent
  Beng: 1.12,  // Bengali similar to Devanagari
  Taml: 1.10,  // Tamil moderate height
  Mymr: 1.15,  // Myanmar tall marks
  Ethi: 1.10,  // Ethiopic moderate
  Khmr: 1.15,  // Khmer subscript consonants
  Guru: 1.12,  // Gurmukhi headline bar
  Mlym: 1.12,  // Malayalam rounded tall forms
  Telu: 1.10,  // Telugu moderate
  Knda: 1.10,  // Kannada moderate
  Sinh: 1.12,  // Sinhala curves
  Laoo: 1.15,  // Lao tone marks
  Geor: 1.0,   // Georgian similar to Latin
};

function getScriptLineHeightFactor(scriptTag: string): number {
  return SCRIPT_LINE_HEIGHT_FACTOR[scriptTag] ?? 1.0;
}
```

#### 5.8.4 联动公式

**基准常量**:
```ts
const BASE_FONT_SIZE = 13;           // 默认字号 px | Default font size
const BASE_LANE_HEIGHT = 54;         // 默认行高 px | Default lane height (= DEFAULT_TIMELINE_LANE_HEIGHT)
```

**方向 A：字号 → 行高（用户通过菜单改字号）**:
```ts
/**
 * 根据新字号和脚本计算推荐行高
 * Compute recommended lane height from font size and script
 */
function computeLaneHeightFromFontSize(
  fontSize: number,
  scriptTag: string,
): number {
  const factor = getScriptLineHeightFactor(scriptTag);
  const raw = BASE_LANE_HEIGHT * (fontSize / BASE_FONT_SIZE) * factor;
  return clampTimelineLaneHeight(Math.round(raw));
}

// 示例 | Examples:
// eng (Latn, 1.0): 13px → 54, 15px → 62, 18px → 75
// tha (Thai, 1.15): 13px → 62, 15px → 72, 18px → 86
// bod (Tibt, 1.25): 13px → 68, 15px → 78, 18px → 94
```

**方向 B：行高 → 字号（用户拖拽行高边界）**:
```ts
/**
 * 根据新行高和脚本反推字号
 * Compute font size from lane height and script
 */
function computeFontSizeFromLaneHeight(
  laneHeight: number,
  scriptTag: string,
): number {
  const factor = getScriptLineHeightFactor(scriptTag);
  const raw = BASE_FONT_SIZE * (laneHeight / BASE_LANE_HEIGHT) / factor;
  // 字号取整到 0.5px，避免频繁跳变 | Round to 0.5px to avoid jitter
  return Math.max(9, Math.min(24, Math.round(raw * 2) / 2));
}

// 示例 | Examples:
// eng (Latn, 1.0): 54px → 13, 80px → 19, 120px → 29→clamped 24
// bod (Tibt, 1.25): 68px → 13, 80px → 15, 120px → 22
```

#### 5.8.5 联动触发路径

**路径 A — 字号改变时自动调整行高**:
```
用户在菜单选"大 (15px)"
  → onUpdate({ fontSize: 15 })
    → updateLayer(layerId, { displaySettings: { ...current, fontSize: 15 } })
    → 同时调用 handleTimelineLaneHeightChange(layerId, computeLaneHeightFromFontSize(15, scriptTag))
    → laneHeights[layerId] 更新到 localStorage
    → "--timeline-lane-height" CSS 变量实时变化
```

**路径 B — 行高拖拽时自动调整字号**:
```
用户拖拽 lane 底边
  → onLaneHeightChange(layerId, newHeight)  [已有路径]
    → 新增: computeFontSizeFromLaneHeight(newHeight, scriptTag)
    → 如果计算出的字号与当前 displaySettings.fontSize 差异 > 0.5px：
      → updateLayer(layerId, { displaySettings: { ...current, fontSize: computedFontSize } })
    → 原有的 localStorage + CSS 变量更新不变
```

**防循环机制**:
```ts
// 用 ref 记录最后操作来源，避免 A→B→A 循环
const lastFontHeightSourceRef = useRef<'font' | 'height' | null>(null);

// 路径 A 触发时
lastFontHeightSourceRef.current = 'font';
// 路径 B 的 Dexie useLiveQuery 回调中，如果 source === 'font'，则跳过反推
```

#### 5.8.6 脚本切换时的行高重算

当用户改变层语言（`languageId` 变更）时，如果 `displaySettings.fontSize` 不变，但脚本从 Latin → Tibetan：
```
原行高: 54 * (13/13) * 1.0 = 54
新行高: 54 * (13/13) * 1.25 = 68  → 自动增长以适配藏文垂直空间
```

在 `TimelineLaneHeader.tsx` 或其包裹组件的语言变更回调中加入：
```ts
const newScript = resolveScriptForLanguage(newLanguageId, orthographies);
const fontSize = layer.displaySettings?.fontSize ?? BASE_FONT_SIZE;
handleTimelineLaneHeightChange(layerId, computeLaneHeightFromFontSize(fontSize, newScript));
```

```
[渲染结果] 文字变大 + lane 变高，保持比例和谐
```

#### 5.9.2 字体变更（完整流程）

```
[用户] → 右键 → 显示样式 → 字体 → "Annapurna SIL"
  ↓
[LayerStyleSubmenu] onUpdate({ fontFamily: 'Annapurna SIL' })
  ↓
[Dexie] updateLayer(layerId, { displaySettings: { ...existing, fontFamily: 'Annapurna SIL' } })
  ↓
[layerDisplaySettingsToStyle] → { fontFamily: '"Annapurna SIL", serif' }
  ↓  
[TimelineAnnotationItem] style={{ ...baseStyle, ...layerStyle }}
  - <span> / <input> 继承 fontFamily
  ↓
[注意事项]
  - 字体变更不自动联动行高（不同字体同字号下高度差异不大）
  - 但脚本变更会联动（§5.8.6），因为脚本级别的垂直 extent 差异显著
```

#### 5.9.3 粗体/斜体切换（完整流程）

```
[用户] → 右键 → 显示样式 → "☐ 粗体"
  ↓
[LayerStyleSubmenu] onUpdate({ bold: true })
  ↓
[Dexie] updateLayer → displaySettings.bold = true
  ↓
[layerDisplaySettingsToStyle] → { fontWeight: 'bold' }
  ↓
[TimelineAnnotationItem] style 合并 → 文字加粗
  ↓
[不联动行高] 粗体/斜体不改变垂直 extent，无需调整行高
```

#### 5.9.4 行高拖拽反推字号（完整流程）

```
[用户] → 拖拽 lane 底部 resize handle（已有交互）
  ↓
[useTimelineLaneHeightResize] pointermove → onLaneHeightChange(layerId, 80)
  ↓
[TranscriptionTimelineMediaLanes] 在 onLaneHeightChange 回调中新增：
  const currentScript = resolveScriptForLanguage(layer.languageId, orthographies);
  const newFontSize = computeFontSizeFromLaneHeight(80, currentScript);
  const oldFontSize = layer.displaySettings?.fontSize ?? BASE_FONT_SIZE;
  if (Math.abs(newFontSize - oldFontSize) > 0.5) {
    updateLayer(layerId, { displaySettings: { ...existing, fontSize: newFontSize } });
  }
  ↓
[渲染结果] lane 变高 + 文字变大（或反过来缩小），保持比例
```

## 八、不做的事

| 不做 | 原因 |
|------|------|
| 引入 jschardet / iconv-lite | 候选集可控，无需额外依赖增加 bundle |
| ~~打包 Web 字体到 bundle~~ | **已规划分层分发策略**（§4.5）：Tier 0 内置核心字体 ~3MB，Tier 1 按需下载脚本字体，Tier 2 读取系统字体 |
| 逐字符样式（rich text） | 解语是结构化标注工具，不是富文本编辑器；样式以层为粒度 |
| 自定义颜色选择器 | 第一版只提供粗体/斜体/字体/字号，颜色可后续扩展 |
| `encodeString` / 任意编码输出 | 浏览器 `TextEncoder` 仅支持 UTF-8，输出永远 UTF-8 |
| 写死全局字体列表 | 字体列表根据层的 `languageId` → `scriptTag` 动态生成，不需要写死 |
| ~~枚举系统字体~~ | **已改为渐进增强**：Chrome/Edge 中通过 `queryLocalFonts()` 可加载设备字体（需用户授权）；Firefox/Safari 仅展示预设列表，功能无损 |
| 固定行高-字号比例 | 不同脚本需要不同比例（SIL Line Metrics），需脚本感知因子 |
| 字号/行高实时联动动画 | 拖拽时逐帧更新已足够流畅，无需额外 transition/animation |
| 使用商业/系统专有字体 | 所有预设字体必须为 OFL/Apache-2.0 等自由许可（§4.4）。Arial、Times New Roman、Helvetica、PingFang SC、Hiragino Sans GB 等不出现在预设列表中，以保证跨平台可分发 |
| IPA 模式展示非 IPA 字体 | 正字法类型为 `phonemic`/`phonetic` 时，自动过滤掉 `ipaSupport: false` 的字体，避免用户选到无法显示 IPA 字符的字体 |
| 修改非文件摄取的 TextDecoder | `aiChatSettingsStorage.ts`（AES-GCM 解密）和 `streamUtils.ts`（SSE 流解析）中的 TextDecoder 不属于文本文件摄取路径，不应改动 |
| 新增 DB 版本号 | `displaySettings` 为可选字段且无需索引，Dexie 对未声明字段透明，无需 `version(16)` |
