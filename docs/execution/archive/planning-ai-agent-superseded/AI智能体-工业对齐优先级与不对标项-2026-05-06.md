# AI 智能体 — 工业对齐 PR 优先级与不对标项（2026-05-06）

## 0. 文档定位

本页把「**应对齐工业成熟产品的子集**」落成 **可排期的 PR 优先级**，并单列「**明确不对标、不排期**」项，防止需求在评审中漂移。

**产品原则（摘要）**：解语 AI 宜强化 **查询型、证据型、可审批、可复盘** 的专用任务智能体；**不**以「更强聊天」或「通用自主编程代理」定义方向。详述见《[语料库（产品定位与执行方案）-2026-04-28](./语料库-产品定位与执行方案-2026-04-28.md)》与《[调研-专用任务AI产品架构对标与解语目标-2026-04-16](../archive/historical-root-docs/调研-专用任务AI产品架构对标与解语目标-2026-04-16.md)》§17。

**与史诗关系**：T1–T4 与 F4 Batch A 的收口事实仍以《[AI智能体工业级史诗-落地方案-2026-05-01](./AI智能体工业级史诗-落地方案-2026-05-01.md)》与《[F4-扩展入口-capability-isolation-epic-2026-05-01](./F4-扩展入口-capability-isolation-epic-2026-05-01.md)》为准；**本页不重复验收细节**，只给出 **后续 PR 队列与边界**。

## 1. PR 优先级队列（建议顺序）

### P0 — 旁路沙箱矩阵持续有效（F4 / T2）

| 目标 | 说明 | 验收提示 |
|------|------|----------|
| 矩阵与代码同步 | 任何新增「用户指令 → session / 记忆虚拟写」入口，按《[F4-扩展入口-受控矩阵-2026-05-05](./F4-扩展入口-受控矩阵-2026-05-05.md)》§2 模板登记并接线 | 矩阵有行；`resolveExecutionPolicy` 族单测 + 定向 Vitest；必要时 E2E（如 `test:e2e:session-sidecar-audit`） |
| 发布与本地构建口径 | 专用构建须注入沙箱相关 VITE；避免默认 `dist` 误跑旁路审计 E2E | CI 与 runbook 与 `playwright.session-sidecar.config.ts` 一致 |

### P1 — 主链门控 / 审计一致性

| 目标 | 说明 | 验收提示 |
|------|------|----------|
| 决策与文案一致 | 阻断/确认/执行路径上，用户可见说明与 `ai_tool_call_decision` 等审计字段可追溯 | 变更涉及 pipeline/audit 时跑相关 Vitest + `gate:release-evidence:governance`（若触证据） |
| 失败分类稳定 | `toolDecisionFailureReason` / release evidence 信号与文档锚点不漂移 | `generate-release-evidence-bundle.test.ts`、governance 脚本 |

### P1 — `propose_changes` 边界内体验与语义

| 目标 | 说明 | 验收提示 |
|------|------|----------|
| B2 边界内加固 | 仅在 ADR-0025 白名单与读模型具备处扩展 rollback；不扩张为「全工具」 | `useAiChat.confirmExecution.proposeBatch`、adapter 单测 |
| 预览 / dry-run / commit | 延续 T3 文档与实现；UI 大改非必须，语义不回归 | `ai-change-transaction-preview-v1.md`、`toolCallDryRun` 相关测 |

### P2 — 证据引用链（RAG / 回答可追溯）

| 目标 | 说明 | 验收提示 |
|------|------|----------|
| 语料库与转写一致 | 导出/复制带可追溯引用（时间码、`corpusRef` 或等价 id）与 AI 回答中的引用策略对齐产品 §1.2 | 与《语料库-产品定位》验收条目对账；必要 E2E/手工记录 |
| 侧栏/脚注可读性 | 在不大改架构前提下，提升「来源可点击、可理解」 | 与 `aiChatRagEnabled` 路径相关的回归测 |

## 2. 不对标项（明确不排期 / 仅方法论参考）

以下 **不作为** 当前产品里程碑；若讨论中出现，默认 **驳回或另立战略议题**，不进入常规 PR。

| 项 | 原因 |
|----|------|
| 通用多智能体 swarm、Devin 式全自动改仓部署 | 与专用任务智能体定位不符；仓库治理方案已刻意保持距离 |
| 云端多租户任务队列、跨设备 durable SLA | 当前真源为桌面本地存储；上云属产品战略变更 |
| 全工具自动 inverse patch / 任意 changeset 事务 | 成本与语言学工具边际收益不匹配；以 ADR-0025 B2 为界 |
| 站内字处理 / 版式引擎 / Obsidian 级插件宇宙 | 语料库与写作路线图已声明非目标；长篇真值在外软 |
| 设计生成类产品能力对标 | 调研 §17.2 已降级 |
| 7×24 线上运维看板、全链路 A/B 商业中台 | 以 release evidence + 可选遥测为主即可 |
| 「最强聊天」与超长自由对话体验对标消费级 ChatGPT | 资源优先任务态、证据、审批与工具正确性 |
