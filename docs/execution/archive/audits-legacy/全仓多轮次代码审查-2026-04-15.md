# 全仓全链路多轮次代码审查（2026-04-15）

**审查方式**：每轮聚焦单一质量/功能维度；工具辅助（检索、深读关键文件、既有 gate 脚本）。主题风险与 backlog 以本文为主；**客观全仓验证**以上述全量审计文档为准。  
**对照计划**：与用户约定的「轮 1～10」审查大纲一致。

---

## 轮 1 — 数据层与持久化契约（Dexie / 迁移 / 导入导出）

**抽样**：`src/db/engine.ts`（版本链至 **v37** M18）、`src/db/io.ts`（`importDatabaseFromJson`）、`scripts/check-segmentation-storage-boundary.mjs`（门禁存在）。

**结论**

- **v37** 通过 `.upgrade` 挂载 `upgradeM18LinguisticUtteranceCutover`，与 ADR/M18 方案一致；`utterances` store 置 `null` 符合 Dexie 删除表语义。
- **导入**：`importDatabaseFromJson` 对非法 JSON、过高 `schemaVersion`、遗留 `utterance_texts` / `utterances` 键显式 **fail-fast**；归一化后对 `layer_units` / `layer_unit_contents` 有默认空数组，降低部分快照缺键风险。
- **事务**：写库在 `dexie.transaction('rw', …)` 内 **bulkPut/clear**，提交前调用 `pruneOrphanUserNotes`，整体 **原子性**良好。
- **注意**：`media_items` 的 `audioDataUrl` → `fetch` → blob 发生在进入事务前的归一化循环；事务失败时不会落库，但会重复拉取 data URL（**P2**，体验/性能向）。

**发现项**

| 级别 | 项 |
|------|-----|
| P2 | 大导入时归一化 + 校验全在事务外遍历，极端大 JSON 可能长时间占主线程；已有测试覆盖 round-trip，**未见**统一「进度/可取消」契约（产品向）。 |

**建议回归**：`npm run test:m18-linguistic-subgraph`、`npm run test:timeline-cqrs-phase9`（与 DB/时间线交叉）。

## 轮 2 — 时间线与读模型（epoch / `localUnitIndex` / 变更集）

**抽样**：`useAiChat.destructiveGate.ts`、`useAiChat.pendingToolCall.ts`（`validateChangeSetEpoch`）、`localContextToolReadModelMeta.ts`、`TranscriptionPage.aiPromptContext` 构建链（测试侧）。

**结论**

- **变更集门控**：`readModelEpochCaptured` 与当前 `timelineReadModelEpoch` 不一致时拒绝应用，与测试 `AiChangeSetProtocol.test.ts` 一致。
- **RAG / 工具**：RAG 使用 **send 时** `promptContext` 的 epoch；本地工具使用 **JIT** `toolContext`；属已知产品取舍（计划文档已写）。

**发现项**

| 级别 | 项 |
|------|-----|
| P2 | `_readModel.unitIndexComplete` 在 `unitIndexComplete === false` 时仍可能呈现为「完整」（`!== false` 语义）；与 RAG「不完整则不校验 hit」一致，但 **文案/字段语义** 易误导排障（文档或字段拆分可后续做）。 |

**建议回归**：`transcriptionDataCaliberConsistency.test.tsx`、`test:timeline-cqrs-phase9`。

## 轮 3 — AI 聊天与本地工具全链路（JIT / list_units 快照 / agent loop 预算）

**抽样**：`useAiChat.streamCompletion.ts`、`localContextTools.ts`、`useAiChat.config.ts`（预算常量）、`agentLoop.ts`。

**结论**

- JIT：`resolveFreshAiContext` 单工具与 batch **每步**刷新，与测试一致。
- **list_units**：`resultHandle` + 快照 TTL/epoch **stale** 门控已实现；非快照路径 `offset` 仍 cap 500，快照路径放宽（此前已修）。
- **Agent loop**：`buildAgentLoopContinuationToolPayload` 将 `tool_result_payload` 压至 `AI_LOCAL_TOOL_RESULT_CHAR_BUDGET`，结构化裁剪 + metric `scope: agent_loop`。

**发现项**

| 级别 | 项 |
|------|-----|
| P2 | `search_units` 空 query → `list_fallback` 反复调用可多次创建快照（内存/metric）；计划 §10 已记 backlog，**未要求本轮改代码**。 |

**建议回归**：`localContextTools.test.ts`、`agentLoop.test.ts`、`useAiChat.streamCompletion.test.ts`。

## 轮 5 — 协作与多端（合并 / 仲裁 / 批次）

**抽样**：`collaborationBetaRuntime.ts`（纯函数合并、chunk 计划、`hashString`）、`collaborationConflictRuntime` 类型引用；测试由 gate 脚本引用。

**结论**

- Beta 路径以 **内存状态机** + 操作日志/仲裁 ticket 为主，适合契约测试；与 IndexedDB 写路径的 **端到端竞态** 未在本轮做手工推演。

**发现项**

|------|-----|

## 轮 6 — 安全与隐私（前端）

**抽样**：全 `src/**/*.tsx` 检索 `dangerouslySetInnerHTML` / `innerHTML =` — **0 命中**。

**结论**

- 未发现典型 **React XSS 注入面**（上述模式）。
- AI 消息、导出 JSON、笔记内容仍应假设 **存储与展示边界** 由组件层转义；若未来引入 Markdown/HTML 渲染，需单独立项审查。

**发现项**

| 级别 | 项 |
