## 五、架构评估

### 5.1 优势

1. **工具调用架构**：Object Adapter Registry 模式（4 适配器 + `ADAPTER_MAP` 分派），新增工具只需 4 步注册，零散发修改风险。
2. **双层意图评估**：`assessToolActionIntent()` 60+ 执行线索词 + 反例排除 + 指代确认(deictic) + 澄清历史感知，在单步架构下已接近极限。
3. **hybrid 三源检索**：向量 + 关键词 + MiniSearch 全文融合，`searchFusionProfiles` 支持场景模板化权重治理。
4. **补偿事务**：`layerAdapter` 的 60 秒补偿回滚机制（创建翻译层→链接失败→自动删除新层）在前端 DB 操作中属高级实践。
5. **测试密度**：770 用例 / 56 AI 源文件 ≈ 13.8 用例/文件，覆盖优秀。

### 5.2 风险

1. **useAiChat.ts 单文件 3502 行**：虽有清晰区域分隔，但重构、code review、并行开发效率受限。
2. **单步工具调用**：无法支持"扫描所有语段并逐个标注"类复杂操作。
3. **文本解析工具调用**：`parseToolCallFromText()` 依赖 LLM 输出格式，无 JSON Schema 保证。
4. **F10 对齐零实现**：作为"手动→半自动"的核心跳板，是最大功能空白。

## 八、结论

解语 AI 智能体从 2026-03-18 评估以来取得了显著进展：

1. **F12 Auto-Gloss** 从 0% → 50%，lexeme 精确匹配引擎已闭环
2. **F28 嵌入** 从 20% → 90%，hybrid 三源检索 + PDF 管线 + 融合权重治理
3. **F32 RAG** 从 5% → 55%，引用溯源跳转 + PDF 查看器已落地
4. **F8 TaskRunner** 从 30% → 95%，统一队列 + cancel/retry/timeout/zombie
5. **工具调用** 从 10 → 14 种，token POS/gloss 操作已注册

**代码健康度良好**：0 TS 错误、770 测试全绿、0 TODO 标记、无循环依赖。死代码已清理 430 行。

**最大空白仍是 F10 MFA/对齐**（10%），这是从"手动标注"到"半自动标注"的核心技术跳板。

**推荐优先级**：先完成 P0 三项低成本任务（persona 自动切换 + token_gloss 落地 + TS 修复），再推进 F28 端到端验证和 F12 Auto-Gloss 增强，为 RAG 和批量标注功能上线扫清最后障碍。
