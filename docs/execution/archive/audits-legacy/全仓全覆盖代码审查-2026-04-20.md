## 1. 机器门禁（machine-gates）

**执行**：`npm run validate`（`typecheck` + `gate:acoustic` + `npm test` + `build:guard`）。

**结果**：2026-04-20 在本工作区 **通过（exit 0）**。

**本轮修复/调整摘要**

- `**SidePaneSidebar`**：`LayerActionPopover` 的 `createLayer` 适配器未转发 `hostTranscriptionLayerIds` / `preferredHostTranscriptionLayerId`（及 `dialect` / `vernacular`），导致侧栏新建翻译层时宿主信息在运行时丢失；已补齐转发，并与 `SidePaneSidebar.interaction.test.tsx` 对齐。

**专项 gate**：本轮以 `validate` 为准；规划中的 `gate:collaboration-cloud` 等可按发布窗口在 CI 或本地叠加，未在此处全量重跑。

## 2. 覆盖率与 E2E（coverage-e2e）

**执行**：`npm run test:coverage`（Vitest + v8，`reportsDirectory: coverage/`，含 `text` + `html`）。

**结果**：**通过（exit 0）**。说明：`EmbeddingCandidateSetPerformanceBaseline` 在覆盖率插桩下显著变慢，已用 `npm_lifecycle_event === 'test:coverage'` 区分「覆盖率跑」与「普通跑」的**墙钟超时与 perf 预算**，避免 coverage 任务误伤；普通 `vitest run` 仍保留 ~8.25s 回归预算。

**低覆盖 + 高风险优先队列（来自 coverage 表摘要，非穷尽）**

以下文件曾在 coverage 摘要中偏低；**已补测或抽离可测契约**（2026-04-20 后续提交）：

- `src/utils/translationRecordingMediaBlob.ts`：`translationRecordingMediaBlob.test.ts`
- `src/utils/segmentPlaybackGuard.ts`、`waveformSelectionUtils.ts`、`waveformDisplayMode.ts`、`waveformVisualStyle.ts`：对应 `*.test.ts`
- `src/workers/vadWorker.ts`：帧概率→语段与线性重采样抽至 `vadWorkerInferenceUtils.ts` 并由 `vadWorkerInferenceUtils.test.ts` 覆盖；Worker 线程仍负责 ONNX / `postMessage`
- `src/workers/acousticExport.worker.ts`：序列化逻辑抽至 `acousticExportSerialization.ts` 并由 `acousticExportSerialization.test.ts` 覆盖；Worker 仅负责消息桥接

仍建议下一轮对 **`src/hooks/useTranscriptionCloudSyncActions.ts` 非 happy path**（弱网、重试、冲突抽屉）做抽检深读或补测。

**Playwright E2E**：本轮未执行 `npm run test:e2e`（规划中为可选）；关键浏览器路径仍建议发版前在具备浏览器依赖的环境跑通。
