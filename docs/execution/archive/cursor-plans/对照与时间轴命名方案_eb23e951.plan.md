# 对照模式与时间轴：更贴切的命名方案

## 已定案：横向模式与纵向模式

**用户选择（最终映射）**：用 **「横向模式」** 取代原 **「时间轴」** 标准视图；用 **「纵向模式」** 取代 **「对照」** 视图。

| 对外名称 | 对应实现（代码标识符可不变） | 产品直觉（供 tooltip 使用） |
|----------|------------------------------|-----------------------------|
| **横向模式** | 标准时间轴 / 多轨编辑：`comparisonViewEnabled` **关** 时的 `TranscriptionTimelineMediaLanes` 或 `TranscriptionTimelineTextOnly` 等 | **时间沿标尺横向延展**，多层轨在同一时间轴上横向铺排与操作。 |
| **纵向模式** | 左右对照 / 对读：`comparisonViewEnabled` **开** 时的 `TranscriptionTimelineComparison` | 主工作流以 **沿列表向下** 扫过成组的对读块为主（原文/译文列仍在界面上左右呈现；「纵向」指 **推进与扫读维度**，非「取消双列」）。 |

**防歧义（建议在 `switchTo*` 的 tooltip 或副标题里写清）**

- **纵向模式**：避免用户以为「只有一列竖排字」——应写明 **「双列对读，按组向下浏览」**。
- **横向模式**：避免用户以为「手机横屏」——应写明 **「多轨时间轴编辑」**；纯文本壳下也可写 **「逻辑时间轴上的多轨编辑」**。

**英文 UI 建议**（需与中文语义对齐，勿直译成易与屏幕方向混淆的 *Portrait/Landscape*）：例如 **Horizontal timeline**（横向模式）与 **Column compare** / **Side-by-side compare**（纵向模式），或并列副标题 *Stacked tracks* vs *Paired columns*。

## 当前用户可见文案（锚点）

- 模式切换短标签：[src/i18n/index.ts](src/i18n/index.ts) 中 `transcription.toolbar.compareView`（现「对照」）、`transcription.toolbar.timelineView`（现「时间轴」）。
- 切换说明：`switchToComparisonView` / `switchToTimelineView`；对照空态：`comparisonRequiresTranslationLayer`；对照区内仍有「原文 / 翻译」「对照列显示」等子文案。

「不直观」通常来自两点：**「对照」未说明是左右双语对齐**；**「时间轴」在纯文本壳下仍是「轨条编辑」**，与「时间轴」一词的直觉（整条时间线）略有偏差。

---

## 方案 A：强调布局（偏 UI 形态）

| 现名 | 建议短名 | 建议长说明（切换按钮 title 或菜单） |
|------|-----------|--------------------------------------|
| 对照 | **双栏对齐** / **左右对齐** | 切换到左右双栏（原文与译文对齐） |
| 时间轴 | **轨条视图** / **分层轨** | 切换到标准分层轨条编辑（波形或纯文本轨） |

**优点**：一眼知道「对照」是双栏；「时间轴」不再冒充「整条产品」。**缺点**：「轨条」略技术向。

---

## 方案 B：强调任务（偏工作流）

| 现名 | 建议短名 | 建议长说明 |
|------|-----------|------------|
| 对照 | **对译** / **逐段对译** | 左右对照编辑译文（按组对齐） |
| 时间轴 | **精编** / **轨上精编** | 在分层时间轨上编辑转写与译文 |

**优点**：突出「对译」是翻译工作、「精编」是细调。**缺点**：「精编」略泛，需与产品其它「精编」概念区分。

---

## 方案 C：强调与波形关系（偏你们现有架构）

| 现名 | 建议短名 | 建议长说明 |
|------|-----------|------------|
| 对照 | **双栏对读** | 下方左右对读；上方仍共用波形 |
| 时间轴 | **波形轨编辑**（有媒体时） / **文本轨编辑**（纯文本时） | 按壳层切换文案更佳：同一键名可固定为 **「标准视图」**，副标题区分波形与纯文本 |

**优点**：与「上方波形 + 下方对照」一致。**缺点**：需壳层分支文案或统一用「标准视图」避免错配。

---

## 方案 D：一对中性总称（最短，适合窄工具栏）

| 现名 | 建议短名 |
|------|-----------|
| 对照 | **双栏** |
| 时间轴 | **标准** |

**优点**：toolbar 最省空间。**缺点**：信息量少，需依赖 tooltip / 首次引导。

## 方案 E：比喻套件（更形象、有画面）

以下成对出现：**左列为「对照」替代**，**右列为「时间轴」替代**；短名适合按钮，**比喻句**适合 tooltip / 空态引导 / 首次使用说明。

### 套件 1：书页与总谱（编辑室隐喻）

| 角色 | 短名（示例） | 比喻一句话（tooltip） |
|------|----------------|-------------------------|
| 对照 | **对页** / **双页** | 像译本与原著**左右对页摊开**，一眼扫过对应关系。 |
| 时间轴 | **总谱** / **多行总谱** | 像管弦**总谱**：多条声部（层）叠在同一时间轴上同时展开。 |

**画面感**：从「读书」切到「指挥读总谱」——前者对齐语义，后者强调多轨并行。**风险**：「总谱」对非乐理用户略冷僻，需一句白话副标（「多轨一起编辑」）。

### 套件 2：河岸与主航道（河流隐喻）

| 角色 | 短名（示例） | 比喻一句话 |
|------|----------------|------------|
| 对照 | **两岸** / **对岸** | 原文与译文像**河的两岸**，中间是时间之流，彼此相望对齐。 |
| 时间轴 | **主航道** / **航道** | 各层像**同一条大河上的多条航道**，船（光标/选区）在时间里上下游走。 |

**画面感**：强、易记。**风险**：「两岸」可能被理解成只有两个块，需 tooltip 点明「仍可多锚点/多组」。

### 套件 3：并排窗与叠影（建筑/光影隐喻）

| 角色 | 短名（示例） | 比喻一句话 |
|------|----------------|------------|
| 对照 | **并排窗** / **双窗** | 像两面**并排的落地窗**，左看原景、右看译意，框对齐、景可对。 |
| 时间轴 | **叠影** / **层影** | 像多张**透明胶片叠在同一张底片**上，每层一条轨，一起压在时间上。 |

**画面感**：叠影非常贴波形/多轨。**风险**：「叠影」略文艺，可作副标题而非主按钮二字。

### 套件 4：传声廊与母带（声音工种隐喻）

| 角色 | 短名（示例） | 比喻一句话 |
|------|----------------|------------|
| 对照 | **传声廊** / **对传** | 像**同声传译的窄廊**：左耳进原话、右耳出译文，身体仍站在同一段时间里。 |
| 时间轴 | **母带** / **多轨母带** | 像录音棚里**尚未拆分的母带工程**：每层一条轨，剪接与标注都在时间线上完成。 |

**画面感**：与「语音转写」产品气质近。**风险**：「母带」略行业向；可作英文 UI 的 *multitrack session* 类对照，中文配白话。

### 套件 5：戏台双屏与侧幕条（剧场隐喻）

| 角色 | 短名（示例） | 比喻一句话 |
|------|----------------|------------|
| 对照 | **双屏** / **戏台双屏** | 像戏台**左右双屏**：左屏演原文、右屏映译文，观众视线横移对齐。 |
| 时间轴 | **侧幕条** / **条幕** | 像舞台**侧幕一条条垂下的提示条**：每层一条，从上到下都是同一场戏（同一时间）。 |

**画面感**：有趣、易传播。**风险**：「侧幕条」生造感略强，更适合营销 copy 而非极简 toolbar。

## 推荐组合（若只选一个默认）

**（已由「纵向模式 / 横向模式」定案取代；以下仅供历史参考。）**

- **偏直观、少行业门槛**：仍首推 **方案 A**（左右对齐 + 分层轨）作「安全默认」。
- **要比喻但仍好懂**：推荐 **套件 1（对页 + 总谱）** 或 **套件 3 的「双窗 + 层影」**——tooltip 写比喻，按钮上可缩短为「对页 | 多轨」。
- **要最大胆、品牌记忆点**：**套件 4（传声廊 + 母带）** 或 **套件 2（两岸 + 主航道）**，建议必须配 **图示或首次引导**，避免单看两字误解。

## 若决定改名：实现范围（供后续迭代）

- **必改**：[`src/i18n/index.ts`](src/i18n/index.ts) 中英词条：`transcription.toolbar.compareView` → **纵向模式**（对照）；`transcription.toolbar.timelineView` → **横向模式**（原时间轴）；`switchToComparisonView` / `switchToTimelineView` 与 `comparisonRequiresTranslationLayer` 等所有用户可见「对照」「时间轴」字样，按上表统一。依赖文案示例：`segmentMissingForSave` 内「请先在时间轴…」宜改为「请先在**横向模式**（多轨编辑）…」等与已定案一致。
- **可选**：执行规划/产品文档中的「对照」「时间轴」用语（如 [docs/execution/plans/对照与多壳时间轴段块统一收口规划-2026-04-19.md](docs/execution/plans/对照与多壳时间轴段块统一收口规划-2026-04-19.md)）与 [docs/execution/plans/左右对照视图模式落地方案-2026-04-19.md](docs/execution/plans/左右对照视图模式落地方案-2026-04-19.md) 标题是否随产品对外名调整。
