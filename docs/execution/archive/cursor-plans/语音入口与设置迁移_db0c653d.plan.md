# 语音入口与设置迁移方案

## 目标与边界
- 保持聊天区现有文本配置与行为不变（Provider/风格/Cost Guard 等不迁移）。
- 在聊天输入区右侧增加/替换语音助手入口按钮。
- 聊天区仅保留语音“关键配置选型”与关键状态（建议：识别模式、实时状态、开始/停止）。
- 非实时语音配置迁入 Settings（如 STT provider、增强参数、超时与连接类配置）。

## 现状锚点（实施文件）
- 聊天入口与输入区：[src/components/ai/AiChatCard.tsx](src/components/ai/AiChatCard.tsx)
- 聊天窗口壳层：[src/pages/TranscriptionPage.ChatWindow.tsx](src/pages/TranscriptionPage.ChatWindow.tsx)
- 语音运行时与注入：[src/pages/TranscriptionPage.AssistantRuntime.tsx](src/pages/TranscriptionPage.AssistantRuntime.tsx)、[src/hooks/useVoiceInteraction.ts](src/hooks/useVoiceInteraction.ts)
- 语音组件：[src/components/VoiceAgentWidget.tsx](src/components/VoiceAgentWidget.tsx)
- Settings 面板：[src/components/SettingsModal.tsx](src/components/SettingsModal.tsx)
- 语音配置持久化：[src/hooks/useVoiceDock.ts](src/hooks/useVoiceDock.ts)
- 样式与文案：[src/styles/panels/ai-chat-composer.css](src/styles/panels/ai-chat-composer.css)、[src/i18n/aiChatCardMessages.ts](src/i18n/aiChatCardMessages.ts)

## 实施方案

### 阶段 1：输入框右侧语音入口重排（仅改入口，不改语音逻辑）
- 在 `AiChatCard` composer 操作区新增语音入口按钮位于发送按钮旁（优先复用现有 `voiceEntry` toggle 行为）。
- 保留原有发送/停止优先级与禁用态逻辑；语音按钮不影响文本发送状态机。
- 若当前语音 runtime 未激活，按钮显示“可启用语音”的 dormant 语义（沿用现有分支）。

### 阶段 2：聊天区仅保留关键语音项
- 在 `VoiceAgentWidget` 或 `voiceDrawer` 渲染层拆分“关键项 vs 非关键项”。
- 聊天区保留：
  - 语音入口与开关
  - 当前语音状态（idle/listening/processing）
  - 识别模式切换（command/dictation/analysis）
- 聊天区移除：provider 细项、增强参数、网络/超时等非实时配置 UI。

### 阶段 3：将非实时语音配置迁入 Settings
- 在 `SettingsModal` 增加“语音助手设置”分区，承接：
  - STT provider 及其参数
  - 本地 whisper/增强参数
  - 超时、连接、默认策略类配置
- 复用 `useVoiceDock` 的存储键与读写逻辑，避免双写与迁移脚本。
- 为设置项增加“与聊天区一致”的即时生效说明（变更后新会话/当前语音会话生效策略写清）。

### 阶段 4：上下文联动与降级策略
- `AssistantRuntime/useVoiceInteraction` 继续作为语音单一状态源。
- Settings 修改后通过 context/hook 回流到语音运行时；若 runtime 未激活，先持久化，激活时加载。
- 保留 feature flag 分支，确保关闭语音能力时 UI 不出现失效控件。

### 阶段 5：验证与回归
- 组件回归：`AiChatCard` 输入区按钮布局、语音入口开关、发送/停止行为。
- 状态回归：dormant -> active 切换、模式切换、语音中断/恢复。
- 设置回归：Settings 修改后持久化正确，刷新后恢复，聊天区实时反映关键状态。
- 文案与样式：i18n key 完整、中文/英文一致、窄宽度下不挤压发送按钮。

## 风险与控制
- 双入口壳层（侧栏/浮窗）行为不一致：同测两条渲染链路。
- 语音入口与发送按钮竞争空间：先最小宽度布局约束，再做响应式降级（图标+tooltip）。
- runtime 未激活时设置不可见：Settings 允许独立编辑并持久化，运行时惰性加载。

## 交付物
- 聊天输入区右侧语音入口已落位。
- 聊天区仅保留关键语音项。
- 非实时语音配置集中在 Settings。
- 对应测试与文档（如有）更新完成。
