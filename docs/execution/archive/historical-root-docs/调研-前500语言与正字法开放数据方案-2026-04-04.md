# 调研：前500语言与正字法开放数据方案（2026-04-04）

## 目标

为项目准备一套可公开内置、可重复生成、可逐步人工校审的“前500语言 + 对应正字法种子”方案，用于后续支持：

- 语言选择
- 正字法选择
- 多书写系统语言的正字法分流
- 输入法 / 字体 / 双向文本 / transform 扩展

## 结论摘要

当前没有我能确认的、同时满足“全球覆盖、开放再分发、持续更新、直接给出前500语言与完整正字法”的单一数据源。

因此项目应采用组合式方案：

1. 用 Wikidata 生成公开可复跑的前500语言候选集
2. 用 Unicode CLDR 提供默认脚本推断
3. 用 ISO 639-3 / ISO 639-1 / Glottocode 做语言实体归一化
4. 用 Wikidata P282 补充多书写系统语言的候选脚本
5. 用 FLORES+、SIL wsresources、Omniglot 等补“高优先级语言”的人工校审正字法

## 已采纳的主数据源

### 1. Wikidata

- 作用：前500语言候选集、语言实体、ISO 639-1/639-3、Glottocode、书写系统候选
- 开放性：CC0
- 官方入口：
  - https://www.wikidata.org/wiki/Wikidata:Data_access
  - https://query.wikidata.org/
- 本项目中的用途：
  - 用 sitelinks 作为“全球通用度代理排序”
  - 用 P220 / P218 / P1394 / P282 提取语言码、Glottocode、书写系统候选

### 2. Unicode CLDR

- 作用：默认脚本、locale 生态、语言脚本关系的现代软件实现参考
- 官方入口：
  - https://cldr.unicode.org/
  - https://cldr.unicode.org/downloads/cldr-48
- 2026-04 调研时效：主页显示 CLDR 48.2 于 2026-03-17 发布
- 本项目中的用途：
  - 用 likelySubtags 作为默认脚本来源
  - 用于给 Wikidata 缺脚本码的语言补默认脚本

### 3. Glottolog

- 作用：语言实体归一化、谱系、Glottocode、语言边界参考
- 开放性：CC BY 4.0
- 官方入口：
  - https://glottolog.org/
  - https://glottolog.org/meta/downloads
- 2026-04 调研时效：下载页显示当前版本 5.3
- 本项目中的用途：
  - 作为语言主表的实体稳定键之一
  - 用于后续和 FLORES+ / 其他研究数据对齐

## 重要补充源

### 4. OPUS

- 官方入口：https://opus.nlpl.eu/
- 调研摘要：2026-04 页面显示覆盖 1005 种语言
- 本项目中的用途：
  - 不直接做正字法来源
  - 作为“数字资源覆盖度”和“语言优先级”补充信号

### 5. Mozilla Common Voice

- 官方入口：https://commonvoice.mozilla.org/en/languages
- 调研摘要：2026-04-04 页面显示语言页最新更新时间为 2026-04-04
- 本项目中的用途：
  - 作为语音资源覆盖与社区活跃度信号
  - 辅助判断某语言是否应进入首批产品优先支持清单

### 6. FLORES+

- 官方入口：https://huggingface.co/datasets/openlanguagedata/flores_plus
- 管理项目：https://oldi.org/
- 许可证：CC BY-SA 4.0
- 调研摘要：数据卡显示当前版本 4.4，覆盖 228 个 language varieties
- 本项目中的用途：
  - 高质量 language-script-variant 样本集
  - 为高优先级语言补充“多正字法 / 多书写系统”细粒度样本

### 7. SIL wsresources

- 官方入口：https://github.com/silnrsi/wsresources
- 许可证：MIT
- 本项目中的用途：
  - 作为语言/脚本/地区级书写系统资源补充源
  - 可用于后续接键盘、mapping、legacy 编码资源

## 当前生成规则

### 排序

- 使用 Wikidata `sitelinks` 作为“全球通用度代理排序”
- 这不是语言人口榜，也不是唯一权威榜
- 但它公开、可复跑、可再分发，适合做产品第一版种子清单

### 默认脚本

- 优先使用 CLDR likelySubtags
- 如果 CLDR 未命中，则回退到 Wikidata `P282 -> P506` 提供的 ISO 15924 脚本码

### 正字法种子

- 当前产物不是最终人工校审正字法库
- 当前是 `script-derived` 级别的正字法种子
- 每个语言至少生成一个 primary seed
- 多书写系统语言会额外生成 secondary seeds
- 全部种子默认标记为 `needs-review`

## 已知限制

1. 这份前500清单是“开放可复跑的产品种子表”，不是语言学终稿。
2. 多正字法语言仍需要人工校审，例如：中文、日语、蒙古语、塞尔维亚语、库尔德语、乌兹别克语、哈萨克语、豪萨语等。
3. 某些语言在 Wikidata 中会列出更细的子脚本或历史文字，当前策略会保留为 secondary seed，后续应收敛为“主推正字法 / 扩展正字法”。
4. 若要上生产，建议把 `needs-review` 分批转换成：
   - `verified-primary`
   - `verified-secondary`
   - `historical`
   - `legacy`
   - `experimental`

## 推荐下一步

1. 先以这份 top500 种子表接入项目的数据结构和 UI。
2. 选前50高优先级语言做人工校审，补正式正字法名称、字体偏好、输入提示、RTL/LTR、transform 规则。
3. 第二阶段引入 FLORES+ 与 wsresources，对多书写系统语言做细化。
4. 第三阶段为正字法建立独立审校流程与版本号，避免以后每次重新生成覆盖人工成果。
