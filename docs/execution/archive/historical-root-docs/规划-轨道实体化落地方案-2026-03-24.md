# 规划：轨道实体化落地方案（2026-03-24）

## 目标
- 将轨道从“渲染期推导状态”升级为“可持久化实体”。
- 让 `track mode` 与 `speaker -> lane` 绑定具备跨会话、跨设备一致性。

## 本次 PoC 已落地
1. 增加媒体维度轨道实体本地存储（LocalStorage）
- 数据结构：`mediaId -> { mode, laneLockMap, updatedAt }`
- 读写模块：`src/services/TrackEntityStore.ts`
- 接入点：`TranscriptionPage.Orchestrator`

2. 兼容策略
- 无历史数据时回退默认：`mode=single`、`laneLockMap={}`
- 读取时做字段清洗，忽略非法轨道索引和非法模式值

## 目标数据库实体（下一步）
建议新增集合：`track_entities`

字段建议：
- `id`: string
- `textId`: string
- `mediaId`: string
- `mode`: `single | multi-auto | multi-locked`
- `laneBindings`: `Record<speakerId, laneIndex>`
- `policyVersion`: number
- `updatedAt`: ISO string

索引建议：
- `mediaId`
- `[textId+mediaId]`（唯一）

## 迁移草案
1. v1（当前）
- 优先读 DB；若无 DB 记录则读 LocalStorage PoC；再回退默认值。

2. v2（切换期）
- 加载时将 LocalStorage 条目写入 DB（仅首轮迁移），写入成功后标记迁移完成。

3. v3（收敛）
- 停止写 LocalStorage，仅保留读取兼容（可设置 1~2 个版本窗口）。

## 风险与缓解
- 风险：媒体切换频繁触发状态覆盖。
- 缓解：按 `mediaId` 键分离存储，并在切换时先恢复再渲染。

- 风险：旧数据污染（非法 lane 索引）。
- 缓解：读取时进行 sanitize，仅保留 `>=0` 的整数索引。

## 验证建议
- 单媒体：切换轨道模式、锁定后刷新页面，状态保持。
- 多媒体：A/B 各自设不同模式与锁定，切换往返不串档。
- 异常值：手动篡改 localStorage 后仍可加载并自动清洗。
