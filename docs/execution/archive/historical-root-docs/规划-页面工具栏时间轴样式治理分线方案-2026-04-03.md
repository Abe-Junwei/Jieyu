# 规划-页面工具栏时间轴样式治理分线方案-2026-04-03

## 1. 目的

在 `panel` 归属线已经完成收口后，把剩余的页面壳、工具栏、时间轴基础设施样式从原有迁移任务中独立出来，避免继续以 `panels/*` 为目标错误迁移跨组件共享样式。

本方案只处理三类样式：

1. 页面壳与槽位样式
2. 工具栏与 portaled toolbar 样式
3. 时间轴基础设施与跨 lane 共享样式

不再把这条线和 `DialogShell` / `PanelSection` / `PanelSummary` 的 panel 收口混在一起推进。

## 2. 边界

### 2.1 属于本线的样式

1. `src/styles/transcription.css` 中仍保留的页面壳、工具栏 portaled 变体、timeline 基础设施
2. `src/styles/ai-hub.css` 中仍保留的 toolbar/timeline 响应式覆盖
3. `src/styles/global.css` 中若仍有页面级残留，需要继续下沉到 `src/styles/pages/*`

### 2.2 不属于本线的样式

1. 单组件独占的业务面板样式
2. 已迁移到 `src/styles/panels/*` 的组件样式
3. 通用原子类与基础 token，例如 `small-text`、`.input`、`.field-grid` 这类共享基础类

## 3. 当前剩余范围

### 3.1 页面/工具栏侧

当前优先关注的块：

1. `app-left-rail-bottom-slot .transcription-wave-toolbar-right-portaled` 相关样式，当前仍在 `src/styles/transcription.css`
2. 与 `transcription-wave-toolbar`、`transcription-wave-toolbar-right-portaled` 相关的页面适配和响应式规则
3. `transcription-ai-observer-*` 这组共享工具栏/状态栏样式

### 3.2 时间轴基础设施侧

当前明确不应进入 `panels/*` 的块：

1. `timeline-scroll` / `timeline-content`
2. `timeline-lane*` / `timeline-annotation*`
3. `lane-link-*` / `waveform-overview-*`
4. `timeline-group-*`

这些块应继续留在页面或 timeline 专属样式治理线，而不是作为组件面板样式处理。

## 4. 目录目标

建议形成以下明确分层：

1. `src/styles/pages/app-shell-layout.css`
   - 页面壳、侧栏槽位、主区域结构
2. `src/styles/pages/transcription-layout.css`
   - 转写页容器与主工作区布局
3. `src/styles/pages/transcription-toolbar.css`
   - 顶部/底部工具栏、portaled toolbar 变体、observer 状态栏
4. `src/styles/pages/transcription-timeline.css`
   - 时间轴滚动容器、lane、annotation、resize、drag/drop、bundle target 等基础设施

如果 `transcription-timeline.css` 一次性切出风险过高，可先按 `timeline-core.css` 与 `timeline-interaction.css` 两步拆分。

## 5. 推进顺序

### 阶段 A: 工具栏与 portaled toolbar 收口

目标：先把 `transcription.css` 中残留的 toolbar-portaled 规则搬到 `src/styles/pages/transcription-toolbar.css`。

完成标准：

1. `app-left-rail-bottom-slot .transcription-wave-toolbar-right-portaled*` 不再出现在 `transcription.css`
2. `architecture guard` 对新归属增加 required/forbidden 规则
3. 现有转写页工具栏回归不退化

### 阶段 B: observer 状态栏收口

目标：把 `transcription-ai-observer-*` 统一纳入工具栏页级样式文件。

完成标准：

1. observer 相关类全部集中在 `src/styles/pages/transcription-toolbar.css`
2. 不再散落在 `transcription.css` 或其他业务 CSS 中

### 阶段 C: timeline 基础设施拆分

目标：把 `timeline-*`、`timeline-lane-*`、`timeline-annotation-*` 等基础设施从 `transcription.css` 收敛到独立 timeline 页级文件。

完成标准：

1. `transcription.css` 不再承担主时间轴基础设施职责
2. timeline 核心块由独立文件承接
3. 页面入口显式导入新的 timeline 样式文件

## 5.1 执行版批次

为避免再次变成“大重构”，本线按 3 个可独立提交的批次推进，每个批次都要求完成“改样式归属 + 补 guard + 跑验证”。

### 批次 1: 工具栏 portaled 变体

目标：把当前仍留在 `src/styles/transcription.css` 的 `app-left-rail-bottom-slot .transcription-wave-toolbar-right-portaled*` 规则迁入 `src/styles/pages/transcription-toolbar.css`。

涉及文件：

1. `src/styles/transcription.css`
2. `src/styles/pages/transcription-toolbar.css`
3. `scripts/architecture-guard.config.mjs`

完成定义：

1. `transcription.css` 中不再出现 `transcription-wave-toolbar-right-portaled` 根规则
2. `transcription-toolbar.css` 明确承接 portaled toolbar 规则与相关 dark/mobile 覆写
3. `architecture guard` 对 `transcription.css` 增加精确 forbidden regex

### 批次 2: observer 状态栏

目标：把 `transcription-ai-observer-*` 整体收进 `src/styles/pages/transcription-toolbar.css`，作为工具栏子系统样式处理。

涉及文件：

1. `src/styles/transcription.css`
2. `src/styles/pages/transcription-toolbar.css`
3. `scripts/architecture-guard.config.mjs`

完成定义：

1. `transcription.css` 中不再出现 `transcription-ai-observer-*`
2. `transcription-toolbar.css` 对 observer 有 required literals
3. [src/components/transcription/toolbar/ObserverStatus.test.tsx](../../../../src/components/transcription/toolbar/ObserverStatus.test.tsx) 通过

### 批次 3: timeline 基础设施首拆

目标：先切出 timeline 最外层结构与交互基础，不一次性搬空全部时间轴样式。

第一批建议先迁以下根块：

1. `timeline-scroll`
2. `timeline-content`
3. `timeline-lane*`
4. `timeline-lane-resize-handle*`
5. `timeline-lane-boundary-highlight*`

暂缓块：

1. `timeline-annotation*`
2. `timeline-text-item*`
3. `timeline-annotation-note-icon*`
4. `lane-link-*`
5. `waveform-overview-*`

涉及文件：

1. `src/styles/transcription.css`
2. 新建 `src/styles/pages/transcription-timeline.css`
3. `src/styles/transcription-entry.css`
4. `scripts/architecture-guard.config.mjs`

完成定义：

1. `transcription-entry.css` 显式导入新的 timeline 文件
2. `transcription.css` 不再承担已迁出的 timeline 外层根块
3. 时间轴核心测试切片保持通过

## 5.2 每批标准动作

每一批都按同一动作序列执行：

1. 先 grep 选择器使用面，确认是页面/工具栏/timeline 共享块，而不是单组件面板块
2. 新文件或目标文件承接样式
3. 从旧文件删除对应根选择器
4. 更新 `scripts/architecture-guard.config.mjs`
5. 跑局部测试切片
6. 跑 `npm run check:architecture-guard`
7. 需要时跑 `npm run build`

## 6. 门禁策略

沿用当前渐进式做法，不做一次性大封锁。

### 6.1 architecture guard

新增建议：

1. 对 `src/styles/pages/transcription-toolbar.css` 增加 toolbar-portaled 与 observer 的 required literals
2. 对未来的 `src/styles/pages/transcription-timeline.css` 增加 timeline root required literals
3. 对 `src/styles/transcription.css` 增加精确 forbidden regex，逐批阻止已迁出的根选择器回流

### 6.2 回归验证

建议继续复用现有 panel gate 之外的独立验证方式：

1. 转写页布局 guard
2. 与工具栏/observer 相关的局部测试切片
3. 需要时再加页面级 build 校验

### 6.3 建议新增脚本

本线稳定后再补脚本，不要求第一批就先建：

1. `check:page-style-governance`
   - 组合 `npm run check:architecture-guard`
2. `test:transcription-toolbar-governance`
   - 组合 layout guard 与 `ObserverStatus.test.tsx`
3. `test:transcription-timeline-governance`
   - 组合 `TimelineLaneHeader.test.tsx`、`TranscriptionTimelineHorizontalMediaLanes.test.tsx`、`TranscriptionTimelineWorkspaceHost.test.tsx`
4. `gate:transcription-style-phase1`
   - 组合 architecture guard + toolbar/timeline 切片 + build

## 7. 结项标准

这条独立线可以认为收口完成，当且仅当：

1. `transcription.css` 中只剩下未拆的少量历史兼容块或确有必要的过渡层
2. 页面壳、工具栏、timeline 基础设施各自有明确文件归属
3. architecture guard 已能阻止已完成批次回流
4. 不再需要借助 `panels/*` 继续吸纳非 panel 样式

额外结项条件：

1. `transcription-toolbar.css` 已完整承接 toolbar 与 observer 规则
2. `transcription-timeline.css` 或等价 timeline 文件已承接首批时间轴基础设施
3. `transcription.css` 的职责收缩到过渡层或剩余未拆的历史块，而不是继续承担主归属

## 8. 执行原则

1. 不把 timeline 基础设施硬塞进 `panels/*`
2. 不把通用原子类误判成页面专属类
3. 每次只切一批明确边界的根选择器
4. 每次改动都要配套 guard 与验证
