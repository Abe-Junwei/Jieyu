## 3. 仍有效的产品分层（V1 / V2 / V3）

在解语既有 AI 对话、快捷键与转写能力之上，语音智能体按能力分层表述仍作为**产品词汇**保留（实现细节以真源文档与代码为准）：


| #      | 能力               | 说明                                       |
| ------ | ---------------- | ---------------------------------------- |
| **V1** | 语音/文字指令 → UI 操作  | 口头指令解析后调用既有 action / tool                |
| **V2** | 实时语音输入 → 语言分析建议  | 持续监听目标语言语音，提供分析类建议（与「分析」模式、多模态编排相关）      |
| **V3** | 语音转文字 → 翻译/标注层填充 | 听写 / STT 结果进入语段层或听写流水线（与 dictation 路径相关） |


## 4. 旧稿中已被事实替代或须纠偏的表述

以下条目在 legacy snapshot 中仍可能出现，**勿再当作当前事实**：

1. **「浏览器端 STT 从零集成」** — 主链已具备 Web Speech、商业 STT、`VoiceInputService` / `SttStrategyRouter` 等；以架构文与 `services/stt/` 为准。
2. **「ChatMessage 仅为 string、无音频通道」** — 多模态与编排策略见 **ADR-0028** 与当前 `useAiChat` / 相关 provider 实现。
3. **独立产品化 `TtsService` 的单一命名** — TTS 边界与策略见 **ADR-0029**、`assistantWebSpeechTts` 等；全盘审查中的阶段描述以后者为准。
4. **非听写意图与聊天主链分裂** — **ADR-0027** 已收敛：`command` / `analysis` 等与文字一致走统一聊天路径（见 `voice-unified-chat-path.md` 文件级索引）。
5. **具体文件行数、包体 KB 数、测试用例计数** — snapshot 内「2026-03-24 实态」等表格已过期；以当前 `npm test` / 构建结果与 `check:architecture-guard` 为准。
6. **§18、附 v3.0 路线图中的周级排期** — 已过期；**排期真源**为《AI智能体-战略规划与下一步》§2.5 / §3.5 与相关执行方案。
