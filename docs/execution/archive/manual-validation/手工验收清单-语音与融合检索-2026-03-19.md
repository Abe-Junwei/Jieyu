# 手工验收清单（语音与融合检索）

## A. 语音链路

1. 打开应用后启用语音输入
2. 说“播放/暂停”，确认执行成功
3. 开启安全模式后说破坏性指令（如删除），确认必须二次确认
4. 切换 mode（command/dictation/analysis）并确认 UI 状态同步
5. 在 dictation 模式输入一段文本，确认内容写入目标位置

验收标准：
- 执行结果与 UI 反馈一致
- 未出现无确认的破坏性执行
- 模式切换无卡死/错乱

## B. 融合检索链路

1. 使用同一查询，在 qa 场景执行检索
2. 使用同一查询，在 terminology 场景执行检索
3. 对比两次结果排序，确认场景差异可见
4. 在 qa 场景下手动设置 keywordWeight，确认可覆盖模板

验收标准：
- 场景切换后排序变化符合预期
- 手动参数优先级高于模板参数
- 无报错与空结果异常

## C. PDF/RAG 链路

1. 触发 PDF 相关检索或引用
2. 点击引用跳转，确认可打开对应预览
3. 验证页码或片段定位逻辑可用（至少一次）
4. 验证失败时的降级提示（如不可用模型/源）存在

验收标准：
- 引用可用且可定位
- 失败路径可解释

## D. 任务调度链路

1. 启动一个可观测的 AI 任务
2. 检查状态流转（pending/running/done 或 failed）
3. 人为制造失败后尝试 retry
4. 验证超时任务的错误提示可读

验收标准：
- 状态更新及时
- retry 生效
- 超时提示清晰

## E. 回归基线

1. 运行 typecheck
2. 运行全量测试

验收标准：
- typecheck 通过
- 全量测试通过
