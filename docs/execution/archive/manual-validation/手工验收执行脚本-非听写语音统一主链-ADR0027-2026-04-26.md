## 用例（与 rollout「QA 抽检」及观测指标对齐）

### CASE 01 — 听写不误触 action（dictation）

- 步骤：转写页打开语音助手，切到 **听写**，口述一句包含「删除」「播放」等指令词的长句（勿真下删除命令）。
- 预期：文本进入听写目标层/区；**不**出现删除句段、播放等 UI action；无错误 toast。

### CASE 02 — command：闲聊进 AI；工具确认与文字一致

- 步骤：切 **指令** 模式，说一句明显非快捷指令的长句（如闲聊或复杂分析请求）；若触发 AI 工具，完成与文字输入相同的确认/拒绝流程各一次。
- 预期：走 AI 聊天主链；`pendingConfirm`、审计与文字路径一致（无「语音旁路」）。

### CASE 03 — 本地 action 快路径（不经 LLM）

- 步骤：**指令** 模式下说「播放 / 暂停」或等价快捷说法（与当前语言路由一致）。
- 预期：低时延触发播放控制；**不**为该类意图误开一轮完整 LLM 聊天（可与网络面板或 mock 对照）。

### CASE 04 — analysis 模式进聊天

- 步骤：切 **分析**，口述一句分析请求。
- 预期：进入与文字一致的 AI 发送路径（`sendToAiChat` / 主链行为与当前实现一致）。

### CASE 05 — 安全模式与破坏性确认（可选）

- 步骤：打开语音 **安全模式**，触发需确认的破坏性意图（若环境允许），确认与取消各一次。
- 预期：`pendingConfirm` 展示；取消不执行；确认后执行且行为数据带 `inputModality: 'voice'`（与 [`voice-unified-chat-path-rollout.md`](../../release-gates/voice-unified-chat-path-rollout.md) 一致）。

### CASE 06 — 用户指令停用（控制台 / 仅 ledger）

- 步骤：通过自然语言或控制台写入一条 **长期** 偏好（如响应风格或工具偏好），确认系统提示或行为生效后，**仅通过停用**该条指令（无对应 `sessionDirectives` 行时亦应生效）。
- 预期：停用后偏好从 prompt/行为中消失；ledger 条目标为 `superseded`。（自动化见 `useAiChat.test.tsx` 中 `deactivateSessionDirective integration`。）
