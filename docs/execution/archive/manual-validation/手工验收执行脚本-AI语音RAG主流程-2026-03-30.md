# 手工验收执行脚本：AI / 语音 / RAG 主流程（2026-03-30）

> 状态：当前主路径
>
> 说明：本脚本用于验收当前转写主工作台中的 AI、语音、RAG/PDF 与任务状态可见性闭环。它替代早期“语音与融合检索”清单中的散点核查方式，强调当前主流程是否可解释、可确认、可写回、可回退。

## 适用范围
- 目标功能：转写主工作台中的 AI 助手、语音助手、RAG/PDF 引用、任务状态与结果写回
- 对应记录文件：`docs/execution/archive/manual-validation/手工验收执行记录-AI语音RAG主流程-2026-03-30.md`
- 建议执行顺序：先自动化基线，再执行手工 10 条主路径用例

## 执行前准备
1. 拉起自动化基线
```bash
npm run typecheck
npm run build
npm test
```
2. 准备数据
- 数据集 A：含正常转写、至少 1 个翻译缺口、至少 1 个低置信句段
- 数据集 B：含 PDF 引用源与可定位页码/片段
- 数据集 C：可触发高风险 AI tool call 的工程（如删除句段 / 删除层）
3. 验收环境
- 桌面浏览器
- 可用麦克风
- 至少一个可用 AI provider 或 mock provider

## 用例执行脚本

### CASE 01 Assistant 状态条与 attention 联动
- 步骤：
  - 打开转写页面并观察 AI 侧栏顶部状态条
  - 切换 assistant / analysis tab，确认状态条始终可见
  - 在存在翻译缺口或 AI 风险的选中项上，确认 assistant tab badge 递增且状态条文案更新
- 预期：
  - 状态条显示“待命 / 建议处理当前选中 / 任务进行中 / 待确认操作”之一
  - assistant attention 数量与当前风险信号一致
- 证据：顶部状态条截图 + tab badge 截图

### CASE 02 语音 command 主路径
- 步骤：
  - 开启语音输入
  - 说“播放”与“暂停”
  - 观察语音状态、播放器动作与状态摘要
- 预期：
  - 语音状态与执行动作一致
  - 无幽灵执行、无模式错乱
- 证据：录屏

### CASE 03 语音 dictation 写入与自动跳转
- 步骤：
  - 切换到 dictation 模式
  - 对当前句段或当前层执行听写
  - 若配置为自动前进，确认选中跳转到下一个句段
- 预期：
  - 文本写入目标正确
  - 写入反馈明确
  - 自动前进不丢焦点
- 证据：录屏 + 写入前后截图

### CASE 04 高风险 AI tool call 待确认闭环
- 步骤：
  - 通过 AI 对话触发删除类操作
  - 确认 assistant tab 自动抢回焦点
  - 在状态条与告警区查看待确认信息
  - 先取消一次，再重新触发并确认一次
- 预期：
  - assistant tab 自动切回
  - 待确认操作、影响范围、确认/取消入口清晰
  - 取消与确认后的 UI 反馈一致
- 证据：录屏 + 待确认状态截图

### CASE 05 AI task / clarify 状态可见性
- 步骤：
  - 输入一个会触发澄清的问题或不完整操作
  - 观察顶部状态条、聊天卡和候选项区
  - 完成澄清后继续执行
- 预期：
  - 状态条显示“任务进行中”或“等待澄清”
  - 候选项与最终执行结果一致
- 证据：录屏 + 状态条截图

### CASE 06 分析模式写回反馈
- 步骤：
  - 切换到 analysis 模式
  - 对当前句段发起语音分析
  - 等待结果写回句段备注
  - 再模拟一次失败路径（如目标缺失 / provider 不可用）
- 预期：
  - 成功时明确提示“已写回”
  - 失败时状态摘要与错误提示一致
- 证据：成功与失败各 1 份截图或录屏

### CASE 07 RAG / PDF 引用跳转主路径
- 步骤：
  - 发起带引用的 AI 问答
  - 点击 PDF citation
  - 验证预览窗口打开，页码或搜索片段定位生效
- 预期：
  - 引用跳转可用
  - 页码或片段定位至少命中一次
- 证据：问答截图 + PDF 预览截图

### CASE 08 RAG / PDF 降级路径
- 步骤：
  - 模拟引用源不可用、provider 不可用或无结果场景
  - 观察侧栏与聊天卡反馈
- 预期：
  - 失败原因可解释
  - 无静默失败
- 证据：错误提示截图

### CASE 09 决策日志与指标条可见性
- 步骤：
  - 连续执行 2-3 次 AI 请求，其中至少一次工具调用
  - 观察聊天卡中的 metrics bar 与 decision logs 区
- 预期：
  - 轮次、成功/失败/澄清等指标有更新
  - 最近决策日志可展开查看
- 证据：指标区与日志区截图

### CASE 10 发布前回归封账
- 步骤：
  - 再次执行 `npm run typecheck`
  - 再次执行 `npm run build`
  - 再次执行 `npm test`
- 预期：
  - 三项全部通过
- 证据：命令行截图
