## CASE 01：ProjectSetupDialog 两字符草稿不提前推断

目标：确认项目创建入口在输入两字符 ISO 时，不会过早回填错误语言名。

步骤：

1. 打开 [src/components/ProjectSetupDialog.tsx](../../../../src/components/ProjectSetupDialog.tsx) 对应的新建项目弹窗。
2. 在“目标语言”代码框中输入 `en`。
3. 不要失焦，先观察语言名称框。
4. 再让代码框失焦。

通过标准：

1. 输入阶段代码框保持 `en`，语言名称不应提前变成“英语”。
2. 失焦后才规范化为 `eng` 并回填正确语言名。

## CASE 02：LayerActionPopover 规范码可继续回删

目标：确认层动作弹窗中，已规范化代码可以继续删除，不会出现删不动或回弹。

步骤：

1. 打开 [src/components/LayerActionPopover.tsx](../../../../src/components/LayerActionPopover.tsx) 的“新建转写层”或“新建翻译层”弹窗。
2. 在语言代码框中输入 `eng`，确认语言名回填为英语。
3. 将代码删到 `en`。
4. 继续删除到空字符串。

通过标准：

1. 从 `eng` 删除到 `en` 时，输入框不能自动跳回 `eng`。
2. 可以继续删除到空值。
3. 删除完成后，语言名与高级标签一并清空，不保留脏值。
