# 代码评审检查清单（术语与边界）

## P0：必须项
- 是否引入了新的 tier 概念到业务域（hooks/components/pages）？若是，必须退回桥接层处理。
- TierDefinitionDocType 导入是否仅出现在允许文件（由 check:tier-boundary-imports 门禁保障）？
- 是否在非白名单业务文件中新增了 segmentation 真表 direct access（`layer_segments` / `layer_segment_contents` / `segment_links` / `layer_units` / `layer_unit_contents` / `unit_relations`）？若是，必须退回 storage-layer helper；由 check:segmentation-storage-boundary 门禁保障。
- 新增业务字段命名是否使用 layerId，而非 tierId？
- 是否在业务域新增了 legacy 字段直接访问（`xxx.tierId`）？若是，必须改为 LayerIdBridgeService helper。
- 是否新增了独立 segment 编辑入口（创建/拆分/合并/删除）？若是，必须退回为 utterance 主轴编辑或挂实验开关。
- 是否执行 `npm run check:architecture-guard`，且没有新增 hard failure？
- 若本次改动命中 hotspot 文件，是否已在同一 PR 内拆分，或已更新 [docs/execution/架构热点台账.md](docs/execution/%E6%9E%B6%E6%9E%84%E7%83%AD%E7%82%B9%E5%8F%B0%E8%B4%A6.md) 并写明回收计划？

## P1：建议项
- 与层编辑相关逻辑是否优先复用 LayerConstraintService / LayerTierUnifiedService？
- 是否在注释或命名中清楚区分 track（模式）与 lane（UI 行）？
- 若本次改动涉及转写首屏、语音、PDF、AI runtime，是否执行 `npm run build:guard` 并检查关键 chunk / CSS 预算？
- 若本次改动属于转写页收尾范围，是否同步核对 [docs/execution/转写页收尾门禁映射-2026-03-30.md](docs/execution/%E8%BD%AC%E5%86%99%E9%A1%B5%E6%94%B6%E5%B0%BE%E9%97%A8%E7%A6%81%E6%98%A0%E5%B0%84-2026-03-30.md) 中对应执行包的自动化门禁、手工用例与封账要求？

## P2：扩散巡检
- 每季度执行一次 tierId 扩散巡检。
- 已批准互操作特例：`useImportExport`、`useTranscriptionSnapshotLoader`（仅用于外部格式兼容）。
- 巡检命令：
  - `npm run report:tierid-diffusion`
  - `npm run check:segmentation-storage-boundary`
- 严格模式（可用于 CI 候选）：
  - `node scripts/report-tierid-diffusion.mjs --strict --max-off-boundary=0`
- 判定规则：
  - 新增 tierId 出现在非桥接/互操作文件 > 0 个，触发专项 review。
  - 专项 review 需给出回收计划（迁移为 layerId 或收敛到桥接层）。
