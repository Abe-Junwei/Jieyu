# 架构热点台账

用于跟踪 `npm run report:architecture-hotspots` 报出的高风险文件。

规则：

1. 任何文件进入 hotspot 区（默认达到硬上限的 85%），同一 PR 内应优先拆分或降复杂度。
2. 若无法在同一 PR 内处理，必须把文件登记到本台账，并给出回收计划。
3. 已登记的热点文件，后续 PR 默认不得继续无上限叠加功能；除非同步完成一次结构收口。
4. hotspot 指标下降后，需同步更新本台账；完全脱离 hotspot 区后可标记为 `已回收`。

## 更新方式

每次运行 `npm run report:architecture-hotspots` 后：

1. 若新增 hotspot，追加一行。
2. 若指标下降，更新“当前指标”。
3. 若已完成拆分并退出 hotspot 区，将“状态”改为 `已回收`。
