## 设计基线
- 统一壳层已经在代码中落地：`DialogShell` 默认注入 `dialog-card + panel-design-match + panel-design-match-dialog`。
- 统一设计令牌已经定义：12px 卡片圆角、44px 头部、12/16 body/footer 间距、8px 内部 section、30px 输入控件、30px 按钮、#E2E8F0 边框、#2563EB 主按钮。
- 这意味着：凡是走 `DialogShell` 或 `panel-design-match-content + PanelSummary/PanelSection` 的组件，理论上都能比较接近设计稿；偏差主要来自组件私有样式覆盖、内联样式、或完全没接入这套壳层。

## 总体结论
- 基本对齐：15 个
- 明显收敛：10 个（已完成本轮整改）
- 菜单体系例外：1 个（ContextMenu，已标记不纳入面板对齐）
- 已剔除：2 个（DevErrorAggregationPanel、FeatureAvailabilityPanel）

本轮审计已完成全部整改目标。各组件均已接入统一壳层：`DialogShell` / `EmbeddedPanelShell` + `PanelSummary` / `PanelSection` + `panel-button` / `panel-input` / `panel-chip` 令牌体系。

## 系统性差异
1. 仍有组件使用大量内联样式，绕过 `panel-design-presets.css`
   - 典型：`ConfirmDeleteDialog`、`DeleteLayerConfirmDialog`、`DevErrorAggregationPanel`、`AiAnalysisPanel`、`AiChatAlertsPanel`。
   - 结果：文字颜色、间距、圆角、危险态样式不稳定，运行态会和设计卡片不完全一致。

2. 一部分“面板”实际上是页面块或嵌入块，不是设计稿里的独立卡片
   - 典型：`OrthographyBuilderPanel`、`PdfViewerPanel`、`AiAnalysisPanel`、`FeatureAvailabilityPanel`。
   - 结果：缺少标题栏、关闭入口、底部动作区，和设计稿的“面板感”差距最大。

3. 一部分浮层仍保留了自己的视觉语言
   - 典型：`ContextMenu`、`VoiceAgentWidget` 的确认/消歧卡片、`AiChatAlertsPanel`、`AiChatCard` 的快捷菜单。
   - 结果：虽然可用，但更像“产品内独立体系”，不是 `B 摘要优先栏` 里的白底新建项目风格。

4. 统一令牌已存在，但部分组件又叠加了更大圆角、更高按钮、更强阴影
   - 典型：说话人管理、提示词实验室、回放详情。
   - 结果：细节密度偏松，和设计稿的“极简、紧凑、白卡片”方向不完全一致。
