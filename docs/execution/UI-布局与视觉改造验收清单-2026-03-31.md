# UI 布局与视觉改造验收清单（2026-03-31）

## 范围
- App Shell：左侧活动栏 + 次级侧边栏 + 顶栏
- 主工作区：转写页固定区与普通页面内容区
- 组件皮肤：按钮、侧栏导航、快捷按钮、弹层层级
- AI 区：聊天区与语音状态条的视觉统一

## 一、视觉验收
- [ ] 左侧活动栏与次级侧边栏视觉一致（边框、圆角、阴影、背景）
- [ ] 顶栏与侧栏为同一主题语法（浅色与深色均一致）
- [ ] 主内容区与侧栏有清晰层级，不出现“拼贴感”
- [ ] 当前功能区激活态显著（颜色、边框、指示条）
- [ ] 最近访问与分组区域视觉弱于主导航，不抢焦点
- [ ] 所有按钮（主按钮/幽灵按钮/危险按钮）状态一致

## 二、交互验收
- [ ] 左侧活动栏点击可正确切换一级路由
- [ ] 次级侧边栏分组可折叠/展开，状态稳定
- [ ] 当前路由在侧栏中准确显示“当前”状态
- [ ] 最近访问按最近路径更新，最多保留 4 项
- [ ] 搜索按钮可触发全局搜索入口
- [ ] 快捷键按钮可打开快捷键面板
- [ ] 焦点态清晰可见（键盘 Tab 可识别）

## 三、响应式验收
- [ ] >=1024px：活动栏 + 次级侧边栏同时显示
- [ ] <1024px：次级侧边栏自动隐藏，仅保留活动栏
- [ ] <640px：活动栏标签隐藏，仅保留图标
- [ ] <640px：顶栏按钮热区仍可点击，不重叠
- [ ] 转写页固定工作区在各断点不被顶栏遮挡

## 四、可访问性验收
- [ ] 侧栏按钮均有可读文本或 aria-label
- [ ] 对比度满足基础可读性（正文、次文本、按钮）
- [ ] 折叠按钮 aria-expanded 状态正确
- [ ] 关键操作支持键盘触达（Tab/Enter）

## 五、工程与回归验收
- [ ] npm run build 通过
- [ ] npm run typecheck 通过
- [ ] 核心回归：转写、标注、分析、写作、词典页面可正常进入
- [ ] AI 侧栏消息区可正常滚动、操作按钮可点击
- [ ] 模态/弹层不会被右侧面板遮挡（z-index 验证）

## 六、风险观察点
- [ ] 顶栏高度变化时，转写主区偏移仍准确
- [ ] 最近访问 localStorage 异常时不会影响页面渲染
- [ ] 深色主题下侧栏 hover/active 不丢失层级
- [ ] 小屏滚动时无“横向抖动”或内容裁切
