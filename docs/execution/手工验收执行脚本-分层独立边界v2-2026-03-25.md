## 用例执行脚本

### CASE 01 开关关闭：读取旧表回退
- 前置：`segmentBoundaryV2Enabled=false`
- 步骤：
  - 打开历史工程（仅旧表有数据）
  - 进入转写页面，观察翻译层文本
- 预期：翻译文本正常展示，无空白回归
- 证据：录屏 + 翻译层截图

### CASE 02 开关开启：读取 v2 优先
- 前置：`segmentBoundaryV2Enabled=true`，准备同 id 的旧表/v2 冲突数据
- 步骤：
  - 打开页面并定位冲突 utterance
  - 对比显示文本是否来自 v2 版本
- 预期：同 id 场景优先显示 v2 内容
- 证据：冲突样例截图（含数据对照）

### CASE 03 新建翻译内容双写一致
- 前置：开启 v2
- 步骤：
  - 在任意翻译层新建文本
  - 触发保存后检查 DB（`utterance_texts` 与 v2 三表）
- 预期：`utterance_texts` 新行存在；`layer_segments/layer_segment_contents` 同步落库
- 证据：DB 面板截图 + 操作录屏

### CASE 04 编辑翻译内容双写一致
- 前置：开启 v2，已有翻译内容
- 步骤：
  - 连续两次编辑同一条翻译
  - 刷新页面复查
- 预期：新内容一致，刷新后不回退旧值
- 证据：编辑前后对比截图

### CASE 05 删除单条翻译内容级联
- 前置：开启 v2
- 步骤：
  - 删除某条翻译文本
  - 检查 `layer_segment_contents` 对应内容是否删除
- 额外检查：确认 `layer_segments` 不存在孤立 segment（该 segment 无任何 `layer_segment_contents.segmentId` 引用）
- 预期：content 删除；若 segment 不再被任何 content 引用，则应被清理
- 证据：删除操作录屏 + 删除后 DB 截图

### CASE 06 删除 utterance 级联清理
- 前置：开启 v2，目标 utterance 有内容与链接
- 步骤：
  - 删除该 utterance
  - 检查 `layer_segments/layer_segment_contents/segment_links`
- 预期：三者对应图谱清理完整，无孤儿残留
- 证据：删除前后 DB 对比截图

### CASE 07 删除 layer 级联清理
- 前置：开启 v2，多层数据已存在
- 步骤：
  - 删除某翻译层
  - 检查该 layer 相关 v2 数据
- 预期：该 layer 下 segment/content/link 清理完整
- 证据：层删除录屏 + DB 对比

### CASE 08 导入数据后 v2 可读
- 前置：准备导入包（含翻译层内容）
- 步骤：
  - 关闭 `segmentBoundaryV2Enabled`
  - 执行导入
  - 开启 `segmentBoundaryV2Enabled`
  - 打开页面检查翻译层显示与快照
- 预期：关闭 v2 导入后再开启，UI 仍可正常读取，关键数量与快照一致
- 证据：导入日志截图 + 页面截图

### CASE 09 恢复流程写入 v2
- 前置：准备恢复快照
- 步骤：
  - 触发恢复
  - 检查恢复内容是否同步进入 v2
- 预期：恢复后 v2 读路径能取到最新文本
- 证据：恢复操作录屏 + DB 截图

### CASE 10 RAG 语料命中 v2 文本
- 前置：开启 v2，准备可检索关键词
- 步骤：
  - 在 AI 对话中触发 RAG 查询
  - 对照返回 snippet
- 预期：返回片段命中 v2 最新文本
- 证据：聊天窗口截图（关键词与命中片段）

### CASE 11 Embedding 检索命中 v2 文本
- 前置：完成嵌入构建或可直接检索
- 步骤：
  - 执行检索查询
  - 对照排序结果与文本来源
- 预期：排序结果包含 v2 最新文本，不回退旧值
- 证据：检索结果截图 + 对照说明

### CASE 12 升级迁移后快照一致性
- 前置：准备升级前样本库
- 步骤：
  - 执行升级链路
  - 导出升级后快照并与基线脚本结果比对
- 预期：关键集合数量与核心字段一致（允许 importer provenance 差异）
- 证据：比对结果截图 + 命令行输出
