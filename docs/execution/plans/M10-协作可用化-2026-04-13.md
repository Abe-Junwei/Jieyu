# M10 协作可用化（2026-04-13）

里程碑：M10（W19-W20，协作 Beta）  
目标：扩展到多层对象编辑，加入协作性能优化与用户提示，验证连续编辑稳定性。

## 1. 里程碑范围

1. 多层对象编辑
- 在同一批次处理多 layer、多 object 编辑操作。
- 冲突发生时保留仲裁入口与日志。

2. 协作性能优化
- 提供批量处理分块计划（chunk），按帧预算控制批次。
- 输出 P95 估算与预算达标判定。

3. 用户提示（谁改了什么）
- 生成协作者提示：按人、按层、按对象计数和最近编辑时间聚合。

4. 连续编辑稳定性
- 对连续快照做结构完整性 + 一致性检查。
- 输出稳定性结论与摘要 digest。

## 2. 核心契约

1. 多层编辑契约
- 批次编辑结果确定性可复现。

2. 性能计划契约
- 分块方案具备可解释的 chunk 结果与预算判定。

3. 协作者提示契约
- 同 actor+layer 的提示需聚合且按最近编辑时间排序。

4. 连续编辑稳定性契约
- 稳定快照输出 stable=true。
- 结构损坏快照输出 stable=false。

## 3. 风险与回滚

1. 风险
- 批处理冲突密集时，仲裁请求可能激增。
- 用户提示聚合不完整会影响可解释性。

2. 回滚动作
- 降级为单层批处理。
- 关闭自动提示，仅保留原始操作日志。

3. 回滚验证
1. npm run gate:m10-collaboration
2. npm run gate:m10-collaboration:shadow
3. 确认 M10 报告恢复到 go
