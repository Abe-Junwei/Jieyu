## 1. 背景与目标

### 1.1 已收敛（不再重复实现）

- **read model**：[`buildTimelineReadModel`](../../../src/pages/timelineReadModel.ts) / `TimelineReadModel` 为语义快照聚合。
- **视口写者**：[`useTimelineViewport`](../../../src/hooks/useTimelineViewport.ts) 包装 [`useZoom`](../../../src/hooks/useZoom.ts)；[`useTranscriptionWaveformBridgeController`](../../../src/pages/useTranscriptionWaveformBridgeController.ts) 消费投影。

### 1.2 仍分叉、本方案要落地的块

| 域 | 问题概要 |
| --- | --- |
| 时间跨度 | `player.duration`、`logicalTimelineDurationSec`、`layout width` 等仍可能多源。 |
| 视口几何 | 像素↔秒、`scrollLeft` 与 WS scroll 同步分散；`useLasso` 等自算组合。 |
| 壳 chrome | `text-only` / `pending_decode` 等与声学相关 class 在多处拼装。 |
| 时间范围交互 | `lassoRect`、`waveLassoRect`、子选区 DOM、`RegionsPlugin` 拖选等多条反馈链。 |
| 改时交互 | `dragPreview` + `snapGuide` 与上述预览平行。 |
| 选集与空壳 | 选集写路径多入口；空状态 CTA 分散（可后阶段）。 |

### 1.3 目标形态（落地后）

```text

  └─ timelineExtentSec（单源可编辑时间轴跨度，只读派生）
  └─ acoustic（已有；为壳 chrome 提供输入；与壳判定共用 layer 口径输入）
  └─ selectionProjection（只读；阶段 F 末起完整）

  └─ projection（zoomPxPerSec、rulerView、…）

  └─ timeRangePreview（拖建 / 框选 / 子选区统一）
  └─ timingEditPreview（拖边 + snap，与 dragPreview 合并）

mapAcousticToTimelineChrome(acoustic)（新纯函数）
  └─ OrchestratorWaveformContent + 横向壳只消费映射结果
```

## 2. 全体阶段共用原则与门禁

### 2.1 原则

1. **读/写分离**：read model 只增加 **只读派生**；可变标量仍只经 **视口写者**（及本方案新增的 **交互写者**）更新。
2. **声学边界**：插件只输出窄事实（时长、解码态、seek、可选波形 scroll）；**不**承载套索/拖建/选区状态。
3. **壳不判业务**：[`TranscriptionTimelineWorkspaceHost`](../../../src/pages/TranscriptionTimelineWorkspaceHost.tsx)、[`OrchestratorWaveformContent`](../../../src/pages/OrchestratorWaveformContent.tsx) **禁止**新增 `if (textOnly)` 式业务分支；只消费投影与 chrome 映射。
4. **每阶段必有**：[`TIMELINE_PARITY_MATRIX_VERSION`](../../../src/pages/timelineParityMatrix.ts) 升版或新增行 + 定向测试 + [`TranscriptionPage.structure.test.ts`](../../../src/pages/TranscriptionPage.structure.test.ts) 门禁之一。

### 2.2 每 PR 必带清单（防回潮）

- [ ] 更新 [`TIMELINE_PARITY_MATRIX_VERSION`](../../../src/pages/timelineParityMatrix.ts) 或新增矩阵行 id；[`timelineParityMatrix.test.ts`](../../../src/pages/timelineParityMatrix.test.ts) 锚点存在。
- [ ] [`TranscriptionPage.structure.test.ts`](../../../src/pages/TranscriptionPage.structure.test.ts) 至少一条与「单源 / 单映射 / 单 writer」相关的结构门禁。
- [ ] CI 或结构测试禁止新增裸 `shell === 'text-only'` **业务**分支（chrome 映射除外）。

## 3. 前置硬化与数据契约

本节为评审中确认的 **P1–P2 改进项**，与 §4 各阶段 **正交**：可单独 PR 交付，但在 **阶段 C（chrome 单映射）** 与 **依赖 `timelineReadModel.acoustic` 的 orchestrator 策略** 之前，**§3.1 建议优先合入**，避免壳与能力位继续分裂。

**本轮已确认结论（执行口径）**

- **结论 A（针对壳层驱动语义）**：当前实现中 `TimelineReadModel` 使用 unit 数量参与壳层判定，属于收敛过程中的实现分叉，**不视为刻意定义“以 unit 存在性驱动 shell”**。落地目标以 layer 维度单源为准。
- **结论 B（针对宿主 link 语义）**：业务与数据契约允许一个翻译层存在多条 host link（含历史脏数据与导入场景）；因此 **§3.2 为必须修复项（required）**，不是可选优化。

### 3.1 壳层判定与 `layersCount` 单源（P1）

**问题**：[`resolveTimelineShellMode`](../../../src/utils/timelineShellMode.ts) 的 `layersCount` 输入不一致——[`buildTimelineReadModel`](../../../src/pages/timelineReadModel.ts) 使用 `unitIndex.totalCount`（unit 数量），[`useTranscriptionTimelineContentViewModel`](../../../src/pages/useTranscriptionTimelineContentViewModel.ts) 使用 `effectiveLayersCount`（与 transcription/translation 列表长度取 max）。在 **有层但尚无 unit** 时，视图可能已是 `text-only`，而 ReadModel 仍可能判 `empty` / `acoustic.state === 'no_media'`，与 [`TranscriptionPage.ReadyWorkspace.tsx`](../../../src/pages/TranscriptionPage.ReadyWorkspace.tsx) 等处用 ReadModel 驱动 `playableAcoustic` 等策略 **分裂**。

**方向**：壳层判定 **单一来源**；**以 layer 维度为主**（与 `effectiveLayersCount` 语义对齐），`buildTimelineReadModel` 与 Timeline 内容 VM **共享同一套输入**（例如从 ReadyWorkspace 只计算一次 `timelineShellLayersCount` 传入二者，或在 read model 构建处统一使用与 VM 相同的派生函数）。

**执行约束**：在该项落地前，禁止新增“基于 `unitIndex.totalCount` 推断 shell 并驱动 toolbar 能力位”的新路径；新增能力门控优先消费与 Timeline 内容 VM 同口径的壳层输入。

**验收**

- 单测或结构测试：`totalCount === 0` 且层列表非空时，ReadModel 的 `acoustic.shell` / `acousticPending` 与 VM 的 `workspaceShell` / `workspaceAcousticPending` **一致**。
- 矩阵行（建议 id：`timeline-shell-layers-count-single-source`）或现有行说明升版。

---

### 3.2 宿主层 link 全量择优（P1）

**问题**：[`resolveSegmentTimelineSourceLayer`](../../../src/hooks/useLayerSegments.ts) 仅解析 `isPreferred ?? links[0]`，**不**在 preferred 失效时尝试其余 link。[`listSegmentTimelineUnitsForLayer`](../../../src/utils/timelineLaneSegmentIteration.ts) 在解析失败时回落 `fallbackUnits`，易出现「轨还在、segment 语义错位」。

**方向**：对同一 `layerId` 的 links：**preferred first**，再按稳定顺序遍历其余；命中 **第一个可解析且宿主满足 `layerUsesOwnSegments`** 的宿主即返回。

**执行约束（required）**：鉴于一翻译层多 host link 为合法状态，本项必须在阶段 D 大规模交互改动前合入，避免把“合法多 link 数据”误退化为 `fallbackUnits`。

**验收**

- 单元测试：preferred 对应宿主缺失或不可解析时，次级 link 可用则 **仍解析到正确 sourceLayer**。

---

### 3.3 独立层 segment 查询并行化（P2）

**问题**：[`useLayerSegments`](../../../src/hooks/useLayerSegments.ts) 对每层 `await listSegmentsByLayerMedia` **串行**，层数上升时切媒体/首屏延迟近似线性累加。

**方向**：`Promise.all` 并行拉取各层，再一次性 `setSegmentsByLayer`；**保留**现有 `loadSequence` 竞态保护。

**验收**

- 行为与数据与串行一致；可选轻量测试 mock 调用顺序或「并发完成后再 commit」断言。

---

### 3.4 segment 宿主解析失败的轻量可观测性（P2）

**问题**：[`listSegmentTimelineUnitsForLayer`](../../../src/utils/timelineLaneSegmentIteration.ts) 在 `sourceLayer` 缺失时 **静默**使用 `fallbackUnits`，线上难以区分「正常无宿主」与 **配置/数据异常**。

**方向**：保留回退；增加 **轻量诊断信号**（任选或组合）：开发环境 `debug` 日志、telemetry 计数、或 **结构化返回值/旁路字段**（便于测试断言），且生产默认低噪。

**验收**

- 至少在 **测试环境** 可断言「解析失败路径被记录一次」；生产路径不刷屏。

## 7. 开放问题（实施前评审）

1. **`timelineExtentSec` 定义**：文献逻辑时长与媒体实际时长同时存在时的优先级（产品拍板后写入 `timelineReadModel` 派生函数注释）。
2. **`layersCount` 精确定义**（与 §3.1 联动）：「层」是否包含未持久化占位、仅翻译壳、或过滤规则；写入单一派生函数的注释与矩阵判据。
3. **`pxPerDocSec` vs `zoomPxPerSec`**：是否在投影层长期并存两名，或 deprecate 其一（需全局替换成本评估）。
4. **RegionsPlugin 是否保留为「命中检测 only」**：与 WaveSurfer 版本升级策略绑定，需在阶段 D 子 PR 中单独立项。
5. **纵向壳**：[`TranscriptionTimelineVerticalView`](../../../src/components/TranscriptionTimelineVerticalView.tsx) 接入 **D** 的时间与 preview 合同的时间点（矩阵 `vertical` 列从 partial → full 的判据）。
