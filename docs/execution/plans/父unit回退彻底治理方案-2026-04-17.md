# 父 unit 回退彻底治理方案

日期：2026-04-17  
适用范围：转写页统一读模型、选择映射、AI 目标解析、自确信度显示与写入路由

---

## 1. 目标与结论

### 1.1 目标

彻底消除“会导致误写或误操作的父 unit 回退”，同时保留“用于边界校验的必要回退”。

### 1.2 一句话结论

- 写路径：禁止隐式父级回退，目标必须显式且可验证。
- 读路径：允许受控回退，但必须可解释、可观测、可拒绝（歧义时不猜测）。
- 时间细分：保留父级边界回退，这是业务约束，不属于遗留兼容。

### 1.3 非目标

- 本期不改 Layer 约束模型（unit / independent-segment / time-subdivision）。
- 本期不重写 TimelineUnitView 读模型。
- 本期不引入新的协同协议字段。

---

## 2. 现状分层审计

### 2.1 必须保留（合理）

1. 时间细分边界回退
- 作用：在 time-subdivision 下确保分裂、合并、创建都不越父级边界。
- 关键位置：
  - src/pages/transcriptionSegmentCreationActions.ts
  - src/pages/useTranscriptionSegmentMutationController.ts
  - src/pages/useTranscriptionSegmentBatchMerge.ts
- 判断：这是领域约束，不是“历史兼容兜底”。

2. 写路径分发器按 kind/editMode 决策
- 关键位置：src/pages/timelineUnitMutationDispatch.ts
- 判断：该路径已是“显式路由”，可继续作为唯一写入口。

### 2.2 必须收敛（高风险）

1. AI owner 推断回退直接参与目标解析
- 关键位置：
  - src/pages/transcriptionAiSelectionResolver.ts
  - src/pages/transcriptionSelectionOwnerResolver.ts
- 风险：在 parentUnitId 缺失且时间重叠场景，推断 owner 可能错选，导致 AI 工具写错对象。

2. 自确信度读路径多候选合并无“歧义拒绝”
- 关键位置：src/pages/useTranscriptionSelfCertaintyController.ts
- 风险：多候选 host 合并后给出确定展示，用户无法判断来源是否可信。

### 2.3 需要守卫（中风险）

1. 选择映射 segment -> parent unit 在多类动作复用
- 关键位置：
  - src/pages/selectionIdResolvers.ts
  - src/pages/useBatchOperationController.ts
  - src/pages/useSpeakerActionScopeController.ts
- 风险：未来若 segment 专属动作误接入 unit 映射，会发生语义漂移。

2. 读模型 semantic shadow（segment 覆盖 parent 语义 key）
- 关键位置：src/hooks/timelineUnitView.ts
- 风险：若入口不带 kind 守卫，调用方容易误把展示行当写目标。

## 3. 目标架构与硬规则

## 3.1 统一规则（硬约束）

1. 写入目标规则
- 任何写操作必须持有“显式目标类型 + 显式目标 id”。
- 禁止通过时间重叠/最近中心点去推断写入目标。

2. 回退可用域
- 仅允许在以下场景使用回退：
  - 时间细分边界验证
  - 只读展示辅助（且可标注不确定）

3. 歧义处理规则
- 当候选父级数量 != 1 且无显式 parentUnitId 时：
  - 写路径：直接失败并提示用户选择。
  - 读路径：返回 ambiguous，不输出确定值。

4. API 分层规则
- Unit-only 动作只能调用 unit 映射 API。
- Segment-only 动作只能调用 segment 目标 API。
- 混合动作必须先拆分再执行，不允许共用隐式映射。

## 3.2 禁止清单

- 禁止新增“先猜父级再写”逻辑。
- 禁止在 dispatch 之外新增写路径分发分支。
- 禁止在 AI 工具目标解析中将 fallback owner 作为写目标默认值。

---

## 4. 改造方案（分阶段）

### Phase A（P0）写目标收口

目标：所有写动作不再依赖父级推断。

实施项：

1. AI 目标解析分离读写语义
- 文件：src/pages/transcriptionAiSelectionResolver.ts
- 动作：
  - 保留 resolveOwnerUnitForAi 用于展示/上下文。
  - 新增 resolveWritableAiTargetId：仅接受显式 segment.id 或显式 ownerUnitId。
  - 当仅有 fallback owner 时返回 undefined，并由上层提示“目标不确定，请手动选择”。

2. 调用点改造
- 文件：src/pages/useTranscriptionAiController.ts
- 动作：
  - selectedSegmentTargetId 改为可写目标语义，不再接受推断 owner。
  - 仅在可写目标存在时向 tool handler 注入 selectedSegmentTargetId。

3. 风险检查器同步
- 文件：src/pages/transcriptionAiToolRiskCheck.ts
- 动作：
  - 增加规则：无显式可写目标时，所有写类 AI 工具进入高风险阻断。

验收：
- AI 在“无显式 parent + 多重重叠”场景不可直接写入。

### Phase B（P1）自确信度回退去歧义

目标：读路径可回退但不可“伪确定”。

实施项：

1. 引入歧义态
- 文件：src/pages/useTranscriptionSelfCertaintyController.ts
- 动作：
  - 新增 host 解析结果结构：exact | fallback-single | ambiguous | none。
  - resolveSelfCertaintyForUnit 在 ambiguous 返回 undefined（或独立标记）。

2. UI 提示增强
- 文件：
  - src/hooks/useTimelineAnnotationHelpers.tsx
  - src/components/TranscriptionTimelineTextOnly.tsx
- 动作：
  - 当为 ambiguous 时不展示 certainty 徽章；可显示轻提示“来源不唯一”。

3. 写入保持现状
- 文件：src/pages/useTranscriptionSelfCertaintyController.ts
- 说明：
  - handleSetUnitSelfCertaintyFromMenu 对 kind=segment 继续直写 segment id。
  - 禁止改回“写入父级聚合 id”。

验收：
- 多候选 host 场景不再显示确定 certainty。

### Phase C（P1）选择映射 API 分轨

目标：防止 unit 映射误用于 segment 专属动作。

实施项：

1. 映射 API 拆分
- 文件：src/pages/selectionIdResolvers.ts
- 动作：
  - 保留 resolveMappedUnitIds（仅 unit-only）。
  - 新增 `resolveSegmentOnlyIds`（并导出别名 `resolveSegmentActionIds` 与方案命名对齐；仅 segment-only，禁止 parent 回退）。

2. 调用方硬约束
- 文件：
  - src/pages/useBatchOperationController.ts
  - src/pages/useSpeakerActionScopeController.ts
  - src/pages/useSpeakerActionRoutingController.ts
- 动作：
  - unit-only 批处理继续使用 mapped unit ids。
  - segment-only 入口显式使用 segment ids。
  - 混合选择拆分为两批执行并分别提示结果。

验收：
- segment-only 操作链路不再经过 resolveSpeakerTargetUnitIdFromUnitId。

### Phase D（P2）可观测性与守卫

目标：让回退行为可监控、可审计、可回归。

实施项：

1. 指标与日志
- 建议指标：
  - parent_fallback_attempt_total
  - parent_fallback_ambiguous_total
  - blocked_write_without_explicit_target_total

2. 架构守卫
- scripts/architecture-guard.config.mjs 增加规则：
  - 写入口禁止调用 resolveFallbackOwnerUnit。
  - AI 写目标解析禁止引用 fallback owner 结果。

3. 测试守卫
- 新增结构测试，确保调用关系不回退。

验收：
- PR 中新增隐式回退写逻辑时，CI 直接失败。

## 7. 灰度与回滚

1. 灰度策略
- 先上 P0（阻断误写），观察 48 小时。
- 再上 P1（展示去歧义 + API 分轨）。
- 最后上 P2（守卫与指标强约束）。

2. 回滚策略
- P0 可单独回滚，不影响既有数据结构。
- P1 若 UI 反馈过严，可仅回滚“提示策略”，不回滚“写路径收口”。
- 禁止回滚到“AI 可通过推断 owner 直接写入”的状态。

---

## 8. 完成定义（DoD）

满足以下全部条件才算完成：

1. 写路径中不存在“推断父级后写入”的代码路径。
2. AI 写操作在无显式目标时全部被阻断并有可理解提示。
3. 自确信度多候选场景不再展示确定结果。
4. segment-only 与 unit-only 动作路径分离且被测试覆盖。
5. 架构守卫可阻止新引入的隐式回退写逻辑。

---

## 9. 备注

本方案与“时间细分父边界校验”不冲突，后者属于必须保留的领域规则。治理目标是移除会导致误写的回退，不是消灭一切回退。
