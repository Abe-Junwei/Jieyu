# M11 跨设备协作治理（2026-04-13）

里程碑：M11（W19-W20，跨设备协作）  
目标：在 M10 基础上落地跨设备协作治理（版本向量、时钟漂移治理、回滚策略、一致性校验）。

## 1. 里程碑范围

1. 版本向量并发判定
- 对跨设备副本建立 vector clock 偏序比较。
- 区分 local-dominates / remote-dominates / equal / concurrent。

2. 时钟漂移治理
- 对跨设备更新时间差进行预算评估。
- 超预算触发回滚信号并输出风险冲突码。

3. 回滚策略
- 基于冲突风险输出软回滚或硬回滚计划。
- 高风险并发删除/更新冲突进入 manual-review。

4. 跨设备一致性校验
- 对同步后副本进行一致性摘要校验。
- 保证可检测、可回滚、可追溯。

## 2. 核心契约

1. 版本向量契约
- 并发关系判定结果必须稳定且可复现。

2. 时钟漂移契约
- 漂移超预算必须可检测并输出治理信号。

3. 回滚契约
- 高风险冲突必须输出 rollback plan。

4. 一致性契约
- 同步后摘要一致率需达到 100%。

## 3. 风险与回滚

1. 风险
- 跨设备时钟漂移导致错误覆盖。
- 并发删除/更新冲突导致数据不可恢复。

2. 回滚动作
- 降级为 M10 同设备协作路径。
- 关闭跨设备自动合并，仅保留人工审查。

3. 回滚验证
1. npm run gate:m11-cross-device
2. npm run gate:m11-cross-device:shadow
3. 确认 M11 报告恢复到 go
