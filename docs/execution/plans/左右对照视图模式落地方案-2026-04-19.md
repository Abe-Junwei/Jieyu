# 转写工作区左右对照视图模式落地方案

## 0. 背景与目标

本方案面向转写工作区新增一种“左右对照视图模式”：

- 上方保留**单一统一波形 / 时间轴**；
- 下方左列显示**转写原文**；
- 下方右列显示**翻译文本**；
- 三者在 hover、选中、定位、播放与编辑上保持一致联动。

目标不是替换现有波形模式或文本模式，而是在其上新增一个**清晰、可控、可渐进落地**的新视图。

---

## 1. 核心结论（先定规则，避免后续返工）

### 1.1 统一波形仍是唯一时间真相源

对照模式下：

- **不复制上方波形**；
- 不做“左原文一套波形、右译文一套波形”；
- 所有文本对照都只是映射到同一套时间轴之上的显示层。

### 1.2 新增“对照组”读模型，而不是立刻改底层表结构

当前数据库中的 `unit`、`segment`、`translation layer`、`speaker`、`bundle` 仍保持现状；
对照模式只在读取与显示层引入一个新的 **alignment-group / 对照组** 概念。

### 1.3 编辑权限必须分离

- **时间编辑 / 拖边 / 合并 / 拆分 / 删除语段**：只在左列原文或上方波形做；
- **翻译文本编辑 / 译音播放与录音**：主要在右列做；
- 不允许在视觉上模糊“我改的是时间，还是译文内容”。

### 1.4 多翻译层默认只展开一个目标层

第一版不建议同时展开多个目标翻译层；默认仅查看一个当前翻译层，其余通过切换查看，避免双栏内部再次爆炸。

---

## 2. 非目标（本方案不直接解决）

- 不在第一版中引入新的持久化对齐关系表；
- 不在第一版中同时支持“多个翻译层并排多栏对照”；
- 不强行在窄屏下维持左右双栏；
- 不在第一版中重写现有波形主工作流。

若未来需要长期沉淀“多行原文 ↔ 一条意译”这类人工关系，再单独立项做持久化对齐层。

---

## 3. 适用场景与目标显示语义


| 场景            | 推荐显示方式                  | 备注               |
| ------------- | ----------------------- | ---------------- |
| 1 行转写 ↔ 1 行翻译 | 左右单行对齐                  | 最基础场景            |
| 多行转写 ↔ 1 行翻译  | 左侧多条子行，右侧一张组级译文卡        | 不复制同一翻译到每条原文     |
| 1 行转写 ↔ 多行翻译  | 左侧一条原文，右侧多条子译文          | 允许展开多条译文         |
| 多说话人共用翻译      | 左侧保持说话人主语义，右侧显示“多说话人”摘要 | 不复制整套 speaker 轨  |
| 多个 bundle     | 按 bundle 分段、折叠或组块展示     | 默认只强调当前活跃 bundle |


---

## 4. 结束态设计原则

1. **组优先于行**：对照模式的最小显示单元是“对照组”，不是平铺单行。
2. **左强右轻**：左侧承担时间、说话人和结构编辑主语义；右侧承担译文与译音交互。
3. **波形单源**：任何点击、hover、播放、定位都回落到统一波形和统一时间范围。
4. **状态双层化**：必须同时存在“组级高亮”和“子项级高亮”，不能沿用单语段单焦点模型。
5. **挂件有优先级**：保存状态、note、self-certainty、音频工具、resize 句柄、overlap 提示等不能自由堆叠。

## 6. WP-A — 视图模式与对照组读模型

### 6.1 当前问题（基线）

当前文本模式与波形模式主要按“layer 顺序 + unit / segment 平铺”理解界面，默认假设：

- 当前只有一个主要活动语段；
- 原文与翻译可按层逐行显示；
- 语段挂件都依附于单条卡片。

这些假设在“多行原文 ↔ 一行翻译”或“多个 speaker / bundle 交织”时会立即失效。

### 6.2 结束态行为

新增一个对照组读模型，至少包含：

- `groupId`
- `bundleRootId`
- `startTime` / `endTime`
- `sourceItems[]`（左列原文子项）
- `targetItems[]`（右列译文子项）
- `speakerSummary`
- `primaryAnchorUnitId`
- `primaryAnchorLayerId`
- `editingTargetPolicy`

实现落点需要明确分层：

- **视图模式开关**应落在 ReadyWorkspace 编排链，与 toolbar props、layout controller、workspace props builder 同层；
- `src/hooks/useTranscriptionUIState.ts` 当前主要承载轨显示模式与翻译草稿聚焦引用，**不应**直接承接整个对照模式编排；
- 对照组建模建议抽成独立的 `buildComparisonGroups(...)` / `useTranscriptionComparisonGroups(...)`，再由 ReadyWorkspace 或现有 view model 注入消费。

### 6.3 分组规则

第一版建议按以下优先级构造：

1. 先按 `bundle` 做大分区；
2. 在 bundle 内，按时间重叠与父层关系构成组；
3. 若存在独立翻译层自带 segments，则按其时间范围归并到最近原文组；
4. 若一组无法稳定归属到单个原文锚点，则仍允许存在，但必须标记为“多锚点组”。

需要特别避免把整套“bundle 分区 + 时间重叠归并 + 多锚点组”逻辑直接塞进 `src/pages/useTranscriptionTimelineController.ts`。该 hook 当前更适合消费结果，而不是承担完整对照组建模。

### 6.4 验收检查清单

- 能稳定生成 1 对 1、多对 1、1 对多三类组；
- 同一组的 hover / selected / active 状态可统一表达；
- 对照组读取不依赖“同索引对齐”；
- segment 模式下，`primaryAnchorUnitId` 必须与波形 region 解析规则一致，统一对齐到 `resolveTranscriptionUnitTarget(...)` / `resolveTranscriptionSelectionAnchor(...)` 所使用的语义 id。

## 7. WP-B — 波形联动与选择模型

### 7.1 结束态语义

无论用户点击：

- 上方波形语段；
- 左列某条原文；
- 右列某条译文；

都应回落到：

1. 一个确定的对照组；
2. 一个确定的子项；
3. 一段确定的时间范围。

### 7.2 必须新增的状态层

不能只保留现有的：

- `selectedTimelineUnit`
- `activeUnitId`
- `focusedLayerRowId`
- `flashLayerRowId`

至少还需补充：

- `activeComparisonGroupId`
- `activeComparisonCellId`
- `comparisonTargetSide: 'source' | 'target'`

### 7.3 风险说明

如果不拆成“组 + 子项”双层状态，会直接引发：

- 波形选中和右侧译文高亮不同步；
- 菜单和便签落到错误对象；
- 播放或 AI 跳转回到错误时间点。

### 7.4 验收检查清单

- 点击左、右、波形三处，焦点对象一致；
- 当前高亮组与当前编辑子项不会相互串扰；
- 多选与批量操作仍能明确作用目标。

## 8. WP-C — 语段挂件与样式收口

### 8.1 当前挂件清单（全面受影响）

以下挂件目前都直接挂在单条语段卡片上，进入对照模式后需要重新排布：

- 文本输入框与 placeholder
- 保存状态点与错误重试
- 说话人语义条 / 说话人标签
- 自我确信度角标
- AI confidence 样式
- note 图标
- 译音录音 / 播放 / 删除控件
- resize 句柄
- overlap cycle 提示
- 右键菜单入口
- hover / selected / active / focus-within 状态

### 8.2 重排建议

#### 左列原文

保留：

- 说话人语义
- 时间编辑句柄
- 结构性操作入口
- overlap / 锁轨提示

#### 右列翻译

保留：

- 译文编辑
- 译音控制
- 轻量状态提示
- note 入口

不建议在右列完整复制 speaker 多轨和时间编辑句柄。

### 8.3 样式级风险

当前文本卡片和波形注释卡片中，大量挂件使用绝对定位；当列宽变窄时，最先出问题的不是文本本身，而是：

- 状态点压住 note 图标；
- self-certainty 与音频工具互相遮挡；
- resize 句柄命中范围覆盖输入框；
- hover 态与 active 态表达不清。

### 8.4 验收检查清单

- 挂件在窄列下无明显重叠；
- 输入框文本不被工具栏挤压；
- hover / active / editing 的视觉层级仍清晰；
- 双栏状态下不会把左列和右列样式串色或串状态。

---

## 9. WP-D — 便签、右键菜单、批量操作与 AI 入口

### 9.1 Note / 备注

当前备注目标主要是：

- unit
- tier annotation（`unitId + layerId`）

从产品语义上，对照模式最终需要支持三种目标：

1. **组级备注**：针对整个对照组；
2. **原文子项备注**：针对左列具体单元；
3. **译文子项备注**：针对右列具体单元。

但当前代码中的 `NoteTarget`、`NotePopoverState` 与存储键规则主要偏向 `unit / tier_annotation`。因此：

- **P0**：先复用既有 `unit / tier_annotation` 语义，确保左右列不会误绑目标；
- **P1**：若要落地真正的组级备注，必须同步扩展 `useNotes` 的 `NoteTarget` 建模与存储键规则，而不只是改 UI 文案。

否则弹层标题、数量统计和引用跳转会在实现中期卡住。

### 9.2 右键菜单

需要把动作拆为两类：

- **结构操作**：删除、拆分、合并、改时，仅对子项或左列锚点开放；
- **内容操作**：加备注、改确信度、译文编辑、音频控制，可在右列开放。

### 9.3 批量操作与 AI 工具

当前批量选择和 AI 工具大多默认只围绕单个 `selectedTimelineUnit` 展开。对照模式下，需要先把 AI 与批量操作统一落到：

- 当前活跃组；
- 当前主锚点子项；
- 或明确勾选的一组子项集合。

### 9.4 验收检查清单

- NotePopover 不会落到错误单元；
- 右键菜单中每一项都能说明“操作的是谁”；
- 批量操作和 AI 跳转不会因为组视图而跑偏。

## 10. WP-E — 多说话人、多 bundle 与窄屏策略

### 10.1 多说话人

原则：

- 左列继续保持说话人主语义；
- 右列只显示轻量继承标签或“多说话人”摘要；
- 不在右列再复制一套完整轨道体系。

### 10.2 多 bundle

建议按 bundle 切段：

- 当前 bundle 强调显示；
- 非活跃 bundle 降噪；
- 允许折叠非当前 bundle，减轻视觉负担。

### 10.3 窄屏降级

小屏不应强推左右双栏，建议：

- 自动改为上下堆叠；
- 或单列切换“原文 / 翻译”。

### 10.4 验收检查清单

- 多说话人语义不重复不冲突；
- 多 bundle 下仍能快速识别当前上下文；
- 窄屏模式下仍可正常编辑与查看。

---

## 11. 推荐实施顺序

1. 先做 **WP-A**：新增对照模式开关与对照组读模型；
2. 再做 **WP-B**：打通统一波形与双栏联动；
3. 然后做 **WP-C**：整理所有 segment 挂件与布局；
4. 再做 **WP-D**：收口便签、菜单、批量操作、AI 入口；
5. 最后做 **WP-E / WP-F**：复杂场景收口与回归验证。

这样可以显著降低一次性改动过大导致的回归风险。

## 15. 最终判断

本功能**可做且值得做**，但第一版必须遵守以下收口边界：

1. 不动底层库结构；
2. 不复制上方波形；
3. 不把所有现有挂件原样平移到右列；
4. 先把“组 + 子项”模型设计清楚，再改 UI。

只要按上述顺序实施，整体风险可控，且能在不破坏现有转写工作区的前提下渐进落地。

## 16. 2026-04-19 执行结果（P1 已落地）

本轮已按 P1 目标完成稳定版收口：

- **P1-1 Note 与菜单适配**：新增 `comparison_group` 备注目标，左右对照组可直接打开组级备注，避免操作对象歧义；
