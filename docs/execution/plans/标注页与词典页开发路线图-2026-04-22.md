## 1. 目标与边界

### 1.1 目标

1. 对标 FLEx 的标注与词典核心能力，形成可落地的最小闭环。
2. 打通转写页、标注页、词典页的数据流，避免并行真源。
3. 支持“每语段一行 + 多层可多实例 + 词汇可选择性入词典”。

### 1.2 非目标

1. 不照搬 FLEx 的桌面式配置流程与重发布链路。
2. 不新增第二套状态中心；标注页优先复用现有转写工作台动作与读模型。

### 1.3 约束

1. 当前产品约束为“转写页作为唯一开放工作台”，其他工作台需渐进开放。
2. 标注页首版按照“复用内核、开放视图”的路径推进。

参考：

- [ADR 0002：转写页作为唯一开放工作台](../../adr/0002-转写页作为唯一开放工作台.md)

### 1.4 与语料库（`/corpus`，旧 `/writing` 重定向）一期：防重复边界

### 1.5 三页里程碑联评（标注 / 词典 / 语料）

**目的**：不合并三份路线图正文，但在 **触及共享读模型、导航、AI、导出或深链** 的里程碑合入前，用 **同一张检查表** 做一次 **联评**（可嵌入发布评审 / PR 描述 / 手工验收记录），避免 **§1.4** 防重复边界被无意打穿。

**建议触发条件（任一即做联评）**

- 跨页刷新或状态同步（与 **§4.2.1** 同精神）：标注 ↔ 转写、词典 ↔ 标注、**语料库 ↔ 转写** 等。  
- 语料库 **只读** 展示 `lexemes` / 命中语段摘要 / `token_lexeme_links` 衍生信息。  
- 导航或 i18n 出现「词典」「术语」「语料」「标注」等 **易混标签** 的增删改。  
- AI：**annotation 侧 suggestion 写库** 与 **语料库侧复制出站** 的工具注册、会话 id、上下文注入发生变更。  
- 导出：FLEx / flextext / **LIFT** 等 **词典包** 与 **语料库剪贴板 / HTML / bundle / 标准交换 profile**（见 **[语料库页面开发路线图 · §8.2](./语料库页面开发路线图-2026-04-22.md)**）的入口、文案或 MIME 策略变更。  
- 深链：`unitId` / `layer` / `lexemeId` **query 形状** 或路由在任两页之间调整。  
- **跨页上下文**：会话恢复、排序/滚动/选中保留、语料 **工作集** 与转写选择在往返中是否 **可预测**（见 **R8**）。

## 4. 统一数据流方案（转写 ↔ 标注 ↔ 词典）

说明：本节默认描述目标态设计；除 4.2 / 4.3 中已存在的统一写链路外，其余多数能力尚未在标注页与词典页页面层落地。来自长文/对话的 **IGT、Resolver、Worker、零拷贝** 等名词与**当前代码**的逐项对照、三档吸收口径见 **§10.1**；**NLP/语言学/工具链** 中不同层次的切分、解析、标注**问题域**及与本路线图的**覆盖/缺口**见 **§10.2**，避免与目标态设计混读为「已实现」或「文献层=产品层」。

### 4.1 单一真源（SSOT）

1. 语段层：layer_units / layer_unit_contents。
2. 词法层：unit_tokens / unit_morphemes。
3. 词典层：lexemes / token_lexeme_links。

禁止新增“标注页专属冗余真源表”。

### 4.2 文本双向同步链路

1. 转写页与标注页都只调用统一写动作：saveUnitText / saveUnitLayerText。
2. 统一进入 syncUnitTextToSegmentationV2，更新 segment graph 与读模型。
3. 协作场景由 cloud sync 包装动作出站，远端变更回放后触发本地刷新。

### 4.2.1 跨页同步触发契约（具体）

1. 标注页在语段文本或分层内容保存成功后，触发同一 unit 的转写页读模型刷新。
2. 转写页在语段文本保存成功后，触发同一 unit 的标注页行刷新。
3. 跨页刷新以 unitId 为最小粒度，不做整页全量重拉。
4. 当同一 unit 正在被本地编辑且收到远端回放时，进入冲突票据流，不直接覆盖编辑区。

### 4.3 词法双向同步链路

1. POS / gloss 更新统一调用 updateTokenPos / updateTokenGloss / batchUpdateTokenPosByForm。
2. 标注页与转写页共享同一 canonical token 数据来源与刷新策略。

### 4.4 词汇层入词典链路

1. 标注页词汇层默认自动入词典；用户可在设置面板切换“自动/手动确认”。
2. 入词典时先按“同语言 + 规范化 lemma”检索候选词条。
3. 若未命中词条：创建 lexeme，并创建 token_lexeme_link。
4. 若命中同 lemma 同语言冲突：默认新建 variant，同时弹出冲突提示，允许用户改选“合并到已有词条”或“新建 variant”。
5. 入词典结果实时反映到词典页与转写页 token 关联状态。

### 4.5 词典页读模型链路（首版）

1. 左侧词条列表数据源为 lexeme 集合，按当前过滤条件与排序规则生成可滚动列表。
2. 点击左侧词条后，右侧上方加载该词条详情（lemma、language、variant、词性/释义等）。
3. 同一次点击触发右侧下方“命中语段列表”查询：通过 token_lexeme_link 反查所有包含该词条的 unit。
4. 命中语段列表每项至少包含 unitId、语段文本摘要、语言、来源上下文（媒体或文献标识）。
5. 词条详情与命中语段列表都以选中 lexemeId 为唯一输入，避免页面内并行状态源。

### 4.6 词条图片与语言附件链路（首版）

1. 词条详情支持附件集合：图片附件（image）与语言附件（audio/document），并记录 languageCode 与显示名称。
2. 附件上传后先写入统一资产存储，再将附件元数据写入词条扩展字段，词条仅持有引用而不内嵌二进制。
3. 图片附件支持缩略图预览；语言附件支持基本播放或下载入口。
4. 附件删除采用“词条引用先删除、资产按引用计数回收”的策略，避免误删共享资源。

### 4.7 词典导出链路（FLEx 风格）

1. 导出入口支持三种范围：当前筛选结果、用户选中词条、全量词典。
2. 导出产物为压缩包：词典主体文件（XML 结构）、附件资源目录、manifest 与导出诊断信息。
3. 词典主体至少包含 lemma、language、variant、词性/释义、语段引用与附件引用。
4. 导出时可选“包含附件/不包含附件”；不包含附件时保留可解析引用与缺失说明。
5. 导出日志需记录条目数、附件数、警告条数，供回归与问题定位。
6. 词法缩写默认遵循 Leipzig Glossing Rules 词表约束（如 `1SG`、`ACC`、`NOM`），并提供大小写规范化与校验提示。
7. 导出 profile 在 FLEx/flextext 基础上预留 LaTeX 学术稿路径（`gb4e`/`expex` 片段级导出），避免标注结果成为生态孤岛。
8. **LIFT（Lexicon Interchange Format）** 作为 **可选导出 profile**（M3+ 或独立小里程碑）：服务 **WeSay / Lexique Pro / SIL 工具链** 用户；须 **复用或扩展现有导出服务**，禁止与 FLEx/flextext 管线并行再造一套；具体字段映射在 M0 映射表预留挂钩。

### 4.8 AI 辅助链路（progress-aware）

1. AI 助手调用上下文包含当前阶段信息：当前媒体/文献、当前 unitId、当前 layerType、当前编辑状态、已完成标注比例与未处理队列。
2. 用户点中语段、token、morpheme 或词条时，可一键触发“AI 分析建议”，输出与当前对象强绑定。
3. AI 输出分三类：分析提示（why）、建议操作（what）、可执行 suggestion（how），并标记置信度。
4. 自动标注仅以 suggestion 形式落地，默认不直接写库；用户确认后才调用现有写动作链。
5. 所有 AI 建议与采纳结果记录到可追踪日志（对象 ID、时间、建议版本、是否采纳）。

### 4.9 转写到标注的零拷贝深视图（Zero-copy Deep View）

1. 标注页作为转写数据的深视图，不复制 segment 文本到第二套真源表。
2. 语段首次进入标注态时，按需懒初始化词法投影（word/morpheme/gloss 视图结构），写入仍走既有统一动作链。
3. 转写与标注双向编辑遵循同一最小刷新粒度（默认 unitId），禁止整页重拉掩盖同步问题。
4. 任一侧修改词形或分词后，另一侧在同一会话内可见；若存在未提交草稿，仍按既有冲突票据与草稿保护策略处理。

### 4.10 防御性引用、删除语义与 UI 韧性

本节约定 **引用断裂时不白屏、不假成功**，与 **[语料库页面开发路线图 · §8.1](./语料库页面开发路线图-2026-04-22.md)**「防御性指针与删除语义」行 **同轨验收**。

1. **真源不变**：`lexemeId` / `tokenId` / `unitId` 等仍以 **§4.1 SSOT** 为准；**禁止**为修 UI 单独新增「第二套词典真源表」。
2. **展示层胖指针（Fat pointer 语义）**：在 **UI 状态、剪贴板草稿、IGT 行缓存** 等层，允许同时持有 `targetId` + 人类可读快照（如末次已知 lemma、末次对齐的语段片段），用于 **目标缺失或软删后的降级渲染**；**采纳写库** 仍以 Resolver 能解析的 id 为准，快照 **不得** 反向覆盖服务端真值。
3. **删除路径**：优先 **软删（`deletedAt` / 墓碑）+ 可诊断断裂态**；物理删除仅允许在 **GC 策略** 与 **协同回放** 评估后分域引入（见语料库 **§8.1**「渐进扩面」）。
4. **跨页**：删除 / 软删 / 合并词条后，须经 **§4.2.1** 约定的事件触发词典「命中语段」、标注 IGT、语料只读摘要等处的 **脏刷新**；**禁止**静默仍显示已失效链接。
5. **验收**：M0–M2 须具备 **悬空引用 / 级联删除** 回归用例清单（断裂态 **错误码 + i18n** 可追溯），与 **§6.5** IGT 降级条款对读。

## 5. 标注页信息架构（首版）

说明：本节描述目标信息架构与功能规格，不代表当前代码已实现；当前实现状态以 **§2** 与 **§6.0** 为准。

### 5.1 行模型

1. 语段按语言拆行为“每种语言一行”（来源于当前媒体或文献范围内的有效 unit/segment）。
2. 同一语段行内不允许跨语言混挂。
3. 行内支持多层并列编辑区。

### 5.2 层模型（首版支持）

1. 词汇层（Lexeme Layer）：词形、词条关联。
2. 词性标准层（POS Layer）：词性编辑与批量改写。
3. 形态层（Morphology Layer）：词素拆分与词素 gloss。
4. 标注层（Annotation Layer）：备注、置信度、标签等。

### 5.3 多实例策略

1. 每类层允许创建多个实例。
2. 实例之间通过 layer id 分离数据，不通过页面内临时结构分叉。
3. 多实例仅允许在同语言行内并存，不允许跨语言实例挂载到同一语段行。

### 5.4 标注页功能描述（首版具体）

1. 页面头部：提供媒体/文献范围切换、语言过滤、冲突面板开关入口（默认开启）。
2. 标注页默认采用纯文本模式：进入页面时不显示波形图，编辑区直接聚焦语段行与分层输入。
3. 行容器：按语段渲染语言行，同一语段可出现多条语言行，每条行头显示 language tag。
4. 行操作：支持折叠/展开、定位到转写页同 unit、显示本行保存状态（保存中/已保存/冲突）。
5. 语段复听：每条语段行提供一键复听按钮（播放/暂停/重播），默认按该语段起止时间播放并在段尾自动停止。
6. 可选波形：用户可通过页面开关临时展开波形辅助视图；关闭后回到纯文本模式，不影响已编辑草稿。
7. 层操作：每个语言行内可新增同类型层实例，支持重命名、删除、排序；不允许跨语言拖拽挂载。
8. 词汇层：token 级显示“已关联词条/未关联词条”状态；默认自动入词典；冲突时弹窗要求用户二选一（合并/新建 variant）。
9. 词性标准层：提供 token 单改与同词形批量改写两种入口，批量改写须显示受影响 token 数量。
10. 形态层：支持词素切分与词素 gloss 编辑，保存后回写 canonical morpheme。
11. 标注层：首版提供 note、self-certainty、tag 的结构化输入位，其他字段待 **§8.2** 待决项定稿后追加。
12. AI 助手入口：点中任一标注对象（语段/词/token/morpheme）后可调用 AI，给出上下文感知分析建议。
13. 自动标注 suggestion：在词性、gloss、词素切分等可机器建议处显示“建议值”，用户可逐条采纳或批量采纳。
14. IGT 编辑区采用 React 受控输入矩阵，不使用 ContentEditable，避免长词与复杂词缀场景下的换行错位与焦点漂移。
15. 键盘流采用严格语义：Word 层 `Space` 前进同层下一词，`Tab` 下沉到 Gloss 层；Gloss 层 `Tab` 前进下一语素；`Shift+Tab` 反向回退，减少鼠标依赖。
16. 在 Word 层输入 `-` 或 `=` 时触发实时自动切分（token/morpheme），并立刻生成对应 Gloss 占位输入位。
17. Gloss 层支持本地预测幽灵文本（灰态建议），用户按 `Enter`/`Tab` 可直接采纳，未采纳不写入正文。

### 5.5 转写页功能描述（与标注页协同）

1. 转写页保持音频播放、语段切分、文本编辑为主职责，不复制标注页全部分层 UI。
2. 每个语段条目提供“打开标注”入口，带 unitId 与当前语言上下文跳转到标注页对应行。
3. 转写页保留轻量词法编辑入口（POS/gloss 快捷改写），复杂多实例编辑统一在标注页完成。
4. 当标注页产生词汇层入词典结果后，转写页 token 的词条关联态即时可见。
5. 当转写页文本更新后，标注页对应语言行自动刷新并保留未提交草稿保护。

### 5.6 词典页功能描述（首版具体）

1. 页面结构：左侧“词条列表”，右侧分上下两块，上方“词条详情”，下方“包含该词条的语段列表”。
2. 左侧词条列表支持至少四种排序：lemma A-Z、lemma Z-A、最近更新优先、使用频次优先。
3. 左侧词条列表支持语言过滤，排序在当前过滤结果内生效。
4. 点击左侧词条后，右侧上方立即展示词条详情；若无详情字段则显示空态占位，不阻断下方语段列表。
5. 右侧下方展示所有命中语段，按“最近编辑优先”默认排序，并允许后续扩展排序。
6. 点击命中语段可跳转到转写页或标注页对应 unit，保持词典页当前选中词条上下文。
7. 当词条发生新增/更新/冲突分流（新建 variant）后，左侧列表、右侧详情、下方语段列表三处联动刷新。
8. 词条详情支持附件区：可添加图片与语言附件，并在详情区预览缩略图、试听或下载。
9. 词典页提供 FLEx 风格导出入口：可导出当前筛选/选中/全量词条，支持附带附件资源。
10. 词典页 AI 建议入口：点中词条后可生成释义优化、例句建议、标签建议与冲突处理建议。

9. 词典页 AI 建议支持释义、例句、标签与冲突处理 suggestion。

验收：

1. 左侧任一排序切换后，词条列表顺序与规则一致且可复现。
2. 点击任一词条后，右上详情与右下语段命中列表在同一次交互内完成刷新。
3. 命中语段列表覆盖该词条全部关联 unit（无漏项），并可跳转回转写页/标注页定位。
4. 新词条创建与冲突分流策略符合预期。
5. token_lexeme_link 可回查、可展示。
6. 词条图片与语言附件可上传、可预览、可删除，且刷新后元数据不丢失。
7. FLEx 风格导出包结构完整，词条字段与附件引用可被校验脚本解析。
8. 词典页 AI suggestion 可被审阅并手动采纳，不发生未确认自动写入。

### M4：协作与冲突收口（1 周，状态：未启动）

### 6.3 采纳边界（避免过度引入）

1. 不引入桌面工具式重配置流程；优先保留当前“页面内轻配置 + 渐进增强”的 Web 体验。

### 6.4 工程闸门与演进主轴（执行化）

**与语料库路线图对齐**：检索 Spike **入级条件**、**OPFS**、**ONNX / 端侧模型**、**协同单主干** 等横向条款以 **[语料库页面开发路线图 · §8.1](./语料库页面开发路线图-2026-04-22.md)** 为 **同一事实源**；本节仅保留 **标注 / 词典特有** 落点与验收，避免两处表格长期漂移。

1. **跨层检索性能分阶段处理**：
  - M1–M2 维持现有只读查询 API 与 `token_lexeme_link` 中间键路径，**不**在一期引入第二套重型检索引擎。  
  - M3+ **若且仅若** 满足 **[语料库页面开发路线图 · §8.1](./语料库页面开发路线图-2026-04-22.md)** 所列 **可复现超预算条件**（或与语料库联合立项 MVP-B），再开 SQLite-WASM+FTS5 / Tantivy WASM **时间盒 Spike**；Spike 结论须回写 PR 与（若影响全局）ADR。
2. **AI 离线降级与端侧模型硬约束**：
  - 标注/词典的 AI suggestion 视为 **可选增强**；**离线**时 **人工编辑 + 手动采纳 + 导出（含 §4.7 产物）** 必须可用。  
  - 任何 AI 不可用状态不得阻断上述主链。  
  - **ONNX Runtime Web / WebGPU** 等端侧重型推理：**P3+ 可选**，须独立 ADR；**不得**作为 M0–M2 阻塞项（与语料库 **§8.1** 一致）。
3. **互操作导出优先复用既有实现**：
  - 词典导出继续以 FLEx/flextext 路径为主，同时与语料库出站联动评估 EAF/TextGrid/TRS 等标准交换格式。  
  - 原则为“复用仓内服务优先”，避免并行导出管线。
4. **Lexeme-Form-Sense 级联安全补强**：
  - 跨页引用遵循 Resolver 生命周期与断裂态策略；删除/软删后必须显式降级，不得静默悬空。  
  - 在 M0-M2 验收中补充“级联删除/悬空引用/断裂态文案”回归用例，确保词典命中语段与标注对象可安全退化。
5. **自动判词口径收敛（禁止绝对规则）**：
  - 将“命中词库即词 / 未命中即非词”改为概率评分：频次、NPMI、左右分支熵/Accessor Variety、形态线索联合打分。  
  - 评分仅输出建议层（High/Medium/Low）；不得作为默认静默切分或静默改写依据。
6. **本地自动分词与提示交互**：
  - 运行时采用本地 Worker 增量计算，优先复用现有 AutoGloss（exact/prefix/substring）与 Leipzig 校验能力形成“建议 + 规范检查”双轨。  
  - 标注 UI 以幽灵文本/波浪线或候选面板呈现建议，用户逐条或批量采纳后才进入既有 token/morpheme 写链，且需记录可审计采纳日志。
7. **协同主干保持单线（不引入并行 CRDT 写链）**：
  - P0–P2 继续采用现有 **Dexie + Supabase** 协同协议主干，**不**引入 **Yjs**（或其他 CRDT）作为 **生产并行写链**；**扁平化 Y.Map 表** 等 CRDT 建模经验 **仅**在「未来若书面批准 PoC」时作为 **技术对照**，**不是**当前主干的交付前提。  
  - 若未来评估 Yjs，仅允许 **时间盒 PoC**，不得绕开现有协议守卫与冲突治理（与 **[语料库页面开发路线图 · §8.1](./语料库页面开发路线图-2026-04-22.md)** 协同行一致）。
8. **IGT 布局与状态库闸门**：
  - 默认布局路线为 Row-based + `@tanstack/react-virtual`，不将 `pretext` 作为一期默认依赖。  
  - 键盘 AST 中间态优先复用现有 hooks/状态机；Zustand 仅在现方案无法满足可量化指标时再评估。
9. **防御性指针分期落地**（与 **§4.10** 对读）：
  - 先实现「引用断裂可诊断 + **§4.10** 快照降级 + 不白屏」，再按对象域推进 `deletedAt` 软删语义扩面。  
  - 迁移期间禁止一刀切全表切换，避免历史数据与协同回放出现混态。

### 6.5 IGT 强化条款（2026-04 并入）

1. **先啃硬骨头顺序**：先落地 **§4.9** 零拷贝深视图与统一刷新契约，再在 M1 打通键盘流最小闭环（`Space`/`Tab`/`Enter`），最后扩展本地自动标注与导出生态。

3. **M1 验收补充**：单语段可纯键盘完成“切分 -> Gloss 输入/采纳 -> 保存 -> 回跳转写”，全程无需鼠标。
4. **M2 验收补充**：本地翻译记忆（Dexie）+ 轻量规则引擎可给出毫秒级建议；离线模式下可完成建议展示与手动采纳闭环。
5. **M3/P2 验收补充**：Leipzig 缩写规范化与校验默认开启；LaTeX（`gb4e`/`expex`）导出 profile 至少提供片段级可用产物。
6. **P2 验收补充**：自动分词建议在离线模式可运行，且“建议可见/可拒绝/可回滚”三态完整，不允许默认静默改库。
7. **P2 验收补充**：删除/断链场景下 IGT 必须可降级渲染并给出可诊断提示，不得出现空白页或静默假成功。

## 7. 风险与应对

1. 风险：层类型语义漂移导致 UI 与数据结构错配。

应对：M0 固化“层类型-字段-动作”对照表，评审后锁定。

1. 风险：词汇层入词典触发重复词条膨胀。

应对：默认新建 variant，并在冲突提示中允许用户显式改选“合并到已有词条”。

1. 风险：协作回放与本地草稿冲突影响体验。

应对：沿用现有冲突票据机制，标注页不新增第二套冲突协议。

1. 风险：附件体积与格式不受控导致词典页卡顿或导出失败。

应对：限制单附件大小与 MIME 白名单；导出前做附件完整性检查并输出诊断。

1. 风险：AI 建议幻觉或语境误判导致误标注。

应对：AI 仅输出 suggestion，默认人工确认；高风险建议需二次确认并保留审计日志。

#### 10.2.2 各层次：问题要点、常见对策、与本品已叙口径的衔接

1. **B 子词 tokenization（NLP 训练/推理）**
  - **问题要点**：多语 tokenizer 对低资源/形态丰富/非拉丁脚本常**更碎、更费上下文**；子词与**语言学词/语素**边界**不一致**。  
  - **常见对策**：多语公平与词表结构（分簇/迁移）、**Unigram/OBPE** 等变体、形态知晓预切分。  
  - **衔接**：与 **IGT/FLEx 词行** 无自动等价；**不得**用「模型子词切分」直接**静默**改写 **§4.1** 真源。
2. **C 无空格词切分**
  - **问题要点**：词界歧义、域迁移、OOV/新词、切分错传导下游。  
  - **常见对策**：有监督/无监督/弱监督/双语 UWS、MDL、**segmentation-free 嵌入**、LLM 启发式分词等。  
  - **衔接**：**§6.4(5)** 禁止「命中词表即真词」的绝对规则；**§6.4(6)** 与 `AutoGloss` 建议+采纳链；**不**在本文将某一算法（如 Morfessor/PMI）**绑定为 DoD**（属 **§10.1 B 档**）。
3. **D IGT/形态+对齐（语言学文献与 NLP 交叉）**
  - **问题要点**：**联合切分与 gloss** 难度高；**Gloss 体例/缩写** 多样、**LGR 允许多种合法写法**；**标签同指/粒度不同/替代分析**；自动系统 **gloss 与语素界错位**、依赖译文漏约束等。  
  - **常见对策**：**Leipzig 规范化+校验**、RDF/链接数据保留不确定性与**多分析**、**显式联合或交错格式** 约束切分与 gloss 对齐、人机关 **建议→采纳**（与 **§4.8、§6.5、§6.4(6)** 一致）。  
  - **衔接**：`LeipzigValidator` / `AutoGlossService` 为代码侧已存在能力（以 **§2.2、§10.1.2** 为准）；**义项稳定 sense 键、多分析并存** 为 **C 档** 待 M0 冻结。
4. **A 声学/时间切分（ELAN 等产品说明与田野方法）**
  - **问题要点**：**segmentation 模式/按键延迟/链式分块**、**叠话/多说话人 tier**、**语篇单位**定义、**静音检测** 参数、与**媒体 offset** 同步等。  
  - **常见对策**：官方手册中的**延迟补偿、split/merge、与 PRAat 等互通**、工作流级规范。  
  - **衔接**：**转写/波形/语段** 为宿主能力；**ELAN 级操作细则** 若进产品，属**别条**与转写/分析页共评，**不**在标注/词典 M1 强绑。
5. **E 句法 parsing（低资源/跨语）**
  - **问题要点**：树库稀缺、**标注体系/词序/脚本** 差异、**跨语迁移** 负面干扰等。  
  - **衔接**：**未**在 M0–M4 列为**默认交付**；若与 Universal Dependencies 等对齐，**须** 独立需求与数据预算。
6. **多标注者、歧义、金标**（HUMEval、学习-with-disagreement 等）
  - **问题要点**：指南不清、**专家分歧**、**多可接受分析**、单一**κ** 掩盖困难项。  
  - **常见对策**：指南迭代、**adjudication**、**保留多标/软标** 或**显式多分析**（与 IGT 文献一致）。  
  - **衔接**：本路线图有 **冲突票据、协作、§4.10**；**IAA 指标、多标并行数据模型** 见 **§10.2.3** 缺口。
7. **工具链互操作/表示**（如 **FLEx/Toolbox/ELAN→IGT 可交换、schema 校验** 的讨论）
  - **衔接**：`FlexService`、LIFT 预留、**§4.7**；**EOPAS、Ligt 工具链** 级互操作**未**作默认范围，**可** 作 **B 档** profile Spike。
