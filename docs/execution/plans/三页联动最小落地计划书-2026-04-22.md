# 三页联动最小落地计划书（标注 / 词典 / 语料）

## 1. 目标与范围

本计划书用于把以下三页的路线图目标转为可排期、可验收的最小实现路径：

1. 标注页（/annotation）
2. 词典页（/lexicon）
3. 语料库页（/corpus，历史 /writing 重定向）

范围边界：

1. 优先落地 P0/P1 最小闭环，不在本计划扩展二期大功能。
2. 坚持单一真源，不新增标注/词典/语料专属冗余真源表。
3. 语料库保持读模型 + 集选 + 出站，不复刻标注/词典写库闭环。

**与路线图/规范的锚点**：跨页刷新合同以 [标注页与词典页开发路线图](./标注页与词典页开发路线图-2026-04-25.md) `§2.2` 为真；事件名/payload/幂等键/回放顺序与三页联评清单统一按 [标注词典语料-治理补充规范](./标注词典语料-治理补充规范-2026-04-25.md#r1-r8-cross-page-checklist) 执行。

**治理口径（2026-04-25 统一）**：
1. 标注/词典执行事实源以 2026-04-25 版路线图为准，2026-04-22 版仅作历史参考。
2. 本计划按产品开发口径推进，目标进入产品开放评审；对外开放仍遵循独立工作台门槛文档中的 `NO-GO/go` 判定。
3. 发布评审证据与工具治理证据按《证据化发布与AI工具治理完整方案》执行。

## 2. 当前实现快照（代码事实）

1. 路由层：已接入三页入口，且 /writing 已重定向至 /corpus。
2. 标注页：当前为 FeatureAvailabilityPanel 占位，未开放编辑工作流。
3. 语料库页：当前为 FeatureAvailabilityPanel 占位，未开放工作集与出站。
4. 词典页：已开放浏览骨架（搜索 + 列表 + 只读详情），未形成三栏联动闭环。
5. 转写页：写链路与 AI/协作能力完整，可作为标注页和词典页的复用内核。

## 3. 落地原则

1. Layer-first：业务域保持 layerId 语义，遵循术语边界。
2. Unit 粒度刷新：跨页同步默认按 unitId 增量刷新，禁止以整页重拉作为默认路径。
3. 深链合同统一：unitId / layerId / lexemeId 语义一致，支持跨页往返上下文恢复。
4. AI 分区：annotation 侧为 suggestion->采纳写库；corpus 侧为复制优先、默认不落盘。
5. 引用生命周期：语料删除或软删后，Resolver 与 UI 必须稳定进入断裂态，不得假成功。

## 7. 验证与门禁

代码门禁：

1. npm run -s typecheck
2. npm run build
3. 首轮：npm run test -- <changed scope>
4. 第二轮（放行前）：npm test（全量阻断）
5. 与三页联评清单同步勾选（R1-R8，新锚点见治理补充规范）

体验门禁：

1. 跨页深链往返上下文不丢失。
2. 保存后跨页增量刷新正确。
3. 删除/软删语料后，引用断裂态稳定可见。

## 8. 风险与对策

1. 风险：跨页状态键名漂移导致返回上下文失效。
对策：先冻结 query/state 合同，再开发页面行为。

2. 风险：语料库误接写库动作，打破边界。
对策：工具注册分区 + 写动作白名单约束。

3. 风险：删除链路导致引用假成功。
对策：Resolver 错误模型与 UI 断裂态同批上线。

4. 风险：标注页直接复制转写重壳导致复杂度过高。
对策：仅复用动作链与读模型，页面壳按最小能力分阶段开放。
