# M12 多副本批量同步治理（2026-04-13）

里程碑：M12（W21-W22，多副本同步）  
目标：在 M11 跨设备治理基础上落地多副本批量同步（分片同步、法定人数、批量回滚、一致性收口）。

## 1. 里程碑范围

1. 多副本批量同步分片
- 对参与副本执行分片计划，保障批量同步可预测。
- 输出 chunk 大小、chunk 数量与尾块规模。

2. 法定人数治理（quorum）
- 对批量同步确认数执行法定人数校验。
- 未达法定人数触发风险信号并阻断自动提交。

3. 批量回滚策略
- 对高风险并发删除/更新与仲裁未收敛场景输出硬回滚。
- 对低风险漂移或确认不足场景输出软回滚。

4. 多副本一致性收口
- 以批量合并结果为基线，执行一致性确认比率评估。
- 保证可检测、可回滚、可追溯。

## 2. 核心契约

1. 批量同步契约
- 分片计划结果稳定可复现。

2. quorum 契约
- 法定人数达标判定必须准确且可复验。

3. 回滚契约
- 高风险批量冲突必须进入 rollback 路径。

4. 多副本一致性契约
- 合并输出必须具备可计算共识比率与冲突清单。

## 3. 风险与回滚

1. 风险
- 多副本批量同步确认不足导致错误扩散。
- 并发删除/更新冲突导致批量结果不可恢复。

2. 回滚动作
- 降级为 M11 双副本同步路径。
- 关闭批量自动提交，仅保留人工审查。

3. 回滚验证
1. npm run gate:m12-multi-replica
2. npm run gate:m12-multi-replica:shadow
3. 确认 M12 报告恢复到 go
