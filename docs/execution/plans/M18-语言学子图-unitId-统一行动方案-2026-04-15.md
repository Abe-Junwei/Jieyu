# M18 — 语言学子图 `unitId` 统一 + **`utterances` 表退场**（**一锤定音，无长期兼容**）

## 0. 执行策略声明（已确认）

本方案采用 **激进切流（big-bang / single cutover）**，与「多 release 双写、shadow diff、字段并存 sunset」**互斥**。

| 原则 | 含义 |
|------|------|
| **单次破坏性升级** | 用户 DB 在 **一个 Dexie major 升级钩子**（或等价单次迁移）内完成：子图改 `unitId`、`utterances` 并入/删除、索引重写。**不**向最终用户交付「仍可读 `utteranceId` / `utterances`」的中间版本。 |
| **代码无长期 fallback** | 合并进主线的实现 **不得** 保留 `where('utteranceId')`、**不得** `db.utterances.get` 业务路径、**不得** feature-flag 双写观察窗；仅允许 **开发分支内** 短暂对比脚本（**不进发布产物**）。 |
| **导入导出** | **导出**仅含 `layer_units` 语义 + 子图 `unitId`。**整库 JSON 导入**仅接受当前应用导出；含遗留 `utterances` 集合或非空子图 `utteranceId` 的快照 **直接拒绝**（与 Dexie 升级迁移分离）。 |
| **协作 / 多端** | **协议与客户端最低版本**与 M18 发布 **同步抬升**；旧客户端 **不兼容** 新 DB（或仅允许「只导出、禁止写」——由 ADR 二选一，**不设长期双向互通**）。 |
| **回滚** | **仅**依赖：升级前用户 **导出备份 / 整库快照**；**不提供**「应用内降级读旧表」——回滚 = 还原备份 + 安装旧版 App。 |

**后果**：工程上必须 **迁移脚本 + 全量测试** 一次到位；产品上必须 **发版说明 + 数据备份提示**。**后续不再维护** `utterances` / 子图 `utteranceId` 的兼容分支。

---

## 1. 背景与目标

### 1.1 问题

**A.** 子图挂在 **`utteranceId`**，与 **`unit` / `layer_units.id`** 不一致。  

**B.** **`utterances`** 与 **`layer_units(unitType='utterance')`** 双根并存。

### 1.2 结束态（与 §0 一致）

- **`utterance_tokens` / `utterance_morphemes`**：仅存 **`unitId`**（= `layer_units.id`），**无 `utteranceId` 列**。  
- **`utterances` 表**：从 Dexie **物理移除**；运行时 **零读写**。  
- **转写行真源**：`layer_units` + `layer_unit_contents`；UI 可用投影类型，**仅**由 LayerUnit 派生。  
- **`timelineUnitMutationDispatch` 的 utterance 语义路径**：实现 **仅** LayerUnit 管线。

### 1.3 非目标

- segment 型 unit 是否承载 token：**仍由 ADR 冻结**（默认 utterance 型宿主）。  
- 其它无关 legacy 表：**另立 M19**。

## 2. 现状盘点（实施前基线）

| 资产 | 当前键 / 真源 | M18 后 |
|------|----------------|--------|
| `utterances` | 独立表 | **删除** |
| `layer_units` utterance 行 | 部分/全量 | **唯一**时间+元数据真源 |
| `utterance_tokens` | `utteranceId` | **`unitId` only** |
| `utterance_morphemes` | `utteranceId` | **`unitId` only** |
| `token_lexeme_links` | `targetId` | 不变（仍指向 token/morpheme id） |

---

## 3. 目标数据契约

- **`unitId`**：`layer_units.id`（utterance 型宿主），与 `TimelineUnitView.id`（`kind==='utterance'`）一致。  
- **子图表**：schema **仅** `unitId`；迁移内 **`unitId := 旧 utteranceId`**，随后 **删除旧列与旧索引**。  
- **禁止**：发布产物中出现「兼容读 `utteranceId`」或「兼容读 `utterances`」。

---

## 4. 分阶段行动（一锤定音编排）

> 下列阶段是 **工作拆分顺序**，**用户可见仅一次版本跃迁**（可标 `BREAKING` / `minDbVersion`）。**不在中间阶段发版给生产用户**（内测渠道可打连续 nightly，但语义上仍是「未完成的 M18」直至一次性合并发布）。

### 4.1 迁移顺序（与 `src/db/migrations/m18LinguisticUtteranceCutover.ts` + Dexie **v37** 一致）

1. 读取 `tier_definitions`，为每条 legacy `utterances` 行解析默认转写层 `layerId`。  
2. **`layer_units` / `layer_unit_contents`**：对每条 legacy utterance 执行 `mapUtteranceToLayerUnit` 并 `put`（与已有 utterance 型 `layer_units` 同 id 则覆盖对齐）。  
3. **子图宿主校验**：`collectM18SubgraphReferencedUnitIds` ∪ 断言每个 host id 在 utterance 型 `layer_units` 或本次合并集合中存在（否则抛错，整事务失败）。  
4. **`utterance_tokens` / `utterance_morphemes`**：逐行 `utteranceId` → `unitId`，去掉 `utteranceId` 字段并 `put`。  
5. **Schema**：`utterances: null` 移除物理表；子图表索引切换为 `[unitId+…]`（由 Dexie `version(37).stores` 声明）。

### Phase 1 — 单次迁移实现（核心）

**状态：已在 `src/db/engine.ts` `version(37)` 落地**，升级入口调用 `upgradeM18LinguisticUtteranceCutover`。

在 **`engine.ts` 单一 `version(n).upgrade`**（或单次可调用 `runM18Migration(db)`）内 **顺序执行**（同一事务或明确子事务顺序）：

1. **确保** 每条 `utterances` 在 `layer_units` 有 **`unitType='utterance'`、同 id** 的行；缺则 **insert**（字段映射 ADR-003）；**内容**写入/对齐 `layer_unit_contents`。  
2. **`utterance_tokens` / `utterance_morphemes`**：新增 **`unitId`**，**`unitId = utteranceId`**，校验每 id 在 `layer_units` 存在；**删除** `utteranceId` 列与依赖索引；**重建** `[unitId+tokenIndex]` 等索引。  
3. **`utterances` 表**：**bulk 删除或 store null**（按 Dexie 能力）；移除 adapter / io 注册。  
4. **失败策略**：任一步抛错 → **整版升级失败**，用户保持旧版本 DB（**不**写半套新表）；要求用户重试前 **备份**（文案在 ADR/发布说明）。

**门禁**：`src/db/migrations/m18LinguisticUtteranceCutover.test.ts` + `m18LinguisticUtteranceCutover.dexie.test.ts`（`npm run test:m18-linguistic-subgraph`）— 多场景、幂等、与 **golden JSON** 关键字段对齐。

### Phase 2 — 应用层同提交改完（无 fallback）

同一合并单元（与 Phase 1 **同一 release tag**）：

- [x] 全仓删除业务侧 **`utterances` 集合** 读写（`JieyuDexie` / `JieyuCollections` / `exportDatabaseAsJson` 无 `utterances`；`importDatabaseFromJson` **拒绝**非空遗留 `utterances` 快照键）。  
- [x] 子图与校验：`utterance_tokens` / `utterance_morphemes` **schema 仅 `unitId`**（`validateUtteranceTokenDoc` 等）；业务查询禁止 legacy `where('utteranceId')`（architecture guard）。  
- [x] **导入**：快照须已为 **LayerUnit 口径**（`layer_units` / `layer_unit_contents`）；子图行须含 **`unitId`**（含 legacy `utteranceId` 则 **抛错**）；**导出**仅遍历当前 `collections`（无 Dexie `utterances` 表）。  
- [ ] **协作**：独立 wire **min version** 常量（若产品需要硬门槛，在协作 ADR / 发布说明中补数字；当前为 per-record `version` 乐观并发，见 ADR-006）。

**门禁**：`npm run typecheck`、`npm run gate:timeline-cqrs-full-migration`、扩展 M18 vitest、手工 smoke（转写/词层/撤销/导入）。

## 6. 风险（激进版特有）

| 风险 | 缓解 |
|------|------|
| 迁移 bug 毁库 | **升级前强制备份提示**；内测全量真实库演练；迁移事务尽量原子 |
| 迁移耗时 | 大库 **进度与可重入**（仍属单次升级，可分段事务但**不**发中间版）；或维护窗口 |
| Undo 快照 | 撤销栈改为 **LayerUnit / unitId** 形状，**同 PR 改完** |
| 用户未备份 | UI **阻断升级**直至确认（可选） |

**不再列为缓解项**：shadow 双读、多 release fallback、旧客户端长期兼容。
