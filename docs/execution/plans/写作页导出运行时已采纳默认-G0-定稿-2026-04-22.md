## 已采纳默认（决策表）

| 决策点 | 选项 | 约定 |
|--------|------|------|
| **谁安装 Pandoc？** | A. 开发者本机 / CI 镜像固定版本；B. 纯 Web 用户无 CLI | **默认 A** 为「真值验收」环境；B 类用户须有 **明确降级**（见下行） |
| **谁安装 Typst（PDF）？** | 与 Pandoc 同属 **A 档**：本机 / CI 固定 **Typst CLI** 版本 | **PDF** 默认 **`typst compile`**；无 Typst 时 **PDF 按钮灰显或缺失** + 安装链（见 ADR 0010 **B 档**） |
| **纯 Web 用户默认** | 下载「md + yaml + asset manifest」zip；本地 README 一键脚本（可含 `typst compile`）；可选付费/自建 Worker | 产品文案 **不得** 暗示浏览器内无依赖完整 CTAN；失败时展示 **可诊断日志片段**（版本号、命令行、退出码）；Pandoc WASM / Typst WASM **仅观察路径**（见下节） |
| **版本归口** | `defaults` / `reference.docx` / Lua / Pandoc **minor**；**Typst compiler minor** 与 `@preview` 等依赖 | CI 与文档同列版本号；升级走变更记录 |
| **失败时 UX** | 非零退出 | 用户可见：**人类可读原因** +「复制诊断包」+ troubleshooting 链接；开发模式可附加完整 stderr |

## 产品对齐（与主路线图一致）

**`pandoc-wasm`**（npm：[pandoc/pandoc-wasm](https://github.com/pandoc/pandoc-wasm)）在浏览器内 **只做受支持子集下的文本类转换**（如 md→HTML、docx、TeX **串** 等）。**PDF** 的 **验收默认** 为 **本机/CI：`typst compile`**（与 Pandoc 衔接细节见 ADR 0010）；**浏览器内 PDF** 仅能通过 **Typst WASM** 等路径作为 **C 档可选增强**，**不得**与 A 档 golden 默认同义。默认 **四出** 承诺为 **docx/html/tex + pdf（Typst）**（A 档）；纯 Web 默认仍是源码包回退与可诊断日志；细节见主路线图 **2.4 产品对齐** 与 ADR 0010。

## 预览与一键环境（与主路线图一致）

- **预览**：默认 **Pandoc `--to html5`（Worker）** 与导出 **同源 `defaults`**，见 ADR 0010「**预览与导出：单一 Pandoc 解析核**」与主路线图 [2.3](./写作页开发路线图-2026-04-22.md#23-预览-vs-导出数学与语言学宏端侧能力边界)。  
- **PDF**：默认 **Typst CLI**，见 ADR 0010 与主路线图 [2.4](./写作页开发路线图-2026-04-22.md#24-导出运行时决策树须在-g0-收口g1-前成文)。  
- **一键环境**：开发者/CI 的 **devcontainer 或 `scripts/` 文档化入口**（须覆盖 **Pandoc + Typst**）为 G0.5 交付物，见 ADR 0010「**官方一键环境**」。
