## 已采纳默认（决策表）

| 决策点 | 选项 | 约定 |
|--------|------|------|
| **谁安装 Pandoc / TeX？** | A. 开发者本机 / CI 镜像固定版本；B. 纯 Web 用户无 CLI | **默认 A** 为「真值验收」环境；B 类用户须有 **明确降级**（见下行） |
| **纯 Web 用户默认** | 下载「md + yaml + asset manifest」zip；本地 README 一键脚本；可选付费/自建 Worker | 产品文案 **不得** 暗示浏览器内无依赖完整 CTAN；失败时展示 **可诊断日志片段**（版本号、命令行、退出码）；Pandoc WASM **仅观察路径**（见下节） |
| **版本归口** | `defaults` / `template.tex` / Lua / Pandoc **minor** 锁版本 | CI 与文档同列版本号；升级走变更记录 |
| **失败时 UX** | 非零退出 | 用户可见：**人类可读原因** +「复制诊断包」+ troubleshooting 链接；开发模式可附加完整 stderr |

## 产品对齐（与主路线图一致）

**`pandoc-wasm`**（npm：[pandoc/pandoc-wasm](https://github.com/pandoc/pandoc-wasm)）在浏览器内 **只做受支持子集下的文本类转换**（如 md→HTML、docx、TeX **串** 等），**不能** 调用完整 **`pdflatex` / `xelatex`** 等引擎，因而 **不能** 作为「Pandoc 在浏览器里一键出 PDF」的对外承诺。纯 Web 默认仍是 **CLI / 后端 Pandoc+LaTeX** 与源码包回退；细节见主路线图 **2.4 产品对齐** 与 ADR 0010。
