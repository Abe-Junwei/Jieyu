## 2. 壳层 `text-only` 强制（`resolveTimelineShellMode`）

### 2.1 行为

当 `verticalViewEnabled && hasLayers` 时短路返回：`shell: 'text-only'`，`acousticPending: false`，`playableAcoustic: false`。

- **意图**：时间轴**宿主档位**不与「tier 内波形轨」矛盾（纵向 tier 不挂横向波形轨）。
- **消费方**：[`useTranscriptionTimelineContentViewModel.ts`](../../../src/pages/useTranscriptionTimelineContentViewModel.ts) → `workspaceShell`（合同）/ `workspaceAcousticPending`（合同）/ `workspaceAcousticChromeState`（全局壳态，与 read model `globalState` 同源）→ [`TranscriptionTimelineWorkspaceHost.tsx`](../../../src/pages/TranscriptionTimelineWorkspaceHost.tsx) 内 `mapAcousticToTimelineChrome`（横向分支的 `acousticShellPending`）。

### 2.2 与 read model 的耦合（历史债务 → **已落地**）

[`buildTimelineReadModel`](../../../src/pages/timelineReadModel.ts) 对壳层判定调用 **两次** [`resolveTimelineShellMode`](../../../src/utils/timelineShellMode.ts)：`contractShellMode` 传入 `verticalViewEnabled`（与宿主合同一致）；`globalShellMode` **不传**纵向开关，得到「媒体 URL + 解码就绪」事实。二者经 [`timelineShellModeResultToAcousticState`](../../../src/utils/timelineShellMode.ts) 映射为：

- **`acoustic.state`**：合同态（纵向下 tier 不挂波形时可为 `no_media`）；
- **`acoustic.globalState`**：全局可播事实（播放器就绪时可为 `playable`，**不因**纵向宿主短路而误判全页不可播）。

编排与顶栏轴状态须消费 **`globalState`**，例如 [`TranscriptionPage.ReadyWorkspace.tsx`](../../../src/pages/TranscriptionPage.ReadyWorkspace.tsx) 中 `playableAcoustic: timelineReadModel.acoustic.globalState === 'playable'`、`acousticState: timelineReadModel.acoustic.globalState`。若以 **`acoustic.state === 'playable'`** 为唯一开关，纵向下仍会表现为「按合同不可播」——此为**已避免的旧误读**，勿再回退。

**与 [《时间轴交互与壳层收敛落地方案》§3.1](./时间轴交互与壳层收敛落地方案-2026-04-21.md#31-壳层判定与-layerscount-单源p1) 的关系（正交）**：该文档中的 **壳层 `layersCount` 单源**解决的是 ReadModel 与 Timeline Content VM 在「**有层但尚无 unit**」时的口径分裂；**不**改变纵向下 [`resolveTimelineShellMode`](../../../src/utils/timelineShellMode.ts) 对 `playableAcoustic: false` 的刻意短路。二者 **正交**——仅做 `layersCount` 收口 **不能**替代双字段方案。

**已定方案（已落地）**：

1. 双字段：`acoustic.state`（合同）+ `acoustic.globalState`（事实）；回归见 [`timelineReadModel.test.ts`](../../../src/pages/timelineReadModel.test.ts)（含纵向分裂用例）。
2. ReadyWorkspace 等消费方已改用 `globalState` 驱动 `playableAcoustic` / `acousticState`；结构守卫见 [`TranscriptionPage.structure.test.ts`](../../../src/pages/TranscriptionPage.structure.test.ts)。

## 3. 历史缺陷与缺口（**已收口**；下文保留根因与选项供追溯）

### 3.1 时间轴套索排除列表未覆盖对读 DOM（P0 → **已完成**）

**实现状态**：`verticalComparisonEnabled` 为真时经 [`transcriptionReadyWorkspaceWaveformBridgeInput.ts`](../../../src/pages/transcriptionReadyWorkspaceWaveformBridgeInput.ts) 传入 **`tierTimelineLassoSuppressed`**，[`useLasso.ts`](../../../src/hooks/useLasso.ts) 在 down / 空击清选 up 路径上整链短路；同时 `isTimelineTierLassoExcludedTarget` 已包含 **`.timeline-paired-reading-view`**（覆盖其下 surface），并与 down / up **共用同一谓词**。回归见 [`useLasso.test.tsx`](../../../src/hooks/useLasso.test.tsx)。

**当时根因（归档）**：`handleLassoPointerDown` 与「未激活套索时的 `pointerUp` 清选」曾以 `.timeline-annotation`、`.timeline-text-item`、`.timeline-lane-label` 等为主；纵向对读大量使用 `.timeline-paired-reading-*`，**曾不在**排除列表中。

**曾有的风险**：

- 说明：`timeline-paired-reading-row-rail-*` 本身是 `button`，历来可被表单类排除覆盖；主要缺口在 `.timeline-paired-reading-*-surface` 等。
- 可能 **`setPointerCapture`** 到 `timeline-scroll`、轻点触发 **`clearUnitSelection()`** 等。

**与矩阵对齐**：[`segment-range-gesture-single-surface`](../../../src/pages/timelineParityMatrix.ts) 对 **vertical** 仍为 **partial**（[`timelineParityMatrixMessages.ts`](../../../src/i18n/timelineParityMatrixMessages.ts) 说明纵向以编辑与导航为主；与横向轨面拖建完全等价仍属阶段 F 范围）。

**已选落地组合（归档）**：

- **产品**：`verticalComparisonEnabled` 为真时禁用 tier 套索链；波形区 wave lasso 不受影响。
- **补强**：`.timeline-paired-reading-view` 排除 + 与禁链并存；测试覆盖 rail、surface、`role="button"`、lane resize handle 等。

### 3.2 编排层 `textTimelineZoomPxPerSec`（P2 → **已完成：字段删除**）

**实现状态**：该合同字段已从编排与类型中**移除**（横纵均不再暴露）；权威缩放以 viewport / 投影为准。断言见 [`transcriptionReadyWorkspaceOrchestratorInput.test.ts`](../../../src/pages/transcriptionReadyWorkspaceOrchestratorInput.test.ts)、[`TranscriptionPage.structure.test.ts`](../../../src/pages/TranscriptionPage.structure.test.ts)。

**历史说明**：曾存在「仅横向写入、纵向不传」的分支，易造成误以为纵向缺数据；因无实际消费点，最终采用 **删除悬空合同**。

### 3.3 `textOnlyProps` 宽进窄用（P1 → **已完成**）

**实现状态**：已引入 [`TranscriptionTimelineVerticalViewInput`](../../../src/pages/transcriptionTimelineWorkspacePanelTypes.ts)；[`TranscriptionTimelineWorkspaceHost.tsx`](../../../src/pages/TranscriptionTimelineWorkspaceHost.tsx) 中 `buildVerticalViewInput` 仅向 [`TranscriptionTimelineVerticalView`](../../../src/components/TranscriptionTimelineVerticalView.tsx) 传入纵向所需字段。

## 6. 验收建议

### 6.1 手动验收 checklist（开 PR / 发版前勾选）

**前置**：有已选媒体；打开**纵向对读**；时间轴上至少有一层转写或翻译。

- [ ] 点 **对读 rail**（行首按钮）：当前句/焦点变化符合预期，**选集不被整页清空**。
- [ ] 点 **source / target 卡片空白**（非 textarea）：不触发「轨面套索」式矩形、**不误清选**。
- [ ] 点 **组与组之间**的纵向空白：同上。
- [ ] 在 **textarea** 内点击、输入、Tab：焦点稳定，**不因轻点失焦而清选**。
- [ ] **波形区**（页面上方声学条内）拖选 / 点选：仍可用；与对读区互不抢成「像在拖 tier 套索」。
- [ ] 关纵向 → 横向 tier **空白轻点**仍可空击清选（与纵向下行为差异可接受）。

1. **手动**：纵向开；点击对读 rail、组块空白、textarea；确认无意外失焦/清选、无「拖一下像轨面套索」的错觉。
2. **自动（P0 最低门槛）**：[`useLasso.test.tsx`](../../../src/hooks/useLasso.test.tsx) 已覆盖 rail、source/target surface、纵向禁链、`role="button"`、lane resize handle、横向空白清选等；与 §3.1 排除谓词 **down/up 共用** 一致。
3. **read model**：[`timelineReadModel.test.ts`](../../../src/pages/timelineReadModel.test.ts) 已含合同 vs `globalState` 纵向分裂用例；ReadyWorkspace 绑定由 [`TranscriptionPage.structure.test.ts`](../../../src/pages/TranscriptionPage.structure.test.ts) 守卫。
