# AI 治理与结构标注收口拆分方案（2026-04-25）

## 1. 拆分原则

1. 不把运行时生成的时间戳、临时 release artifact 或审计 fixture 污染混入提交；只保留口径变化导致的稳定产物差异。
2. 每组提交都应能独立解释“为什么需要”，并附对应聚焦测试。
3. C 阶段能力默认 behind-flag；除 `aiMemoryRecallShapeTelemetryEnabled` 外，不改变默认主链路行为。
4. 本方案是提交/PR 边界建议，不代替代码与架构文档作为事实源。

## 3. Final Review Checklist

1. `release-evidence-cost-guard-trend-gate.json` 的 root `summary.total/skipped` 从 `5` 到 `11` 是预期口径变化：它现在统计完整 `evidenceIndex`，包括 dry-run 下跳过的 P1 证据项。
2. `docs/execution/audits/*.ndjson` 与自动生成 M5/M6 报告不应包含本地测试时间戳污染。
3. `featureFlags` 中除 `aiMemoryRecallShapeTelemetryEnabled` 外，C 阶段新能力默认必须为 `false`。
4. Extension trust、voice manifest、memory broker、intent confidence gate 都必须能在 flag disabled 时保持旧行为。
5. Structural profile list 查询不得把无关 scope 的 asset 带入 language/project 查询。

## 4. Backlog

1. `src/components/ai/AiChatCard.tsx` 当前接近 architecture guard 行数上限（1745/2000）。本轮不混入重构；下一轮建议单独拆出 card 子组件或状态 hook。
2. C 阶段从 behind-flag 切到默认启用前，需要至少两个 release evidence 周期证明：
   - memory broker recall shape 稳定；
   - extension health/deny reason 分布可解释；
   - voice provider health 降级路径稳定；
   - intent confidence gate 没有提高误澄清率；
   - background task/session cleanup 无残留。
