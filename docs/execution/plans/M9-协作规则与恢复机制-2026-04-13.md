# M9 协作规则与恢复机制（2026-04-13）

里程碑：M9（W17-W18，M8 后续扩展）  
目标：协作规则与恢复机制（优先级策略、仲裁入口与操作日志、断网重连一致性校验）

## 1. 里程碑范围

1. 冲突优先级策略
- 目标：在字段/实体/会话冲突并存时，给出确定性优先级顺序。
- 策略：`critical > high > medium > low`，其中实体冲突与实体ID错配视为 `critical`。

2. 仲裁入口与操作日志
- 目标：在自动策略不安全时，收敛到可审计的仲裁流程。
- 策略：检测到 `critical` 冲突时，自动把 `last-write-wins` 升级为 `manual-review`。
- 审计：仲裁请求与仲裁决策均生成日志条目并可按时间序排序。

3. 断网重连一致性校验
- 目标：离线编辑回连后，确认合并结果无结构损坏。
- 策略：先做结构完整性检查，再做 digest 一致性检查。

## 2. 核心契约

1. 冲突优先级契约
- 输入同一组冲突，排序结果必须稳定且可重复。

2. 仲裁决策契约
- 存在 `critical` 冲突时，优先级策略必须强制转入 `manual-review`。

3. 重连一致性契约
- 结构合法数据：一致性校验输出 `hasStructuralDamage=false`。
- 结构损坏数据：一致性校验输出 `hasStructuralDamage=true`。

## 3. 风险与回滚

1. 风险
- 优先级策略误判，导致不该自动合并的冲突被放行。
- 重连校验遗漏结构损坏，导致后续数据污染。

2. 回滚动作
- 将仲裁默认策略降级为 `manual-review`。
- 暂停自动重连合并，仅保留一致性检查与人工确认。

3. 回滚验证
1. `npm run gate:m9-collaboration`
2. `npm run gate:m9-collaboration:shadow`
3. 检查 M9 报告是否恢复 `P0=0` 与 `go`
