# 混合模式录音后 STT 回填执行方案

日期：2026-04-20（同日补入与仓库代码路径对齐的评估结论：白名单、`saveUnitLayerText` 语义、STT 调用方式、localStorage 键名）  
适用范围：转写页时间轴三壳（媒体轨 / 纯文本 / 对照）中的译文录音控件与回填链路

---

## 0. 目标与确认约束

### 0.1 目标

在混合模式（mixed）译文行中，用户录完音后可点击 STT 按钮，将现有录音转写为文本，并回填到当前层当前语段。

### 0.2 已确认约束（本方案硬约束）

1. 继续采用最小白名单改动策略，仅改与该能力直接相关文件。
2. STT 按钮仅 mixed 可见；audio-only 与 text-only 不显示该按钮。
3. 按钮可见性仅由 `layer.modality` 判定，不以当前译文行 `item.modality`（audio/text/mixed）单独判定。
4. 转写引擎策略为 whisper-local 优先，失败后回退 commercial。
5. 本文档阶段不执行代码，不触发实现与提测命令。

### 0.3 非目标

1. 不重构整套语音智能体或 VoiceAgent 生命周期。
2. 不调整现有录音保存/删除能力定义。
3. 不引入新的跨层数据模型或协同协议字段。

## 2. 产品行为规格（落地口径）

### 2.1 按钮可见规则

1. 仅当 `layer.modality === 'mixed'` 时可见 STT 按钮；该规则与当前行 `item.modality` 无关。
2. 无录音资源时按钮不可点或不显示（以控件最终样式策略为准）。
3. 录音中状态下禁用 STT，避免并发录音与转写。

### 2.2 执行动作

1. 点击 STT 后读取当前行对应录音资源。
2. 先调用 whisper-local provider。
3. whisper-local 失败时自动回退 commercial provider。
4. 转写成功后写入当前 layer + 当前 unit 的文本字段。
5. 转写失败时给出明确错误态与可重试入口。

### 2.3 写入语义

1. 仅回填当前层，不跨层写入。
2. 仅回填当前语段（或该行绑定的 recording scope 语义目标）。
3. **不得**在无甄别情况下假定「调用 `saveUnitLayerText` 即可完成回填」。当前 `saveUnitLayerText` 选取已有译文时仅匹配 `modality === 'text' || modality === 'mixed'`，**不包含**刚录音后的 **`modality === 'audio'`** 译文行；若直接调用，存在选不中现有行、**新建一条纯 text 译文**而与录音行分叉的风险。
4. 推荐回填策略（二选一，实现前在代码中择一并写清注释）：
   - **A（优先）**：在 `useTranscriptionVoiceTranslationActions` 的转写逻辑中，按与 `saveVoiceTranslation` 相同的数据模型，**定位当前 `translationAudioMediaId` 对应的 `LayerUnitContentDocType` 行**，原地写入 `text` 并将 `modality` 升为 `'mixed'`，再经现有 `syncUnitTextToSegmentationV2` / `layer_unit_contents.put` 等与录音保存一致的路径落库；或
   - **B**：收窄或扩展 `saveUnitLayerText`（`useTranscriptionUnitActions`）使在存在 **audio 译文行且带 `translationAudioMediaId`** 时合并更新为 mixed，而非新建行（改动面略大于 A，但长期统一「有文则 mixed」语义）。
5. 回填成功后，若 UI 依赖 `translationDrafts` / 行内 `text` 展示，应通过现有 state 更新路径使 **draft 与持久化文本一致**，避免用户看到旧草稿覆盖转写结果（具体以各壳现有 draft 键规则为准）。

## 3. 技术落点设计

### 3.1 Action 层

在 `useTranscriptionVoiceTranslationActions` 增加 `transcribeVoiceTranslation`（命名可在实现时微调），职责如下：

1. 从当前录音翻译记录定位 media（优先 `MediaItem.details.audioBlob`，与控件侧 `resolveMediaAudioBlob` 语义对齐）。
2. 执行 STT（whisper-local → commercial fallback）。
3. **按 §2.3 选定策略**将识别结果写入当前译文行（优先原地升级 audio → mixed，禁止 silent 分叉出新 text 行）。
4. 返回明确结果状态（成功 / 失败 / 无音频可转写）。

**STT 调用方式（与 0.3 非目标一致）**：采用 **`LocalWhisperSttProvider` + `src/services/stt` 工厂创建的 commercial `CommercialSttProvider`** 的 `transcribe(audioBlob, lang, …)` 编排回退链；**不**通过整套 `VoiceInputService` 实时麦生命周期完成离线 blob 转写（避免误用实时路径、减少与 VoiceAgent 的耦合）。

### 3.2 透传链

与现有 `saveVoiceTranslation` / `deleteVoiceTranslation` **真实**依赖顺序一致，扩展透传须覆盖：

1. `useTranscriptionVoiceTranslationActions`（实现 raw 能力）
2. **`useTranscriptionUnitActions`**（解构 voice hook 并纳入对外返回；**不可跳过**）
3. `useTranscriptionActions`
4. `useTranscriptionMutexActionWrappers`（建议对「读 blob + 网络 STT + 写库」整体包在 `runWithDbMutex` 内，与现有录音互斥策略一致；若 STT 仅只读 DB 再写，仍建议与录音类 mutation 同 mutex，避免竞态）
5. `useTranscriptionData`
6. `transcriptionReadyWorkspacePropsBuilders`（`SharedLaneFields` / `buildSharedLaneProps`）
7. `TranscriptionPage.ReadyWorkspace`（`lanePropsInput` 注入）

目标是三壳（媒体轨 / 纯文本 / 对照）共用同一 callback 来源，避免某视图漏接。

### 3.3 控件层

TimelineTranslationAudioControls 增加 STT 交互参数：

1. onTranscribeRecording
2. transcribing
3. transcribeDisabled

并在下列挂载点按 **layer.modality === mixed** 规则透传（与 **§1.3** 排除项一致）：

1. `TranscriptionTimelineMediaTranslationRow`（媒体轨译文子行；经 `TranscriptionTimelineHorizontalMediaLanes` 注入 props）
2. `TranscriptionTimelineTextTranslationItem`
3. `TranscriptionTimelineTextOnly`（纯文本壳内联控件）
4. `TranscriptionTimelineVerticalView`

### 3.4 配置读取

与 STT 相关的 **localStorage** 键与 `useVoiceDock.ts` 保持一致（勿与仅表示浮窗位置的 `jieyu.voiceDock.pos` 混淆）：

| 用途 | 键名 |
|------|------|
| 商用 STT 提供方与参数 | `jieyu.voiceAgent.commercialStt` |
| 本地 Whisper 服务 URL / model | `jieyu.voiceAgent.localWhisper` |
| STT 后处理增强（若实现阶段需要） | `jieyu.voiceAgent.sttEnhancement` |

**实现策略**：不在 action 中 `useVoiceDock()`，避免 UI hook 反向耦合；可选路径为（执行时择一并在 PR 说明）：在 **`useTranscriptionVoiceTranslationActions.ts` 内**内联与 `useVoiceDock` 等价的 JSON 解析（接受与 hook 的轻微重复）；或将解析提取为 **`src/utils/` 下只读纯函数**并令 `useVoiceDock` 与该 util 共用（若采用 util，须把该新文件纳入白名单或单独发起极小「抽公共配置读取」提交，避免方案外扩散）。

## 5. 验收标准

### 5.1 功能验收

1. mixed 行可见 STT；非 mixed 不可见。
2. 已录音可转写并回填。
3. whisper-local 失败后可自动回退 commercial。
4. 回填结果写到当前层当前语段，无跨层污染。
5. 录音后 **audio-only** 译文行经 STT 回填后，库中 **不出现** 与同一录音并行的第二条孤立纯 text 译文行（验收 §2.3 策略生效）。

## 6. 风险与回滚策略

### 6.1 主要风险

1. 配置读取与 provider 构造路径不一致导致按钮可点但不可转写。
2. 三壳透传漏接导致某视图行为缺失（尤其 **纯文本壳 `TranscriptionTimelineTextOnly`** 与 **`TranscriptionTimelineHorizontalMediaLanes`** 父级透传）。
3. 转写中与录音中并发引发状态竞争。
4. **误用 `saveUnitLayerText` 回填 audio-only 译文行**，导致库中出现重复译文行或录音与文本脱节（见 §2.3）。

### 6.2 回滚策略

1. 若阶段 B 出现跨壳不一致，先回滚控件显示策略为仅单壳灰度。
2. 若 provider 回退链不稳定，保留 whisper-local 单链并显式报错，不 silent fallback。
3. 任何回滚仅限白名单文件，不扩散到多宿主主线改动。
