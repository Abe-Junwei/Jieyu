## 2. 接线模板（最小集）

1. **决策**：在写路径前调用 `resolveExecutionPolicy` 族或 `resolveBackgroundToolSandboxDecision`，输入含 `profile` / `workspaceRoot` / `path` / `authorizedWriteDirs`（与 `backgroundToolSandbox` 类型一致）。
2. **审计**：拒绝 / ask / allow 均写入现有 `ai_messages` 审计字段（或扩展字段），metadata 含 **action**、**reason**。
3. **测试**：`resolveExecutionPolicy.test.ts` 或 sandbox 单测补一条代表入口；必要时 Vitest 集成钩子。
4. **证据**：dry-run `gate:release-evidence:governance` 可见 **skipped / ready_or_partial** 与索引摘要；新入口若需独立卡片，在 `generate-release-evidence-bundle.mjs` 跟进（另开任务）。
