# F4 扩展入口 — 受控矩阵（2026-05-05）

对应史诗：《[F4 扩展入口 capability isolation（2026-05-01）](./F4-扩展入口-capability-isolation-epic-2026-05-01.md)》。本表与《[AI 智能体工业级史诗 — 落地方案（2026-05-01）](./AI智能体工业级史诗-落地方案-2026-05-01.md)》**T2** 对齐，用于 **Batch A 盘点** 与后续接入排序。

## 1. 入口与策略真源

| 入口（能力） | 类型 | 策略 / 沙箱真源 | 审计 / 可观测 | 风险 | 状态 |
|-------------|------|-----------------|---------------|------|------|
| 主链工具调用（streaming / pipeline） | 主链 | `resolveUserDirectivePolicyDecision`（`src/ai/policy/resolveExecutionPolicy.ts`）；调用点 `useAiChat.toolDecisionPipeline.ts`、`useAiChat.streamCompletion.ts` | `ai_tool_call_decision` 等 | 高 | **已受控** |
| 后台记忆抽取 flush | 后台写（虚拟路径） | `resolveAiChatBackgroundMemorySandboxPolicy`（同上 policy 模块）→ `resolveBackgroundToolSandboxDecision` | `ai_background_memory_extraction`、C5 release evidence | 中高 | **已受控** |
| 其它后台工具 / 扩展旁路 | 待盘点 | 应走 `resolveBackgroundToolSandboxDecision` 或统一 policy 包装；**禁止**裸写 | 与 F4 史诗 §2 成功标准一致 | 视入口 | **待 Batch A/B 逐项填实** |

## 2. 接线模板（最小集）

1. **决策**：在写路径前调用 `resolveExecutionPolicy` 族或 `resolveBackgroundToolSandboxDecision`，输入含 `profile` / `workspaceRoot` / `path` / `authorizedWriteDirs`（与 `backgroundToolSandbox` 类型一致）。
2. **审计**：拒绝 / ask / allow 均写入现有 `ai_messages` 审计字段（或扩展字段），metadata 含 **action**、**reason**。
3. **测试**：`resolveExecutionPolicy.test.ts` 或 sandbox 单测补一条代表入口；必要时 Vitest 集成钩子。
4. **证据**：dry-run `gate:release-evidence:governance` 可见 **skipped / ready_or_partial** 与索引摘要；新入口若需独立卡片，在 `generate-release-evidence-bundle.mjs` 跟进（另开任务）。

## 3. Batch A 下一步（建议顺序）

1. 在代码库中 **grep** `Dexie` / `persist` / `insertAudit` 与 `ai` 同路径，列出候选旁路，回填上表「其它后台」行。
2. 选 **1–2 个** P0 未受控写入口，按 §2 模板接线并补单测。
3. 更新本矩阵「状态」列与 F4 史诗 **§4 实施清单** 勾选项。
