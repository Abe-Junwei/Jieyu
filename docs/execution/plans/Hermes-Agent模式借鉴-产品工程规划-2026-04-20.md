# Hermes Agent 模式借鉴 — 产品工程规划

**状态**：草案（待评审后定版为「采纳 / 分期采纳 / 搁置」）  
**日期**：2026-04-20  
**溯源**：对照 [NousResearch/hermes-agent](https://github.com/NousResearch/hermes-agent) README 与公开文档；解语定位为 **本地优先** 的濒危语言研究协作应用（见仓库根 `README.md`）。  
**范围**：借鉴其 **信息架构、开发者体验、技能与记忆思想、迁移与安全叙事**；不引入 Hermes 运行时、不默认建设即时通讯网关。

---

## 1. 动机与定位

Hermes Agent 强调：统一入口的命令体验、可发现文档、技能闭环、跨会话检索与摘要、可 dry-run 的迁移、`doctor` 式自检、多后端与模型切换抽象、以及「自动执行」场景下的审批与安全分层。

解语已有较完整的 **转写 AI 接地与本地工具** 工程（见 `docs/execution/plans/AI-上下文接地与本地工具改进方案-2026-04-15.md`）与协作相关执行计划。本规划回答：**在不改变产品核心的前提下，哪些 Hermes 式能力值得分期建设，以及如何验收。**

---

## 2. 目标与非目标

### 2.1 目标

1. **文档与 onboarding**：新协作者与「人类 + Agent」同事能在短路径内找到 **快速上手 → 配置与安全 → 参考（环境变量/脚本）**；与仓库治理规则（`.cursor/rules/jieyu-docs-governance.mdc`）一致。
2. **技能（Skills）治理**：沉淀 **可发现、可版本讨论、边界清晰** 的研究/工程技能（与 [agentskills.io](https://agentskills.io) 思想对齐即可，不要求协议级兼容）。
3. **数据演进可信度**：本地库与标注结构升级时具备 **可预览迁移** 与回滚叙事（思想对标 `hermes claw migrate --dry-run`）。
4. **可选的长期知识层**：若产品需要「项目内笔记 / 标注日志 / 审计记录」的跨会话检索，引入 **全文检索 + 摘要索引** 的产品级方案占位（实现技术选型留到阶段内 ADR）。
5. **条件触发的安全默认**：若未来增强「AI 或脚本对数据的写路径」，默认走向 **显式批准、最小权限工作区、可取消任务**（叙事对标 Hermes Security 文档分层，实现贴合解语现有门控与协同模型）。

### 2.2 非目标（本规划不承诺交付）

- 建设 Telegram/Discord 等 **消息网关** 作为一等入口（除非产品路线单独立项）。
- 引入 Hermes Python 栈或复刻其子 Agent 运行时。
- 将解语整体改造成「通用自进化 Agent 产品」。
- RL 轨迹、训练管线等与濒危语言核心用户价值弱相关的研究设施。

## 3. 设计原则

1. **本地优先与隐私**：任何「记忆 / 检索 / 自动化」默认在本地完成；上云须单独 ADR 与用户知情控制。
2. **事实源在 architecture / ADR / 代码**：执行计划仅描述 **如何推进与验收**；落地后的长期事实写入 `docs/architecture/` 或 `docs/adr/`。
3. **小步可逆**：每一阶段可独立合并，避免「大文档 + 大重构」同 PR。
4. **与现有 AI 链路正交**：不削弱 `timelineReadModelEpoch`、本地工具 JIT、变更集门控等既有契约；新增能力须写明与 `useAiChat` 链路的交互边界。

## 4. 分阶段路线图

### Phase 0 — 对齐与基线（约 0.5–1 人日）

**交付物**

- 在 `docs/architecture/` 增加一篇 **短文**（或更新现有索引）：说明解语与「通用 Agent 产品」的边界，以及本规划的 **分期表入口链接**（避免口头传播失真）。
- 梳理现有文档入口：`README.md`、`AGENTS.md`、`docs/execution/plans/README.md`、关键 ADR 列表；标出 **缺口**（例如：环境诊断命令是否集中成一页）。

**验收**

- `npm run check:docs-governance` 通过。
- 新读者能在 **10 分钟内** 从根 README 走到「如何 typecheck / test / build:guard」与「协作相关计划索引」。

---

### Phase 1 — 开发者体验与「doctor」叙事（约 1–3 人日）

**交付物**

- 新增或扩展 **单一命令/脚本**（置于 `scripts/`，命名与现有 `package.json` 脚本风格一致）：聚合检查 Node 版本、`.env` 关键项提示、地图代理可选配置说明等（**只读诊断**，默认不改机器状态）。
- 在 `package.json` 增加对应 npm script（例如 `npm run doctor`），并在根 README **简短** 引用。

**验收**

- 新克隆仓库的贡献者在未读完全部文档时，运行 doctor 能得到 **可操作的下一步**（缺什么、去哪读）。
- 不引入新的生产依赖重资产；诊断失败时 exit code 与 CI 策略在方案中写明。

---

### Phase 2 — 技能（Skills）目录与发现（约 2–5 人日）

**交付物**

- 在仓库内约定 **技能索引**（可利用现有 `.claude/skills` 与 Cursor skills，但需一份 **人类可读索引**：放在 `docs/architecture/` 或 `docs/execution/governance/`，以治理规则为准）。
- 每条技能包含：**触发场景、输入/输出边界、与哪些模块耦合、维护者/最后审阅日期**。
- 与 `docs/execution/plans/README.md` 互链：标出「哪些计划落地后应反哺技能条目」。

**验收**

- 任意一条「转写对照 / 协作冲突 / 地图代理」类高频任务，能在索引中 **一跳** 找到对应 skill 或 ADR。
- 评审记录：至少一次团队或核心维护者走读，确认无敏感信息写入技能正文。

---

### Phase 3 — 数据迁移：dry-run 与回滚叙事（约按需求触发，3+ 人日）

**触发条件**：下一次 **用户可见** 的本地 schema 或批量数据结构破坏性升级前，必须完成本阶段最小集。

**交付物**

- 迁移工具或子命令：**dry-run**（输出将改动的统计与样例）、**执行**、**备份/导出钩子**（与现有 Dexie/协同方案对齐，具体实现引用当时代码基线）。
- 在 `docs/adr/` 或 `docs/architecture/` 记录 **迁移不变量**（例如：unitId 稳定性、协同 epoch 规则）。

**验收**

- 在测试数据集上 dry-run 与实跑结果一致；具备 **可重复** 的回归用例（Vitest 或脚本级）。
- 发布说明模板中增加「迁移」小节检查项。

---

### Phase 4 — 长期知识层：检索与摘要（可选，立项后 1–2 周级）

**触发条件**：产品确认需要「跨会话项目笔记 / 标注审计 / 研究日志」且现有文件搜索不足。

**交付物**

- ADR：**存储位置、索引粒度、与转写 unit 的关联键、隐私与删除策略**。
- 最小实现：**全文检索**（技术选型在 ADR 中定：如 SQLite FTS、IndexedDB 全文、或外挂本地搜索引擎）+ **可选摘要**（批处理或按需，须定义成本上限）。

**验收**

- 典型查询 p95 延迟与索引占用在 ADR 中公示；卸载/清除数据路径可测。

---

### Phase 5 — 自动执行安全默认（随写能力增强而启用，持续）

**触发条件**：出现「AI 或脚本批量写库 / 调外部 API / 执行子进程」任一能力面向默认用户开放。

**交付物**

- 与协作门控、AI 变更集确认流 **对齐** 的审批与审计设计（写入 ADR）。
- UI/CLI 层：**可取消的长任务**、危险操作二次确认、操作日志导出（范围随隐私策略）。

**验收**

- 威胁建模简表（STRIDE 或等价）存档于 `docs/execution/audits/` 或 ADR 附录。
- 关键路径 E2E 或集成测试覆盖「拒绝 / 超时 / 撤销」。

## 5. 与现有执行方案的关系

| 现有文档 | 关系 |
|----------|------|
| `AI-上下文接地与本地工具改进方案-2026-04-15.md` | Phase 4–5 不得破坏其 JIT 与工具元数据契约；新增检索层须定义与 `timelineReadModelEpoch` 的一致性。 |
| 协作类 M9–M14 等计划 | Phase 5 的审批与恢复机制应与协作规则共用词汇，避免两套「确认」模型。 |
| `docs/execution/plans/README.md` | 本文件落地后保持索引更新。 |

## 6. 风险与缓解

| 风险 | 缓解 |
|------|------|
| 文档膨胀、与代码脱节 | 计划文档标注状态；事实迁入 architecture；大变更跑 `report:docs-link-debt`。 |
| Phase 4 索引与安装包体积 | ADR 中约束索引范围与压缩策略；默认关闭重型特性。 |
| doctor 脚本误判环境 | 只报告不自动修复；CI 与本地行为文档化。 |
