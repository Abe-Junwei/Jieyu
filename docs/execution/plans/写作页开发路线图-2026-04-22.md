## 二、产品目标与非目标

### 2.1 目标（可验收）

1. **格式与编辑**：Markdown 为主；通用编辑能力；**CodeMirror 6** 首发编辑面。  
2. **语言学**：gloss、句法树 / 谱系树、长表、元音图、声明式图码 / 白板 / raw LaTeX 逃逸口；导出映射 **expex / forest / vowel / longtable** 等（模板**白名单**）。  
3. **导出**：**docx、HTML、LaTeX 源码** 为硬需求；**Pandoc + Lua filter + `defaults` + `reference.docx`** 为主线。  
4. **长篇**：多章 vault + 书级合并；**禁止**整本上千页单一富文本 DOM 为默认（见 [第三节](#三技术架构四层--唯一主图) 与 [第八节](#八风险与待决-adr)）。  
5. **语例与 AI**：`corpus-example` 块 **`corpusRef` 必填**；跨项目只读 resolve；与 **`useAiChat` + `tool_call` + `writingDocAdapter`** 同构；默认 **不写回转写**。  
6. **一致性语义**（对齐 XLingPaper 产品层）：语例互引、书目一致、缩写表一致（技术路径见 [附录 B](#附录-b--xlingpaper-对标产品语义--技术映射)）。

### 2.2 非目标（首版不承诺）

Obsidian 全插件与图谱；浏览器内任意 CTAN；docx 与 PDF 矢量级像素一致；默认 EPUB；**真 PDF 导出承诺**；无 Pandoc 环境下的「一键真 PDF」——**具体交付路径见下节决策树**，并以 `docs/adr/` 文档冻结。

### 2.3 预览 vs 导出：数学与语言学宏（端侧能力边界）

> **结论先行**：**KaTeX** 适合 **主流数学公式** 的快速预览；**不承诺**在浏览器内实时排版 **`expex` / `forest` / `vowel` 等完整 LaTeX 宏包语义**（与「不承诺任意 CTAN」一致）。这些宏的 **真渲染** 以 **Pandoc 文本导出链（docx/html/tex）** 为真源。

| 能力 | 编辑器内预览（默认） | 导出 / CI |
|------|----------------------|-----------|
| `$...$` / `$$...$$` 等常见数学 | **KaTeX**（G1 切片 B） | Pandoc + LaTeX/KaTeX 依模板 |
| **raw LaTeX** 中的 `expex` / `forest` / `vowel` 等 | **不保证** WYSIWYG；可 **原样/折叠** +「仅在导出生效」提示 | **Pandoc + 模板白名单** |
| 树 / 元音图等 **非 LaTeX** `kind` 块 | **结构化预览**（SVG、简化树、或外链图）；与 [附录 D](#附录-d--图码供应链降级矩阵g3-前定案) 一致 | 映射到 **CTAN 包**（由 ADR 冻结） |

**可选评估（G0.5 时间盒，非默认）**：

- **MathJax 4**：若团队需要 **更广的 LaTeX/MathML 子集** 预览，可做与 KaTeX 的 **并排 spike**（体积、SSR、与 rehype 安全策略一并记录）；**不**自动替换 KaTeX。  
- **Pandoc 端侧预览**：在 **G1.5** 评估是否需要对特定 `kind` 块补充端侧占位图缓存；不引入第二导出引擎。

### 2.4 导出运行时决策树（须在 G0 收口、G1 前成文）

> **目的**：避免 G1.5 在「能本地 demo」与「用户可复现导出」之间摇摆；**须在 G0 结束前列出默认路径 + 责任人 + 失败 UX**，并写入专项 ADR（见 [第六节 · 6.2](#62-计划-adr须落入-docsadr编号择机)）。**书面定稿**另见 [写作页导出运行时已采纳默认-G0-定稿-2026-04-22.md](./写作页导出运行时已采纳默认-G0-定稿-2026-04-22.md)。

**产品对齐（硬性，非措辞）**：**`pandoc-wasm` 只做文本格式转换**（如 md→HTML、受支持的 docx、**TeX 串**等 **依 WASM 能力子集**），**不能**在浏览器内拉起完整排版引擎，因而 **不能**把「浏览器里点一下 = 真 PDF」写进对外承诺。默认导出承诺收敛为 **docx/html/tex 源码**；纯 Web 提供源码包与可诊断回退，并写入 [ADR 0010](../../adr/0010-writing-workspace-export-runtime.md)。

| 决策点 | 选项 | 约定 |
|--------|------|------|
| **谁安装 Pandoc？** | A. 开发者本机 / CI 镜像固定版本；B. 纯 Web 用户无 CLI | **默认 A** 为「真值验收」环境；B 类用户须有**明确降级**（见下行） |
| **纯 Web 用户默认** | 下载「md + yaml + asset manifest」zip；本地 README 一键脚本；可选付费/自建 Worker | 产品文案**不得**暗示浏览器内无依赖完整 CTAN；失败时展示**可诊断日志片段**（版本号、命令行、退出码）；`pandoc-wasm` 仅作观察路径（见 [G0.5](#五分期路线执行主线)） |
| **浏览器内编译观察路径** | **`pandoc-wasm`**（npm / [pandoc/pandoc-wasm](https://github.com/pandoc/pandoc-wasm)，随 Pandoc **3.9** 线） | 进入 G0.5/G1.5 **时间盒 spike**；**WASM 能力 ⊂ CLI**：不承诺真 PDF、**Lua/JSON filter 能力受限**、无任意 shell——故仅作 HTML/docx/TeX 子集与体积/冷启动评估；结论写入 [附录 E](#附录-e--技术栈与上游调研) |
| **版本归口** | `defaults` / `reference.docx` / Lua / Pandoc **minor** 锁版本 | CI 与文档同列版本号；升级走变更记录 |
| **失败时 UX** | 非零退出 | 用户可见：**人类可读原因** +「复制诊断包」+ 链到 troubleshooting；开发模式可附加完整 stderr |

**G0 完成定义（导出向）**：上表默认列（A + 版本归口 + 失败 UX）**书面签字**；[ADR 0010](../../adr/0010-writing-workspace-export-runtime.md) 标题与 owner 已定，正文可在 G1 并行补齐。

## 三、技术架构（四层 · 唯一主图）

| 层 | 选型 | 职责边界 |
|----|------|----------|
| **1. 编辑** | **仅 CodeMirror 6** + `@codemirror/lang-markdown`；**WYSIWYG-ish** 需求用 **Decoration / `WidgetType` / 块级内嵌视图** 在 **Markdown 真源**上实现（对标 Obsidian  live preview 思路；细则见 [附录 F](#附录-f--cm6-单轨与-obsidian-式体验的-decorationwidget-指引)） | 键入、选区、折叠、搜索；**不承担**印刷分页与全书版心；**不**引入 **ProseMirror / TipTap / Milkdown** 第二套文档模型（避免字符串真源 ⟷ PM `Fragment` **双向同步焦油坑**） |
| **2. 语义** | **Markdown + YAML** → **unified/remark（mdast）** + **Zod** | 块 `kind`、**`corpusRef`**、gloss schema；**预览** rehype + sanitize |
| **3. 协同与 AI** | v1 自动保存；v2 **Supabase 协同协议**（`project_changes` / `project_snapshots` / `project_presence`） | **AI**：`writing` persona；**`writingDocAdapter`**；会话 **`writingSessionId`** 与转写会话隔离 |
| **4. 排版** | **Pandoc**；可选 **Quarto（Pandoc 编排器）** | **TOC、交叉引用、脚注、书目版式、图表编号** 以导出引擎为真相 |

**虚拟列表与动态行高**：优先复用仓库已依赖的 [`@tanstack/react-virtual`](https://tanstack.com/virtual/latest)（`package.json` 已有）；**不默认引入** [`@chenglou/pretext`](https://github.com/chenglou/pretext)（见前置清单第 5 条：仅当对照实验证明不可替代时再议）。

**G4 协同真源（硬约束）**：**规范真源**始终是 **Markdown 源码（及可序列化的 mdast 投影）**；共享协同以 **Supabase 协同协议对象** 为真源，编辑器侧仅做投影；**不得**另立 PM 文档树为持久化真源。

## 四、文档与块合同（实施须落 ADR）

| 主题 | 要点 |
|------|------|
| **方言** | GFM/CommonMark + 扩展（脚注、数学、directive）**列表化锁定** |
| **书级** | **磁盘表现**：`writing-project.yaml`（`chapters` 顺序、合并入口、`defaults` 覆盖）。**多人协同时**：以 **Supabase 协同对象快照**（章节顺序 / 合并入口等结构化字段）为共享真源，**禁止**多人直接对「整份 YAML 文本」并发写入；**导出 / 落盘** 时再 **序列化** 为人类可读 YAML。**防腐**：写出 YAML **之前**，必须经过 **`ManifestSchema.parse(...)`**（Zod）；解析失败须 **阻断写盘**并给出可诊断错误，避免协同脏数据落成 **语法损坏的 YAML**。 |
| **单文件** | 软上限 → 仅告警与拆章引导 |
| **`kind` 枚举** | `syntax-tree` \| `phylogeny` \| `vowel-chart` \| `table-data` \| `diagram-engine` \| `embed-whiteboard` \| `raw-latex` \| **`corpus-example`** 等 |
| **`corpus-example`** | **必填** `corpusRef`（版本化 URI，如 `jieyu:corpus:v1:…`）；可选冗余字段与 `corpusRef` **冲突则保存失败**；无 `corpusRef` 旧块 **一次性迁移**；跨项目 **只读** resolve；未挂载项目 → **可诊断断裂态**；与转写侧 **Citation Resolver Core 同源语义**（**不**直接依赖 `TranscriptionPage.citationJump.ts` 页面模块；扩展 `corpusRef` 入口）——**文法与模块边界以 [ADR 0011](../../adr/0011-writing-corpus-ref-and-citation-jump.md) 为准** |
| **gloss 正文块** | 字段级 Zod；导出 expex 等 |
| **资产** | 白板 / 预渲染图：**blob + id**；正文引用；`embed-whiteboard` 在 G3 前冻结 **持久化格式（SVG/JSON）** 与三出映射策略（PDF/TeX/docx） |
| **raw LaTeX** | 受控 fenced；安全与审计策略 |
| **GrammarDoc ↔ 写作库（默认策略）** | 正文与 vault **不**直接变异步 Grammar 权威表；**只读**复用已有类型与全局资产（如 `bibliographic_sources`）；语例 **一律**经 **`corpusRef`** 跨项目，**不**将 Grammar 段落树嵌入 md 作为第二真源；若需引用语法条目，**仅**允许外链式 id / 只读快照（例外见 ADR） |

## 附录 F · CM6 单轨与「Obsidian 式」体验的 Decoration/Widget 指引

> **定位**：在 **不引入第二文档模型**（见第三节）的前提下，说明如何用 **CodeMirror 6 原生扩展点** 做出「源码为主、关键块内嵌呈现」的交互，思路对齐 **Obsidian 的 live preview / 内嵌块** 类产品，而非 1:1 复刻 Obsidian 插件体系。  
> **非代码**：本附录不写具体组件实现；实现阶段须另配单测、性能预算与 a11y 走查。

### F.1 能力分层（与真源关系）

| 层级 | CM6 机制 | 典型用途 | 与 `md` / `mdast` 关系 |
|------|-----------|----------|-------------------------|
| **轻量样式** | `Decoration.mark` / `line decoration` | 标题层级色、待办勾选态、`corpus-example` 边框高亮 | 由 **语法高亮或 mdast 映射** 驱动 **稳定 range**；不持久化额外状态 |
| **块级占位/内嵌** | `Decoration.replace` + **`WidgetType`** | 数学预览小块、折叠 `raw-latex`、语例卡片壳、`kind` 块工具条 | **Widget 为投影**：内容仍以 **底层 Markdown 文本** 为真源；编辑命令 **回写文本**（或通过 `StateEffect` 触发统一 pipeline） |
| **大块 DOM / 画布** | `WidgetType` +（必要时）**iframe / shadow 根** | 白板缩略、谱系图交互预览 | **只读或受限编辑**；写回须走 **明确事务**（避免与键入竞争） |

**原则**：凡可 **纯文本 + 装饰** 表达的，**不要**上 Widget；Widget 数量与 **viewport 内可见块** 绑定，须与 [第七节](#七验收指标汇总须可测) 长文性能指标一起验收。

### F.2 与 mdast 管线的衔接（防漂移）

1. **单一解析入口**：块级 `kind`、front matter 的识别结果应由 **同一套** `mdast`（或等价）管线产出 **「范围 → 语义」** 映射，再转换为 **DecorationSet** 更新；避免 CM6 内再写一套 ad-hoc 正则与 remark 长期分叉。  
2. **更新策略**：大文件上 **增量** 更新 Decoration（按变更区间 / debounced 重解析），禁止每次键入 **全文档** 重建 Widget。  
3. **与 [2.3](#23-预览-vs-导出数学与语言学宏端侧能力边界) 对齐**：`expex`/`forest` 等 **不在此附录承诺 WYSIWYG**；对应 Widget 默认展示 **标签 +「仅导出生效」** 或 **结构化 `kind` 预览**。

### F.3 协同（Supabase 协议）下的注意事项

- **远程变更回放** 到编辑器时，Decoration 的 range 必须随 **增量补丁** 正确失效/重建；**禁止**在 Widget 内缓存 **过期 document 偏移**。  
- **Undo 边界**：Widget 触发的文本编辑须进入 **同一 EditorState 事务**，并与 `project_changes` 回放顺序一致，避免出现「改了文本但 Widget 仍显示旧快照」。

### F.4 可访问性与焦点

- 块内按钮（插入语例、跳转引文）须有 **`tabindex` / `role` / 键盘操作** 设计；避免 **焦点陷阱** 在 Widget DOM 内无法回到 CM6。  
- 与全局 **快捷键**（`mod+b` 等）冲突时，须在 ADR 或交互说明中 **显式列外**。

### F.5 对外链接（实现前阅读）

- [CodeMirror 6 参考手册](https://codemirror.net/docs/ref/)（`Decoration`、`WidgetType`、`EditorView`、`StateField`）  
- [Obsidian 开发者文档](https://docs.obsidian.md/)（产品级模式参考；**非** API 兼容承诺）

### F.6 CM6 `WidgetType` 与 React 生命周期（桥接合同）

> **问题**：CM6 **自行管理** Widget 对应 DOM 的挂载/回收；若在 `WidgetType` 内直接 **`createRoot(...).render(<App />)`**，在 **滚动、Decoration 重建、协同变更** 时易出现 **重复根、意外卸载、状态丢失、内存泄漏**——与 React 19 的协调模型 **冲突风险高**。

**合同（G1 须书面定稿，可测）**：

1. **默认倾向**：Widget DOM 内 **仅保留稳定占位节点**（如带 `data-writing-widget-id` 的空 `div`）；富交互 UI 由 **编辑器外层 React 树** 通过 **`createPortal`**（或等价 **单例外层挂载**）**传送**到占位符；占位符 **销毁** 时外层 **同步卸载** 对应 Portal（由 **Zustand/Context + 订阅** 或 **CM6 `destroy` 钩子** 驱动，具体实现 PR 写明）。  
2. **推荐实现形态（G1）**：使用 **极轻量全局 store**（如 **Zustand**）。**`WidgetType` 在 `toDOM` / mount 路径** 向 store **注册** `{ id, domNode, kind, … }`；在 **`destroy` / 卸载路径** **注销**。**外层单一 React 容器** `useStore` 订阅后执行 **`widgets.map(w => createPortal(<Rich … />, w.domNode))`**，形成 **单向数据流**：CM6 只负责 **占位 DOM 生命周期**；React 只负责 **Portal 子树**，避免二者 **竞态双写**同一子树。  
3. **例外路径**：若某块 **完全非 React**（纯 DOM / canvas / iframe），须在 ADR 记录 **为何不建 React 根**，并保证 **无泄漏**（`destroy` 释放 listener/WebGL 等）。  
4. **禁止**：在 Widget 内 **隐式**创建多个 `createRoot` 而无 **与文档版本/块 id 绑定** 的清理策略。
