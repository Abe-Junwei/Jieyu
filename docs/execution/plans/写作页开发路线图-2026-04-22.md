**硬约束（违反即须改 ADR 或本路线图，不得静默偏离）**

- **编辑真源**：Markdown 字符串 + **仅 CodeMirror 6**；**禁止** ProseMirror / TipTap / Milkdown **第二文档模型**（[第三节](#三-技术架构-四层-唯一主图)、[附录 F](#附录-f-cm6-单轨与-obsidian-式-体验的-decoration-widget-指引)）。  
- **导出真值**：**Pandoc CLI + Typst CLI + 锁版本模板** 为验收默认；承诺 **docx / html / tex 与 pdf（Typst）**；**完整 CTAN / raw LaTeX 语言学版式** 仍以 **TeX 导出 + 本地 LaTeX** 为真值，**Typst PDF** 为 **声明子集**（见 [2.4](#2-4-导出运行时决策树-须在-g0-收口-g1-前成文)、[0010](../../adr/0010-writing-workspace-export-runtime.md)）。`**pandoc-wasm` / Typst WASM** 在 **导出承诺与 golden 档位**上 **仅为 C 档**观察或增强路径；**纯 Web 下预览实现载体 vs 档位语义** 须与 **[0010 · 消歧义](../../adr/0010-writing-workspace-export-runtime.md#纯-web-下预览引擎与分档语义-消歧义)**、下条「预览真值」**同读**，避免误读为「禁止预览 Worker 使用 WASM」。

## 二、产品目标与非目标

### 2.1 目标（可验收）

1. **格式与编辑**：Markdown 为主；通用编辑能力；**CodeMirror 6** 首发编辑面。
2. **语言学（产品真值）**：**结构化 `kind` 块 + Zod** 为编辑器内可交互与可验收的主叙事；**导出**经 Pandoc/Lua 映射到 **expex / forest / vowel / longtable** 等（模板**白名单**）。`**expex`/`forest` 等 LaTeX 宏**在 md 中可为 **raw 逃逸**或 **历史导入**，**不**承诺浏览器内与 CTAN 完全一致的实时 WYSIWYG（与 [2.3](#2-3-预览-vs-导出-数学与语言学宏-端侧能力边界)、附录 B 一致）。
3. **导出**：**docx、HTML、LaTeX 源码、PDF** 为硬需求；**Pandoc + Lua filter + `defaults` + `reference.docx**` 为 docx/html/tex 主线；**PDF** 默认 `**typst compile**`（与 Pandoc 的衔接：如 **md → `.typ` → PDF**，具体命令在实现 PR / 0010 冻结）。
4. **长篇**：多章 vault + 书级合并；**禁止**整本上千页单一富文本 DOM 为默认（见 [第三节](#三-技术架构-四层-唯一主图) 与 [第八节](#八-风险与待决-adr)）。
5. **语例与 AI**：`corpus-example` 块 `**corpusRef` 必填**；跨项目只读 resolve；与 `**useAiChat` + `tool_call` + `writingDocAdapter**` 同构；默认 **不写回转写**。
6. **一致性语义**（对齐 XLingPaper 产品层）：语例互引、书目一致、缩写表一致（技术路径见 [附录 B](#附录-b-xlingpaper-对标-产品语义-技术映射)）。
7. **交稿与版式体验基线**：**不**以 **Microsoft Word 全功能对标**为验收前提（修订模式、域、复杂分节与全样式矩阵等 **非首版承诺**）。首版以 **macOS Pages 在学位论文常见场景** 中的 **可达能力子集** 为 **产品与手工验收满意线**——例如：默认字体与段落观感、常见标题层级与列表、基本分页与节、**导出 PDF（Typst）**、**导出 docx（Pandoc）** 供导师往返、页眉页脚 **常见子集**（具体以 **G1.5 前** 书面清单为准，并链 [第七节](#七-验收指标汇总-须可测)、[2.9](#2-9-面向-语言学语料学位论文-的用户路径与-ai-整合目标)）；**超出子集** 仍按 [2.4](#2-4-导出运行时决策树-须在-g0-收口-g1-前成文) **分档**、可诊断降级与模板白名单处理。

### 2.2 非目标（首版不承诺）

**与 Microsoft Word 功能对齐全量**：**非目标**；首版交稿与页级体验以 **macOS Pages 级常规论文用法** 的 **可达子集** 为 **满意基线**（清单与验收挂钩见 [2.1](#2-1-目标-可验收) 第 7 点）。**Obsidian 全插件与图谱**；浏览器内任意 CTAN；**Typst PDF 与 LaTeX/CTAN 导出像素级一致**；docx 与 **Typst PDF** 矢量级像素一致；默认 EPUB；**零本地用户与 A 档 CLI golden 版式完全等价的 PDF**（见 [2.4](#2-4-导出运行时决策树-须在-g0-收口-g1-前成文) **C 档**）；无 Pandoc **且** 无 Typst 环境下的「一键完整四出」——**具体交付路径见下节决策树**，并以 `docs/adr/` 文档冻结。

### 2.3 预览 vs 导出：数学与语言学宏（端侧能力边界）

> **结论先行（默认采纳）**：**与导出共享单一 Pandoc 解析核**。正文 Markdown 的 **默认 HTML 预览** 由 **同一套 `defaults`/Lua 策略** 下的 `**pandoc --to html5**`（或锁定的 `pandoc-wasm` 子集）在 **Web Worker** 产出，经 **iframe sandbox**（或等价隔离）展示；**不以** remark/rehype 作为必须与 docx/html/tex 导出语义一致的第二套 HTML 管线。CM6 侧 **源码高亮** 可用 `@codemirror/lang-markdown` / Lezer；**unified/remark** 若存在，**仅限**折叠、轻量范围映射等 **非权威** 用途。**预览任务调度**（debounce、可取消、禁止默认每键全量等）以 [ADR 0010 · HTML 预览触发策略](../../adr/0010-writing-workspace-export-runtime.md#html-预览触发策略-工程约束) 为准。  
> **数学与宏**：Pandoc 导出的 HTML 已可内嵌或外链公式渲染策略；**不承诺**浏览器内对 `**expex` / `forest` / `vowel` 等完整 CTAN 语义** 的实时 WYSIWYG（与「不承诺任意 CTAN」一致）。**raw LaTeX** 块以 **「仅导出生效」+ 折叠/标签** 为默认 UX。


| 能力                                              | 编辑器内预览（默认）                                                  | 导出 / CI                                                                                 |
| ----------------------------------------------- | ----------------------------------------------------------- | --------------------------------------------------------------------------------------- |
| **正文 Markdown（含 GFM 扩展等已启用项）**                  | **Pandoc → HTML5**（Worker；与导出同源配置）                          | 同核 **Pandoc** → docx/html/tex；**PDF** → **Typst**（[2.4](#2-4-导出运行时决策树-须在-g0-收口-g1-前成文)） |
| `$...$` / `$$...$$` 等数学                         | **由 Pandoc HTML 路径呈现**（依模板/filter）；不另立必须与导出分叉的 KaTeX 管线为真值  | Pandoc + 模板                                                                             |
| **raw LaTeX** 中的 `expex` / `forest` / `vowel` 等 | **不保证** WYSIWYG；**原样/折叠** +「仅在导出生效」                         | **Pandoc + 模板白名单**                                                                      |
| 树 / 元音图等 **非 LaTeX** `kind` 块                   | **结构化预览**（SVG、简化树、或外链图）；与 [附录 D](#附录-d-图码供应链降级矩阵-g3-前定案) 一致 | 映射到 **CTAN 包**（由 ADR 冻结）                                                                |


**可选评估（G0.5 时间盒，非默认）**：

- **MathJax 4**：仅当 **Pandoc HTML** 子集仍不足且通过评审时，可做 **叠加层** spike；**不得**替代 Pandoc 为预览真值。  
- **占位图缓存**：对重图 `kind` 块可在 Worker 外链 **预渲染 SVG**（与附录 D 一致）；**不**引入第二导出引擎。

### 2.3.1 绕开 LaTeX 运行态，仅导出 TeX 源码

> **目的**：主路径**不**依赖 TeX 发行版与 `pdflatex`，降低日常与会 gate 机工具链负担；**仍**满足合作方/期刊要 `**.tex` 源文件** 的交付，且与 [2.4](#2-4-导出运行时决策树-须在-g0-收口-g1-前成文)、[ADR 0010](../../adr/0010-writing-workspace-export-runtime.md) 分档叙述一致。


| 环节                    | 做法                                                                                                                                  |
| --------------------- | ----------------------------------------------------------------------------------------------------------------------------------- |
| **编辑**                | 以 **Markdown + 结构化 `kind` + Zod** 为主；`expex` / `forest` 等可落在 **受控 raw 块**或经 **filter 从 `kind` 生成 LaTeX 片段**（实现 PR 定稿）。              |
| **HTML 预览**           | **Pandoc → html5**（[2.3](#2-3-预览-vs-导出-数学与语言学宏-端侧能力边界)），**不**跑 TeX。                                                                 |
| **默认 PDF**            | **Typst**（[2.4](#2-4-导出运行时决策树-须在-g0-收口-g1-前成文)），**不**以 `pdflatex` 为应用内默认。                                                           |
| **交付 LaTeX 源码**       | 对**同一份** md 跑 **Pandoc `--to latex`（+ `defaults` / 模板 / Lua）**，得到 `**.tex` 纯文本**；该步骤为**文本格式变换**，**不要求** 安装 **TeX 引擎** 才能「点导出 .tex」。 |
| **从 `.tex` 再出纸面 PDF** | 由用户在 **本机 TeX / Overleaf** 等环境完成，**不** 纳入 **C 档「零安装」** 承诺；与 **A 档**「完整本地工具链」的说明在 UI/文档中 **显式区分**。                                   |


**与「语言学完整版式」关系**：`kind` / filter 在 **export tex** 路径上可生成/保留 **期刊模板可编译** 的 LaTeX；在 **Typst PDF** 路径上可为**声明子集**并对 **raw LaTeX** 做**显式降级**（见 2.3 表、0010）——**两轨**均从同一真源出，**不**要求日常编辑端「已装全 CTAN」。

### 2.4 导出运行时决策树（须在 G0 收口、G1 前成文）

> **目的**：避免 G1.5 在「能本地 demo」与「用户可复现导出」之间摇摆；**须在 G0 结束前列出默认路径 + 责任人 + 失败 UX**，并写入专项 ADR（见 [第六节 · 6.2](#6-2-计划-adr-须落入-docs-adr-编号择机)）。**书面定稿**另见 [写作页导出运行时已采纳默认-G0-定稿-2026-04-22.md](./写作页导出运行时已采纳默认-G0-定稿-2026-04-22.md)。

**产品对齐（硬性，非措辞）**：`**pandoc-wasm` 只做文本格式转换**（如 md→HTML、受支持的 docx、**TeX 串**等 **依 WASM 能力子集**）。**A 档 PDF** 须由 **Typst CLI `typst compile**` 产出；**Typst WASM** 仅可作为 **C 档**子集路径，**不得**与 A 档 golden 默认同义。**不得**向用户承诺「仅浏览器、零本地」即可得到与 **Pandoc+Typst CLI** **完全等价**的四出版式。默认 **A 档**导出承诺为 **docx/html/tex + pdf（Typst）**；纯 Web 提供源码包与可诊断回退，并写入 [ADR 0010](../../adr/0010-writing-workspace-export-runtime.md)。


| 决策点                 | 选项                                                                                          | 约定                                                                                                                                                                                                                                                              |
| ------------------- | ------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **谁安装 Pandoc？**     | A. 开发者本机 / CI 镜像固定版本；B. 纯 Web 用户无 CLI                                                       | **默认 A** 为「真值验收」环境；B 类用户须有**明确降级**（见下行）                                                                                                                                                                                                                         |

| **纯 Web 用户默认**      | 下载「md + yaml + asset manifest」zip；本地 README 一键脚本（可编排 `typst compile`）；可选付费/自建 Worker        | 产品文案**不得**暗示浏览器内无依赖完整 CTAN；失败时展示**可诊断日志片段**（版本号、命令行、退出码）；**导出侧** `pandoc-wasm` / **Typst WASM** 仅作 **C 档**观察路径（见 [G0.5](#五-分期路线-执行主线)）；**预览侧** 在纯 Web 下的 **实现载体与档位消歧义** 见 [0010 · 消歧义](../../adr/0010-writing-workspace-export-runtime.md#纯-web-下预览引擎与分档语义-消歧义) |
| **浏览器内编译观察路径**      | `**pandoc-wasm**`；**Typst 编译器 WASM**（C 档可选）                                                 | 进入 G0.5/G1.5 **时间盒 spike**；**WASM 能力 ⊂ CLI**；**不**替代 A 档 **PDF golden**；**Lua/JSON filter** 等受限项结论写入 [附录 E](#附录-e-技术栈与上游调研)                                                                                                                                     |
| **版本归口**            | `defaults` / `reference.docx` / Lua / Pandoc **minor**；**Typst 编译器 minor** 与 `@preview` 等依赖 | **文档真源**：[ADR 0010 · 版本表](../../adr/0010-writing-workspace-export-runtime.md#版本表-单一工程真源)；CI 与 README **引用或生成自该表**；升级走变更记录                                                                                                                                       |
| **失败时 UX**          | 非零退出                                                                                        | 用户可见：**人类可读原因** +「复制诊断包」+ 链到 troubleshooting；开发模式可附加完整 stderr                                                                                                                                                                                                   |


**G0 完成定义（导出向）**：上表默认列（A + 版本归口 + 失败 UX）**书面签字**；[ADR 0010](../../adr/0010-writing-workspace-export-runtime.md) 标题与 owner 已定，正文可在 G1 并行补齐。

**分期建议（可与现有 G 并行立项，不在此改编号）**

- **G1.5～G3**：**「AI 编辑面 v1」**——改写 / 续写 / 侧栏评审的 **patch UX**、只读 adapter 工具链、与 **G1.5** 四出 / **G3** 写回型 `tool_call` 对齐。  
- **G2 起**：**「长篇与书级生成工作流」**——章队列、未决 diff 列表、与 manifest / 虚拟列表 / TTFF **同一验收表**更新（[2.8.3](#2-8-可访问性-事故手册-大文档边界与横向收口)、[2.7.4](#2-7-体验-安全-迁移与生态补充)）。  
- **合同与 ADR**：**默认**在 **[0011](../../adr/0011-writing-corpus-ref-and-citation-jump.md)** 与 **写作合同包**（[2.5](#2-5-写作合同包-全仓块语义单一正本)）中增补 **「模型写回允许的块形状 / 禁止静默改写 `corpusRef`」** 等条款；若 AI 载荷与协同交叉复杂，再 **修订 0012 / 0016** 或 **新开 0017+**（登记见 [6.2](#6-2-计划-adr-须落入-docs-adr-编号择机) 骨架列规则）。

#### 非目标与边界（须写进帮助与 PR 自检）

- **不承诺「单提示一键终稿」**：全书 / 整部参考语法 = **多年期、多人、多轮 CI** 的工程，本节仅定义 **工作流上限**与 **人机分责**。  
- **不替代机构学术诚信政策**：界面与文档须提示 **用户最终负责**、**披露生成辅助**（若学校/期刊有要求）；产品可提供 **「本段含模型辅助」** 类元数据或导出字段，**不**做法务裁决。  
- **预览与导出语义**：AI 只改 **md / 合同块**；**HTML 预览与四出**仍以 **Pandoc（+Typst PDF）** 为真值（[2.3](#2-3-预览-vs-导出-数学与语言学宏-端侧能力边界)、[0010](../../adr/0010-writing-workspace-export-runtime.md)）。  
- **与导师协作**：首版 **G4.0** 为 **章级 / 文档级** 协同，**不**默认等同 **Word 修订模式 / 字符级共编**；若要做「批注式」改稿，须 **独立产品决策 + ADR**。

| 层             | 选型                                                                                                                                                                                                                          | 职责边界                                                                                                                                                                                            |

> **定位**：在 **不引入第二文档模型**（见第三节）的前提下，说明如何用 **CodeMirror 6 原生扩展点** 做出「源码为主、关键块内嵌呈现」的交互，思路对齐 **Obsidian 的 live preview / 内嵌块** 类产品，而非 1:1 复刻 Obsidian 插件体系。  
> **非代码**：本附录不写具体组件实现；实现阶段须另配单测、性能预算与 a11y 走查。**验收级清单** 与里程碑节奏见 [2.8.1](#2-8-可访问性-事故手册-大文档边界与横向收口)、[第七节 · a11y](#七-验收指标汇总-须可测)。**长文 + 重型 Widget** 的 **视口门控** 见 **[F.7](#f-7-重型-widget-与视口-推荐)**（与 [2.8.3](#2-8-可访问性-事故手册-大文档边界与横向收口) 同轨）。

### F.1 能力分层（与真源关系）


| 层级              | CM6 机制                                   | 典型用途                                    | 与 `md` / `mdast` 关系                                                                       |
| --------------- | ---------------------------------------- | --------------------------------------- | ----------------------------------------------------------------------------------------- |
| **轻量样式**        | `Decoration.mark` / `line decoration`    | 标题层级色、待办勾选态、`corpus-example` 边框高亮       | 由 **语法高亮或 mdast 映射** 驱动 **稳定 range**；不持久化额外状态                                             |
| **块级占位/内嵌**     | `Decoration.replace` + `**WidgetType`**  | 数学预览小块、折叠 `raw-latex`、语例卡片壳、`kind` 块工具条 | **Widget 为投影**：内容仍以 **底层 Markdown 文本** 为真源；编辑命令 **回写文本**（或通过 `StateEffect` 触发统一 pipeline） |
| **大块 DOM / 画布** | `WidgetType` +（必要时）**iframe / shadow 根** | 白板缩略、谱系图交互预览                            | **只读或受限编辑**；写回须走 **明确事务**（避免与键入竞争）                                                        |


**原则**：凡可 **纯文本 + 装饰** 表达的，**不要**上 Widget；Widget 数量与 **viewport 内可见块** 绑定，须与 [第七节](#七-验收指标汇总-须可测) 长文性能指标一起验收。

### F.2 与解析管线的衔接（防漂移）

1. **块级 Decoration 入口**：`kind`、front matter 的 **范围 → 语义** 映射应由 **同一套轻量管线**（可选 mdast/扫描；**非** HTML 真值）产出，再转换为 **DecorationSet**；避免 CM6 内 ad-hoc 正则与合同层长期分叉。
2. **HTML 预览真值**：与 [2.3](#2-3-预览-vs-导出-数学与语言学宏-端侧能力边界) 一致，**须**走 **Pandoc Worker**；Decoration 管线 **不得**承担「必须与 docx 导出像素级一致」的 HTML 生成。
3. **更新策略**：大文件上 **增量** 更新 Decoration（按变更区间 / debounced 重解析），禁止每次键入 **全文档** 重建 Widget；**Pandoc 预览**须 **debounce** 与 **可取消** 任务。
4. **与 [2.3](#2-3-预览-vs-导出-数学与语言学宏-端侧能力边界) 对齐**：`expex`/`forest` 等 **不在此附录承诺 WYSIWYG**；对应 Widget 默认展示 **标签 +「仅导出生效」** 或 **结构化 `kind` 预览**。

### F.3 协同（Supabase 协议）下的注意事项

- **远程变更回放** 到编辑器时，Decoration 的 range 必须随 **增量补丁** 正确失效/重建；**禁止**在 Widget 内缓存 **过期 document 偏移**。  
- **Undo 边界**：Widget 触发的文本编辑须进入 **同一 EditorState 事务**，并与 `project_changes` 回放顺序一致，避免出现「改了文本但 Widget 仍显示旧快照」。

### F.4 可访问性与焦点

- 块内按钮（插入语例、跳转引文）须有 `**tabindex` / `role` / 键盘操作** 设计；避免 **焦点陷阱** 在 Widget DOM 内无法回到 CM6。  
- 与全局 **快捷键**（`mod+b` 等）冲突时，须在 ADR 或交互说明中 **显式列外**。

### F.5 对外链接（实现前阅读）

- [CodeMirror 6 参考手册](https://codemirror.net/docs/ref/)（`Decoration`、`WidgetType`、`EditorView`、`StateField`）  
- [Obsidian 开发者文档](https://docs.obsidian.md/)（产品级模式参考；**非** API 兼容承诺）

### F.6 CM6 `WidgetType` 与 React 生命周期（桥接合同）

> **问题**：CM6 **自行管理** Widget 对应 DOM 的挂载/回收；若在 `WidgetType` 内直接 `**createRoot(...).render(<App />)`**，在 **滚动、Decoration 重建、协同变更** 时易出现 **重复根、意外卸载、状态丢失、内存泄漏**——与 React 19 的协调模型 **冲突风险高**。

**合同（G1 须书面定稿，可测）**：

1. **默认倾向**：Widget DOM 内 **仅保留稳定占位节点**（如带 `data-writing-widget-id` 的空 `div`）；富交互 UI 由 **编辑器外层 React 树** 通过 `**createPortal`**（或等价 **单例外层挂载**）**传送**到占位符；占位符 **销毁** 时外层 **同步卸载** 对应 Portal（由 **Zustand/Context + 订阅** 或 **CM6 `destroy` 钩子** 驱动，具体实现 PR 写明）。
2. **推荐实现形态（G1）**：使用 **极轻量全局 store**（如 **Zustand**）。`**WidgetType` 在 `toDOM` / mount 路径** 向 store **注册** `{ id, domNode, kind, … }`；在 `**destroy` / 卸载路径** **注销**。**外层单一 React 容器** `useStore` 订阅后执行 `**widgets.map(w => createPortal(<Rich … />, w.domNode))`**，形成 **单向数据流**：CM6 只负责 **占位 DOM 生命周期**；React 只负责 **Portal 子树**，避免二者 **竞态双写**同一子树。
3. **例外路径**：若某块 **完全非 React**（纯 DOM / canvas / iframe），须在 ADR 记录 **为何不建 React 根**，并保证 **无泄漏**（`destroy` 释放 listener/WebGL 等）。
4. **禁止**：在 Widget 内 **隐式**创建多个 `createRoot` 而无 **与文档版本/块 id 绑定** 的清理策略。

### F.7 重型 Widget 与视口（推荐）

> **问题**：长文档中若 **大量** `WidgetType` 同时 mount **重型 React**（谱系图、元音图、富语例卡），即使用 Portal + 单容器，仍可能 **撑爆主线程内存** 与布局成本。

**推荐（G1 起纳入评审，可测）**：

1. **视口门控**：对 **非轻量** Widget，在 `toDOM` 或外层订阅中使用 `**IntersectionObserver`（或 CM6 可见区间回调）**：**仅当块进入视口（± 可选 margin）** 时才 **注册 Portal / mount React**；**滚出** 则 **卸载子树** 并保留 **轻量占位**（骨架或静态标签），避免「千页 × 百块」同时存活。
2. **与 [2.8.3](#2-8-可访问性-事故手册-大文档边界与横向收口) 同轨**：门控策略的 **阈值、内存峰值** 写入性能说明或与 **Worker 上限** 同一验收表。
3. **不得**以视口优化为由 **绕过** [F.6](#f-6-cm6-widgettype-与-react-生命周期-桥接合同) 的 **destroy 清理** 与 **a11y 焦点**（[F.4](#f-4-可访问性与焦点)）。
