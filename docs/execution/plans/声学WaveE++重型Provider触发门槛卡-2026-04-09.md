# 声学 WaveE++ 重型 Provider 触发门槛卡（2026-04-09）

> 文档角色：执行门槛卡。用于决定是否从当前 Wave E 产品化收口版进入 Wave E++（MFA / pyannote 等重型 provider 实施）。

## 1. 适用范围

仅适用于以下能力进入实现排期：

1. 词/音素级强制对齐（如 MFA、WhisperX 对齐链）。
2. 多说话人分离与标注（如 pyannote diarization 链）。
3. 需要独立算力、部署与密钥治理的外部声学 provider。

## 2. 进入条件（全部必须满足）

1. 业务触发：存在明确交付需求，且默认本地链路无法满足（必须给出样本和失败证据）。
2. 数据触发：至少 2 组目标语料完成基线评测，并形成可复现指标目标。
3. 运维触发：已明确部署形态、成本预算、容量上限与故障处置责任人。
4. 安全触发：已具备生产级 secret 管理方案（托管、轮换、审计、最小权限）。
5. 合规触发：许可证、数据合规和外部服务条款已完成审查并记录结论。
6. 回退触发：失败时可无损回退到本地 provider，且主链路行为可验证。

## 3. 否决条件（任一命中即不进入）

1. 需求仅为探索性调研，尚无明确交付窗口。
2. 指标目标不清晰，或无法提供可复现评测集。
3. secret 管理仍依赖本地明文或临时人工分发。
4. 无法保证 provider 故障不影响默认本地链路。
5. 预估成本超预算且无替代方案。

## 4. 最小提交物清单（评审前必须齐备）

1. 需求说明：目标场景、收益、失败样例、上线窗口。
2. 评测报告：基线方法、样本、指标、误差区间。
3. 方案文档：provider 契约、状态机、降级/回退策略。
4. 安全方案：密钥来源、权限模型、轮换策略、审计策略。
5. 运维方案：容量评估、告警规则、SLA/SLO、故障演练。
6. 风险清单：技术、成本、合规、数据质量风险与缓解措施。

## 5. 决策机制

1. 评审频率：按月评审一次，必要时加开临时评审。
2. 角色：产品负责人、技术负责人、运维负责人、安全负责人共同签字。
3. 结论：`go` / `no-go` / `defer` 三态，必须附理由与下一次复审时间。

## 6. go 后首批验收标准

1. 功能验收：目标能力在限定语料上达到评审设定阈值。
2. 稳定性验收：provider 异常时自动回退本地链路且不破坏主流程。
3. 安全验收：密钥不落地明文，轮换与审计链路可演示。
4. 观测验收：错误分型、延迟、成功率与回退率可观测。
5. 退出验收：可执行一键回退到 Wave E 产品化收口版。

## 7. 当前结论（2026-04-09）

1. 当前状态：`no-go`（暂缓重型 provider 实施）。
2. 原因：生产级 secret 管理与团队配置分发尚未闭合，且当前需求未达到必须引入重型链路的门槛。
3. 下一次复审：下一轮 Phase 4+ 规划评审会。
