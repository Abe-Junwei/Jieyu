## 1. 产品原则与「不对标」

解语 AI 强化 **查询型、证据型、可审批、可复盘** 的专用任务智能体；**不**以「更强聊天」或「通用自主编程代理」定义方向。

**明确不排期（讨论默认驳回或上升为战略议题）**：


| 项                             | 原因                  |
| ----------------------------- | ------------------- |
| 通用多智能体 swarm、Devin 式全自动改仓     | 与定位不符；治理方案刻意保持距离    |
| 云端多租户任务队列、跨设备 SLA             | 当前真源为桌面本地；上云属战略变更   |
| 全工具自动 inverse patch           | 以 ADR-0025 B2 白名单为界 |
| 站内字处理 / 版式引擎 / Obsidian 级插件宇宙 | 语料库/写作路线图已声明非目标     |
| 设计生成类产品对标                     | 调研已降级               |
| 7×24 线上运维中台、消费级「最长聊天」对标       | 资源优先任务态与工具正确性       |

#### 3.4 工业开关发布策略（V1，已统一）

| 开关 | local | dogfood | staging | prod |
| --- | --- | --- | --- | --- |
| `aiBackgroundToolSandboxEnabled` | 环境变量控制 | 默认开 | 默认开 | 默认关 |
| `aiBackgroundMemorySessionWriteQuotaEnabled` | 默认关 | 默认开 | 默认开 | 默认关 |
| `aiToolCallExecutorAutoRetryEnabled` | 默认关 | 默认开 | 默认开 | 默认关 |

配套执行规则（V1）：

1. **Sandbox 生产临时开启权限**：个人开发场景采用单人决策；单次最长 24h，到期自动回关。
2. **会话写入配额参数**：`aiBackgroundMemorySessionWriteQuotaMax` 首发维持 `12`；每周最多调整一次，单次调整幅度不超过 25%。
3. **配额超限行为**：拒绝写入 + 用户可见提示 + 审计留痕；不采用静默降级。
4. **自动重试边界**：仅对执行器抛错/超时生效，且仅重试 1 次；`ok:false` 与破坏性工具不重试。
5. **升 prod 门槛**：连续 2 个周期通过 release evidence 复核（含 `agent-evals` 全通过，`toolDecisionFailureSignals` 保持 `ready_or_partial` 且无回退信号）。
6. **自动回滚触发**：任一关键闸失败、审计字段缺失、执行错误率异常上升时，立即回关对应开关。
7. **变更留痕**：每次开关策略变更需同步更新本计划修订记录与 release evidence 产物。
8. **变更节奏**：每周最多 1 次策略变更；发版前 48h 冻结策略。

### 3.6 垂直智能体产品化路线（证据型语言资料 agent，本轮暂缓）

本节在当前执行窗口内暂不推进；待 §3.0 前四项完成并通过门禁后再恢复为活跃项。

详见《[AI智能体-垂直产品化架构与落地方案-2026-05-05.md](./AI智能体-垂直产品化架构与落地方案-2026-05-05.md)》。该路线来自对 NotebookLM、Looppanel、Harvey / Lexis+ Protégé、医疗 scribe 产品、ELAN / FLEx / Language Forge 等垂直智能体与语言资料工具的调研：成熟范式不是「更自由的聊天」，而是 **限定来源、精确引用、模板化专业产出、人在环采纳、工作流嵌入、审计可复盘**。

**补充 workflow 登记草案**（语料多选对比/摘要/扩写、token·Leipzig·IGT、时间轴卫生、声学选区简报、PDF 区+语料联合问答、正字法、项目 next steps、听写后清理等，供 `VerticalWorkflowRegistry` 后续扩表）：见同文档 **[§10](./AI智能体-垂直产品化架构与落地方案-2026-05-05.md)**（章节标题「待纳入 VerticalWorkflowRegistry 的补充工作流（草案）」）；与《语料库-产品定位》D 区、《标注词典语料-治理补充规范》R4 分轨要求见该节 §5.5 / §10.2。

**战略判断**：解语应对标 `NotebookLM + ELAN/FLEx + Looppanel + Lexis Protégé` 的交集，产品化为 **证据型语言资料智能体**。AI 的基础输出单位从自由回答升级为 `EvidencePacket`：每个事实性结论或候选建议都能回到语段、层、词条、笔记、文档片段或音频时间码。

**架构增量（不替代现有主链）**：


| 构件                           | 作用                                                                       | 与现状关系                                     |
| ---------------------------- | ------------------------------------------------------------------------ | ----------------------------------------- |
| `CorpusSourceSet`            | 限定一次 agent run 可读取的项目材料、语段集、笔记与文档                                        | 承接语料库一期 source / export 定位                |
| `EvidencePacket`             | 引用、原文片段、位置、置信度与 reason 的统一证据包                                            | 接在 RAG、read model、tool audit 与 preview 后  |
| `AiToolRegistry / Workbench` | 工具 schema、scope、trust、preview、dry-run、executor 的目录化真源                    | P0 先做 shadow registry，避免立刻重写执行链           |
| `VerticalWorkflowRegistry`   | 语段问答、标注 QA、词典候选、导出计划、田野笔记等 workflow 定义                                   | 继续经 `useAiChat` 主链与 policy 门禁执行           |
| `AgentRunState`              | 长 workflow 的 created / running / waiting_for_user / applying / done 等状态图 | 复用 `TaskRunner`，短只读问答不强行 durable 化        |
| `AdoptionQueue`              | 候选建议的采纳、忽略、复制、导出、跳转证据                                                    | 真写入仍走 `propose_changes` / confirm / audit |
| `VerticalAgentEval`          | 引用准确、scope 正确、无静默写、采纳率等领域评测                                              | 扩展 agent-evals 与 release evidence         |


**推荐落地顺序**：


| 阶段     | 交付                                                                                                        | 退出条件                                                              |
| ------ | --------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------- |
| **P0** | `EvidencePacketV0`、`VerticalWorkflowRegistry` 四个 workflow（`segment_qa` / `annotation_qa` / `lexeme_candidates` / `elan_flex_compatibility`）、`AiToolRegistry` shadow map、10-15 条垂直 eval seed | 工具名集合与 schema 不漂移；只读回答 / QA finding 可生成证据包                        |
| **P1** | 语段问答 Agent、标注 QA Agent、AdoptionQueue MVP                                                                  | 每个事实性结论至少 1 个 evidence；候选不静默写主链                                   |
| **P2** | 语料库 NotebookLM 化 source set、词典候选 Agent、ELAN/FLEx compatibility report                                     | source set 可保存到会话；互通报告可导出；写入均需确认                                  |
| **P3** | 田野语音采集前/中/后 workflow、多角色审阅、垂直 release evidence 指标                                                         | 与 ADR-0027/0028、`check:voice-agent-pre-merge`、release evidence 对齐 |


**边界**：不做通用 swarm，不复制 ELAN/FLEx 全量编辑器，不默认云端知识库，不让 AI 静默替代研究者判断；所有写操作保持 proposal / confirm / audit 路径。

### 3.7 远期备选方向：自演化智能体（P3+ 评估窗口）

> **定位**：本节为远期架构备选，非当前执行项。待 §3.0 前四项完成且 P0–P2 评测基线建立后再评估是否启动。详见 `docs/architecture/self-evolving-agents-survey-analysis.md`。

**核心判断**：Jieyu 当前处于 MAO（多智能体编排）向 MASE（多智能体自演化）过渡的前半段。已有静态工作流注册表、工具集合、记忆模块、评测基线，但所有配置均为手动设计，无基于交互数据的自动优化闭环。

**与现有组件的衔接点**：

| 现有组件 | 自演化增强方向 | 评估窗口 |
|----------|----------------|----------|
| `adaptiveInputProfile` | 基于任务成功率的 Prompt 自动优化 | P2+ |
| `sessionMemory` / `projectAiMemory` | 自适应检索策略（BM25/向量/关键词动态选择） | P2+ |
| `localContextTools` | 工具调用成功率追踪 → 动态调整工具指南权重 | P1b–P2 |
| `verticalWorkflowRegistry` | 基于意图置信度的动态路由 + 高频组合模板积累 | P2+ |
| `modelSelectionFeedback` | 从规则驱动升级为数据驱动 | P2+ |

**安全约束**：任何自适应功能必须先满足「Endure」原则（安全边界不退化），且必须有 P2 建立的 baseline 数据支撑。不自演化 unless 有 baseline。
