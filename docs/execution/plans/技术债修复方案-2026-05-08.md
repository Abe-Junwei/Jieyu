**验收**：`npm run check:docs-governance`

### 阶段 B — Source Set 未完成能力（1～2 个功能 PR，可与 A 并行）

| 步骤 | 内容 | 验收 |

| B3 | 在约定运行时入口调用失效检测（对齐 TODO P1-3）；UI 可见失效态。 | 组件或 hook 测试 |
| B4 | 为 `toRuntimeCorpusSourceSet` / `fromRuntimeCorpusSourceSet` 接真实调用方后移除或收紧 TODO（P1-4）。 | `rg TODO\\(P1-4\\)` 无残留或仅剩跟踪 issue 链接 |

**依赖**：B2 为 B3/B4 的基础；B4 须在调用链存在后合入。

### 阶段 C — ARCH-7 转写编排减厚（多个小 PR）

| 步骤 | 内容 | 验收 |
| --- | --- | --- |
| C1 | PR 描述附 `npm run report:architecture-hotspots` 输出片段作基线。 | 无新增 WARN |
| C2 | 每次单域下沉：迁入既有 `useTranscription*Controller` 或 `*InputBuilder` 等，**页面仅减编排、行为不变**。 | `npm run check:architecture-guard`；相关结构测试若已有则更新 |
| C3 | 在台账中更新 ARCH-7 进度文案（部分完成 → 已缓解 / 已关闭由团队定义）。 | 文档 PR + governance |

### 阶段 D — Knip / depcheck / TS（专项或随功能 PR 捎带）

| 步骤 | 内容 | 验收 |
| --- | --- | --- |
| D1 | `knip` 与 CI 使用同一版本（`devDependencies` + `npx knip`）；评估删除 `knip.json` 中 `ignoreBinaries` 的 `knip` 项。 | CI `quality` 与本地 `npx knip --no-progress` 通过 |
| D2 | 每次从 `ignore` / `ignoreIssues` 移除 1～3 项：修死代码、补入口或测试引用；优先处理 **duplicate exports**（改代码优于长期 ignore）。 | Knip 通过；配置变短 |
| D3 | `noUnusedParameters`：先单包或单目录试点（独立 PR），再全仓打开。 | `npm run typecheck` |

### 阶段 E — 文档链接债务（并行、低阻塞）

| 步骤 | 内容 | 验收 |
| --- | --- | --- |

| E2 | 每 PR 修一类路径（如仅 `docs/execution/archive/`）。 | `check:docs-governance` |

### 阶段 F — 供应链与例外注释（节奏化）

| 项 | 内容 |
| --- | --- |
| F1 | 定节奏执行 `npm audit`，区分 prod（`audit:prod`）与 dev；可修复项入独立 PR。 |
| F2 | 新增 `eslint-disable` / `@ts-expect-error` 必须附理由；周期性审计数量，禁止静默扩散。 |

## 3. 验收命令矩阵（每 PR 至少子集）

| 命令 | 何时必跑 |
| --- | --- |
| `npm run typecheck` | 所有改代码 PR |
| `npm run check:architecture-guard:core` 或 `npm run check:architecture-guard` | 阶段 C、或任何可能影响体量/钩子的改动 |
| `npm run check:docs-governance` | 阶段 A、B1、E2、及任何 `docs/` 改动 |
| `npx knip --no-progress` | 阶段 D |
| `npm run check:depcheck` | 改动 `package.json` / `.depcheckrc.json` 时 |
| 定向 `vitest run …` | 阶段 B/C 在 PR 描述中显式列出 |

全量 `npm test`：发版前或大改动后按仓库惯例执行。

## 5. 关闭本方案的条件（可选）

当满足以下**且**台账已更新时，可将本文 `status` 改为 `superseded` 并指向新的滚动计划文件：

- ARCH-7 在台账中达到团队定义的「已关闭 / 已缓解」；
- Source Set TODO(P1-3)/(P1-4) 已接线或已迁移至其他计划真源；
- Knip `ignore*` 收敛至团队认可的剩余集合并有书面例外清单；
- `noUnusedParameters` 已全仓开启或明确放弃并列 ADR/治理说明。
