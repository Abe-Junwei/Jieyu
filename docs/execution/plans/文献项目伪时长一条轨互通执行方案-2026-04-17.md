# 文献项目伪时长一条轨互通执行方案

日期：2026-04-17  
适用范围：Jieyu 无真实音频或后置音频对齐的文献项目  
目标：以“一条单调时间轴 + start/end/text”作为统一交换语义，在不改动主数据结构的前提下，打通文献项目互通与后续媒体对齐。

---

## 1. 结论

本方案采用“伪时长一条轨（logical timeline）”作为文献项目默认时间基线，并与现有互通链路保持一致：

1. 持久层继续使用 `startTime/endTime`，不引入第二套序数分段模型。
2. 新建文献项目自动注入占位媒体与默认总时长，满足现有写路径的媒体门禁。
3. UI 显式标记逻辑时间轴，避免用户误读为声学秒。
4. 导入导出继续围绕区间语义运行，后续真实音频通过线性映射完成对齐。

---

## 2. 参照方案与互通抽象

### 2.1 行业参照（抽象层）

1. 字幕类（SRT / WebVTT）
   - 抽象核心是时间区间 + 文本块。
   - 实际工作流通常可脱离真实音频先完成编辑与交换。

2. Audacity Labels
   - 交换文件核心是 `start/end + label text`。
   - 标签文件可独立于音轨存在，音轨更多是上下文。

3. Praat TextGrid
   - 自带 `xmin/xmax` 时间域，可先建时间域再挂真实声学对象。

4. ELAN EAF
   - 以时间槽与注释层为基础，典型互通最终仍回到时间区间 + 文本。

### 2.2 对 Jieyu 的可执行含义

1. “秒”在文献项目中可定义为逻辑坐标，不必等价于真实声学时间。
2. 互通优先时，时间区间模型比纯序号模型更易接入现成工具链。
3. 真实媒体后置接入时，不需要推翻文献阶段产生的数据。

## 4. 目标边界

### 4.1 本期目标

1. 文献项目在无真实音频时可完整进行分段、编辑、导入导出。
2. 保持与 EAF/TextGrid/Flex/TRS/Toolbox 的时间区间互通稳定。
3. 项目级显式区分 `document timeline` 与 `media aligned timeline`。
4. 后续支持把逻辑时间映射到真实媒体时间。

### 4.2 非目标

1. 本期不引入第二套“纯序号分段”持久化模型。
2. 本期不做词级/字级自动强制对齐。
3. 本期不要求所有外部格式都原生支持逻辑时间声明（通过侧车元数据兜底）。

---

## 5. 方案设计（最小改动）

### 5.1 项目时间语义标记

在项目/文本元数据新增轻量字段（不改主分段结构）：

1. `timelineMode`: `document` | `mediaAligned`
2. `logicalDurationSec`: number
3. `timebaseLabel`: `logical-second`
4. `timeMapping`（可选，后置）:
   - `offsetSec`
   - `scale`

说明：
- 分段仍只存 `startTime/endTime`。
- 语义通过元数据解释，而不是新增并行时间字段。

### 5.2 新建文献项目注入占位轨

1. 创建项目时自动创建占位 `MediaItem`（可无真实文件）。
2. 默认总时长建议：30min（可配置）。
3. 所有写路径继续依赖“已选媒体”，避免改动大量控制器门禁逻辑。

### 5.3 UI 语义防误读

1. 时间轴头部显示 `Logical Timeline` 标识。
2. 导出入口显示提示：当前时间为逻辑坐标，非声学对齐时间。
3. 可选显示模式：时间码 / 段序号（仅视图层切换，不改存储）。

### 5.4 互通策略

1. 继续复用现有 EAF/TextGrid/Flex/TRS/Toolbox 导出。
2. 导出包附带简短说明（README 或 header 注释）：
   - `timelineMode=document`
   - `timebase=logical-second`
3. 后续新增 SRT/WebVTT 适配层时，仍输出标准时间码；额外语义通过 sidecar/注释声明。

### 5.5 真实媒体后置对齐

使用统一线性映射：

$$
t_{real} = offset + scale \cdot t_{doc}
$$

1. 默认 `offset=0`, `scale=1`。
2. 批量重映射时保持区间单调与非负约束。
3. 映射过程保留可回滚记录（一次操作一个 revision）。

### 5.6 当前执行进度（2026-04-18，已验证）

已完成：

1. 文献项目默认写入 `timelineMode=document`、`logicalDurationSec` 与 `timebaseLabel`。
2. 时间轴头部已增加逻辑时间标签，导出入口已增加逻辑时间提示。
3. 导入导出链路已保留逻辑时间语义，文献项目可继续走既有单轨 `start/end` 互通模型。
4. 已落地映射校准闭环：
   - 预览文档时间到实际时间的区间变化；
   - 编辑 `offsetSec/scale` 并应用；
   - 一键回滚上一版映射；
   - 持久化多步历史；
   - 在项目中心点击历史项即可回填校准表单。
5. 当前验证结果：
   - 定向回归：3 个测试文件、122 个测试全部通过；
   - 架构守卫：通过；
   - 生产构建：通过。

本轮收口后，逻辑时间模式已经具备“创建 → 编辑 → 导入导出 → 预览校准 → 回滚恢复”的连续可用链路，后续增强仅保留为体验优化，不再是主流程阻塞项。

## 6. 风险与缓解

1. 风险：用户把逻辑秒当成声学秒。
   - 缓解：UI 常驻标签 + 导出提示 + 项目设置显式开关。

2. 风险：外部工具二次传播后丢失“逻辑时间”语义。
   - 缓解：导出 sidecar 元数据与导入时提示校验。

3. 风险：后续媒体对齐导致历史注释错位。
   - 缓解：映射操作统一走事务与可回滚 revision；批量前预览。

4. 风险：为兼容逻辑模式引入双轨代码分叉。
   - 缓解：坚持“同一 `start/end` 存储模型 + 元数据解释”，不复制业务流程。

## 7. 分期执行

### Phase A（P0）：语义落标 + 占位轨闭环

1. 新建文献项目自动注入占位媒体。
2. 写入 `timelineMode=document` 与默认 `logicalDurationSec`。
3. 时间轴与导出菜单增加逻辑时间提示。

验收：
- 无真实音频可完成建段、改段、导出。

### Phase B（P1）：互通稳定性与文档声明

1. 为现有导出链路统一补充逻辑时间说明。
2. 导入时识别并保留逻辑时间模式元数据。
3. 补齐回归测试（导入导出 round-trip）。

验收：
- EAF/TextGrid/Flex/TRS/Toolbox round-trip 不破坏分段顺序与文本内容。

### Phase C（P2）：后置媒体对齐映射

1. 增加 `offset/scale` 映射工具与预览。
2. 批量映射写入事务，支持回滚。
3. 映射后导出与搜索定位保持一致。

验收：
- 映射前后数据可互逆（在容差范围内）。

1. 文献项目创建后可立即分段，无需真实媒体文件。
