## 使用说明

1. **拍板后**：在对应 ADR（尤其 0029）、`docs/execution/release-gates/` 下相关 rollout / checklist、以及 `docs/architecture/桌面端浏览器支持策略.md`（若涉 TTS）更新结论与本表「状态」列。
2. **键入路由**若选「变更」：必须先合并 **新 ADR** 再动代码，避免与 0028 读模型冲突。
3. 本表不替代 `npm run check:docs-governance` 所约束的其它 governed 文档；仅作 **拍板索引**。
