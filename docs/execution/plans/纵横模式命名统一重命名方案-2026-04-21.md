# 纵横模式命名统一重命名方案（2026-04-21）

**状态**：可执行草案（待实施，已按 2026-04-21 评审修订）
**日期**：2026-04-21
**适用范围**：转写页**壳层**纵向/横向模式命名统一（含 MediaLanes 组件链）；与视频预览几何、组内对齐拓扑正交。

---

## 1. 目标

1. 将**壳层**纵向模式（原「对照 / 双列对读」视图）的工程命名从 comparison 收敛为 **vertical**（用户向仍称「纵向模式」）。
2. 将**壳层**横向模式（原标准多轨时间轴）的用户与关键符号收敛为 **horizontal**。
3. 将 `TranscriptionTimelineHorizontalMediaLanes` 的组件命名链升级到 horizontal 语义（或采用第 3.2 节备选短名）。
4. 采用硬切策略：不保留旧符号、旧键名与旧存储键回落（见第 7 节用户沟通）。

---

## 2. 决策与边界

### 2.1 术语表（必须先对齐，禁止混用）


| 概念         | 含义                     | 典型符号 / 位置                                                                             | 与本方案关系                                                |
| ---------- | ---------------------- | ------------------------------------------------------------------------------------- | ----------------------------------------------------- |
| **壳层模式**   | 整页时间轴壳：多轨横向 vs 双列纵向对读  | `comparisonViewEnabled`、toolbar 切换、workspace 存储键                                      | **本次重命名核心**                                           |
| **视频布局**   | 有视频时预览区相对时间轴的几何（上/左/右） | `VideoLayoutMode`、`jieyu:video-layout-mode`、`WORKSPACE_VIDEO_LAYOUT_MODE_STORAGE_KEY` | **正交，不在本次改名范围**                                       |
| **组内对齐拓扑** | 某一纵向组内一对多/多对一等关系形状     | `pairedReadingLayoutMode`、`data-paired-reading-layout`                                       | **域语义仍为「对照/对齐」**；不得与壳层 `vertical` 混成同一 i18n 前缀（见 3.3） |


### 2.2 禁止合并清单

1. **不得**将 `VideoLayoutMode` / `videoLayoutMode` / `jieyu:video-layout-mode` 改为「vertical」等壳层用词。
2. **不得**将组拓扑 `balanced | one-to-many | …` 重命名为「vertical*」类名，以免与壳层纵向混淆。
3. **不得**使用裸符号 `VerticalGroup` 表示「对读组」——必须保留「成组对读/对齐」可读性（见 3.1）。

### 2.3 已确认决策

1. 改名深度：文案 + 变量/类型/组件名（在 2.2 约束内）。
2. 兼容策略：壳层存储键与相关 i18n **硬切**；不实现旧键静默回落（可选：发布说明中提示偏好重置一次）。
3. horizontal 范围：优先统一**壳层与主链组件**命名；`timeline` 一词在全仓其他语境中暂不做替换。

### 2.4 本次不做（默认范围）

1. 不做目录级搬迁与大规模文件树重排。
2. **默认**不做 CSS class / `data-`* 全量替换（见第 5 节 Batch F 可选债）；仅当壳根节点或切换逻辑强绑定时做**最小联动**。
3. 不做 data-testid 全量替换（仅测试因重命名失败时最小修复）。

### 2.5 文档联动（与历史规格并存）

实施收尾须更新交叉引用，避免执行规划与代码名长期分叉：

1. [对照与多壳时间轴段块统一收口规划-2026-04-19.md](./对照与多壳时间轴段块统一收口规划-2026-04-19.md)：在文首或「用户向命名」段增加「工程名已迁移见本方案」脚注；历史段落可保留「对照」作叙述别名。
2. 本文件完成后，在相关 ADR / architecture 条目中增加**一行**指向本方案（不必改写历史 ADR 正文）。

## 3. 重命名映射（硬切）

### 3.1 纵向壳层：comparison -> vertical（组件与状态）

1. `TranscriptionTimelineVerticalView` → `TranscriptionTimelineVerticalView`（文件名同步改，imports 全量更新）。
2. `useTranscriptionTimelineComparisonChrome` → `useTranscriptionTimelineVerticalChrome`（实现文件 `useTranscriptionTimelineVerticalChrome.ts`）。
3. `**ComparisonGroup` → `VerticalReadingGroup`**（**修订**：禁止 `VerticalGroup`——易与任意「纵向分组」混淆；若更强调域模型可用 `PairedReadingGroup`，二选一在 PR 描述中固定）。
4. `BuildComparisonGroupsInput` → `BuildVerticalReadingGroupsInput`（与 3 所选组名一致）。
5. `buildComparisonGroups` → `buildVerticalReadingGroups`。
6. `comparisonViewEnabled` → `verticalViewEnabled`。
7. `comparisonViewActive` → `verticalViewActive`。
8. `comparisonFocus` → `verticalPaneFocus`（**修订**：较 `verticalFocus` 更明确表示壳内焦点，避免与别处的 focus 混淆）。
9. `updateComparisonFocus` → `updateVerticalPaneFocus`。
10. `resetComparisonFocus` → `resetVerticalPaneFocus`。

### 3.2 横向壳层：MediaLanes 命名升级（主选 + 备选短名）

**主选（与初版一致，语义最直白）：**

1. `TranscriptionTimelineHorizontalMediaLanes` → `TranscriptionTimelineHorizontalMediaLanes`
2. `TranscriptionPageTimelineMediaLanesProps` → `TranscriptionPageTimelineHorizontalMediaLanesProps`
3. 本地别名 `MediaLanesProps` → `HorizontalMediaLanesProps`

**备选（若行宽 / 可读性压力过大，可在 Batch C 前经评审二选一）：**

- `TranscriptionTimelineHorizontalMediaLanes` → `TranscriptionTimelineTrackStack`（强调「多轨堆叠」）
- Props → `TranscriptionPageTimelineTrackStackProps`

选定后须在 PR 与本文档修订记录中固定，避免混用。

### 3.3 i18n 键名（修订：壳层与组内拓扑拆分）

**壳层 / 工具条（建议前缀 `transcription.workspaceShell` 或保留 `toolbar` 下子键）：**

1. `transcription.toolbar.compareView` → `transcription.toolbar.verticalView`
2. `transcription.toolbar.timelineView` → `transcription.toolbar.horizontalView`
3. `transcription.toolbar.switchToComparisonView` → `transcription.toolbar.switchToVerticalView`
4. `transcription.toolbar.switchToTimelineView` → `transcription.toolbar.switchToHorizontalView`
5. `transcription.toolbar.comparisonRequiresTranslationLayer` → `transcription.toolbar.verticalViewRequiresTranslationLayer`（或 `…verticalLayoutRequires…`，与键长度权衡）

**组内对读 UI（不得整体并入 `transcription.vertical.`*）：**

- 原 `transcription.comparison.columnMode`、`allColumns`、`emptyGroups` 等描述的是**双列对读区内**控件与提示，建议新前缀：`transcription.pairedReading.`* 或 `transcription.verticalLayout.`*（表示「纵向壳内的布局」，而非壳开关）。
- `**transcription.comparison.`* → `transcription.vertical.`* 整表映射撤销**：避免与壳层 `verticalView` 同前缀导致误读。

实施时维护一张「旧键 → 新键」表，一次性替换调用点并删除旧键。

### 3.4 持久化键（壳层专用）

1. `jieyu:workspace-comparison-view` → `jieyu:workspace-vertical-view`
2. `WORKSPACE_COMPARISON_VIEW_KEY` → `WORKSPACE_VERTICAL_VIEW_KEY`
3. `WORKSPACE_COMPARISON_VIEW_STORAGE_KEY` → `WORKSPACE_VERTICAL_VIEW_STORAGE_KEY`

**明确不改**：`jieyu:video-layout-mode` 及 `WORKSPACE_VIDEO_LAYOUT_MODE_STORAGE_KEY` 等视频布局键（见 2.2）。

### 3.5 辅助模块与文件名（补全初版遗漏）

以下与纵向对读强相关，Batch B 须与主组件**同批**打通，避免出现「主组件已 Vertical、helpers 仍 Comparison」的长期分叉：

1. [src/components/transcriptionTimelineVerticalViewHelpers.tsx](./src/components/transcriptionTimelineVerticalViewHelpers.tsx) → 建议重命名为 `transcriptionTimelineVerticalViewHelpers.tsx`（或保留 `PairedReading` 域名片）。
2. [src/utils/verticalReadingHostFilter.ts](./src/utils/verticalReadingHostFilter.ts)：若仅服务纵向壳内过滤，可改为 `verticalReadingHostFilter.ts`；若仍被横向链共用，则优先**仅改导出与类型名**并在文件头注释标明服务范围，避免误改横向语义。

## 7. 风险与注意事项

1. **存储键硬切**会重置用户此前保存的「是否纵向壳」偏好；接受为已知行为，并在 *docs/发布说明- 或 changelog** 中写一句用户可见说明（Batch G）。
2. **i18n 键硬切**：须维护旧→新表并一次性删旧键，避免运行期缺键；**组内键与壳层键分前缀**可降低漏改后的语义错误。
3. 纵向主组件与 helpers 文件体量大，实施时**禁止夹带**业务逻辑重构。
4. 本方案为命名统一，**不修改**多宿主业务语义与数据模型行为。
5. **全仓 `rg comparison` 不适合作为完成标准**——误报多；以第 6.2 节白名单为准。
