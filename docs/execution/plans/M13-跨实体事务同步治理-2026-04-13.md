# M13 跨实体事务同步治理（2026-04-13）

里程碑：M13（W23-W24，跨实体事务同步）  
目标：在 M12 多副本批量同步基础上落地跨实体事务同步（分片提交、原子收口、事务回滚、跨实体一致性）。

## 1. 里程碑范围

1. 跨实体事务分片提交
- 对事务内实体集合执行分片计划与有序提交。
- 输出 chunk 大小、chunk 数量与尾块规模。

2. 事务原子性收口
- 对事务内每个实体的同步结果做原子性聚合。
- 任一实体阻断即事务整体进入 rollback。

3. 事务级回滚策略
- 对跨实体冲突、重复实体、法定人数不足场景输出硬回滚。
- 对局部可恢复场景输出软回滚路径。

4. 跨实体一致性评估
- 输出事务级共识比率与阻断实体列表。
- 保证可检测、可回滚、可追溯。

## 2. 核心契约

1. 事务同步契约
- 跨实体事务执行结果稳定可复现。

2. 原子性契约
- 任一阻断实体必须触发事务级 rollback。

3. 回滚契约
- 高风险事务冲突必须进入 hard-rollback。

4. 跨实体一致性契约
- 事务输出需具备一致性比率与冲突汇总字段。

## 3. 风险与回滚

1. 风险
- 跨实体事务部分提交导致状态分裂。
- 重复实体或高风险冲突导致事务不可恢复。

2. 回滚动作
- 降级为 M12 单实体批量同步路径。
- 关闭事务自动提交，仅保留人工审查。

3. 回滚验证
1. npm run gate:m13-transaction-sync
2. npm run gate:m13-transaction-sync:shadow
3. 确认 M13 报告恢复到 go
