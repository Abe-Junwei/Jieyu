# M8 协作三段式演进图（2026-04-13）

里程碑：M8（W15-W16）  
目标：协作演进起步（异步 -> 同设备实时 -> 跨设备协作）

## 1. 三段式演进图

1. Stage A（异步协作）
- 输入：导入/导出产物（快照包、差异包）。
- 冲突语义：字段级、实体级冲突可检测。
- 解决策略：LWW 自动合并 + manual-review 兜底。

2. Stage B（同设备实时）
- 输入：多标签并发编辑事件流。
- 冲突语义：新增会话级并发窗口冲突（session overlap）。
- 解决策略：会话冲突先告警，再合并。

3. Stage C（跨设备协作）
- 输入：跨端同步消息与版本向量。
- 冲突语义：保留字段/实体/会话三级，并扩展网络时钟漂移治理。
- 解决策略：进入 CRDT/OT 选型，先保持冲突可检测与可回滚。

## 2. 冲突语义分层

1. 字段级（field）
- 同一实体同一字段值不一致。

2. 实体级（entity）
- 一端删除，另一端继续更新（delete-update）。

3. 会话级（session）
- 实时阶段内多个会话在重叠窗口并发写入。

## 3. 回滚预案

1. 回滚触发
- RC 报告出现 P0（检测率/一致率不达标）。

2. 回滚动作
- 协作功能降级到异步导入导出模式。
- 关闭实时冲突自动合并，保留人工审核。

3. 回滚验证
- 运行 `npm run gate:m8-collaboration`。
- 运行 `npm run gate:m8-collaboration:shadow`。
- 确认 RC 报告恢复到 `go`。

## 4. 技术选型结论（M8 阶段）

1. 当前结论
- M8 不直接引入 CRDT/OT 实装，先完成语义与门禁收口。

2. 进入 M9 前提
- 连续两轮 RC 报告 `P0=0`。
- 冲突一致率稳定 >= 99.9%。
