# 16周执行版路线图（成熟产品级平台 V2）

日期：2026-04-12  
范围：当前仓库主线（转写工作台 + AI 助手 + 语言资产）

## 0. 目标升级（按你给定的成熟产品对齐）

本路线图从“工程拆分清单”升级为“四层架构目标”，并以 16 周落地：

1. 数据内核层（Data Kernel）
2. 领域应用层（Domain Application）
3. 体验与交付层（Experience Delivery）
4. 扩展与协作层（Ecosystem Collaboration）

核心价值：
1. 不只拆文件，而是建立可验证、可回放、可回滚的内核治理。
2. 不让页面继续直连底层细节，改为应用服务编排。
3. 不把监控当展示板，升级为发布 SLO 门禁。

## 2. Phase A（P0，W1-W4）：稳定内核，不改行为

### A.1 目标
1. 让核心数据路径“可验证、可回放、可回滚”。
2. 把 E2E 从“补测试”升级为“发布门禁”。
3. 所有重构必须有行为对比证据，不允许隐式行为漂移。

### A.2 W1-W2（M1）执行方案

1. 交付物
- Schema 不变量测试模板（字段约束、索引约束、关系约束）。
- Migration 回放基线（至少覆盖最新 5 个版本迁移）。
- 行为快照模板（重构前后同输入同输出对比）。

2. 执行步骤
- 梳理 `src/db` 全集合不变量矩阵。
- 建立迁移回放测试入口，固定样本快照。
- 建立“重构不改行为”快照脚本与报告格式。

3. 量化验收指标
- 核心集合不变量覆盖率 >= 90%。
- 迁移回放成功率 = 100%。
- 重构行为差异（P0）= 0。

### A.3 W3-W4（M2）执行方案

1. 交付物
- 导入导出幂等性测试（同一快照多次导入结果一致）。
- 主路径 E2E 门禁清单（转写创建/编辑/保存/导出）。
- 发布前行为对比报告（自动生成）。

2. 执行步骤
- 增加 import/export round-trip 测试矩阵（含异常样本）。
- 把主路径 E2E 接入 CI 门禁。
- 建立“门禁失败 -> 阻断发布”规则。

3. 量化验收指标
- 导入导出幂等一致率 = 100%。
- 主路径 E2E 成功率 = 100%。
- 关键路径回归缺陷逃逸数 = 0。

4. 执行状态（2026-04-12）
- 执行记录：`docs/execution/archive/milestone-records/M2-执行记录-2026-04-12.md`
- 门禁清单：`docs/execution/release-gates/M2-主路径E2E门禁清单-2026-04-12.md`
- 自动报告：`docs/execution/archive/audits-auto/M2-主路径行为对比报告-自动生成.md`

### A.4 风险与回滚（Phase A）
1. 主要风险
- 迁移修复扩散影响历史数据兼容。
- 门禁上线初期导致交付速度下降。

2. 回滚策略
- 保留上一个可回放 schema checkpoint。
- 门禁分层启用：先告警 1 周，再切阻断。
- 任一 P0 行为差异出现，立即回滚至上个基线点。

## 3. Phase B（P1，W5-W8）：领域边界收口

### B.1 目标
1. 建立 Domain Application 层，页面不再拼底层 hooks。
2. 用绞杀者模式迁移，降低一次性替换风险。
3. 建立“反向依赖禁止”，防止耦合回流。

### B.2 W5-W6（M3）执行方案

1. 交付物
- 应用服务目录与契约定义（按转写、AI、语言资产分域）。
- 页面编排迁移清单（按优先级排序）。
- 依赖方向规则（lint/架构守卫）。

2. 执行步骤
- 为每个核心页面定义单一应用服务入口。
- 新增功能一律走应用服务。
- 建立禁止依赖规则：`page -> app -> domain -> data`。

3. 量化验收指标
- 新增需求经应用服务接入率 = 100%。
- 页面直接调用底层数据 API 的新增次数 = 0。
- 架构守卫违规项周环比下降 >= 50%。

4. 执行状态（2026-04-12）
- 应用服务骨架：`src/app/`（TranscriptionAppService / AiAppService / LanguageAssetsAppService + 公共契约）
- 页面迁移清单：`src/app/m3MigrationChecklist.ts`（57 文件，按域 × 优先级排序）
- 依赖方向守卫：`scripts/architecture-guard.config.mjs`（M3 规则段：app 层反向依赖禁止 + 新文件 db 导入冻结）
- 门禁命令：`npm run gate:m3-dependency-direction`
- 基线数据：page→db 46 文件 / page→services 34 文件 / component→db 29 文件

### B.3 W7-W8（M4）执行方案

1. 交付物
- 首批绞杀迁移完成（至少 2 个高流量页面）。
- 旧入口转发层与废弃计划。
- 领域服务回归测试集。

2. 执行步骤
- 把旧页面入口改为转发新应用服务。
- 标记废弃 API（deprecate + deadline）。
- 为域服务补齐回归（成功、失败、边界）。

3. 量化验收指标
- 核心页面应用服务接入率 >= 70%。
- 旧入口调用占比降至 <= 30%。
- 域服务回归通过率 = 100%。

4. 执行状态（2026-04-12）
- 执行记录：`docs/execution/archive/milestone-records/M4-执行记录-2026-04-12.md`
- 旧入口废弃计划：`docs/execution/plans/M4-旧入口废弃计划-2026-04-12.md`
- 首批迁移页面：`src/pages/useTranscriptionProjectMediaController.ts`、`src/pages/useTranscriptionSegmentMutationController.ts`
- 转发层：`src/app/transcriptionLegacyForwarding.ts`

- 门禁命令：`npm run gate:m4-domain-closure`

### B.4 风险与回滚（Phase B）
1. 主要风险
- 迁移并行期间双路径不一致。
- 领域划分不稳定导致重复抽象。

2. 回滚策略
- 保留旧入口开关，逐页切换。
- 发现业务偏差，先回切页面入口，再修应用服务。
- 不合格抽象当周回收，不进入下周。

## 6. 发布门禁分级（全周期执行）

1. P0（阻断发布）
- 主路径 E2E 失败。
- 数据一致性/迁移回放失败。
- 安全边界失效（权限越权、扩展越权）。

2. P1（允许灰度，必须告警）
- 关键性能指标超阈但有降级路径。
- 非核心路径回归失败但有替代流程。

3. P2（可延期）
- 轻微视觉/非关键交互差异。
- 文档/体验细节改进项。
