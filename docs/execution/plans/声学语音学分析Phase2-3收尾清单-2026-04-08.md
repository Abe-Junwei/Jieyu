## 2. 已完成项，不再重复列入待办

以下能力已经进入主线，不应再作为 Phase 2 / 3 的新增目标重复立项：

1. 本地 acoustic runtime、进度状态与 VAD runtime 状态。
2. waveform / spectrogram 的统一 readout 与最小 live inspector。
3. local slice 与归一化 tone / intensity contour。
4. spectral centroid、rolloff、zero-crossing、flatness、loudness、MFCC 等 descriptor 摘要。
5. 近似 Formant、元音空间摘要与最小散点表达。
6. 当前选区 CSV / JSON 导出。
7. IndexedDB 持久缓存、stale / fresh 分层与校准脚本基础骨架。

## 4. 明确不在本清单内的内容

以下内容属于 Phase 4 或更后阶段，不应混入本清单：

1. provider-neutral 的外部声学 backend 适配层。
2. `MFA`、`pyannote`、外部 WhisperX 对齐或远程研究 provider 的正式接入。
3. 把默认浏览器本地声学链路替换为外部服务。

## 5. 推荐实现顺序

如果只按最小返工路径推进，建议顺序如下：

1. 先做 Phase 2-A：冻结 + 双点比较 + inspector 工作区收口。
2. 再做 Phase 3-C：把只读 runtime 参数升级为可控研究参数。
3. 再做 Phase 3-A：补 Formant backend、平滑与校准闭环。
4. 最后做 Phase 3-B：把导出升级到 Praat / 科研互操作层。

原因：

1. 先收口 inspector，才能让后续 Formant 与导出有稳定承载面。
2. 先把参数与缓存语义做清楚，后续校准与导出才不会反复返工。
3. Formant 和科研导出都依赖校准口径，不适合在当前近似版本上直接做重 UI。

Phase 2 / 3 历史差距项已在当前范围内完成闭环；本清单保留为“曾经缺口 -> 已闭合”的追踪记录，不再作为新增待办来源。
