## 1. 执行目标

在不破坏当前转写主工作流的前提下，把声学语音学分析的 Phase 1 拆成三步：

1. 先打通数据层。
2. 再上主波形区只读增强。
3. 最后补频谱交互与缓存收口。

本清单只覆盖 Phase 1a / 1b / 1c 的执行目标；当前仓库虽已提前落入部分 Phase 2 / 3 本地能力，但这些内容不改变本清单的文档边界。

## 2. 共同约束

所有批次都必须满足：

1. 不新增第四种主显示模式。
2. 不破坏现有切分、播放、lasso、snap、note indicator 与 RegionActionOverlay 交互。
3. 新声学能力以结构化结果输出，不把逐帧数组直接灌进 AI prompt。
4. 复杂逻辑下沉到 service / worker / bridge，不塞回页面编排层。
5. Phase 1 默认不引入 GPL / AGPL 运行时依赖；如后续评估 Essentia.js 或其他受限许可证库，需单独完成合规确认。
6. Phase 1 的 F0 实现以 YIN 原论文与公开算法描述为依据，不对照 aubio GPL 源码做移植。
7. 若路线继续朝本地优先收口，则核心分析必须在浏览器本地可运行、可缓存、可离线复用，不以远程声学分析服务为前提。

## 3. Phase 1a：数据层先行

### 3.1 目标

完成声学分析底座，不接 UI。

### 3.2 建议新增文件

1. `src/services/acoustic/acousticAnalysis.worker.ts`
2. `src/services/acoustic/AcousticAnalysisService.ts`
3. `src/utils/acousticOverlayTypes.ts`

### 3.3 任务清单

1. 定义 worker 请求与响应协议。
2. 定义 `AcousticFrame`、`AcousticFeatureResult`、`AcousticAnalysisSummary`、`AcousticAnalysisConfig`。
3. 明确 PCM 获取路径：先支持从 WaveSurfer 已解码缓冲区取数，再预留 worker 独立解码入口。
4. 为 MVP 路径补 ready 时序保护：只有在 WaveSurfer ready 且已解码 PCM 实际可用时才允许启动分析，否则返回 `missing` 或延迟重试。
5. 选定 F0 算法口径：Phase 1 以 YIN 为主，pYIN 仅作后续兼容方向，不在 1a 增加额外复杂度。
6. 实现 intensity 基础指标与 reliability 字段。
7. 落最小缓存策略：`mediaId + analysisConfigHash`，并补单媒体上限、全局上限与 LRU / stale 优先淘汰规则。
8. 若保留本地优先路线，则在协议与缓存结构中预留算法版本、模型版本与本地持久化版本字段。

### 3.4 非目标

1. 不改工具栏。
2. 不改 waveform shell。
3. 不画任何 overlay。
4. 不接入 AI prompt。

### 3.5 验收

1. 在 80Hz、120Hz、200Hz、300Hz 的纯正弦波输入下，F0 估算中位绝对误差不高于 3Hz 或 2%，两者取较宽者。
2. 不同 config 能生成不同缓存键。
3. service 层不依赖页面组件状态。
4. WaveSurfer 尚未 ready 或 decoded PCM 为空时，service 不抛出未处理异常，而是返回可降级状态。

### 3.6 当前状态

1. 已完成：类型定义、核心算法、worker 协议、service 基础缓存、WaveSurfer / media URL 分析主路径、IndexedDB 持久缓存与 stale/fresh 分层淘汰、最小单元测试。
2. 仍可继续完善：更显式的 `missing` / `unstable` 细分状态建模，以及更多 PCM 来源复用策略。

## 6. AI 接入预留

Phase 1 不要求完整 AI 落地，但必须预留稳定结构：

1. `AcousticAnalysisSummary` 可被序列化为紧凑 prompt 字段。
2. `buildTranscriptionAiPromptContext` 后续可直接并列挂入 `acousticSummary`。
3. 不把逐帧数组直接注入 prompt context。

建议序列化格式：

```txt
acousticSummary(selectionSec=1.20-3.40, f0Min=120, f0Max=280, f0Mean=195, contourType=rising, intensityPeak=-12.0dB, reliability=0.82, hotspots=pitch_break@2.10s|intensity_peak@2.80s)
```
