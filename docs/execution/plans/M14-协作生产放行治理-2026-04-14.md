# M14 协作生产放行治理

## 目标

- 聚合 M10-M13 自动生成门禁报告，形成最终协作生产放行结论
- 将上游阶段的 `go` / `go-with-gray` / `no-go` 统一映射到发布阶段建议
- 为灰度与观察项生成回滚观察清单

## 交付物

- `src/collaboration/collaborationPromotionRuntime.ts`
- `src/collaboration/collaborationPromotionRuntime.test.ts`
- `scripts/check-m14-collaboration-promotion-foundation.mjs`
- `scripts/report-m14-collaboration-promotion-gate.mjs`
- `package.json` 中的 `check/test/report/gate:m14-*` 脚本

## 判定原则

1. 任一上游阶段为 `no-go` 或存在 P0，则 M14 为 `no-go`
2. 无 P0 但存在 `go-with-gray` 或 P1，则 M14 为 `go-with-gray`
3. 仅存在 P2 时允许 `go`，但需输出观察清单
