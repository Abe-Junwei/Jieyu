# M4 旧入口废弃计划（2026-04-12）

## 1. 目标

在绞杀迁移过渡期保留兼容入口，同时明确退场节奏，避免长期双轨。

## 2. 废弃范围

过渡转发层：`src/app/transcriptionLegacyForwarding.ts`

## 3. 映射关系（旧入口 -> 新入口）

1. `legacyResolveAutoSegmentCandidates` -> `getTranscriptionAppService().resolveAutoSegmentCandidates`
2. `legacyCreateProject` -> `getTranscriptionAppService().createProject`
3. `legacyImportAudio` -> `getTranscriptionAppService().importAudio`
4. `legacyDeleteProject` -> `getTranscriptionAppService().deleteProject`
5. `legacyDeleteAudio` -> `getTranscriptionAppService().deleteAudio`
6. `legacySplitSegment` -> `getTranscriptionAppService().splitSegment`
7. `legacyMergeAdjacentSegments` -> `getTranscriptionAppService().mergeAdjacentSegments`
8. `legacyDeleteSegment` -> `getTranscriptionAppService().deleteSegment`

## 4. 时间表

1. 2026-04-12：引入转发层并在首批高流量页面接入。
2. 2026-05-15：完成第二批页面迁移并把转发层调用比例降到 < 20%。
3. 2026-06-30：移除 `transcriptionLegacyForwarding.ts`，仅保留 app 服务入口。

## 5. 风险与门禁

1. 风险：并行期双入口语义漂移。
2. 控制：
- 所有旧入口仅做无逻辑转发，不承载业务分支。
- 新增/改动能力必须优先写在 app 服务层。
- 增加硬门禁：`npm run check:transcription-legacy-forwarding-usage`（禁止新增调用扩散）。
- 发布前执行 `npm run gate:m4-domain-closure`。
