# 1. 背景

四条史诗轨道（T1–T4）在 **§7 退出条件** 与执行记录上已与当前实现对齐。下一阶段宜收敛为 **可度量的一条主 OKR**，避免无目标扩 scope。

# 2. 主 OKR（草案）

**O：F4 Batch A 完成「至少 2 个新增后台/旁路写入口」的 capability isolation，并在审计与 governance 下可验证。**

- **KR1**：受控矩阵中至少 **2** 行从「待填」变为 **已受控**（见 [F4-扩展入口-受控矩阵-2026-05-05.md](./F4-扩展入口-受控矩阵-2026-05-05.md)）。
- **KR2**：每条新增入口具备 **Vitest** 决策断言 + **metadata action/reason** 可回放。
- **KR3**：`npm run gate:release-evidence:governance` 在含该审计源的 dry-run 下 **无结构回归**；必要时补 release evidence 卡片或索引摘要（与 F4 史诗 §5 一致）。

# 3. 本迭代可执行任务（3 条）

1. **盘点**：按矩阵 §3.1 grep 回填「其它后台」候选列表，并在矩阵中标注 **P0 / P1**。
2. **接入 #1**：选最高优先级旁路，按矩阵 §2 模板落地 + 单测。
3. **接入 #2**：第二条入口同上；更新 F4 史诗 **§4 实施清单** 状态与本文 KR 勾选。

# 4. 非目标（本 OKR 周期内不做）

- 全量重写 `useAiChat` 或 `AiChatCard`（架构热点另立 **技术债** 迭代）。
- 任意工具级自动 inverse patch（仍以 ADR-0025 B2 边界为准）。
