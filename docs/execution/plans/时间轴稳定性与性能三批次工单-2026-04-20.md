## 1. 工单 A（先做）

工单编号：A
标题：稳定 speakerVisualByUnitId 空值兜底引用，避免无意义重算

### 1.1 目标

1. 消除默认空对象导致的引用抖动。
2. 降低无意义 useMemo 失效概率。
3. 不改变业务行为与渲染结果。

### 1.2 影响范围

1. [src/components/TranscriptionTimelineVerticalView.tsx](src/components/TranscriptionTimelineVerticalView.tsx)
2. 可选同批收口：[src/components/TranscriptionTimelineTextOnly.tsx](src/components/TranscriptionTimelineTextOnly.tsx)
3. 相关测试：[src/components/TranscriptionTimelineVerticalView.test.tsx](src/components/TranscriptionTimelineVerticalView.test.tsx)

### 1.3 非目标

1. 不改分组算法。
2. 不改 speaker 数据来源。
3. 不做任何样式与交互改造。

### 1.4 执行步骤

1. 移除参数解构中的默认空对象写法。
2. 在模块级引入稳定 EMPTY_RECORD 常量。
3. 在组件内部进行空值归一化。
4. 确认依赖数组仅依赖真实输入引用。
5. 运行目标测试并做一次手工 smoke。

### 1.5 验收标准

1. speaker 标签显示与分组行为完全一致。
2. 目标测试通过。
3. 无新增 lint 或 type 问题。

### 1.6 建议验证命令

1. npx vitest run src/components/TranscriptionTimelineVerticalView.test.tsx

### 1.7 风险与回滚

1. 风险等级：低。
2. 风险点：个别分支依赖 undefined 与空对象差异。
3. 回滚：仅回滚 [src/components/TranscriptionTimelineVerticalView.tsx](src/components/TranscriptionTimelineVerticalView.tsx) 与可选收口文件。

### 1.8 PR 建议

1. fix(timeline): stabilize empty speaker visual fallback reference

## 2. 工单 B（第二批）

工单编号：B
标题：媒体轨分层可见语段预计算，减少层循环内重复 filter

### 2.1 目标

1. 优化多层多语段下的渲染与筛选性能。
2. 保持功能行为不变。
3. 产出可复核的简单性能回归结果。

### 2.2 影响范围

1. [src/components/TranscriptionTimelineHorizontalMediaLanes.tsx](src/components/TranscriptionTimelineHorizontalMediaLanes.tsx)
2. 测试：[src/components/TranscriptionTimelineHorizontalMediaLanes.test.tsx](src/components/TranscriptionTimelineHorizontalMediaLanes.test.tsx)

### 2.3 非目标

1. 不改重叠组算法语义。
2. 不改录音回填逻辑。
3. 不改时间轴交互协议。

### 2.4 执行步骤

1. 预计算 layerId 与 speakerFilterKey 维度的可见语段映射。
2. 层循环改为直接命中预计算映射。
3. 保持折叠、展开、筛选语义一致。
4. 补齐最小必要测试覆盖。
5. 做一次前后对比的简单性能回归。

### 2.5 简单性能回归口径

1. 固定同一大文档 fixture（多层、多语段）。
2. 采集三项指标：首渲染、切换 speaker 过滤、展开重叠组。
3. 每项执行 20 次，记录 p50 与 p95。
4. PR 描述附 before 与 after 对比表。

### 2.6 验收标准

1. 媒体轨行为与现有测试预期一致。
2. 至少一项关键操作 p95 有明确下降。
3. 无新增功能回归。

### 2.7 建议验证命令

1. npx vitest run src/components/TranscriptionTimelineHorizontalMediaLanes.test.tsx

### 2.8 风险与回滚

1. 风险等级：中。
2. 风险点：筛选条件漏算或缓存失效导致显示异常。
3. 回滚：仅回滚预计算映射相关提交，保留测试增强提交。

### 2.9 PR 建议

1. perf(timeline): precompute visible segments per layer and speaker filter
