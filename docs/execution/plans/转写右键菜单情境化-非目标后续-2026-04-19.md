# 转写右键菜单情境化：非目标后续任务（跟踪清单）

本文档承接主方案中**明确列为非目标**、但产品若希望「全表面统一情境」时需单独立项的工作。主轨已实现：`ContextMenuState.menuSurface` / `layerType`、`buildTranscriptionUnitContextMenuItems`、主转写轨层头 `headerMenuPreset` 与工具栏 `uttOps` 与翻译层对齐等。

## 1. 波形空白区右键

- **现状**：波形 region 上语段右键已写入 `menuSurface: 'waveform-region'`；**空白区域**无统一 `ctxMenu` 策略。
- **建议方向**：定义 `menuSurface: 'waveform-empty'`（或等价命名），与 `waveform-region` 分支并列；项集限于缩放/播放/选区等与空白语义一致的命令。
- **依赖**：波形交互控制器、与 `TranscriptionOverlays` 或独立 `ContextMenu` 的接入点。

## 2. 侧栏（如 `SidePaneSidebar`）层 / 语段右键

- **现状**：侧栏若有独立 context menu，与转写工作区 `ctxMenu` 链路分离。
- **建议方向**：复用同一 `ContextMenuState` 形状或共享 `buildTranscriptionUnitContextMenuItems` 的输入投影层，避免第三套拼装逻辑。

## 3. MediaLanes 及其他时间轴变体

- **现状**：`TranscriptionTimelineHorizontalMediaLanes` 翻译层头已显式 `headerMenuPreset="layer-chrome"`；媒体模式主转写轨经 `TranscriptionTimelineMediaTranscriptionLane` 使用 `layer-chrome-plus-track`。
- **后续**：若 MediaLanes 增加**轨内单元**右键或其它入口，应对齐 `menuSurface` / `layerType` 写入规范。

## 4. 验收与治理

- 每完成一条子任务：补 vitest（存在性/不存在性）与手动回归矩阵一行。
- 架构决策若涉及跨表面统一：考虑 `docs/adr/` 短文记录「情境枚举」扩展规则。
