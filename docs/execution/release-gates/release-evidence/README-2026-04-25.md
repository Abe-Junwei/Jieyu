# release-evidence latest alignment (2026-04-25)

## Dual gate model（权威口径）

发布与默认 CI 对 release evidence 采用 **两道闸**，**顺序执行、均需通过**（见 ADR [`0024-release-evidence-core-governance-dual-gate.md`](../../../adr/0024-release-evidence-core-governance-dual-gate.md)）：

| 闸口 | npm 入口（示例） | 职责 |
|------|------------------|------|
| **Core（enforce）** | `npm run -s gate:release-evidence:core` / `gate:release-evidence:full` | `generate-release-evidence-bundle.mjs` 在 **enforce** 下生成整包；结构、卡片契约、`degraded` 等硬断言。 |
| **Governance（dry-run + check）** | `npm run -s gate:release-evidence:governance` | 再生成 dry-run JSON，由 `check-release-evidence-governance.mjs` 校验 **costGuard.trend** 与 **actionApprovalCenter** 策略阈值。 |

二者**不是**互斥的「单一口径」；运维话术应写 **core + governance**。

## CostGuard Trend Strict Gate
- Local strict commands:
  - `npm run -s gate:release-evidence:core:require-cost-guard-trend-ready`
  - `npm run -s gate:release-evidence:full:require-cost-guard-trend-ready`
- Local structure-only command:
  - `npm run -s gate:release-evidence:cost-guard-trend`
- CI workflow_dispatch toggle:
  1. Open Actions -> CI workflow.
  2. Click Run workflow.
  3. Set `require_cost_guard_trend_ready=true`.
  4. Keep `run_release_evidence_full` optional (`false` for core-only run, `true` for full run).
- Enablement policy:
  - First run in shadow mode via workflow_dispatch toggle only.
  - Promote to regular strict gate only after two consecutive release cycles with `costGuard.trend.compareReady=true` and `pointCount>=2`.
  - Roll back to structure-only gate if any cycle regresses to `compareReady=false`.

## Action Approval Gate (M1)
- Local structure-only command:
  - `npm run -s gate:release-evidence:action-approval`
- Local strict command:
  - `npm run -s gate:release-evidence:action-approval:strict`
- Usage scope:
  - These standalone action-approval gate commands are retained for manual diagnostics and focused troubleshooting.
  - They are not the default CI entrypoint after unified governance gate rollout.
- Strict thresholds (current defaults):
  - `max-pending-rate=0.70`
  - `max-blocked-rate=0.80`
  - `max-failed-rate=0.20`
  - `require-high-risk-signal=true` (must include `riskTiers.high > 0`)
- Governance policy:
  - Start with structure-only gate in daily runs.
  - Promote strict gate after two consecutive cycles without false positives.
  - Roll back to structure-only if data quality regresses (e.g. missing risk tiers).

## Unified Governance Gate
- Local structure-only command:
  - `npm run -s gate:release-evidence:governance`
- Local strict command:
  - `npm run -s gate:release-evidence:governance:strict`
- **样本量下限（strict）**：`check-release-evidence-governance.mjs` 支持 `--min-approval-total=<n>`（整数 `>=1`），要求 `actionApprovalCenter.summary.total >= n`，避免极小 `total` 偶然满足比例类阈值。`check:release-evidence:governance:strict` 已默认附带 **`--min-approval-total=2`**（与当前 strict 种子 fixture 一致）。非 strict 默认不传，以免阻断仅含 0–1 条审批样本的本地 dry-run。
- **何时开 strict**：在 Actions 手动 `workflow_dispatch` 勾选 `require_cost_guard_trend_ready` 或 `require_action_approval_strict`（见 `.github/workflows/ci.yml`）；发布前本地跑 `gate:release-evidence:governance:strict` 做同等收紧。
- Strict seed source:
  - `docs/execution/release-gates/release-evidence/release-evidence-governance-strict-seed.ndjson`
  - Used to guarantee two-day trend comparability plus high-risk approval signal in strict dry-run gate.
- CI wiring:
  - `release-evidence-core-gate` and `release-evidence-full-gate` now use governance gate as the single combined check entrypoint (cost-guard + action-approval).
- CI policy:
  - Treat governance gate as the authoritative release-evidence gate in CI.
  - Keep standalone action-approval gate commands for local/manual drill-down only.
- Combined strict semantics:
  - Requires `costGuard.trend.compareReady=true`.
  - Enforces Action Approval thresholds (`pending/blocked/failed` rates + `riskTiers.high > 0`).
