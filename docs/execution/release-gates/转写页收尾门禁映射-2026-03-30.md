# 转写页收尾门禁映射（2026-03-30）

> 目的：把“执行包 -> 风险点 -> 自动化门禁 -> 手工验收 -> 封账产物”映射到仓库现有资产，避免收尾阶段只做代码不做证据。

> 口径更新（2026-04-25）：正式改口径为“新自动门禁体系替代 CASE 勾选阻断”。手工 CASE 保留为抽样验收材料，不再作为封账必经阻断项。

## 一、总规则

1. 每个执行包都必须同时满足代码、自动化、文档、手工验收四类产物。
2. 任一执行包如果缺少对应门禁，不得标记为完成。
3. 若本次改动命中重模块，除聚焦测试外还必须执行 `npm test` 与 `npm run typecheck`。
4. 命中转写首屏、语音、AI、PDF 的改动，默认追加 `npm run build`；必要时追加 `npm run build:guard`。
