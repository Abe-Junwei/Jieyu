# M6 发布门禁清单（2026-04-12）

里程碑：16周执行版路线图 M6（W11-W12）  
目标：将 SLO 阈值转为可执行分级门禁，并自动产出 go/no-go 判定。

## 1. 分级规则

## P0（阻断）
- build 失败（构建成功率 < 100%）。
- 主路径成功率缺失（`business.e2e.main_path_success_rate` 无可用样本）。
- 主路径成功率低于阈值（`p95 < 0.99`）。

## P1（灰度）
- AI 首包延迟缺样本，或 `ai.chat.first_token_latency_ms.p95 > 8000ms`。
- 转写关键操作时延缺样本，或 `business.transcription.segment_action_latency_ms.p95 > 300ms`。

## P2（观察）
- 样本量不足（默认建议每个核心指标 `sampleCount >= 20`）。

## 2. 门禁入口

1. M5 观测门禁：`npm run gate:m5-observability`
2. M6 发布门禁（强制）：`npm run gate:m6-release`
3. M6 发布门禁（shadow）：`npm run gate:m6-release:shadow`
4. CI enforce 作业：`.github/workflows/ci.yml` -> `m6-release-gate`

## 3. 自动产物

1. 自动判定报告：`docs/execution/release-gates/M6-发布门禁判定报告-自动生成.md`
2. 指标事件流水：`docs/execution/audits/M5-观测指标事件流水-v1.ndjson`
3. 趋势报告：`docs/execution/audits/M5-观测趋势报告-v1-YYYY-MM-DD.md`

## 4. 判定语义

1. enforce 模式：出现任一 P0 即 no-go，并返回非零退出码。
2. shadow 模式：出现 P0 仅告警，不阻断流水线。
3. 若无 P0 但存在 P1：判定为 go-with-gray（允许灰度发布）。
4. M6 判定默认按本次门禁 `runId` 过滤事件，避免历史样本混算。
5. 样本量判定使用 `sampleCount`（优先取事件标签，其次回退到 1）。
6. synthetic-test-probe 仅作为来源说明，不参与 P0/P1/P2 分级。

## 5. 执行顺序（发布前）

1. 执行 `npm run gate:m6-release`（强制门禁）。
2. 查看 `docs/execution/release-gates/M6-发布门禁判定报告-自动生成.md`。
3. 若结论为 `no-go`，先修复 P0 项后再重跑；若为 `go-with-gray`，按灰度策略发布。
