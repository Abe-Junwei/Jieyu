# M2 主路径 E2E 门禁清单（2026-04-12）

里程碑：16周执行版路线图 M2（W3-W4）  
目标：将“转写创建 / 编辑 / 保存 / 导出”主路径收敛为可执行门禁，并在 CI 中阻断失败发布。

## 1. 门禁入口

1. 本地命令：`npm run gate:m2`
2. CI 作业：`.github/workflows/ci.yml` -> `m2-mainpath-gate`；Playwright 关键路径见同文件 `e2e-critical-paths-pr`（PR 仅 Chromium）与 `e2e-critical-paths-full`（push 至 `main`/`dev` 三引擎）。
3. 失败语义：任一套件失败即返回非零退出码，按 P0 阻断发布。

## 2. 覆盖矩阵

| 路径 | 对应测试文件 | 验收点 |
|---|---|---|
| 转写创建 | `src/pages/useTranscriptionSegmentCreationController.test.tsx` | 句段创建、约束路由、失败提示 |
| 转写编辑 | `src/pages/useTranscriptionSegmentMutationController.test.tsx` | 拆分/合并/删除、约束保护、错误透出 |
| 转写保存 | `src/hooks/useTranscriptionPersistence.test.tsx` | 冲突保护、写入一致性、持久化路径 |
| 转写导出 | `src/hooks/useImportExport.export.test.tsx` | EAF 等导出路径、LayerUnit 导出行为 |
| 导入导出幂等 | `src/db/importExportRoundTripIdempotency.test.ts` + `src/db/importDatabaseFromJson.test.ts` | 同快照重复导入一致、异常样本不污染 |

## 3. 判定规则

1. 阻断（P0）：
- 任一套件 `failed`。
- 任一矩阵指标未达 100% 通过。

2. 允许继续：
- 上述全部矩阵测试 `passed`。
- 自动报告成功落盘。

3. 证据文件：
- 自动行为对比报告：`docs/execution/archive/audits-auto/M2-主路径行为对比报告-自动生成.md`
- 原始 JSON：`.tmp/m2/*.vitest.json`

## 4. 执行顺序（发布前）

1. 执行 `npm run gate:m2`。
2. 查看自动报告中“发布判定”。
3. 若为“阻断”，先修复失败测试，再重新执行门禁。
