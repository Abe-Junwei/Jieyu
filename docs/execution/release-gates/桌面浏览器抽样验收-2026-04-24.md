# 桌面浏览器抽样验收（国内双核）

**频率**：建议 **每季度** 或 **重大发版前** 至少执行一次；若某版本改动涉及媒体、权限、存储或路由，应追加一轮。

**事实源**：产品级承诺见 [桌面端浏览器支持策略](../../architecture/桌面端浏览器支持策略.md)。

## 环境

| 项 | 要求 |
| --- | --- |
| 操作系统 | Windows 10/11 |
| 浏览器 | 360 安全/极速、QQ 浏览器、搜狗高速浏览器 **之一**（轮换即可） |
| 模式 | **仅** Chromium **极速 / 高速** 模式；**禁止** IE / 兼容模式 |
| 构建 | 与待发版相同的 **生产构建**（`npm run build` + `npm run preview` 或等价托管） |

## 最小用例（勾选）

- [ ] 打开首页 `/`，侧栏与设置可进入。
- [ ] 进入 `/transcription`，完成加载后出现工作区（波形/时间轴区域或空项目提示其一即可）。
- [ ] 浏览器控制台 **无**持续刷新的红色 CSP / 脚本加载失败（偶发扩展噪声可注明并排除）。
- [ ] （可选）导入一小段测试音频或打开已有项目，确认波形或占位不白屏。

## 真 Safari（macOS，与 WebKit E2E 互补）

**频率**：与上表同轮；或任一版本改动 **媒体 / 权限 / 存储 / 路由 / 构建入口** 时追加。

| 项 | 要求 |
| --- | --- |
| 系统 | 仍接收安全更新的 **macOS**；Safari **大版本** 不低于产品策略建议（与 [桌面端浏览器支持策略](../../architecture/桌面端浏览器支持策略.md) 一致）。 |
| 构建 | 与待发版相同的 **生产构建**（`npm run build` + `npm run preview` 或等价托管）。 |

**最小用例（勾选）**

- [ ] 打开首页 `/` 与 `/transcription`，与上文「最小用例」等价通过。
- [ ] **翻译条录音**（若该版本涉及录音）：录一段停止后，侧栏/媒体列表出现条目，**文件名后缀** 与浏览器实际编码一致（Chrome 多为 `.webm`，Safari 常见 `.m4a` / `audio/mp4`）。
- [ ] 控制台无 **CSP / eval** 类持续报错（与 CI Firefox CSP 用例目标一致）。

## 证据

- 记录：浏览器 **精确版本号**、**内核版本**（如有）、测试日期、测试人。
- 附件：关键界面截图 1～2 张或短视频链接（存于团队约定位置即可）。

## 与 CI 的关系

- **CI**：`quality` 在完整 `npm test` 前会先跑 **`npm run check:css-compat`**（`backdrop-filter` 按文件探针）；PR 上跑 **Chromium** E2E；`main` / `dev` 推送跑 **Chromium + Firefox + WebKit**（见 `.github/workflows/ci.yml`）。WebKit **不等于** 桌面 Safari 的全部行为。
- **本清单**：覆盖 **国内套壳浏览器 + 极速模式** 与 **真 Safari**，CI 不替代，仅互补。
