# 面板化语言资产 Phase 1 任务拆解（2026-04-07）

## 1. Phase 1 目标

在不重写现有语言输入交互语义的前提下，完成以下收敛：

1. `LanguageIsoInput` 不再依赖同步全量 `langMapping` 搜索。
2. 语言名称 baseline 不再必须以超大 TS 对象进入前端运行时。
3. 语言元数据、正字法、桥接三类面板后续都可以复用同一套异步语言搜索基础设施。

本阶段只处理“语言搜索能力底座”，不一次性改完整个面板 UI。

## 2. 非目标

本阶段不做：

1. 不重写 `LanguageIsoInput` reducer 的编辑语义。
2. 不一次性重构整个 `langMapping.ts`。
3. 不处理正字法 / 桥接 baseline 的 JSON 分片。
4. 不直接删除旧同步搜索实现，先走双轨兼容。

## 3. 当前代码切入点

### 3.1 输入组件

当前 [src/components/LanguageIsoInput.tsx](src/components/LanguageIsoInput.tsx) 依赖：

1. [src/utils/languageInputReducer.ts](src/utils/languageInputReducer.ts)
2. 其中 reducer 直接调用 [src/utils/langMapping.ts](src/utils/langMapping.ts) 的 `searchLanguageCatalog`

这意味着建议列表必须同步可得，进而把大语言名基线拖进运行时。

### 3.2 baseline 数据与现成 JSON

生成脚本 [scripts/generate-language-name-indexes.mjs](scripts/generate-language-name-indexes.mjs) 已经产出：

1. `public/data/language-support/language-display-names.core.json`
2. `public/data/language-support/language-query-aliases.json`
3. `public/data/language-support/language-query-index.{locale}.json`

Phase 1 应优先复用这些现有 JSON，不要重新设计数据源。

## 4. 推荐实施顺序

## Task 1：引入异步语言搜索服务壳层

新增：

1. `src/services/LanguageCatalogSearchService.ts`

首版职责：

1. 懒加载 `language-query-index.{locale}.json`
2. 懒加载 `language-display-names.core.json`
3. 提供搜索候选接口
4. 提供按 ID / 批量 ID 查详情接口
5. 合并 runtime cache 中的用户覆盖结果

建议接口：

```ts
export type LanguageSearchSuggestion = {
  id: string;
  languageCode: string;
  primaryLabel: string;
  secondaryLabel?: string;
  matchedLabel: string;
  matchedLabelKind: 'local' | 'native' | 'english' | 'alias' | 'code';
  matchSource: string;
  rank: number;
};

export async function searchLanguageCatalogSuggestions(...): Promise<LanguageSearchSuggestion[]>
export async function lookupLanguageCatalogEntryById(...): Promise<LanguageCatalogEntry | null>
export async function lookupLanguageCatalogEntriesByIds(...): Promise<LanguageCatalogEntry[]>
```

首版要求：

1. query 为空时不发请求。
2. 按 locale 缓存 query index。
3. 详情 core JSON 全局单例缓存。
4. exact code / alias 命中优先。
5. 对用户覆盖条目优先排序。

验收：

1. service 层测试可证明无需 import `languageNameCatalog.generated.ts` 也能返回结果。

## Task 2：抽出 `LanguageIsoInput` 建议源适配层

新增：

1. `src/hooks/useLanguageSearchSuggestions.ts`

职责：

1. 接收 `query + locale + context`
2. 做 debounce
3. 管理取消过时请求
4. 提供 `loading / suggestions / error`

建议接口：

```ts
type UseLanguageSearchSuggestionsInput = {
  query: string;
  locale: LanguageSearchLocale;
  enabled?: boolean;
  projectLanguageIds?: readonly string[];
};
```

实现约束：

1. 两字符 provisional code 输入时，不要提前触发“强收敛”为最终选项。
2. 输入变短的退格路径必须保留当前 reducer 语义。
3. 请求回包晚于新 query 时必须丢弃。

验收：

1. hook 单测覆盖 debounce、取消、缓存命中、过时结果丢弃。

## Task 3：让 reducer 接受“外部 suggestions 注入”

改造：

1. [src/utils/languageInputReducer.ts](src/utils/languageInputReducer.ts)

目标不是重写 reducer，而是移除它对同步 `searchLanguageCatalog(...)` 的硬编码依赖。

建议策略：

1. reducer 仍负责状态机。
2. 搜索结果由外部通过新 event 注入。
3. reducer 保留自动选择、blur、code 规范化逻辑。

新增 event 建议：

1. `suggestionsRequested`
2. `suggestionsResolved`
3. `suggestionsFailed`

最低改法：

1. 对 `nameChanged` 不再直接调用 `searchLanguageCatalog`
2. 只更新 draft 和 status
3. 由组件层 `useEffect` 监听 query 后触发异步 suggestions 回填

验收：

1. 现有 `languageInputReducer` 回归测试不退化。
2. reducer 不再 import `searchLanguageCatalog`。

## Task 4：改造 `LanguageIsoInput` 组件接入异步建议

改造：

1. [src/components/LanguageIsoInput.tsx](src/components/LanguageIsoInput.tsx)

做法：

1. 组件内部接入 `useLanguageSearchSuggestions`
2. 把 hook 结果通过 reducer event 注入
3. suggestion list UI 基本保持原样

UI 语义保持：

1. 键盘上下高亮不变
2. Enter 选中不变
3. blur 行为不变
4. code 输入合法性语义不变

新增 UI 状态：

1. suggestion loading hint
2. 空结果时稳定占位，不抖动

验收：

1. 现有 `LanguageIsoInput` 交互测试继续通过。
2. 输入名称时建议仍能出现。
3. 输入 code 时不因异步回流打断编辑。

## Task 5：为 `langMapping` 保留只读同步兜底，但退出主搜索路径

改造：

1. [src/utils/langMapping.ts](src/utils/langMapping.ts)

目标：

1. 保留 `resolveLanguageCodeInput`、`formatLanguageDisplayName` 等同步工具。
2. 逐步淡化 `searchLanguageCatalog` 的一线调用角色。
3. 给旧调用方留兼容窗口。

建议：

1. 在注释中明确 `searchLanguageCatalog` 为 legacy sync fallback。
2. 新入口一律走 `LanguageCatalogSearchService`。

验收：

1. 没有新代码再直接把 `searchLanguageCatalog` 用作输入建议源。

## Task 6：先把语言元数据页面的一个入口切到新搜索能力

首个试点建议选择：

1. [src/pages/LanguageMetadataWorkspacePage.tsx](src/pages/LanguageMetadataWorkspacePage.tsx) 中“新建自定义语言 / 编辑语言代码”入口

验收：

1. 语言元数据页相关测试继续通过。
2. 搜索建议改走异步服务后，不再需要全量 baseline 同步装载。

## 5. Phase 1 结束时的代码状态

完成后应满足：

1. `LanguageIsoInput` 主搜索路径不再同步依赖 `searchLanguageCatalog`
2. 语言建议结果来自异步 service
3. baseline 数据通过 public JSON 懒加载
4. `langMapping` 仍保留同步解析工具，但不再承担主建议源职责

## 6. 建议的最小提交切分

建议拆成 4 个提交，而不是一个超大提交：

1. `feat(language-search): add async catalog search service`
2. `refactor(language-input): decouple reducer from sync catalog search`
3. `feat(language-input): wire async suggestions into LanguageIsoInput`
4. `refactor(language-metadata): migrate first panel entry to async search`

## 7. 当前最推荐的起手实现

如果现在直接开始写代码，最稳的第一刀是：

1. 先实现 `LanguageCatalogSearchService`
2. 先不碰 reducer
3. 先写 service 测试，验证 JSON 懒加载 + 查询排序 + runtime cache 合并

原因：

1. 这一步不破坏现有输入语义。
2. 可以先把数据入口和搜索能力独立出来。
3. 后续 reducer / 组件改造时就只是在切换调用源，而不是一边造底座一边改交互。
