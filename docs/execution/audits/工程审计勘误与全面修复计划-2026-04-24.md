## 1. 事实勘误

以下结论曾出现在审计摘要或对话记录中；**以当前仓库为准**。


| 原表述（错误或不完整）                                                                                                           | 勘误后的结论                                                                                                                                                                                                                 | 依据（便于复查）                                       |
| --------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------- |
| 「缺少 CI / `.github/workflows` 目录不存在」                                                                                   | 仓库**已存在** GitHub Actions：`[.github/workflows/ci.yml](../../../.github/workflows/ci.yml)`，对 `pull_request` 与 `main` / `dev` 的 `push` 触发；内含多 job（如 `quality`、`docs-governance`、门禁与 E2E 等）。                               | 打开该 workflow 即可核对 `on:` 与 `jobs:`。             |
| 「`vite.config.ts` 的 `manualChunks` 只引用 `@xenova/transformers`，与 `package.json` 的 `@huggingface/transformers` 矛盾、属死代码」 | `manualChunks` 中**同时**匹配 `/@huggingface/transformers/` 与 `/@xenova/transformers/`，统一打入 `transformers-vendor`。依赖已迁移到 `@huggingface/transformers`；`@xenova` 分支为**历史兼容**（若 lockfile 中已无 `@xenova`，该分支可删，但不宜表述为「整段配置未更新」）。 | `vite.config.ts` 内 `transformers-vendor` 条件。   |
| 「转写恢复快照仅存 `localStorage`，受 5–10MB 限额」                                                                                 | **错误。** 崩溃/恢复快照由 `[src/services/SnapshotService.ts](../../../src/services/SnapshotService.ts)` 写入**独立 IndexedDB 库** `jieyu_recovery` 的 `snapshots` 表；并有 UTF-8 序列化大小上限（默认 8MB），超限会跳过并清理陈旧快照。                            | 读 `RecoveryDexie` 与 `saveRecoverySnapshot` 实现。 |
