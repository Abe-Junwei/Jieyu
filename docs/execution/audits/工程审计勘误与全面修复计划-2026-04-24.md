# 工程审计勘误与全面修复计划

本文档纠正 2026-04-24 前后多 Agent「工程健康度 / 性能 / 离线」审计中的**事实错误或不完整表述**，并给出与仓库现状对齐的**全面修复路线图**。与代码缺陷类整改（`[docs/remediation-plan-2026-04-24.md](../../remediation-plan-2026-04-24.md)` 中的 CRITICAL / HIGH / ARCH）**正交**：可并行跟踪、分别验收。

## 1. 事实勘误

以下结论曾出现在审计摘要或对话记录中；**以当前仓库为准**。


| 原表述（错误或不完整）                                                                                                           | 勘误后的结论                                                                                                                                                                                                                 | 依据（便于复查）                                       |
| --------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------- |
| 「缺少 CI / `.github/workflows` 目录不存在」                                                                                   | 仓库**已存在** GitHub Actions：`[.github/workflows/ci.yml](../../../.github/workflows/ci.yml)`，对 `pull_request` 与 `main` / `dev` 的 `push` 触发；内含多 job（如 `quality`、`docs-governance`、门禁与 E2E 等）。                               | 打开该 workflow 即可核对 `on:` 与 `jobs:`。             |
| 「`vite.config.ts` 的 `manualChunks` 只引用 `@xenova/transformers`，与 `package.json` 的 `@huggingface/transformers` 矛盾、属死代码」 | `manualChunks` 中**同时**匹配 `/@huggingface/transformers/` 与 `/@xenova/transformers/`，统一打入 `transformers-vendor`。依赖已迁移到 `@huggingface/transformers`；`@xenova` 分支为**历史兼容**（若 lockfile 中已无 `@xenova`，该分支可删，但不宜表述为「整段配置未更新」）。 | `vite.config.ts` 内 `transformers-vendor` 条件。   |
| 「转写恢复快照仅存 `localStorage`，受 5–10MB 限额」                                                                                 | **错误。** 崩溃/恢复快照由 `[src/services/SnapshotService.ts](../../../src/services/SnapshotService.ts)` 写入**独立 IndexedDB 库** `jieyu_recovery` 的 `snapshots` 表；并有 UTF-8 序列化大小上限（默认 8MB），超限会跳过并清理陈旧快照。                            | 读 `RecoveryDexie` 与 `saveRecoverySnapshot` 实现。 |
| 「`dexie.open()` 完全无 catch、必白屏」                                                                                        | **不完整。** `_createDb()` 内 `await dexie.open()` 无局部 try/catch，但 `getDb()` 对单例 Promise 有 `.catch` 清理并重抛；**仍缺**面向用户的「库损坏 / 无法打开」专用 UI 与恢复引导（建议单独立项，勿与「完全无错误传播」混为一谈）。                                                       | `src/db/engine.ts` 中 `getDb` / `_createDb`。    |


以下表述**属实**，审计可保留：

- `@tanstack/react-virtual` 此前在 `src/` 无 import；**现已**用于侧栏语段列表（`SidePaneSidebarSegmentList`，≥48 条虚拟化）；横向时间轴仍以自定义裁剪为主。
- ~~`@types/language-tags` 误放在 `dependencies`~~（已移至 `devDependencies`）。
- `src/i18n/index.ts` **双语同文件**，体量大，默认全量进入引用方 bundle。
- `index.html` **无** `preload` / `preconnect` 资源提示（需在 CSP 允许前提下设计）。
- ~~**无** PWA~~：已最小接入 `vite-plugin-pwa`（见下 Phase C-1/C-2）。
- **无** `vite-plugin-compression` 等预压缩产物；`build.minify` 依赖 Vite 默认即可，非「未压缩」。
- `country-state-city` **node_modules 约 17MB**；代码侧仅少数文件引用。
- `CollaborationClientStateStore` 写 `localStorage` 时 `**catch` 静默吞错**（含 `QuotaExceededError`），出站状态可能丢失（**属实**，与恢复快照存储介质无关）。
- ~~`MaterialSymbol` **无** `aria-hidden`~~（已默认 `aria-hidden`，可 props 覆盖）；基础对话框壳默认 **无** `role="dialog"` / `aria-modal` 等（可通过调用方传入 `divProps` 补，但默认态薄弱）。
- Playwright E2E **未**集成 `@axe-core/playwright`。
- ~~无 `.nvmrc` / `package.json` 无 `engines`~~（已补 `.nvmrc` 与 `engines`）。
- ~~根目录无 `CONTRIBUTING.md`~~（已新增）；**仍无** `.husky`（与 CI 存在不矛盾）。

## 2. 全面修复计划（按主题与阶段）

阶段划分原则：**先低风险、高确定收益** → **再架构级或需产品决策** → **最后大颗粒（PWA、虚拟列表）**。每项建议标注责任域（前端 / 构建 / 文档 / 运维）。

### 2.1 Phase A — 依赖与类型卫生（约 0.5–1 周）


| ID  | 动作                                                                                                                                                                                                               | 验收标准                                                                                                                                                                                          |
| --- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| A-1 | 将 `@types/language-tags` 从 `dependencies` 移至 `devDependencies`。                                                                                                                                                  | **已落地**：`package.json` 已调整；`npm ls --omit=dev` 不含该类型包。                                                                                                                                                          |
| A-2 | 核实 `iso-639-3` npm 包是否仍被运行时代码或脚本引用；若无，从 `dependencies` 移除并更新锁文件。                                                                                                                                                 | **已落地**：`src/` 无 import；仅 `scripts/generate-language-name-indexes.mjs`、`scripts/build-iso6393-country-baselines.mjs` 使用。包已移至 **`devDependencies`**（运行时使用生成种子 `iso6393Seed`）；`npm ci` + 数据脚本在 CI 仍可用。                                                                                                                                                               |
| A-3 | **二选一**：移除未使用的 `@tanstack/react-virtual`，**或**在转写/语料长列表路径落地虚拟化（与 `[docs/adr/0017-jieyu-core-workbench-architecture-adaptation.md](../../adr/0017-jieyu-core-workbench-architecture-adaptation.md)` 路线图一致时需设计评审）。 | **已部分落地（2026-04-24）**：`[SidePaneSidebarSegmentList.tsx](../../../src/components/SidePaneSidebarSegmentList.tsx)` 在语段数 ≥ 48 时对侧栏列表使用 `useVirtualizer` + `measureElement`；横向时间轴仍非虚拟列表，可按性能数据再扩。 |
| A-4 | 若 `package-lock.json` 中已不存在 `@xenova/transformers`，删除 `vite.config.ts` 中对 `/@xenova/transformers/` 的匹配分支，避免误导。                                                                                                   | **已落地**：`manualChunks` 仅匹配 `@huggingface/transformers`；`npm run build` 通过。                                                                                                                                                                   |
| A-5 | 评估 `country-state-city`：保留 / 换轻量数据源 / 生成静态 JSON；**禁止**仅因体积删功能而不迁移数据。                                                                                                                                             | 语言元数据工作台地理选择行为不变或有书面迁移说明。                                                                                                                                                                     |
| A-6 | 评估 `react-select` 替换为原生或自研轻量选择器（仅限当前唯一调用点路径）。                                                                                                                                                                    | bundle 体积或交互无障碍不劣化；E2E/单测覆盖关键交互。                                                                                                                                                              |
| A-7 | 核对 `@opentelemetry/`* 主版本与 exporter 组合，避免重复打包子路径（按需升级到一致的主线）。                                                                                                                                                    | `npm ls` 无意外多版本；OTel 冒烟测试仍通过。                                                                                                                                                                 |


### 2.2 Phase B — 构建、加载与运行时性能（约 1–2 周）


| ID  | 动作                                                                                                               | 验收标准                               |
| --- | ---------------------------------------------------------------------------------------------------------------- | ---------------------------------- |
| B-1 | **i18n 按 locale 拆包**：例如 `dictionaries/zh-CN.ts`、`dictionaries/en-US.ts`，入口按 `detectLocale()` 或用户选择动态 `import()`。 | 主入口 chunk 不再静态拉取双语全文；切换语言一次懒加载。    |
| B-2 | 在 `index.html` 增加**不违背 CSP** 的 `preconnect` / `preload`（自托管字体、关键 CSS 等仅 `self` 源）。                               | Lighthouse / 网络瀑布可见提示；CSP 不放宽通配。   |
| B-3 | Spike：`iso6393Seed` / `languageNameCatalog` 类大生成物改为 JSON + `fetch` 或构建为独立静态资源，主线程按需解析。                           | 构建时间与首屏 JS 体积下降或可量化；离线场景有明确加载失败降级。 |
| B-4 | 引入 `vite-plugin-compression`（或 CI 侧 brotli）生成 `.br`/`.gz`；可选开启 `build.reportCompressedSize`。                     | 发布流水线或静态托管可使用预压缩文件。                |
| B-5 | 评估 `VadMediaBackend.browser` 等于 `main.tsx` 的 eager 副作用链（若审计指出首屏拉重）：改为懒加载或路由后加载。                                  | 首屏 bundle 或执行轨迹对比报告存档。             |

| ID  | 动作                                                                                                              | 验收标准                                                                                                                                                                                                                                                                                                                                                                                                         |
