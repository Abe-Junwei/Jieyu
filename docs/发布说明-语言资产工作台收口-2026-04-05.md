# 发布说明：语言资产工作台收口（2026-04-05）

## 版本范围
- 模块：语言元数据工作台、正字法工作台、正字法桥接工作台、语言目录运行时读模型、语言资产导航
- 目标：完成语言资产区从规划态到可用态的整体收口，消除 `/lexicon/*` 历史语义与生成目录直读残留

## 主要变更
1. 语言资产入口完成独立化
- 左侧栏已形成独立的“语言资产”二级入口区
- 子入口固定为：`语言元数据`、`正字法`、`正字法桥接`
- 主工作台与资产入口完成分层，避免继续把资产能力塞回词典工作台

2. 路由完成收口
- 正式路由：
  - `/assets/language-metadata`
  - `/assets/orthographies`
  - `/assets/orthography-bridges`
- 旧 `/lexicon/orthographies` 仅保留兼容重定向

3. 语言元数据治理完成 MVP
- 支持 built-in baseline / override / custom entry 三层模型
- 支持保存前校验：canonical tag、display locale、ISO 字段、`user:` 命名空间约束
- 支持 alias 冲突检测与高频歧义词拦截
- 支持审计历史记录：变更原因、changedFields、before/after patch、sourceRef

4. 运行时读取完成迁移
- `src/data/languageNameCatalog.ts` 改为只消费统一读快照
- 语言名称、别名、查询入口、解析逻辑统一收口到 Service 投影 + 同步读模型
- App 启动会主动刷新语言目录读模型，避免“必须先发生写操作”后才能看到持久化语言资产

5. 正字法与桥接职责完成拆分
- 正字法工作台保留正字法本体编辑与目录元数据维护
- 桥接维护已独立到正字法桥接工作台
- 正字法页不再内嵌桥接管理器，仅保留带当前目标正字法上下文的跳转入口

6. 生成链角色完成降级
- `scripts/generate-language-name-indexes.mjs` 现在明确是 baseline seed generator / snapshot builder
- `public/data/language-support/*.json` 与 generated catalog 继续作为内置基线与调试可视产物
- 运行时不再把生成文件视为唯一事实源

## 影响面
- 影响语言资产相关信息架构、运行时语言目录读取、正字法/桥接工作台职责边界
- 不影响既有词典词条浏览主路径
- 保留旧正字法链接兼容，避免外部引用立即失效

## 增量修复（2026-04-10）
1. 语言资产弹层壳层收敛
- 在不改变遮罩语义的前提下，语言资产三类面板改为仅保留 EmbeddedPanelShell 作为可见主壳
- 外层 Modal 宿主继续负责 portal、focus trap、Esc/遮罩关闭与 role=dialog 可访问性语义

2. 透明度回归修复
- 修复了语言资产工作台局部“意外透明”问题，根因是样式中使用 `var(--color-white)` 但 token 未定义，导致 `color-mix(...)` 背景声明失效
- 在主题 token 中补齐 `--color-white: #ffffff`（亮色/暗色两套 root）
- 保持语言资产弹层遮罩与其他面板一致，不采用全遮挡 opaque overlay

3. 回归保障
- App 交互回归覆盖 Esc 关闭、遮罩点击关闭、面板切换后关闭
- 新增样式结构守卫，锁定 `--color-white` token 定义与三类工作台壳层对白色 token 的依赖，防止未来再次退化为透明
