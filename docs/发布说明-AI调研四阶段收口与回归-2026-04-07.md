# 发布说明：AI 调研四阶段收口与回归（2026-04-07）

## 版本范围
- 模块：Transformers.js v4 本地嵌入 runtime、Arctic/E5 默认模型决策、CrisperWhisper 时间戳评测工具、Analysis 占位路由治理
- 目标：完成四个 AI 研究项的代码修复、真实回归与文档收口

## 主要变更
1. Arctic 实模型 A/B 入口已打通
- 修复 `EmbeddingModelABTest.test.ts` gated 分支的错误导入
- `npm run perf:embedding-model-ab:real` 已可直接复跑
- 2026-04-07 复跑结论仍为保留 `Xenova/multilingual-e5-small` 作为默认本地嵌入模型

2. Transformers runtime helper 已收敛浏览器 / Node 分支
- `transformersRuntimeConfig.ts` 统一处理浏览器无 GPU 回退 `wasm`、Node 无 GPU 回退 `cpu`
- `useBrowserCache` 仅在 cache storage 可用时开启
- Node 默认 `cacheDir` 收敛为 `.jieyu-models`，避免再向根目录写缓存目录

3. CrisperWhisper 时间戳评测可信度已修正
- `timestampEvalUtils.ts` 的 boundary F1 改为基于词级 LCS 对齐后的 start/end 边界计算
- 避免“词内容不匹配，但全局边界刚好重合”时分数虚高
- 当前产品结论仍是“暂不接入运行时”，这次修复的是评测可信度，不是推进接入

4. AI 独立工作台口径继续维持 `NO-GO`
- `/analysis` 继续由 `FeatureAvailabilityPanel` 说明页承接
- 占位路由不再被误解为“已开放独立工作台”

## 影响面
- 影响本地 embedding runtime、实模型 benchmark 入口、时间戳评测可信度与 AI 页面治理口径
- 不改变当前产品默认模型，不新增 CrisperWhisper runtime，也不开放独立 AI 工作台

## 用户可感知功能
- 转写页的正字法入口继续按模态方式覆盖在当前工作区之上；缺少背景路由状态时，关闭路径会自动回到转写页，避免出现空白背景或侧栏状态错乱
- 波形/AI 分析侧现在会在媒体加载后尝试为当前音频生成 VAD 语音段缓存；后续同一媒体再次进入时，可直接复用缓存结果补充分析叠加信息
- AI 分析面板的 stats 视图现在会显示当前媒体的 VAD 缓存状态，包括是否命中、运行引擎和当前缓存段数，便于直接判断当前媒体是否已完成 VAD 预热
- whisper-server 转写请求继续输出 `verbose_json`，置信度和调试信息来源更稳定

## 内部稳定性改进
- Transformers.js runtime helper 已统一浏览器与 Node 的设备回退策略，并收敛缓存目录与 browser cache 开关判断
- Arctic 实模型 A/B 路径已恢复可直接复跑，当前结论仍是保留 E5-Small 为默认本地嵌入模型
- CrisperWhisper 时间戳评测的 boundary F1 已改为词级 LCS 对齐后计分，避免词不对但边界偶然重合时虚高
- VAD 缓存现已同时覆盖录音输入与已加载媒体文件两条路径，缓存元信息按实际运行引擎记录
- VAD 缓存服务已补订阅能力，媒体预热完成后 AI 分析面板可自动刷新状态，不需要重新打开页面
