# 桌面端浏览器支持策略

本文定义 **解语（Jieyu）Web 应用** 在 **桌面端** 对用户与支持的浏览器承诺边界；与 [CSS 浏览器兼容矩阵](./CSS浏览器兼容矩阵.md)（样式特性与 CI 校验）互为补充：**本页管「用哪些浏览器打开产品」**，矩阵管「CSS 特性如何降级」。

复核时应一并查看业界主流产品的公开表述与 Web 平台的 **Baseline** 叙事，避免策略与外部现实脱节。

## 业界参考（截至 2026-04 的公开策略摘要）

以下仅作**对标与复核来源**，不构成对第三方的权利义务引用；各厂商可随时调整条款，以官方页面为准。

| 来源 | 要点 |
| --- | --- |
| [Google Workspace 支持的浏览器](https://support.google.com/a/answer/33864) | **Chrome**：支持**最新**稳定版（自动更新）；**Firefox、Safari、Edge**：支持**当前版与上一版**，新版本发布后滚动前移、最旧第三版移出支持。要求开启 Cookie 与 JavaScript。 |
| [Microsoft 365 网页版适用的浏览器](https://support.microsoft.com/office/which-browsers-work-with-microsoft-365-for-the-web-ad1303e0-a318-47aa-b409-d3a5eb44e452) | 桌面端建议使用 **Windows / macOS / Linux** 上各浏览器的**较新版本**；**IE 11** 自 2021-08 起不再支持。部分场景（如 Outlook 网页版）对 **Safari** 有**最低大版本**要求（官方文档曾明确 **Safari 16 及以上** 等表述，随文档更新可能变化）。 |
| [Outlook 网页版支持的浏览器](https://support.microsoft.com/office/supported-browsers-for-outlook-on-the-web-and-outlook-com-ca350265-6284-4682-9abd-85fc2bd37934) | 强调使用**最新版本**；对过旧浏览器会引导升级（例如 2024-08 起对极旧 Chrome / Safari / Firefox / Edge 拦截错误页的策略，见该页说明）。 |
| [Salesforce Lightning 支持浏览器](https://developer.salesforce.com/docs/platform/lwc/guide/get-started-supported-browsers.html)（并参见各发行说明中的 Supported browsers） | **Chrome、Firefox、Safari、Edge（Chromium）** 的**最新稳定版**方向；**IE 11**、**非 Chromium 的旧 Edge** 不支持；**Edge 的 IE 模式**不支持。 |

**归纳**：大型企业级 Web 产品普遍以 **Chrome / Edge（Chromium）/ Firefox / Safari** 为桌面基线；**IE 与旧 Edge** 已退场；版本表述上常见 **「最新」** 或 **「当前 + 上一大版本」** 两档；**Safari** 因与 macOS 绑定，业界常单独写**最低 Safari 大版本**或「随受支持系统」。

**工程侧横向参考**（非用户承诺文案）：[Web Platform Baseline](https://web.dev/baseline) 以 Chrome、Edge、Firefox、Safari（含移动端）为核心浏览器集合，用 **Newly available / Widely available** 描述特性可互用程度；工具链可通过 Browserslist 的 `baseline …` 查询与产品策略对齐（见 [Browserslist 与 Baseline](https://web.dev/blog/baseline-digest-sep-2025)）。**本产品桌面策略不绑定**某一 Baseline 年号，但新特性评审时可对照 Baseline 与 [Can I use](https://caniuse.com/) 做风险分级。

## 本产品承诺范围

### 平台范围

- **仅桌面**：运行于 **Windows 10/11** 与 **仍接收 Apple 安全更新的 macOS** 上的 **独立浏览器窗口**。
- **不包含**：手机与平板浏览器、各类 App **内嵌 WebView**、游戏机与电视浏览器等。此类环境**不在支持范围内**；如有机构合同另议，以合同为准。

### 国际主流浏览器（Windows 与 macOS）

在各自厂商的 **稳定（Stable）渠道** 下，支持以下浏览器的 **当前稳定大版本** 及其 **紧邻的上一稳定大版本**（即常说的 **n 与 n-1**；随自动更新滚动）：

| 浏览器 | 说明 |
| --- | --- |
| Google Chrome | 稳定渠道 |
| Microsoft Edge | **Chromium** 版稳定渠道；**不支持**旧版 EdgeHTML **Edge** |
| Mozilla Firefox | 稳定渠道；若用户环境为 **Firefox ESR**，则以 Mozilla 对该 ESR 分支的官方支持声明为准，且**能力上应不低于**本表对 Firefox 稳定版的 n/n-1 预期 |
| Apple Safari | 随上述受支持 macOS 安装的 Safari；**不建议**使用低于 **Safari 16** 的组合（与 Microsoft 365 等常见网页应用门槛同量级）；**无法升级系统/Safari 的旧设备**视为不在支持范围 |

### 国内常见浏览器（仅 Windows，附加条件）

在 **同时满足** 下列全部条件时，视为满足本产品的 Chromium 侧兼容预期：

| 浏览器 | 条件 |
| --- | --- |
| 360 安全浏览器 / 360 极速浏览器 | **仅极速模式**（Chromium / Blink）；**不支持** IE 内核或兼容模式 |
| QQ 浏览器 | **仅高速 / 极速模式**（Chromium / Blink）；**不支持** IE 兼容模式 |
| 搜狗高速浏览器 | **仅高速模式**（Chromium / Blink）；**不支持** IE 兼容模式 |

**内核版本**：上述浏览器的 Chromium 内核须 **不低于** 本策略对 **Google Chrome** 所隐含的 **最低稳定大版本**（与 n-1 对齐）。过旧内核请升级浏览器或改用 Chrome / Edge。

### 明确不支持

- **Internet Explorer** 全系列；**Microsoft Edge（EdgeHTML）**；**Edge Chromium 的 IE 模式**（与 Salesforce 等业界表述一致：不将 IE 模式列为支持配置）。
- **隐私模式 / 无痕模式** 下因浏览器策略导致的存储、媒体自动播放、权限差异：**不单独保证**与常规模块等价（若未来需像部分 SaaS 一样完全排除无痕，再单独开 ADR）。
- **虚拟桌面 / 远程会话**（如 Citrix、VMware 等）中浏览器或 GPU 能力受限时：**尽力而为**，与 [Google Workspace 说明](https://support.google.com/a/answer/33864) 类似，**不保证**与本地物理机完全一致。

### 基础设置

- 须启用 **JavaScript** 与 **Cookie**（含本地存储类能力所依赖的同源存储策略）。
- 不因 User-Agent 中的厂商字样做业务分支；以 **标准 Web API + 特性检测** 为准。

### 研发与测试建议

- **构建下限**：`package.json` 的 **`browserslist`**（`last 2` × Chrome / Edge / Firefox / Safari，并含 **`Firefox ESR`**）与上文 **n / n-1** 及 Firefox ESR 表述对齐，供 Vite 等工具链做转译与兼容基线；调整策略时须同步改该字段。
- **Zod 与 CSP**：`src/zodJitlessBootstrap.ts` 向 `globalConfig` 写入 `jitless`（先于 `main` 执行，见 [ADR 0021](../adr/0021-zod-jitless-strict-csp.md) 的 Vite `rollupOptions.input` + `transformIndexHtml` post 注入），避免 Zod v4 的 `Function` 构造器探测与严格 `script-src` 冲突。
- **自动化 E2E**：`playwright.config.ts` 对 **Chromium、Firefox、WebKit** 各一套 `project`。**GitHub Actions**：`pull_request` 仅跑 **Chromium**（`npm run test:e2e:chromium`）；`push` / `workflow_dispatch` 至 `main`/`dev` 跑三引擎 `npm run test:e2e`。浏览器二进制目录使用 `actions/cache` 缓存 `~/.cache/ms-playwright`（key 含 `package-lock.json` 哈希）。本地全量同上；仅跑 Chromium 时用 `npm run test:e2e:chromium` 或 `npx playwright test --project=chromium`。
- **国内浏览器**：CI **不**覆盖 360 / QQ / 搜狗；发版前按 [桌面浏览器抽样验收](../execution/release-gates/桌面浏览器抽样验收-2026-04-24.md) 手工轮换验证。
- **手工回归**：周期性地在 **360 极速 Chromium 模式** 抽样验证关键流程；QQ、搜狗可轮换。
- **TLS / 证书**：关注操作系统与浏览器信任库更新（例如行业对旧根证书的弃用）；异常连接优先从环境证书链排查。
