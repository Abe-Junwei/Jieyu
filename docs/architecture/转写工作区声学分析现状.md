# 转写工作区声学分析现状

> 文档角色：长期有效的当前现状文档。用于描述转写工作区内声学分析能力当前已经实现了什么、仍缺什么，以及代码主要落点。

## 1. 当前定位

截至当前仓库状态，解语的声学分析能力已经超过“仅 Phase 1 最小闭环”的状态，可理解为“Phase 1 完成 + Phase 2 / Phase 3 的本地子集已进入主线”，但还不是完整的科研级声学分析工作台。

当前定位应理解为：

1. 已具备浏览器本地优先的最小声学分析底座。
2. 已具备主波形区的只读声学叠加入口与最小渲染闭环。
3. 已补最小频谱只读交互、AI 声学摘要注入、声学热点提取与 IndexedDB 持久缓存。
4. 已在现有 analysis sidebar 内补出独立 acoustic tab，用于承载当前选区 acoustic summary、诊断提示、live inspector、slice、tone、descriptor、formant / vowel-space 概览与导出入口。
5. 已把 waveform / spectrogram hover 数值读数收口为统一角落 readout，并让 spectrogram / split 下的最小读数不再强依赖声学叠加开关。
6. 尚未完成带冻结、多点比较、更细粒度谱参数控制与成熟 backend 校准的重型 inspector 能力。

### 1.1 Phase 视角下的当前状态

基于当前代码落点，推荐使用以下 phase 判断：

1. Phase 1a：已完成。
2. Phase 1b：已完成。
3. Phase 1c：已完成最小闭环。
4. Phase 2：部分完成，当前以本地 inspector / tone / runtime 形态存在。
5. Phase 3：部分完成，当前以 descriptor / MFCC / 近似 Formant / 元音空间摘要 / 导出 / 校准脚本形态存在。
6. Phase 4：基本未开始。

## 2. 已落地能力

### 2.1 本地分析底座

当前仓库已落地以下声学分析基础模块：

1. `src/utils/acousticOverlayTypes.ts`
2. `src/services/acoustic/acousticAnalysisCore.ts`
3. `src/services/acoustic/acousticAnalysis.worker.ts`
4. `src/services/acoustic/AcousticAnalysisService.ts`

这些模块共同提供：

1. `AcousticOverlayMode`、`AcousticFrame`、`AcousticAnalysisConfig`、`AcousticAnalysisSummary` 等基础类型。
2. 基于 YIN + RMS/dB 的最小浏览器端声学分析核心。
3. worker 化执行入口。
4. fetch、decode、downmix、worker dispatch、内存缓存与 IndexedDB 持久缓存的统一 service 封装。
5. 基于分析结果的最小 acoustic hotspots 提取。

当前默认技术路线是：

1. 浏览器本地运行。
2. 不依赖远程声学分析服务。
3. 不引入 GPL / AGPL 作为 Phase 1 默认运行时依赖。

### 2.2 主波形区只读增强

当前主波形区已经落地以下能力：

1. `none / f0 / intensity / both` 模式切换。
2. 运行时状态持久化。
3. 主波形区 F0 / intensity 最小叠加渲染。
4. 基础图例与加载态表达。

主要代码挂点如下：

1. `src/pages/useWaveformRuntimeController.ts`
2. `src/components/WaveformToolbar.tsx`

### 2.4 已落地的 Phase 2 / 3 本地子集

当前仓库已经落地，但旧文档容易低估的能力包括：

1. acoustic runtime 进度、算法窗口、阈值与 VAD 运行状态展示。
2. live acoustic inspector，把 waveform / spectrogram hover readout 投影到右侧 acoustic tab。
3. local slice 视图，提供当前观察点附近窗口的 pitch / intensity / reliability / trend 汇总。
4. 归一化 tone / intensity contour 的最小图表化表达。
5. spectral centroid、rolloff、zero-crossing、flatness、loudness、MFCC 等描述符摘要与当前点读数。
6. 近似 Formant、元音空间摘要与最小元音图散点表达。
7. 当前选区 CSV / JSON 导出。
8. 对齐 Praat 参考样本的校准脚本与样例数据。

## 3. 当前边界

当前已经明确，但尚未完成的能力包括：

1. 主波形区与频谱区统一 hover readout 的进一步视觉与信息层级打磨。
2. 已有独立 acoustic tab，但仍未进入带冻结、多点比较与高级谱参数控制的完整 inspector 形态。
3. 当前 Formant / 元音图仍以近似估计与摘要表达为主，尚未形成成熟 backend 校准后的科研级工作流。
4. 更丰富的跨面板热点复用、批量比较与科研导出能力。

因此，当前代码不应被描述为“已完成声学分析工作台”，但也不应再被描述为“仅完成 Phase 1 最小闭环”；更准确的表述是“已完成 Phase 1，并提前落地了 Phase 2 / 3 的一部分本地能力”。

## 4. 当前运行约束

当前声学实现遵循以下约束：

1. 不新增第四种主显示模式。
2. 主声学层仍附着在现有 waveform / split 视图内。
3. 主声学层默认不抢占既有波形区编辑交互。
4. 当前重点仍是主波形区只读增强，不是复杂谱编辑器。
5. AI 仍未消费逐帧声学数据，只消费紧凑的声学摘要与热点摘要。
