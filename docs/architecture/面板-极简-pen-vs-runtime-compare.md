# 面板-极简 pen vs runtime 对比 runbook

> 说明：本文档从原 `.claude/skills/pen-language-assets-compare/SKILL.md` 迁入（2026-05-13）。当时此能力以 Claude Code skill 形式存在；因项目改为 Cursor + Copilot + Kimi 三工具统一，skill 形式不再适用，转为通用 runbook 文档。任何 AI 工具在执行"设计稿 vs 运行态"对比任务时按本文执行。

## 目标产出 | Outcome

产出一份"设计稿 vs 实际运行"差异清单，覆盖 `面板—极简.pen` 中"语言资产-section"里的三个面板，并给出可执行修复建议与优先级。

## 适用场景 | When To Use

- 用户要求对照 `.pen` 与运行页面，找视觉或交互偏差
- 发布前 UI 回归检查，尤其是语言资产区域
- 修改后怀疑出现"设计一致性漂移"

## 输入要求 | Inputs

- `.pen` 文件路径（默认：`面板—极简.pen`）
- 运行页面可访问方式（本地 dev server 地址或启动命令）
- 语言资产区域在运行页面中的入口路径（如路由或按钮）

## 默认约定 | Defaults

- 运行基线：本地 dev 页面
- 对齐强度：视觉像素级严格对齐（在结构/交互之外，重点关注 spacing、字号、颜色与边框细节）

## 执行流程 | Procedure

1. 确认运行基线 | Confirm runtime baseline
   - 启动或连接运行页面。
   - 打开目标页面，确保可以看到"语言资产"区域。
   - 若有登录/初始化步骤，先完成到可对比状态。

2. 读取设计稿结构 | Read design structure from .pen
   - 打开 `面板—极简.pen`。
   - 定位到"语言资产-section"。
   - 找到其中三个面板，记录每个面板的名称、层级、核心文案与关键控件。

3. 建立一一映射 | Build panel mapping
   - 为三面板建立 `设计面板 -> 运行面板` 映射。
   - 若名称不一致，用"位置、标题、核心功能"三因子进行匹配。

4. 逐面板对比 | Compare panel-by-panel
   - 对每个面板按以下维度检查：
     - 信息架构（标题、分组、字段顺序）
     - 视觉样式（间距、尺寸、字体层级、颜色、边框/阴影；要求严格对齐）
     - 交互行为（点击、展开收起、禁用态、hover/focus）
     - 状态覆盖（空态、加载态、错误态、有数据态）
     - 文案一致性（术语、标点、大小写、中英文）

5. 形成差异报告 | Produce diff report
   - 用"面板维度表"输出差异。
   - 每条差异包含：证据、影响、建议修复、优先级（P0/P1/P2）。
   - 标记"阻断发布项"与"可延期项"。

## 分支决策 | Decision Branches

- 如果在 `.pen` 中找不到"语言资产-section"
  - 尝试同义关键词（语言资产、Language Assets、asset、workspace）。
  - 若仍找不到，报告阻塞并请求用户指定节点或截图。

- 如果运行页面暂不可访问
  - 优先尝试启动命令（如 `npm run dev`）。
  - 若仍失败，改为"仅设计侧结构审计"并明确缺失运行侧证据。

- 如果面板数量不是 3
  - 记录实际数量，并以"预期 3 / 实际 N"作为首条高优先级差异。

## 质量门槛 | Quality Criteria

- 覆盖三面板，不能遗漏。
- 每个面板至少给出 1 条"结构"与 1 条"交互/状态"结论。
- 所有差异必须可复现（含位置或步骤）。
- 报告包含优先级和建议，不只描述现象。

## 完成检查清单 | Completion Checklist

- [ ] 已确认并记录三面板映射关系
- [ ] 已完成设计侧与运行侧逐项对比
- [ ] 已输出差异表（含证据/影响/建议/优先级）
- [ ] 已标注阻断发布项与可延期项
