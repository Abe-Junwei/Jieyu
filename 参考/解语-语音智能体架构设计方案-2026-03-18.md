# 解语 · 语音智能体架构设计方案

> 版本: v2.3 · 2026-03-19（含 Ollama Whisper Local · Phase 2 STT 引擎更新）

---

## 一、设计目标

在现有 AI 对话（11 tool calls + 3 persona + RAG + streaming）和快捷键体系（20 action, 2 scope）基础上，新增一个 **应用级语音智能体（Voice Agent）**，提供三大核心能力：

| # | 能力 | 说明 |
|---|------|------|
| **V1** | 语音/文字指令 → UI 操作 | 用户口头说"删除当前句段""播放下一个"等，Agent 解析意图后调用已有 action/tool 执行 |
| **V2** | 实时语音输入 → 语言分析建议 | 持续监听目标语言语音，实时提供 POS/gloss/翻译等建议 |
| **V3** | 语音转文字 → 翻译/标注层填充 | 对选中语段录制口述翻译或标注，STT 后直接写入对应 layer 的 utterance_texts |

|--------|------|

## 四、核心模块设计

### 4.1 VoiceInputService — 语音采集与识别

```
src/services/VoiceInputService.ts
```

**职责**: 统一管理麦克风权限、音频流、STT 转写，屏蔽底层差异。

#### 4.1.1 STT 策略（三层 fallback）

| 优先级 | 引擎 | 场景 | 延迟 | 成本 |
|--------|------|------|------|------|
| 1️⃣ | **Web Speech API** (`SpeechRecognition`) | V1 指令识别、V3 口述翻译 | ~200ms | 免费 |
| 2️⃣ | **Ollama Whisper Local**（whisper-* 模型） | V3 高精度转写 | ~3-8s（本地 GPU） | **免费 · 离线可用** |
| 3️⃣ | **Gemini Multimodal**（inlineData audio） | V2 实时语言分析 | ~1-3s | API 计费 |

```typescript
// ── 类型定义 ──

type SttEngine = 'web-speech' | 'whisper-local' | 'gemini-multimodal';

interface SttResult {
  text: string;
  lang: string;
  isFinal: boolean;          // Web Speech API interim vs final
  confidence: number;        // 0-1
  engine: SttEngine;
  audioBlob?: Blob;          // 仅 whisper-local/gemini 模式保留原始音频
  alternatives?: Array<{     // v2.1: STT 候选列表（最多 maxAlternatives 个）
    text: string;
    confidence: number;
  }>;
}

interface VoiceInputConfig {
  lang: string;              // BCP-47 语言代码, e.g. 'zh-CN', 'ja-JP'
  continuous: boolean;       // 持续监听 or 单次识别
  interimResults: boolean;   // 是否返回中间结果
  preferredEngine: SttEngine;
}

// ── 公共 API ──

class VoiceInputService {
  /** 当前是否在监听 */
  readonly listening: boolean;

  /** 启动语音输入 */
  start(config: VoiceInputConfig): void;

  /** 停止语音输入 */
  stop(): void;

  /** 事件回调 */
  onResult: (result: SttResult) => void;
  onError: (error: { code: string; message: string }) => void;
  onStateChange: (listening: boolean) => void;

  /** 检测浏览器能力 */
  static isWebSpeechSupported(): boolean;
}
```

#### 4.1.2 Web Speech API 实现要点

```typescript
// 浏览器兼容
const SpeechRecognitionClass =
  window.SpeechRecognition ?? window.webkitSpeechRecognition;

// 配置
const recognition = new SpeechRecognitionClass();
recognition.lang = config.lang;          // 必须设置，否则随系统语言
recognition.continuous = config.continuous;
recognition.interimResults = config.interimResults;
recognition.maxAlternatives = 1;

// 关键事件
recognition.onresult = (event: SpeechRecognitionEvent) => {
  for (let i = event.resultIndex; i < event.results.length; i++) {
    const result = event.results[i];
    this.onResult({
      text: result[0].transcript,
      lang: config.lang,
      isFinal: result.isFinal,
      confidence: result[0].confidence,
      engine: 'web-speech',
    });
  }
};

recognition.onerror = (event) => {
  // 'no-speech' / 'audio-capture' / 'not-allowed' / 'network'
  this.onError({ code: event.error, message: event.message });
};

// Chrome 在 continuous 模式下会静默停止 → 自动重启
recognition.onend = () => {
  if (config.continuous && this.listening) {
    recognition.start();
  }
};
```

#### 4.1.3 Ollama Whisper Local 集成（V3 高精度场景）

```typescript
/**
 * 通过 Ollama provider 转写音频（使用 whisper-* 模型）。
 *
 * Ollama whisper 模型接受音频文件输入，返回文本。
 * 模型列表: llama3.2-vision(视觉), llama3.2(文本), whisper-base,
 * whisper-small, whisper-medium, whisper-large-v3
 *
 * 优势: 免费 · 离线可用 · 无 API 计费 · 适合低资源语言
 * 延迟: ~3-8s（取决于模型大小和硬件）
 */
async function transcribeWithOllamaWhisper(
  audioBlob: Blob,
  ollamaBaseUrl: string,   // e.g. 'http://localhost:11434'
  model: string = 'whisper-small',
  lang?: string,           // BCP-47, whisper 模型可接受 language hint
): Promise<SttResult> {
  const formData = new FormData();
  formData.append('file', audioBlob, 'recording.webm');
  formData.append('model', model);
  if (lang) formData.append('language', lang);

  const resp = await fetch(`${ollamaBaseUrl}/api/audio/transcriptions`, {
    method: 'POST',
    body: formData,
  });

  if (!resp.ok) {
    throw new Error(`Ollama Whisper failed: ${resp.status} ${await resp.text()}`);
  }

  const json = await resp.json();
  return {
    text: json.text ?? '',
    lang: lang ?? 'unknown',
    isFinal: true,
    confidence: 1.0,
    engine: 'whisper-local',
    audioBlob,
  };
}
```

**注意**: Ollama Whisper 使用 `/api/audio/transcriptions` 端点（不是 `/v1/audio/transcriptions`）。确保 Ollama 版本 >= 0.1.30 并且已拉取对应模型:

```bash
# 拉取 Whisper 模型（首次 ~40MB-1.4GB，取决于模型大小）
ollama pull whisper-small    # 最小模型，推荐 fieldwork 使用
ollama pull whisper-medium   # 更高精度，需要更多内存
ollama pull whisper-large-v3 # 最高精度，需要 6GB+ GPU

# 验证模型已安装
ollama list
```

**职责**: 将 STT transcript 分类为 4 种意图之一，优先用本地规则匹配，fallback 到 LLM。

#### 4.2.1 意图类型

```typescript
type VoiceIntentKind = 'action' | 'chat' | 'dictation' | 'analysis';

interface VoiceIntent {
  kind: VoiceIntentKind;
  confidence: number;

  // action 类型专有
  actionId?: ActionId;        // 映射到 KeybindingService 的 ActionId
  actionParams?: Record<string, unknown>;

  // dictation 类型专有
  text?: string;              // 口述内容
  targetLayerId?: string;     // 目标写入层

  // chat / analysis 共用
  chatText?: string;          // 发送给 LLM 的文本
  personaKey?: AiSystemPersonaKey;
}
```

#### 4.2.2 本地规则引擎（零延迟，零成本）

```typescript
// ── 指令模式库 ──

interface CommandPattern {
  patterns: RegExp[];
  actionId: ActionId;
  extract?: (match: RegExpMatchArray) => Record<string, unknown>;
}

const COMMAND_PATTERNS: CommandPattern[] = [
  // ── 播放控制 ──
  { patterns: [/^(播放|暂停|play|pause)$/i],                    actionId: 'playPause' },
  { patterns: [/^(下一[个段]|next)/i],                           actionId: 'navNext' },
  { patterns: [/^(上一[个段]|prev|previous)/i],                  actionId: 'navPrev' },

  // ── 编辑操作 ──
  { patterns: [/^(删除|delete)(当前)?(句段|segment)?$/i],         actionId: 'deleteSegment' },
  { patterns: [/^(撤销|undo)$/i],                                actionId: 'undo' },
  { patterns: [/^(重做|redo)$/i],                                actionId: 'redo' },
  { patterns: [/^(合并)(上一个|前一个|prev)/i],                   actionId: 'mergePrev' },
  { patterns: [/^(合并)(下一个|后一个|next)/i],                   actionId: 'mergeNext' },
  { patterns: [/^(分割|split)(句段|segment)?$/i],                 actionId: 'splitSegment' },
  { patterns: [/^(标记|mark)(句段|segment)?$/i],                  actionId: 'markSegment' },

  // ── 视图/搜索 ──
  { patterns: [/^(搜索|查找|search|find)$/i],                    actionId: 'search' },
  { patterns: [/^(全选|select\s?all)$/i],                         actionId: 'selectAll' },
];

// ── 路由逻辑 ──

function routeIntent(
  transcript: string,
  mode: VoiceAgentMode,
): VoiceIntent {
  const text = transcript.trim();

  // 1️⃣ 口述模式下直接归类为 dictation
  if (mode === 'dictation') {
    return { kind: 'dictation', confidence: 1.0, text };
  }

  // 2️⃣ 本地正则匹配 → action
  for (const cmd of COMMAND_PATTERNS) {
    for (const re of cmd.patterns) {
      const m = text.match(re);
      if (m) {
        return {
          kind: 'action',
          confidence: 0.95,
          actionId: cmd.actionId,
          actionParams: cmd.extract?.(m),
        };
      }
    }
  }

  // 3️⃣ 以"分析""建议""这个词"等开头 → analysis
  if (/^(分析|建议|这个词|这句话|analyze|suggest)/i.test(text)) {
    return { kind: 'analysis', confidence: 0.8, chatText: text };
  }

  // 4️⃣ 默认 → chat（发给 LLM 做意图识别 + 对话）
  return { kind: 'chat', confidence: 0.5, chatText: text };
}
```

#### 4.4.4 STT 置信度视觉编码（v2.1 新增）

根据 `SttResult.confidence` 值对识别结果施加不同颜色指示，帮助用户快速判断是否需要人工确认：

| 置信度范围 | 颜色 | 行为 |
|------------|------|------|
| ≥ 0.85 | 🟢 绿色 | 直接执行/写入，无需确认 |
| 0.70 – 0.84 | 🟡 黄色 | 显示识别文本 + “确认执行?” 提示 |
| < 0.70 | 🔴 红色 | 显示 alternatives 下拉列表，用户选择或重新说 |

```typescript
// VoiceAgentWidget.tsx 中的置信度色码逻辑
function getConfidenceColor(confidence: number): 'green' | 'yellow' | 'red' {
  if (confidence >= 0.85) return 'green';
  if (confidence >= 0.70) return 'yellow';
  return 'red';
}

// 低置信度时显示候选列表
function renderAlternatives(result: SttResult) {
  if (result.confidence >= 0.70 || !result.alternatives?.length) return null;
  return (
    <select onChange={e => onSelectAlternative(e.target.value)}>
      <option>{result.text} ({(result.confidence * 100).toFixed(0)}%)</option>
      {result.alternatives.map((alt, i) => (
        <option key={i} value={alt.text}>
          {alt.text} ({(alt.confidence * 100).toFixed(0)}%)
        </option>
      ))}
    </select>
  );
}
```

> **与 safeMode 的交互**: 当 `safeMode: true` 时，破坏性操作无论置信度多高均需确认。

#### 直接 Action（本地执行，零延迟）

| 语音指令示例 | ActionId | 说明 |

| "下一个" / "next" | `navNext` | 导航到下一句段 |
| "上一个" / "previous" | `navPrev` | 导航到上一句段 |
| "删除" / "delete" | `deleteSegment` | 删除选中句段 |
| "撤销" / "undo" | `undo` | 撤销 |
| "重做" / "redo" | `redo` | 重做 |
| "合并上一个" | `mergePrev` | 合并上一句段 |
| "合并下一个" | `mergeNext` | 合并下一句段 |
| "分割" / "split" | `splitSegment` | 在播放头位置分割 |
| "标记" / "mark" | `markSegment` | 标记句段起止 |
| "搜索" / "search" | `search` | 打开搜索 |
| "全选" / "select all" | `selectAll` | 全选句段 |

#### LLM Tool Call（通过 AI Chat 执行）

| 语音指令示例 | Tool Name | 说明 |
|-------------|-----------|------|
| "转写这句为'...'" | `set_transcription_text` | 设置转写文本 |
| "翻译成'...'" | `set_translation_text` | 设置翻译文本 |
| "新建一个日语转写层" | `create_transcription_layer` | 创建层 |

| "关联翻译层" | `link_translation_layer` | 关联层 |

### 5.2 安全策略

- **破坏性操作确认**: `deleteSegment`, `delete_layer` 等仍需用户确认（复用已有 `allowDestructiveToolCalls` 机制）
- **误触防护**: action 类意图 `confidence < 0.8` 时，显示确认 toast 而非直接执行
- **撤销能力**: 所有通过语音触发的写操作记录到 undo 栈

### 14.2 P0 改进详细设计

#### I1: TTS 语音反馈输出

**问题**: 当前方案是"哑巴 Agent"——只能听指令、不能说话回复。所有成熟语音助手都有语音输出。对于语言学分析场景，TTS 尤其关键：用户可以在操作波形时听到 AI 回复，无需转头看屏幕。

**方案**: 利用 Web Speech Synthesis API（浏览器原生，零成本）+ 可选云端 TTS。

```typescript
// ── src/services/TtsService.ts ──

type TtsEngine = 'web-speech' | 'openai-tts';

interface TtsConfig {
  engine: TtsEngine;
  lang: string;           // BCP-47
  rate: number;            // 0.5 - 2.0, default 1.0
  voice?: string;          // Web Speech voice name
  enabled: boolean;        // 全局开关
}

class TtsService {
  private synth = window.speechSynthesis;
  private currentUtterance: SpeechSynthesisUtterance | null = null;

  /** 朗读文本 */
  speak(text: string, config: TtsConfig): void {
    this.cancel(); // 打断上一句
    if (!config.enabled) return;

    const utt = new SpeechSynthesisUtterance(text);
    utt.lang = config.lang;
    utt.rate = config.rate;
    if (config.voice) {
      const voice = this.synth.getVoices().find(v => v.name === config.voice);
      if (voice) utt.voice = voice;
    }
    this.currentUtterance = utt;
    this.synth.speak(utt);
  }

  /** 取消当前朗读 */
  cancel(): void {
    this.synth.cancel();
    this.currentUtterance = null;
  }

  /** 获取可用声音列表 */
  getVoices(lang?: string): SpeechSynthesisVoice[] {
    const voices = this.synth.getVoices();
    return lang ? voices.filter(v => v.lang.startsWith(lang)) : voices;
  }

  readonly speaking: boolean; // = this.synth.speaking
}
```

**集成到 useVoiceAgent**:

```typescript
// 在 action 执行完毕后播报结果
case 'action':
  executeAction(intent.actionId);
  ttsService.speak(`已执行: ${actionLabel}`, ttsConfig);
  break;

// AI 分析完成后口述结论（可选截取前 N 字）
case 'analysis':
  const result = await sendAiMessage(intent.chatText);
  ttsService.speak(truncate(result, 200), ttsConfig);
  break;

// dictation 模式下回读确认
case 'dictation':
  saveTextTranslation(...);
  ttsService.speak(`已写入: "${intent.text}"`, ttsConfig);

```

**用户可配置**: TTS 开关、语速、声音偏好都放入 `VoiceAgentSettings`。

#### I4: 多轮对话上下文

**问题**: 原方案每次语音输入都是独立的，无法理解指代关系：
- "删除这句" → ok
- "也删除下一句" → ？（"也"和"下一句"需要理解上一轮执行了什么）
- "把刚才删的撤销" → ？（需要知道"刚才"做了什么）

**对标**: OpenAI Realtime API 维护完整 conversation items 列表；Alexa 的 session attributes。

**方案**: 在 useVoiceAgent 中维护一个 **短期操作栈**（VoiceSession）：

```typescript
interface VoiceSessionEntry {
  timestamp: number;
  transcript: string;         // 用户原始语音
  intent: VoiceIntent;        // 解析后的意图
  result: string;             // 执行结果描述
  affectedIds?: string[];     // 受影响的实体 ID
}

interface VoiceSession {
  entries: VoiceSessionEntry[];           // 最近 N 轮（默认 10）
  startedAt: number;
  /** 会话超时: 静默 5 分钟后自动清空 */
  timeoutMs: number;
}

// ── 在 IntentRouter 中注入上下文 ──

function routeIntent(
  transcript: string,
  mode: VoiceAgentMode,
  session: VoiceSession,       // ← 新增参数
): VoiceIntent {
  // ...正则匹配逻辑不变...

  // chat fallback 时，将最近 3 轮操作注入 LLM 上下文
  const recentContext = session.entries.slice(-3).map(e =>
    `[${e.intent.kind}] "${e.transcript}" → ${e.result}`
  ).join('\n');

  return {
    kind: 'chat',
    confidence: 0.5,
    chatText: `[近期语音操作]\n${recentContext}\n\n[当前指令] ${text}`,
  };
}
```

**效果**: LLM 收到上下文后能正确理解 "也删除下一句"（看到上一轮是 deleteSegment）、"撤销刚才的"（看到上一轮操作）。

```typescript
// ── 导出可重放操作序列 ──

interface ReversibleCommand {
  action: string;           // ActionId 或 tool name
  args: Record<string, unknown>;
  timestamp: number;
}

function exportReplaySequence(
  session: VoiceSession,
  lastN: number = 5,
): ReversibleCommand[] {
  return session.entries
    .slice(-lastN)
    .filter(e => e.intent.kind === 'action' || e.intent.kind === 'tool')
    .map(e => ({
      action: e.intent.kind === 'action' ? e.intent.actionId : e.intent.toolName,
      args: e.intent.kind === 'tool' ? e.intent.toolArgs : {},
      timestamp: e.timestamp,
    }));
}

// → exportReplaySequence(session, 1) → 取最近一条可执行操作 → 重放
```

```typescript
/** 尝试将 languageId 映射为 BCP-47；失败时返回 undefined */
export function toBcp47(languageId: string): string | undefined {
  // 直接查表
  const direct = ISO639_TO_BCP47[languageId.toLowerCase()];
  if (direct) return direct;

  // 如果已经是 BCP-47 格式 (xx-YY)，直接返回
  if (/^[a-z]{2,3}(-[A-Z]{2})?$/.test(languageId)) return languageId;

  return undefined;
}
```

**v2.1 新增: 上下文感知槽位预填（prefillSlots）**

```typescript
// ── 根据项目上下文自动预填槽位，减少澄清对话轮次 ──

function prefillSlots(
  pending: PendingSlotFill,
  context: {
    transcriptionLayers: TranscriptionLayerDocType[];
    translationLayers: TranslationLayerDocType[];
    selectedLayerId?: string;
  },
): PendingSlotFill {
  const filled = { ...pending.filledSlots };
  const remaining = [...pending.missingSlots];

  // 规则 1: 如果项目只有一个转写层，自动填入 languageId
  if (remaining.includes('languageId')) {
    const allLangs = [
      ...context.transcriptionLayers.map(l => l.languageId),
      ...context.translationLayers.map(l => l.languageId),
    ];
    const unique = [...new Set(allLangs)];
    if (unique.length === 1) {
      filled['languageId'] = unique[0];
      remaining.splice(remaining.indexOf('languageId'), 1);
    }
  }

  // 规则 2: 如果当前选中了某个 layer，推断 modality
  if (remaining.includes('modality') && context.selectedLayerId) {
    const isTrans = context.translationLayers.some(l => l.id === context.selectedLayerId);
    filled['modality'] = isTrans ? 'translation' : 'transcription';
    remaining.splice(remaining.indexOf('modality'), 1);
  }

  return {
    ...pending,
    filledSlots: filled,
    missingSlots: remaining,
    clarificationPrompt: remaining.length > 0
      ? `请问${remaining[0] === 'languageId' ? '什么语言' : remaining[0]}？`
      : '', // 全部填完，无需澄清
  };
}
```

#### I8: 音效反馈（Earcon）

**方案**: 使用 Web Audio API 合成简短音效，无需加载音频文件：

```typescript
// ── src/services/EarconService.ts ──

class EarconService {
  private ctx: AudioContext | null = null;

  private getCtx(): AudioContext {
    if (!this.ctx) this.ctx = new AudioContext();
    return this.ctx;
  }

  /** 激活声: 短促升调 beep */
  playActivate(): void {
    this.playTone(880, 0.08);   // A5, 80ms
  }

  /** 成功声: 双音 ding */
  playSuccess(): void {
    this.playTone(523, 0.06);   // C5, 60ms
    setTimeout(() => this.playTone(784, 0.1), 80);  // G5, 100ms
  }

  /** 错误声: 低沉 buzz */
  playError(): void {
    this.playTone(220, 0.15);   // A3, 150ms
  }

  private playTone(freq: number, duration: number): void {
    const ctx = this.getCtx();
    const osc = ctx.createOscillator();
    const gain = ctx.createGain();
    osc.frequency.value = freq;
    osc.type = 'sine';
    gain.gain.setValueAtTime(0.15, ctx.currentTime);
    gain.gain.exponentialRampToValueAtTime(0.001, ctx.currentTime + duration);
    osc.connect(gain).connect(ctx.destination);
    osc.start();
    osc.stop(ctx.currentTime + duration);
  }
}
```

**集成**: 麦克风激活 → `playActivate()`；指令执行成功 → `playSuccess()`；识别失败 → `playError()`。

### 15.1 六维对标评分

### 15.2 差距分析

| 差距 | 根因 | 弥合策略 |
|------|------|---------|
| **TTS 自然度差距最大**（MOS 3.0 vs 4.5） | Web Speech Synthesis 为拼接合成，机械感重 | 浏览器阶段接受；桌面版引入 Piper TTS（MOS 4.0+）→ Fish Speech（MOS 4.3+）（见§17） |

| 缺乏全双工 | 浏览器 Web Speech API 为半双工 | P2 阶段引入 OpenAI Realtime API（I10）；桌面版可直连 |

### 15.3 解语独有优势（竞品不具备）

1. **垂直领域工具集成**：20 个键绑定操作 + 11 个 AI 工具 + 3 个 persona 切换 → 语音直接操作语料标注，豆包/千问无此能力
2. **低资源语言覆盖**：Whisper 支持 99 种语言，含大量豆包/千问未覆盖的语种（如 Yoruba、Quechua、Khmer 等）
3. **数据溯源**：所有语音操作记录 provenance（actorType + method + sttEngine + timestamp），可审计、可回溯
4. **零延迟简单指令**：正则匹配 <5ms，比云端 NLU 快 20-60 倍
5. **隐私保护**：桌面版可完全离线运行（本地 Whisper + 本地 Piper TTS + Ollama LLM）

### 15.4 总体评价

| 场景 | 评分 | 核心判断 |
|------|------|---------|
| 通用助手体验 | 3/5 | 无法匹敌豆包/千问的通用对话流畅度 |
| **垂直标注工作流** | **4/5** | 语音指令+口述翻译+分析请求的三合一闭环，**在语言学标注领域独一无二** |
| 低资源语言田野场景 | **5/5** | 离线 + 99 语言 + 数据主权 + 领域工具 → **超越通用助手** |

### 16.4 Tauri IPC 通信设计

```typescript
// ── 前端调用 Tauri 命令 ──

import { invoke } from '@tauri-apps/api/core';

// STT: 发送音频 Blob → Whisper.cpp sidecar
async function transcribeLocal(audioData: ArrayBuffer, lang: string): Promise<SttResult> {
  const result = await invoke<{ text: string; confidence: number }>('whisper_transcribe', {
    audio: Array.from(new Uint8Array(audioData)),
    lang,
    model: settings.localWhisperModelSize,
  });
  return {
    text: result.text,
    lang,
    isFinal: true,
    confidence: result.confidence,
    engine: 'whisper-local',
  };
}

// TTS: 文本 → Piper sidecar → 音频播放
async function speakLocal(text: string, lang: string): Promise<void> {
  const audioData = await invoke<number[]>('piper_speak', { text, lang });
  const buffer = new Uint8Array(audioData).buffer;
  const ctx = new AudioContext();
  const audioBuffer = await ctx.decodeAudioData(buffer);
  const source = ctx.createBufferSource();
  source.buffer = audioBuffer;
  source.connect(ctx.destination);
  source.start();
}
```

## 十七、TTS 引擎进阶路线

> TTS 自然度是与豆包/千问差距最大的维度（MOS 3.0 vs 4.5）。以下为三阶梯进方案。

### 17.1 引擎对比

| 引擎 | MOS 评分 | 体积 | 硬件需求 | 许可证 | 适用阶段 |
|------|---------|------|---------|--------|---------|
| **Web Speech Synthesis** | 3.0-3.5 | 0（系统内置） | CPU | — | Phase 1-2（浏览器） |
| **Piper TTS** | 3.8-4.2 | ~50 MB | CPU | MIT | **Phase 3（推荐首选）** |
| **Fish Speech** | 4.3-4.5 | ~1 GB | GPU 推荐 | Apache-2.0 | Phase 4（中文优化） |
| **Coqui XTTS v2** | 4.2-4.5 | ~2 GB | GPU 必需 | CPML | Phase 5（声音克隆） |
| **GPT-SoVITS** | 4.5+ | ~1.5 GB | GPU 必需 | MIT | 未来（5秒声音克隆） |

### 17.2 Piper TTS 详细方案（Phase 3 桌面版推荐）

**为什么是 Piper**：
- C++ 实现，可直接编译为 Tauri sidecar（无 Python 依赖）
- CPU 即可运行，实时率 > 10x
- MIT 许可证，商业可用
- 支持 20+ 种语言预训练模型
- MOS 提升 +1.0（从 3.0→4.0+）

```rust
// ── src-tauri/src/piper.rs ──

use tauri::command;
use std::process::Command;

#[command]
async fn piper_speak(text: String, lang: String) -> Result<Vec<u8>, String> {
    let model_path = get_model_path(&lang)?;
    let output = Command::new("piper")
        .args(["--model", &model_path, "--output_raw"])
        .stdin(std::process::Stdio::piped())
        .stdout(std::process::Stdio::piped())
        .spawn()
        .map_err(|e| e.to_string())?;
    // pipe text to stdin, read raw audio from stdout
    // ...
    Ok(audio_bytes)
}
```

### 17.3 Fish Speech 详细方案（Phase 4 中文场景）

**为什么是 Fish Speech**：
- 中文 MOS 4.3-4.5，接近豆包水平
- 支持零样本语音克隆（3 秒参考音频即可）
- Apache-2.0 许可证
- 推理速度 ~150 tokens/s（GPU）

**与 Piper 的互补**：Piper 覆盖多语言日常播报；Fish Speech 在需要高自然度中文输出时按需切换。

### 17.4 TTS 阶梯路线总结

```
Phase 1-2 (浏览器)          Phase 3 (桌面版)         Phase 4+ (深度集成)
────────────────────      ────────────────────      ────────────────────
Web Speech Synthesis  →   Piper TTS (Sidecar)  →   Fish Speech / XTTS
MOS 3.0-3.5               MOS 3.8-4.2               MOS 4.3-4.5
零成本                     +50MB 模型                +1GB 模型 + GPU
20+ 语言                   20+ 语言                  中文最优 + 克隆能力
```

> **v2.2 未来可选注记**: [Cartesia Sonic](https://cartesia.ai) 提供 <100ms 流式 TTS，声音自然度极高，适合全双工打断式对话（barge-in）场景。但属于商业闭源 API，不符合离线/隐私优先原则。待 Phase 5 全双工场景成熟后可重新评估（或待其开源替代方案出现）。
