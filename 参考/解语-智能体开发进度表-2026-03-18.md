## 2) 关键判断

1. 智能体能力当前已经不是“空白功能”，而是“转写页内的重能力簇”。
2. 这些能力的最大问题不再是“能不能做”，而是“是否已从转写页大编排层中抽成稳定产品模块”。
3. 因此后续排期不应继续把 F31/F32/Voice Agent 当作独立横向功能线推进，而应优先服务于“转写页内核收口”和“其余工作台解锁前置条件”。

---

## 3) 分项进度与缺口

## F8 AI任务调度与状态管理

### 已完成

- 全局统一调度器已稳定运行，embedding / auto-gloss / AI 任务均可复用同一状态机。
- 任务取消、重试、超时与 UI 刷新已完成，并已通过全量回归。
- 任务状态已能在转写页 AI 侧栏展示，而不是隐藏在服务层。

### 当前缺口

- 缺 taskType 级别的分级并发控制。
- 缺“用户交互任务优先、后台批处理让步”的统一优先级策略。
- 缺按任务类型配置的 timeout / retry 模板。

### 建议下一步

1. 把 F8 的剩余工作作为“转写页任务中心”能力实现，而不是单独新增页面。
2. 先补优先级与并发治理，再决定是否独立开放分析页的任务工作台。

---

## F9 Provider抽象层

### 已完成

- LLM Provider、商业 STT Provider、Embedding Provider 已形成可扩展抽象。
- 近期已补 deferred runtime 契约缺口与运行时收敛问题，减少“类型能过、运行时报错”的风险。

### 当前缺口

- 缺 provider 健康探针与失败自动降级策略。
- 缺按场景切换 provider 的产品级配置面板，而不仅是开发导向的选择器。

### 建议下一步

1. 把 provider health / fallback 做成统一基础设施。
2. 让转写页中的 AI 与语音能力都复用同一套 provider 诊断与降级策略。

---

## F10 强制对齐（MFA/WhisperX/WebMAUS）

### 已完成

- 方案与优先级已明确：MFA 为本地高精度主链，WhisperX 为快速服务端通道，WebMAUS 作为远程兜底或验证链路。

### 当前缺口

- 代码侧尚无完整对齐 runtime、任务类型、结果回填 UI、失败回退流程。
- 转写页里也还没有“对齐任务卡片 -> 回填 -> 人工修正”的端到端体验。

### 建议下一步

1. 先在转写页内完成远程最小链路（WhisperX 或 WebMAUS 二选一）验证交互。
2. 再规划 Tauri sidecar + MFA，把高精度本地链路接入。

---

## F31 LLM对话与上下文感知

### 已完成

- 对话、流式输出、工具调用、persona/context 注入、ProjectObserver 与 AI 助手侧栏均已在转写页里工作。
- 近期审计修复后，语音助手与 AI 助手之间的上下文、确认链路和面板展开行为也已更一致。

### 当前缺口

- Prompt 实验与对话能力仍主要依附在转写页，不是独立工作台。
- 缺受控的多步 agent loop。
- 缺更清晰的“保留上下文重试 / 分支对话 / 回放审计”产品流。

### 建议下一步

1. 先把 F31 继续沉到转写页稳定工作流中，而不是急着独立开放分析页。
2. 对多步 loop 采取 2-3 步上限的受控策略，避免工具链失控。

---

## F32 项目知识库 RAG

### 已完成

- 多源检索（utterance / note / pdf / schema）与 hybrid 融合已落地。
- PDF 解析、嵌入、引用跳转、预览与片段高亮已有可用实现。
- 这些能力已真实嵌入转写页 AI 侧栏，而非停留在服务层 Demo。

### 当前缺口

- AnalysisPage 本身尚未开放，因此 RAG 仍是“转写页内部能力”，不是独立分析工作台。
- 缺独立文献管理闭环、检索模板治理、脚注风格产品化与用户可配置检索策略。

### 建议下一步

1. 继续围绕转写页中的“问答、校对、术语查证”三类场景做检索模板化。
2. 待转写页收口后，再把 F32 抽成独立分析页的主能力。

## 4) 语音智能体专项

### 当前已完成

- Web Speech / 本地 Whisper / 商业 STT 的统一接入与切换。
- 命令意图路由、LLM fallback、alias 学习、低置信度消歧、破坏性操作确认。
- 语音面板、Dock、AI 助手侧栏的统一 UI 接口。
- 2026-03-29 已补齐消歧候选在 start/stop/confirm/cancel/switchMode 中的生命周期清理，避免旧会话状态残留。

### 当前未完成

- TTS 服务与完整语音播报闭环。
- dictation -> 写入 -> 自动前进 的产品化打磨。
- 分析模式的结果回填自动化与可视反馈。
- 桌面 sidecar / 全双工 / 唤醒词常驻等 Phase 3 能力。

### 当前判断

- 语音智能体已具备“转写页内可用”水位。
- 但它仍不是独立产品线，应继续按“服务转写页主流程”的原则推进。
